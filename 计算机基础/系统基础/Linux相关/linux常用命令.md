### chmod补充：权限模式参数
#### 1. 数字模式（八进制模式）
最常用的方式，使用3位八进制数表示权限：

| 数字 | 权限 | 说明       |
| :------------------------------------------ | :------------------------------------------ | :------------------------------------------------ |
| 0                                           | ---                                         | 无任何权限    |
| 1                                           | --x                                         | 只有执行权限   |
| 2                                           | -w-                                         | 只有写权限    |
| 3                                           | -wx                                         | 写和执行权限   |
| 4                                           | r--                                         | 只有读权限    |
| 5                                           | r-x                                         | 读和执行权限   |
| 6                                           | rw-                                         | 读和写权限    |
| 7                                           | rwx                                         | 读、写、执行权限 |

**数字模式结构：**

```plain
chmod ABC file
```

+ A= 用户(User/Owner)权限
+ B= 组(Group)权限
+ C= 其他用户(Other)权限

**示例：**

```plain
chmod 755 script.sh    # 用户:rwx, 组:r-x, 其他:r-x
chmod 644 document.txt # 用户:rw-, 组:r--, 其他:r--
chmod 700 private.txt  # 只有用户有全部权限，其他用户无任何权限
```

#### 2. 符号模式（字母模式）
更直观的方式，使用字母表示权限：

**用户类别：**

+ u: 文件所有者(User)
+ g: 文件所属组(Group)
+ o: 其他用户(Other)
+ a: 所有用户(All)，相当于ugo

**操作符号：**

+ +: 添加权限
+ -: 移除权限
+ =: 设置精确权限（覆盖原有权限）

**权限类型：**

+ r: 读权限
+ w: 写权限
+ x: 执行权限

**符号模式示例：**

```plain
# 添加执行权限给所有用户
chmod a+x script.sh

# 给文件所有者添加写权限
chmod u+w file.txt

# 移除组和其他用户的写权限
chmod go-w document.txt

# 设置精确权限：用户有全部权限，组有读权限，其他用户无权限
chmod u=rwx,g=r,o= file.txt

# 组合操作
chmod u+x,go-rwx secret.txt
```

## 重定向与管道

| 命令                                                                      | 说明                                                             |
| ----------------------------------------------------------------------- | -------------------------------------------------------------- |
| cut -d  '，' -f 1 name.txt >1.txt                                        | 输出为1.txt覆盖之前的内容    |
| cut -d  '，' -f 1 name.txt>>1.txt                                        | 追加输出到文件1.txt       |
| cut -d  '，' -f 1 name.txt 2>                                            | 错误输出               |
| cut -d  '，' -f 1 name.txt `2>>`                                         | 追加错误信息             |
| cut -d  '，' -f 1 name.txt2>&1                                           | 错误与标准输出合并          |
| cat `<`文件                                                               | 被动读取文件             |
| cat < name.txt输出name.txt（cat被动）          cat name.txt 打开name.txt（cat主动） |                                                                |
| wc -l `<<`END                                                           | 从标准输入读取直到 END，然后统计 |
| \|                                                                      | 管道：前一个命令的输出作为后一个的输入                                            |
