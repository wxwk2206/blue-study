# Claude Code 终端版本使用指令

> 整理日期：2026-05-29。Claude Code 更新比较快，遇到命令不一致时，优先看 `claude --help`、会话内 `/help` 和官方文档。

## 1. 安装与检查

Claude Code 是终端里的 AI 编程助手。使用前需要 Claude Pro、Max、Team、Enterprise、Claude Console 账号，或企业云服务商接入；免费 Claude.ai 账号默认不包含 Claude Code。

### Windows PowerShell

```powershell
irm https://claude.ai/install.ps1 | iex
claude --version
claude doctor
```

### Windows CMD

```cmd
curl -fsSL https://claude.ai/install.cmd -o install.cmd && install.cmd && del install.cmd
claude --version
claude doctor
```

### macOS / Linux / WSL

```bash
curl -fsSL https://claude.ai/install.sh | bash
claude --version
claude doctor
```

### 其他安装方式

```bash
# macOS Homebrew，stable 通道
brew install --cask claude-code

# Windows WinGet
winget install Anthropic.ClaudeCode

# npm，全局安装；需要 Node.js 18+
npm install -g @anthropic-ai/claude-code
```

Windows 原生使用时建议安装 Git for Windows，这样 Claude Code 可以使用 Bash 工具；如果没有安装 Git for Windows，会回退到 PowerShell 工具。需要 Linux 工具链或沙箱能力时，优先放在 WSL 2 里运行。

## 2. 登录、登出、账号状态

第一次直接运行 `claude` 会引导浏览器登录。

```bash
# 启动并按提示登录
claude

# 显式登录
claude auth login

# 使用 Claude Console 账号，按 API 用量计费
claude auth login --console

# 预填邮箱或强制 SSO
claude auth login --email your@email.com
claude auth login --sso

# 查看登录状态
claude auth status --text

# 登出
claude auth logout
```

进入交互会话后也可以用：

```text
/login
/logout
/status
```

## 3. 启动 Claude Code

```bash
# 进入项目目录后启动交互会话
cd /path/to/project
claude

# 启动时带上第一个问题
claude "解释这个项目的目录结构"

# 一次性问答：输出结果后退出，适合脚本
claude -p "解释这个函数的作用"

# 管道输入，Linux/macOS/WSL
cat logs.txt | claude -p "分析这些报错"
```

PowerShell 管道示例：

```powershell
Get-Content .\logs.txt | claude -p "分析这些报错"
```

## 4. 继续或恢复会话

```bash
# 继续当前目录最近一次会话
claude -c
claude --continue

# 继续最近一次会话，并直接追加任务
claude -c -p "检查是否还有类型错误"

# 恢复指定会话，可以用会话 ID 或名字
claude -r "auth-refactor"
claude --resume "auth-refactor"

# 启动时给会话起名，方便后续恢复
claude -n "fix-login-bug"

# 在当前恢复点创建新会话分支
claude --resume "auth-refactor" --fork-session
```

交互会话内：

```text
/resume
/rename 新名字
/clear
/compact
```

## 5. 常用启动参数

```bash
# 指定模型；常用别名  haiku低 / sonnet中 / opus高
claude --model sonnet

# 指定推理强度
claude --effort low
claude --effort medium
claude --effort high
claude --effort xhigh
claude --effort max

# 从计划模式启动，只探索和制定方案，不直接改文件
claude --permission-mode plan

# 让 Claude 额外访问其他目录
claude --add-dir ../lib ../docs

# 在隔离 git worktree 中工作
claude --worktree feature-login
claude -w feature-login

# 输出详细日志
claude --verbose
claude --debug
```

一次性模式常用参数：

```bash
# 限制最多轮数
claude -p --max-turns 3 "只分析，不修改：这个模块的入口在哪里？"

# JSON 输出，适合脚本处理
claude -p "总结变更" --output-format json

# 流式 JSON 输出
claude -p "运行代码审查" --output-format stream-json --verbose

# 替换或追加系统提示词
claude -p --system-prompt "你是 Python 安全审计助手" "审计这个文件"
claude -p --append-system-prompt "回答必须简洁" "解释这个函数"

# 最小模式，跳过 hooks、skills、plugins、MCP、CLAUDE.md 等自动发现
claude --bare -p "解释当前目录"
```

## 6. 会话内斜杠命令

在 Claude Code 交互界面里，输入 `/` 可以查看当前可用命令；不同平台、账号和版本显示的命令可能不同。

| 命令                 | 用途                                 |
| ------------------ | ---------------------------------- |
| `/help`            | 查看帮助和可用命令                          |
| `/init`            | 为项目生成 `CLAUDE.md` 项目说明             |
| `/memory`          | 编辑或查看项目记忆                          |
| `/config`          | 打开设置界面                             |
| `/permissions`     | 管理工具权限规则                           |
| `/plan`            | 进入计划模式，适合大改动前先让 Claude 拆方案         |
| `/model`           | 切换模型                               |
| `/effort`          | 调整推理强度                             |
| `/add-dir 路径`      | 会话中额外授权一个工作目录                      |
| `/context`         | 查看上下文占用情况                          |
| `/compact`         | 压缩长会话上下文                           |
| `/clear`           | 开新对话，旧会话仍可恢复                       |
| `/resume`          | 恢复历史会话                             |
| `/diff`            | 查看当前未提交改动                          |
| `/code-review`     | 审查当前 diff，可加 `--fix` 让 Claude 修复问题 |
| `/security-review` | 从安全角度审查当前分支改动                      |
| `/review`          | 本地审查 PR                            |
| `/rewind`          | 回退到某个检查点或总结部分历史                    |
| `/tasks`           | 查看后台任务                             |
| `/background`      | 把当前会话转为后台执行                        |
| `/usage`           | 查看用量、成本或额度                         |
| `/doctor`          | 诊断安装和运行环境                          |
| `/exit`            | 退出当前终端会话                           |

## 7. 后台任务和多会话管理

```bash
# 直接后台运行一个任务，命令会返回 session id
claude --bg "调查测试为什么偶发失败"

# 查看后台 agent
claude agents
claude agents --json

# 接回某个后台会话
claude attach <session-id>

# 查看后台会话日志
claude logs <session-id>

# 停止或移除后台会话
claude stop <session-id>
claude rm <session-id>
```

交互会话内可以用 `/background` 把当前会话放到后台，用 `/tasks` 查看当前会话里的后台任务。

## 8. 权限和安全

Claude Code 默认会自动读取项目文件；执行 shell 命令、修改文件等高影响操作通常会请求确认。常见权限模式：

| 模式 | 说明 |
| --- | --- |
| `default` | 标准模式，首次使用需要确认 |
| `acceptEdits` | 自动接受文件编辑和常见文件操作 |
| `plan` | 只探索和制定计划，不直接改源文件 |
| `auto` | 自动批准部分工具调用，并做后台安全检查 |
| `dontAsk` | 未预先允许的工具自动拒绝 |
| `bypassPermissions` | 跳过大多数权限提示，只建议在容器或虚拟机里用 |

启动时指定：

```bash
claude --permission-mode plan
claude --permission-mode acceptEdits
```

临时允许或禁止工具：

```bash
# 允许部分只读或低风险命令
claude --allowedTools "Bash(git status)" "Bash(git diff *)" "Read"

# 禁止危险命令
claude --disallowedTools "Bash(rm *)" "Bash(git push *)"
```

建议：

- 大改动前先 `git status`，保持能回滚。
- 普通本机环境不要随便用 `--dangerously-skip-permissions` 或 `bypassPermissions`。
- `.env`、SSH 私钥、云密钥等敏感文件建议在 `/permissions` 里加 deny 规则。
- 让 Claude 提交、推送、删除文件前，先用 `/diff` 检查变更。

## 9. 常见工作流

### 第一次接入项目

```text
claude
/init
解释这个项目的目录结构，指出入口文件、技术栈、运行方式
/permissions
```

### 修 bug 或加功能

```text
claude -n fix-login-bug
/plan 修复登录失败后白屏的问题
按照刚才的计划实现
/diff
/code-review --fix
运行项目已有测试并总结结果
```

### 只做代码审查

```text
claude
/diff
/code-review high
/security-review
```

### 长会话防止上下文太满

```text
/context
/compact 保留当前任务目标、已修改文件、待验证事项
```

### 脚本里调用

```bash
claude -p "总结当前 git diff 的主要风险" --output-format json
claude -p --max-turns 2 "只给出修复建议，不要编辑文件"
```

## 10. 更新、版本和卸载

```bash
# 原生安装可手动立即更新
claude update

# 安装或切换指定版本 / 通道
claude install latest
claude install stable
claude install 2.1.89

# Homebrew 更新
brew upgrade claude-code

# WinGet 更新
winget upgrade Anthropic.ClaudeCode

# npm 更新，不要用 sudo npm install -g
npm install -g @anthropic-ai/claude-code@latest
```

卸载按安装方式选择：

```bash
# npm
npm uninstall -g @anthropic-ai/claude-code

# Homebrew
brew uninstall --cask claude-code

# WinGet
winget uninstall Anthropic.ClaudeCode
```

Windows 原生安装删除配置示例：

```powershell
Remove-Item -Path "$env:USERPROFILE\.claude" -Recurse -Force
Remove-Item -Path "$env:USERPROFILE\.claude.json" -Force
```

Linux/macOS/WSL 删除配置示例：

```bash
rm -rf ~/.claude
rm -f ~/.claude.json
```
