# 1.介绍
docker就是应用容器化技术给应用程序封装独立的运行环境，每个运行环境就是一个容器
**核心概念**
镜像（Image） - 只读模板，包含应用运行所需的一切（代码、运行时、库、环境变量等）
容器（Container） - **镜像的运行实例**，可被启动、停止、删除
仓库（Repository） - 存储和分发镜像的地方，如Docker Hub [https://hub.docker.com](https://hub.docker.com)
Dockerfile - 定义如何构建镜像的脚本文件
**特点**
轻量级 - docker容器之间共享系统内核，区别于虚拟机
可移植性 - 在任意支持 Docker 的环境中运行
快速启动 - 容器启动时间通常只需几秒甚至毫秒级
版本控制 - 镜像支持类似 Git 的版本管理
# 2.安装
## Linux
Linux中下载docker，具体见：[https://get.docker.com/](https://get.docker.com/)
```shell
curl -fsSL https://get.docker.com -o install-docker.sh
sudo sh install-docker.sh
```
## Windows
Windows中安装docker
1.在启用或关闭Windows功能中设置
![[Pasted image 20260501193020.png|401]]
2.执行完重启电脑，然后更新 WSL：
```powershell
wsl --update --web-download
wsl --set-default-version 2
```
3.安装 Docker Desktop
**方式 1**：下载安装包：Docker Desktop 官网
双击 Docker Desktop Installer.exe 运行，按提示安装即可，勾选 Use WSL 2 instead of Hyper-V。
**方式 2**：命令行安装
先下载安装包到本地，然后在 PowerShell 管理员窗口执行：
```powershell
# 基础静默安装
Start-Process 'Docker Desktop Installer.exe' -Wait install

# 带参数安装（指定安装目录、快捷方式等）
# 例如：安装到 D 盘，不创建桌面快捷方式
Start-Process 'Docker Desktop Installer.exe' -Wait install --installation-dir "D:\Docker" --no-shortcut
```
CMD 里可以用：
```cmd
start /w "" "Docker Desktop Installer.exe" install
```

# 3.常见命令
```shell
#Linux命令行中加sudo，Windows中管理员运行
#作用：下载最新版的 Nginx 官方镜像。docker.io是Docker Hub 的官方域名，是默认的公共镜像仓库，library 是官方镜像的命名空间，冒号后面为版本号，不写时默认为latest。
#拉取镜像
docker pull docker.io/library/nginx:latest
docker pull docker:latest #简写
docker pull docker #最简写法

docker images              # 查看本地镜像
docker images -q           #只看ID

docker ps                  # 查看正在运行的容器
docker ps -a               #查看所有容器

docker rmi <ID或镜像名称>   #rmi=remove+images删除镜像，后面跟镜像名称或者ID

docker rm  <容器ID或名称>   #删除容器，区别于rmi。删除正在运行的容器要加参数-f，表示强制

docker run -d nginx        # 运行容器，-d分离模式，让容器在后台运行，不会阻塞当前窗口
docker run -p 80:80        #端口映射 -p 主机端口:容器端口
docker run -v 宿主机目录:容器内目录 #-v 挂载卷
docker run -e   变量名称=值 #往容器中传入环境变量
docker run --name 名称  #自定义命名
docker run -it --rm 镜像名称 #-it可以让控制台进入容器进行交互，--rm当容器停止时就把容器删除
docker run --restart always(unless-stopped) 镜像名称#配置容器在停止时的重启策略，手动停止的容器不会尝试重启

docker create #类似于docker run，但是只创建，不启动

 #创建一个自定义名称的卷。如果不指定名称，Docker 会自动生成一个随机名称
docker volume create my-vol
#查看当前系统中所有的数据卷列表
docker volume ls
#查看卷的详细信息，最关键的信息是 Mountpoint（在宿主机上的存储路径）
docker volume inspect my-vol
#删除指定的卷。注意：只有未被任何容器使用的卷才能被删除
docker volume rm my-vol
#一键删除所有未被任何容器引用的“悬空”卷，用于释放磁盘空间
docker volume prune -a

docker start <容器ID或名称>      #启动容器
docker stop <容器ID或名称>       # 停止容器

docker inspect <容器ID或名称>    #查看容器信息，查看的内容扔给ai分析就行了
docker logs <容器ID或名称> -f      #查看日志，-f等于--follow，表示追踪输出

docker build -t myapp .    # 构建镜像
```

Docker 容器（基于Linux 容器）的**两大底层核心技术**：Namespaces 和 Cgroups
- Namespaces（命名空间）：实现 “视图隔离”
- Cgroups（控制组）：实现 “资源管控”
```shell
#常用指令
docker exec <容器ID或名称> linux命令   #输入linux命令
docker exec -it <容器ID或名称> /bin/sh #进入容器内部执行Linux命令
```

**network相关命令**
```shell
docker network ls #查看网络
docker network inspect bridge #看详情
docker network create (host/none/bridge...) #创建网络
--network (host/none/bridge...) #容器指定网络
```

**容器编排技术**
- Docker Compose 单机编排，只能单台机器上编排多个容器
- 用 **docker-compose.yml** 一次性启动多个容器（nginx、数据库、项目一起起）
- 不能跨多台服务器
将想要执行的docker命令告诉ai，ai来生成docker compose文件
```shell
docker compose up -d #启动容器
docker compose stop #停止容器
docker compose start #启动容器
docker compose down #停止并删除容器
```
