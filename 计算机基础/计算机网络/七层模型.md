### **1. 物理层（Physical Layer）**
+ **功能**：负责在物理媒介上传输**原始比特流**（0和1），定义电气、机械、时序和接口规范。
+ **关键设备/协议**：网线（双绞线、光纤）、网卡（NIC）、中继器（Repeater）、集线器（Hub）、物理接口标准（如RJ45）。
+ **数据单位**：**比特（Bit）**
+ **关注点**：电压高低、光信号闪灭、引脚定义、传输速率、物理拓扑等。

### **2. 数据链路层（Data Link Layer）**
+ **功能**：在**直接相连**的节点间提供可靠的数据传输。负责**帧封装/解封**、**物理地址（MAC地址）寻址**、**差错控制**（校验码如CRC）、**流量控制**、**访问控制**（谁能在共享媒介上发送数据）。
+ **关键设备/协议**：交换机（Switch）、网桥（Bridge）。协议如以太网（Ethernet MAC子层）、PPP、HDLC。
+ **数据单位**：**帧（Frame）**
+ **关注点**：如何在共享媒介（如以太网）上避免冲突？如何检查数据是否损坏？

### **3. 网络层（Network Layer）**
+ **功能**：负责**逻辑地址（IP地址）寻址**，在**不同网络之间**为数据包选择**最佳路径（路由）**进行传输。实现互联网的主干逻辑。
+ **关键设备/协议**：路由器（Router）。协议如IP（IPv4/IPv6）、ICMP（ping）、IGMP、OSPF、BGP。
+ **数据单位**：**包/分组（Packet）**
+ **关注点**：这个包应该走哪条路才能从源IP到达目标IP？如何应对网络拥堵？

### **4. 传输层（Transport Layer）**
+ **功能**：提供**端到端（End-to-End）** 的数据传输服务，确保数据的**可靠性、顺序性、完整性**。负责**端口**寻址（标识进程）、**分段（发送端）** 与**重组（接收端）**、**流控**和**差错控制（如超时重传）**。
+ **关键协议**：TCP（Connection-oriented， 可靠。 如HTTP, FTP）、UDP（Connectionless， 不可靠但快。如DNS, 视频流）。
+ **数据单位**：**段（Segment）**（TCP） / **数据报（Datagram）**（UDP）
+ **关注点**：发送的数据有没有丢？顺序有没有乱？接收方处理能力跟得上吗？（TCP） / 够不够快？（UDP）

### **5. 会话层（Session Layer）**
+ **功能**：建立、管理、终止**会话（Session）** 。提供全双工、半双工、同步点（如在传输大文件时设置断点续传标记）等功能。
+ **关键概念/协议**：会话令牌管理、RPC（远程过程调用）、部分应用协议会管理自己的会话（如HTTP Cookie、TLS握手）。
+ **关注点**：通信双方的对话如何开始、维护（身份验证）、恢复和结束？

### **6. 表示层（Presentation Layer）**
+ **功能**：解决不同设备**数据表示差异**的问题。负责**数据转换、加密/解密、压缩/解压缩**。
+ **关键协议**：SSL/TLS（加密）、ASCII/Unicode转换、JPEG/MPEG压缩。
+ **关注点**：确保应用程序能“看懂”对方发送的数据（比如图片不会被当做乱码显示）。

### **7. 应用层（Application Layer）**
+ **功能**：为用户（应用程序）提供**网络访问接口**，实现具体的**应用功能**。
+ **关键协议**：HTTP（网页）、HTTPS（加密网页）、FTP（文件传输）、SMTP（发邮件）、POP3/IMAP（收邮件）、DNS（域名解析）、SSH（安全远程登录）、MQTT（物联网）等。
+ **关注点**：用户或程序需要完成什么样的网络任务（访问网页、发邮件等）

---

### **核心思想与总结**
+ **分层解耦**：每一层只需知道自己**如何与紧邻的上/下层交互**，而不需了解其他层的复杂细节（定义清晰的**服务和接口**），降低了网络设计的复杂性。
+ **封装与解封装**：发送数据时，从应用层开始，每一层在**上层数据前添加本层控制信息（头部Header，有的层也有尾部Trailer）**。接收数据时，从物理层开始，每一层**读取并剥离对等层的控制信息**，得到本层数据，然后向上层交付。
+ **标准化**：促进不同厂商设备/软件之间的互操作性。
+ **关键层次**：在现实网络（尤其是Internet）中，**传输层（TCP/UDP）** 和 **网络层（IP）** 是通信的绝对核心。
