## Quiet Outline 插件展开 Bug 排查与修复

**问题**：Quiet Outline 设置 `expand_level: 5`（默认展开到 h5），但部分文件打开时层级不展开。

**插件信息**：obsidian-quiet-outline v0.5.14，位于 `.obsidian/plugins/obsidian-quiet-outline/`

**排查过程**：
1. 读取 `data.json` 配置，发现关键三项：`expand_level:"5"`、`persist_md_states:true`、`auto_expand_ext:"disable"`
2. 发现 `markdown-states.json`（133KB）缓存了每个文件的 `expandedKeys` 数组，大部分文件是 `expandedKeys: []`（空数组=全折叠）
3. 扒 main.js 源码，定位核心判断逻辑：
   ```js
   n.settings.persist_md_states && r ? _(r) : h(m.value)
   // persist 开启 且 有存储记录 → 恢复存储状态；否则用 expand_level
   ```
4. 根因：存储逻辑兜底 `|| []`，而 JS 中空数组 `[]` 是 truthy，导致已打开过的文件永远走恢复分支，`expand_level` 被完全忽略

**修复**：将 `data.json` 中 `persist_md_states` 从 `true` 改为 `false`。代价是失去"记住每个文件展开状态"功能，但 `expand_level: 5` 可稳定生效。

**关键文件**：
- 配置：`.obsidian/plugins/obsidian-quiet-outline/data.json`
- 状态缓存：`.obsidian/plugins/obsidian-quiet-outline/markdown-states.json`
- 源码：`.obsidian/plugins/obsidian-quiet-outline/main.js`（压缩，618KB）

**教训**：上次凭直觉编了"时序竞争"理论，没看源码。这次真正读 main.js 才定位到 `[]` truthy 这个 JS 陷阱。排查插件 bug 必须读源码，别猜。
