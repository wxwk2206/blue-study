# 1.大模型部署

大模型部署主要有三种方式，各有优缺点：

---
#### 1.1 本地部署
- **优点**：数据安全、自主可控、长期成本低
- **缺点**：初始成本高、需长期维护、性能受限
#### 1.2 官方开放 API
- **优点**：前期成本低、无需部署和维护、随时访问
- **缺点**：隐私不能保障、长期成本高、可控性差
- **示例平台**：
    - 通义千问，文心一言，豆包
#### 1.3 云服务平台
- **优点**：前期成本低、无需部署和维护、选择度高
- **缺点**：安全及隐私不能保障、长期成本高
- **示例平台**：
    - 阿里云百炼，百度千帆，火山引擎
---

**API说明**

API：应用程序编程接口（**Application Programming Interface**），是软件间的标准化的 “桥梁”，允许开发者无需知晓内部细节即可调用外部功能或数据。
# 2.本地部署
 Ollama是一个在本地运行、管理大语言模型的工具。官网：[https://ollama.com/](https://ollama.com/)

 **Ollama 自定义安装完整步骤（Windows）**

将 Ollama 不安装在 C 盘的**命令行安装方法**和**模型存储路径配置**，核心是通过命令指定安装目录，避免占用系统盘空间。

 1. **命令行自定义安装**（避免默认装 C 盘）

- 前提：下载好 `OllamaSetup.exe`，不要双击运行。
- 操作：
    1. 打开 `OllamaSetup.exe` 所在文件夹，在地址栏输入 `cmd` 回车，打开命令行。
    2. 执行命令：
- ` OllamaSetup.exe /DIR=要安装的目录位置`
示例（安装到 D 盘）：
`OllamaSetup.exe /DIR=D:/develop/ollama`
- 作用：强制将 Ollama 主程序安装到指定路径，而非默认的 C 盘用户目录。
**** 2. 配置模型存储环境变量（分离模型文件）

安装完成后，需要配置环境变量，让 Ollama 把下载的大模型存到指定位置（避免 C 盘被占满）：

- 环境变量  ：`OLLAMA_MODELS=`D:/ollama/models
**最常用本地部署指令速查

```
# ollama 查看已安装模型
ollama list

# ollama 删除模型
ollama rm llama3.1

# ollama 手动拉模型
ollama pull llama3.1

# 以 API 方式运行
ollama serve

# 直接对话
ollama run llama3.1
```
