##  一、Spring描述
 为了解决企业应⽤开发的复杂性⽽创建的轻量级开源框架 

Spring⼜称为容器, 使⽤Spring开发可以将Bean对象, Dao组件对象, Service组件对象等交给Spring容器来管理 

这样使得很多复杂的代码在Spring中开发却变得⾮常的优雅和简洁, 有效的降低代码的耦合度, 极⼤的⽅ 便项⽬的后期维护、升级和扩展  

##  二、Spring作⽤ 
实现解耦合, 解决Java对象和模块之间的耦合  

##  三、Spring的特点  
###  1.IOC(控制反转)  (核心)
```plain
Inversion of Control简称IOC
降低了组件之间的耦合性, 实现了软件各个层之间的解耦
通过Spring提供的IOC容器,将对象之间的依赖关系交由Spring进⾏控制,避免硬编码所造成的程序耦合
有了Spring以后,就可以不⽤⾃⼰去⼿动实例化对象等操作和编写属性⽂件解析等这些很底层的需求代码
可以更专注于上层的应⽤
```

###  2.AOP(⾯向切⾯编程)  (核心)
```plain
Aspect Oriented Programming简称AOP
通过Spring提供的AOP功能,可以很⽅便的进⾏⾯向切⾯编程
许多不容易实现的功能,⽤传统⽅式OOP的⽅式实现⽐较困难,但是使⽤了AOP之后,可以轻松应付
```

###  3.声明式事务的⽀持  
```plain
在Spring中,我们可以从编程式事务代码中解脱出来
通过声明式的⽅式,灵活地进⾏事务的管理,提⾼开发效率和质量
编程式事务: 在业务代码中添加事务控制代码，这样的事务控制机制就叫做编程式事务
声明式事务: 通过xml或者注解配置的⽅式达到事务控制的⽬的，叫做声明式事务
```

### 4. ⽅便程序的测试  
```plain
可以⽤⾮容器依赖的编程⽅式进⾏⼏乎所有的测试⼯作
在Spring⾥, 测试不再是麻烦的操作,⽽是随⼿可做的事情
例如: 通过Spring使⽤junit,可以通过注解的⽅式优雅的测试Spring
```

###  5.⽅便集成各种优秀框架  
```plain
Spring不排斥各种优秀的开源框架
相反使⽤Spring可以降低各种框架的使⽤难度
Spring提供了对各种优秀框架的⽀持例如: Struts、Hibernate、Hessian、Quartz JPA Mybatis IBatis
```

###  6.Spring的⾼度开发性 
```plain
Spring并不强制应⽤完全依赖于Spring,开发者可以⾃由选择Spring的部分或者全部
```

###  7.降低 Java EE API 的使⽤难度  
```plain
Spring对很多难⽤的Java EE API(如:JDBC、JavaMail、远程调⽤等)
提供了⼀个薄薄的封装层,通过Spring的简易封装,这些Java EE API的使⽤难度⼤⼤降低
```

###  8.⾮侵⼊式  
```plain
基于Spring开发的应⽤中的对象可以不依赖于Spring的API
```

###  9.依赖注⼊  
```plain
Dependency Injection简称DI
是指依赖的对象不需要⼿动调⽤setXX⽅法去设置,⽽是通过配置赋值
```

###  10.容器  
```plain
Spring是⼀个容器,因为它包含并且管理应⽤对象的⽣命周期
```

####  10.1 组件化  
```plain
Spring实现了使⽤简单的组件配置组合成⼀个复杂的应⽤
在Spring中可以使⽤和Java注解组合这些对象
```

####  10.2 ⼀ 站式  
```plain
在IOC和AOP的基础上可以整合各种企业应⽤的开源框架和优秀的第三⽅类库
(实际上Spring⾃身也提供了表述层的SpringMVC和持久层的Spring JDBC)
```

## 六、核心特性代码示例
### 1. IOC/DI（控制反转 / 依赖注入）
#### （1）组件类（Bean）
```java
// 1. 实体类
public class User {
    private Integer id;
    private String name;
    // getter/setter/toString
}

// 2. Dao 层（数据访问层）：@Repository 标记为数据访问Bean
@Repository
public class UserDao {
    // 模拟数据库查询
    public User getUserById(Integer id) {
        User user = new User();
        user.setId(id);
        user.setName("Spring 示例用户");
        return user;
    }
}

// 3. Service 层（业务逻辑层）：@Service 标记为业务Bean，@Autowired 注入依赖
@Service
public class UserService {
    // 依赖注入 UserDao（无需手动 new）
    @Autowired
    private UserDao userDao;

    public User getUser(Integer id) {
        return userDao.getUserById(id);
    }
}
```

#### （2）Spring 配置类（替代 XML）
```java
// @Configuration 标记为配置类，@ComponentScan 扫描指定包下的Bean
@Configuration
@ComponentScan("com.spring.demo") // 替换为你的包路径
public class SpringConfig {
}
```

#### （3）测试类（验证 IOC 容器）
```java
@RunWith(SpringJUnit4ClassRunner.class)
@ContextConfiguration(classes = SpringConfig.class) // 加载配置类
public class IocTest {
    @Autowired
    private UserService userService;

    @Test
    public void testIoc() {
        User user = userService.getUser(1);
        System.out.println(user); // 输出：User{id=1, name='Spring 示例用户'}
    }
}
```

### 2. AOP（面向切面编程）
#### （1）切面类（日志切面示例）
```java
// @Aspect 标记为切面类，@Component 纳入IOC容器
@Aspect
@Component
public class LogAspect {
    // 切入点：匹配 com.spring.demo.service 包下所有方法
    @Pointcut("execution(* com.spring.demo.service.*.*(..))")
    public void servicePointcut() {}

    // 前置通知：方法执行前执行
    @Before("servicePointcut()")
    public void beforeLog(JoinPoint joinPoint) {
        String methodName = joinPoint.getSignature().getName();
        System.out.println("【前置日志】执行方法：" + methodName);
    }

    // 后置通知：方法执行后执行（无论是否异常）
    @After("servicePointcut()")
    public void afterLog(JoinPoint joinPoint) {
        String methodName = joinPoint.getSignature().getName();
        System.out.println("【后置日志】方法执行完毕：" + methodName);
    }
}
```

#### （2）测试 AOP 效果
运行上面的 `IocTest.testIoc()`，输出：

```plain
【前置日志】执行方法：getUser
【后置日志】方法执行完毕：getUser
User{id=1, name='Spring 示例用户'}
```

### 3. 声明式事务
#### （1）配置事务管理器（SpringConfig 中添加）
```java
@Configuration
@ComponentScan("com.spring.demo")
@EnableTransactionManagement // 开启声明式事务
public class SpringConfig {
    // 模拟数据源（实际开发用 Druid/C3P0 等）
    @Bean
    public DataSource dataSource() {
        DriverManagerDataSource dataSource = new DriverManagerDataSource();
        dataSource.setDriverClassName("com.mysql.cj.jdbc.Driver");
        dataSource.setUrl("jdbc:mysql://localhost:3306/test?useSSL=false");
        dataSource.setUsername("root");
        dataSource.setPassword("123456");
        return dataSource;
    }

    // 事务管理器
    @Bean
    public PlatformTransactionManager transactionManager(DataSource dataSource) {
        return new DataSourceTransactionManager(dataSource);
    }
}
```

#### （2）Service 层添加事务注解
```java
@Service
public class UserService {
    @Autowired
    private UserDao userDao;

    // @Transactional 标记该方法需要事务（异常时回滚）
    @Transactional
    public void addUser(User user) {
        // 模拟数据库插入（实际用 MyBatis/JPA）
        System.out.println("添加用户：" + user);
        // 模拟异常（触发事务回滚）
        if (user.getId() == 0) {
            throw new RuntimeException("用户ID不能为0");
        }
    }
}
```

#### （3）测试事务
```java
@Test
public void testTransaction() {
    try {
        User user = new User();
        user.setId(0); // 触发异常
        user.setName("测试事务");
        userService.addUser(user);
    } catch (Exception e) {
        System.out.println("事务回滚：" + e.getMessage());
    }
}
```

### 4. Web 模块（Spring MVC）
#### （1）Spring MVC 配置类
```java
@Configuration
@EnableWebMvc // 开启 Spring MVC 功能
@ComponentScan("com.spring.demo.controller") // 扫描控制器
public class MvcConfig implements WebMvcConfigurer {
    // 配置视图解析器（跳转 JSP 示例）
    @Bean
    public InternalResourceViewResolver viewResolver() {
        InternalResourceViewResolver resolver = new InternalResourceViewResolver();
        resolver.setPrefix("/WEB-INF/views/"); // JSP 路径前缀
        resolver.setSuffix(".jsp"); // 后缀
        return resolver;
    }
}
```

#### （2）控制器类
```java
// @Controller 标记为控制器，@RequestMapping 映射请求路径
@Controller
@RequestMapping("/user")
public class UserController {
    @Autowired
    private UserService userService;

    // 接收 GET 请求：/user/get/{id}
    @GetMapping("/get/{id}")
    public String getUser(@PathVariable Integer id, Model model) {
        User user = userService.getUser(id);
        model.addAttribute("user", user); // 传递数据到视图
        return "userDetail"; // 跳转到 WEB-INF/views/userDetail.jsp
    }

    // 接收 POST 请求：/user/add
    @PostMapping("/add")
    @ResponseBody // 返回 JSON（无需跳转视图）
    public String addUser(@RequestBody User user) {
        userService.addUser(user);
        return "添加成功";
    }
}
```

#### （3）Web 容器初始化（替代 web.xml）
```java
public class WebAppInitializer implements WebApplicationInitializer {
    @Override
    public void onStartup(ServletContext servletContext) {
        // 加载 Spring 配置
        AnnotationConfigWebApplicationContext context = new AnnotationConfigWebApplicationContext();
        context.register(SpringConfig.class, MvcConfig.class);

        // 注册 DispatcherServlet（Spring MVC 核心）
        ServletRegistration.Dynamic dispatcher = servletContext.addServlet(
            "dispatcher", new DispatcherServlet(context)
        );
        dispatcher.setLoadOnStartup(1);
        dispatcher.addMapping("/"); // 拦截所有请求
    }
}
```

## 七、关键说明
1. **IOC/DI**：通过 `@Component` 及其衍生注解（`@Repository`/`@Service`/`@Controller`）将类交给 Spring 管理，`@Autowired` 自动注入依赖，无需手动 `new` 对象。
2. **AOP**：通过 `@Aspect` 定义切面，`@Pointcut` 定义切入点，`@Before`/`@After` 等注解定义通知，无需修改业务代码即可增强功能（如日志、权限）。
3. **声明式事务**：`@EnableTransactionManagement` 开启事务，`@Transactional` 标记需要事务的方法，Spring 自动管理事务的提交 / 回滚。
4. **Spring MVC**：`@Controller` 处理请求，`@RequestMapping` 映射路径，`Model` 传递数据到视图，`@ResponseBody` 返回 JSON。
