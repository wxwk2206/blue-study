## 一、环境搭建
若未搭建 JSP 环境，需先完成环境配置。

前置操作（必须执行）：

1. 对准 `webapp` 目录右键；
2. 将 `webapp` 目录标记为 “源码根”。

## 二、Filter 概述
### 1. 作用
Filter 又称过滤器，配置在 MVC、Servlet 和 JSP 请求之前，主要用于：

+ 后端权限控制；
+ 统一的 Http 请求参数过滤；
+ 统一处理 XSS、SQL 注入、SSRF 等攻击检测。

### 2. 核心接口与生命周期
+ 核心接口：`javax.servlet.Filter`，需实现 3 个核心方法；
+ 生命周期：
    1. 初始化（`init` 方法）：Web 应用启动时创建 Filter 实例，执行一次，用于读取配置、初始化资源；
    2. 过滤（`doFilter` 方法）：客户端请求匹配拦截规则时触发，执行实际过滤逻辑；
    3. 销毁（`destroy` 方法）：Filter 实例销毁前调用，用于释放资源。

## 三、Filter 配置方式
Filter 有两种配置方式：注解配置（`@WebFilter`）和 `web.xml` 配置，核心配置项一致。

### 1. 核心配置项说明

| **配置项** | **作用** | **可选值 / 示例** |
| --- | --- | --- |
| `filterName` | 过滤器名称（建议与类名一致） | `GlobalFilter`<br/>、`HelloFilter` |
| `urlPatterns` | 配置拦截的资源路径 | 1. 指定资源：`/hello.jsp`；2. 目录匹配：`/Servlet/*`；3. 后缀匹配：`*.jsp`；<br/>4. 通配符（拦截所有）：`/*` |
| `initParams` | 初始化参数（键值对） | 可配置多个，如 <br/>`origin=@WebFilter`、<br/>`testName=testValue` |
| `dispatcherTypes` | 拦截类型（枚举） | 默认 `REQUEST`（请求），<br/>可选 `FORWARD`（转发）、<br/>`INCLUDE`（页面包含）、<br/>`ASYNC`（异步）、<br/>`ERROR`（出错） |
| `asyncSupported` | 是否支持异步请求 | 默认 `false` |


### 2. 注解配置（`@WebFilter`）
直接在 Filter 类上通过注解配置，无需 `web.xml` 映射。

#### 2.1常用配置项
```java
//常⽤配置项
//参数: urlPatterns
//作⽤: 配置要拦截的资源以指定资源匹配,例如:"/hello.jsp"
//以⽬录匹配,例如:"/Servlet/*"
//以后缀名匹配,例如:"*.jsp"
//通配符,拦截所有web资源,例如:"/*"
//例⼦:
@WebFilter(
// 过滤器的名字
// ⼀般来说保持和类名⼀致即可
filterName = "/GlobalFilter",
// 拦截所有web资源
urlPatterns = "/*")
public class GlobalFilter implements Filter {
```

```java
//参数: initParams
//作⽤: 配置初始化参数
//例⼦:
@WebFilter(
// 过滤器的名字
// ⼀般来说保持和类名⼀致即可
filterName = "/GlobalFilter",
// 初始化的参数,这⾥初始化了两个
initParams = {
@WebInitParam(name = "origin", value = "@WebFilter"),

})

```

```java

//作⽤: 配置拦截的类型，可配置多个
//默认为DispatcherType.REQUEST
//其中DispatcherType是个枚举类型，有下⾯⼏个值
//FORWARD //转发的
//INCLUDE //包含在⻚⾯的
//REQUEST //请求的
//ASYNC //异步的
//ERROR //出错的
//例⼦:
@WebFilter(
filterName = "/GlobalFilter",
// 设置拦截
dispatcherTypes = {
DispatcherType.REQUEST,
DispatcherType.ASYNC,
DispatcherType.ERROR
})
public class GlobalFilter implements Filter {
```

#### 2.2 完整示例
```java
package com.Filter;

import javax.servlet.*;
import javax.servlet.annotation.WebFilter;
import javax.servlet.annotation.WebInitParam;
import java.io.IOException;

// 注解配置：名称、拦截所有资源、初始化参数、多拦截类型
@WebFilter(
    filterName = "GlobalFilter",
    urlPatterns = "/*",
    initParams = {
        @WebInitParam(name = "origin", value = "@WebFilter"),
        @WebInitParam(name = "testName", value = "testValue")
    },
    dispatcherTypes = {
        DispatcherType.REQUEST,
        DispatcherType.INCLUDE,
        DispatcherType.FORWARD,
        DispatcherType.ERROR,
        DispatcherType.ASYNC
    }
)
public class GlobalFilter implements Filter {

    // 初始化方法：Web应用启动时执行一次
    @Override
    public void init(FilterConfig config) throws ServletException {
        // 获取初始化参数
        String origin = config.getInitParameter("origin");
        String testName = config.getInitParameter("testName");
        System.out.println("GlobalFilter类起源: " + origin);
        System.out.println("GlobalFilter类testName值: " + testName);
    }

    // 过滤核心方法：请求匹配时执行
    @Override
    public void doFilter(ServletRequest request, ServletResponse response, FilterChain chain) throws IOException, ServletException {
        response.setCharacterEncoding("GBK");
        response.getWriter().println("我是全局过滤器: GlobalFilter类");
        chain.doFilter(request, response); // 放行：允许访问目标资源
    }

    // 销毁方法：Filter实例销毁前执行
    @Override
    public void destroy() {
        System.out.println("GlobalFilter类,被摧毁了");
    }
}
```

### 3. `web.xml` 配置
通过 `web.xml` 文件配置 Filter 映射关系，适合需要精确控制执行顺序的场景。

```xml
//设置Filter
//作⽤: 定义这个Filter等待其它程序调⽤
//例如:
<filter>
  <!-- 类名 -->
  <filter-name>HelloFilter</filter-name>
  <!-- 所在的包路径 -->
  <filter-class>com.Filter.HelloFilter</filter-class>
</filter>
//例⼦:
<?xml version="1.0" encoding="UTF-8"?>
<web-app version="2.5"
  xmlns="http://java.sun.com/xml/ns/javaee"
  xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
  xsi:schemaLocation="http://java.sun.com/xml/ns/javaee http://java.sun.com/xml/ns/javaee/web-app_2_5.xsd">
  <filter>
    <!-- 类名 -->
    <filter-name>HelloFilter</filter-name>
    <!-- 所在的包路径 -->
    <filter-class>com.Filter.HelloFilter</filter-class>
  </filter>
</web-app>
```

#### 3.2 完整示例
1. Filter 类（`HelloFilter.java`）：

```java
//在这⽬录: src->main->webapp->com->Filter
//创建⼀个⽂件: HelloFilter.java
package com.Filter;

import javax.servlet.*;
import java.io.IOException;

public class HelloFilter implements Filter {

    @Override
    public void init(FilterConfig config) throws ServletException {
        String origin = config.getInitParameter("origin");
        String testName = config.getInitParameter("testName");
        System.out.println("HelloFilter类起源: " + origin);
        System.out.println("HelloFilter类testName值: " + testName);
    }

    @Override
    public void doFilter(ServletRequest request, ServletResponse response, FilterChain chain) throws IOException, ServletException {
        response.setCharacterEncoding("GBK");
        response.getWriter().println("站点网址:http://~.com");
        chain.doFilter(request, response); // 放行
    }

    @Override
    public void destroy() {
        System.out.println("HelloFilter类,被摧毁了");
    }
}
```

2. `web.xml` 配置：

```xml
//通过在web.xml也可以很⽅便的完成配置映射
//打开WEB-INF⽬录下的web.xml 修改为如下样⼦:
<?xml version="1.0" encoding="UTF-8"?>
<web-app version="2.5"
  xmlns="http://java.sun.com/xml/ns/javaee"
  xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
  xsi:schemaLocation="http://java.sun.com/xml/ns/javaee http://java.sun.com/xml/ns/javaee/web-app_2_5.xsd">

  <!-- 1. 定义Filter -->
  <filter>
    <filter-name>HelloFilter</filter-name> <!-- 与filter-mapping中名称一致 -->
    <filter-class>com.Filter.HelloFilter</filter-class> <!-- 类的全路径 -->
    <!-- 初始化参数 -->
    <init-param>
      <param-name>origin</param-name>
      <param-value>web.xml</param-value>
    </init-param>
    <init-param>
      <param-name>testName</param-name>
      <param-value>testValue</param-value>
    </init-param>
  </filter>

  <!-- 2. 配置Filter映射 -->
  <filter-mapping>
    <filter-name>HelloFilter</filter-name>
    <url-pattern>/TomcatTest/*</url-pattern> <!-- 拦截/TomcatTest目录下所有资源 -->
    <!-- 拦截类型 -->
    <dispatcher>REQUEST</dispatcher>
    <dispatcher>FORWARD</dispatcher>
    <dispatcher>INCLUDE</dispatcher>
    <dispatcher>ERROR</dispatcher>
  </filter-mapping>

  <!-- 可选：配置Servlet（用于测试） -->
  <servlet>
    <servlet-name>HelloServlet</servlet-name>
    <servlet-class>com.Servlet.HelloServlet</servlet-class>
  </servlet>
  <servlet-mapping>
    <servlet-name>HelloServlet</servlet-name>
    <url-pattern>/TomcatTest/HelloServlet</url-pattern> <!-- 访问路径 -->
  </servlet-mapping>
</web-app>
```

#### 3.3 注意事项
+ `filter-name` 必须与 `filter-mapping` 中的名称一致；
+ `url-pattern` 为 `/TomcatTest/*` 时，仅拦截 URL 中包含 `/TomcatTest/` 的请求（如 `http://127.0.0.1:8081/mavenJspTest_war/TomcatTest/HelloServlet`），不包含该路径的请求不拦截。

## 四、多个 Filter 的执行顺序
1. 注解配置的 Filter：执行顺序由类名的字母顺序决定（如 `AFilter` 比 `BFilter` 先执行）；
2. `web.xml` 配置的 Filter：执行顺序由 `<filter-mapping>` 标签的声明顺序决定（先声明先执行）；
3. 优先级：`web.xml` 配置的 Filter 优先级高于注解配置的 Filter（先执行）；
4. 建议：若需确保执行顺序，优先使用 `web.xml` 配置。
