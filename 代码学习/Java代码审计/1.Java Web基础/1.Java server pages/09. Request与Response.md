## 一、核心概念
在 JSP 开发中，**请求（Request）** 与**响应（Response）** 是核心交互机制，Java 通过两个封装类实现：

+ **HttpServletRequest**：代表客户端的 HTTP 请求，封装了请求头、参数、客户端信息等所有请求相关数据，通过其提供的方法可获取请求详情。
+ **HttpServletResponse**：用于服务端向客户端返回响应，可设置响应内容、状态码、响应头、Cookie 等，实现对客户端请求的反馈。

二者常作为doGet()、doPost()等方法的参数直接使用，是前后端交互的核心载体。

## 二、HttpServletRequest 常用方法（请求对象）
### 1. 参数获取

| **方法** | **说明** |
| --- | --- |
| getParameterNames()` | 获取所有请求参数名 |
| getParameter(String name)` | 通过参数名获取单个值（重复参数返回第一个） |
| getParameterValues(String name)` | 通过参数名获取所有值（返回 String 数组） |


### 2. 请求路径与 URL 相关

| **方法** | **说明** |
| --- | --- |
| getRequestURL().toString()` | 获取完整请求 URL（含协议、域名、端口、路径） |
| getRequestURI()` | 获取客户端请求的字符串地址（不含协议、域名、端口） |
| getContextPath()` | 返回项目名称（根目录项目返回空字符串） |
| getServletPath()` | 返回客户端请求的脚本文件路径 |
| getPathInfo()` | 返回传递到 Servlet 的额外路径（无则返回 NULL） |
| getQueryString()` | 获取 URL 中附带的查询参数（如?id=1&name=test） |


### 3. 客户端与服务器信息

| **方法** | **说明** |
| --- | --- |
| getMethod()` | 获取请求方式（GET/POST） |
| getRemoteAddr()` | 获取客户端 IP 地址 |
| getRemoteHost()` | 获取客户端主机名 |
| getRemotePort()` | 获取客户端使用的网络端口号 |
| getLocalAddr()` | 获取 WEB 服务器 IP 地址 |
| getLocalName()` | 获取 WEB 服务器主机名 |


### 4. 属性操作（请求域数据共享）

| **方法**                                  | **说明**                          |
| --------------------------------------- | ------------------------------- |
| setAttribute(String name, Object value) | 向 request 对象添加属性（键值对）           |
| getAttribute(String name)               | 通过名称获取 request 对象的属性值           |
| getAttributeNames()                     | 获取所有属性名（返回 Enumeration<String>） |
| removeAttribute(String name)            | 移除 request 对象的指定属性              |


### 5. 其他常用方法

| **方法**                  | **说明**                                |
| ----------------------- | ------------------------------------- |
| getCookies()            | 获取客户端所有 Cookie（返回 Cookie 数组）          |
| getSession()            | 获取 Session 会话对象                       |
| getHeaderNames()        | 获取所有请求头名称（返回 Enumeration<String>）     |
| getHeader(String name)  | 获取指定名称的请求头值                           |
| getHeaders(String name) | 获取指定名称的所有请求头值（返回 Enumeration<String>） |
| getContentLength()      | 获取请求体（Body）的长度                        |
| getCharacterEncoding()  | 获取请求的字符编码方式                           |
| getInputStream()        | 获取请求的输入流对象                            |


## 三、HttpServletResponse 常用方法（响应对象）
### 1. 响应基础配置
| **方法**                                | **说明**                                           |
| ------------------------------------- | ------------------------------------------------ |
| setContentType(String type)           | 设置响应的 MIME 类型（如text/html`<br/>、application/json） |
| setCharacterEncoding(String encoding) | 设置响应的字符编码（如UTF-8）                                |
| getWriter()                           | 获取响应打印流对象（用于输出字符内容）                              |
| getOutputStream()                     | 获取响应流对象（用于输出二进制内容）                               |


### 2. Cookie 与响应头操作
| **方法**                                | **说明**                           |
| ------------------------------------- | -------------------------------- |
| addCookie(Cookie cookie)              | 向响应中添加 Cookie（同名 Cookie 会新增，不覆盖） |
| addHeader(String name, String value)  | 向响应头添加键值对（同名会新增，不覆盖）             |
| setHeader(String name, String value)  | 设置响应头键值对（同名会覆盖原有值）               |
| addDateHeader(String name, long date) | 添加日期类型响应头（同名会新增）                 |
| setDateHeader(String name, long date) | 设置日期类型响应头（同名会覆盖）                 |


### 3. 响应状态与跳转
| **方法**                                    | **说明**                                                                     |
| ----------------------------------------- | -------------------------------------------------------------------------- |
| setStatus(int statusCode)                 | 设置响应状态码（如 302 重定向、304 缓存控制、200 成功）                                         |
| sendError(int statusCode, String message) | 发送错误响应（含状态码和错误信息）                                                          |
| sendRedirect(String url)                  | 实现请求重定向（如sendRedirect("index.jsp")`<br/>或sendRedirect("http://baidu.com")） |


## 四、实战小例子
### 1. 项目目录结构
```xml
⽬录结构
├── src
│   └── main
│     └── webapp
│       └── com
│         └── Servlet
│           ├── ...
│           └── TestServlet2.java
│       └── WEB-INF
│         └── web.xml
│       └── index.jsp
```

### 2. 核心代码（TestServlet2.java）
```java
package com.Servlet;

import javax.servlet.annotation.WebServlet;
import javax.servlet.http.HttpServlet;
import javax.servlet.http.HttpServletRequest;
import javax.servlet.http.HttpServletResponse;
import java.io.IOException;

@WebServlet("/TestServlet2")
public class TestServlet2 extends HttpServlet {
    public TestServlet2() {
        super();
    }

    @Override
    protected void doGet(HttpServletRequest request, HttpServletResponse response) throws IOException {
        // 1. 获取请求参数id
        String id = request.getParameter("id");
        // 2. 向响应输出内容
        response.getWriter().write("id: " + id);
    }
}
```

### 3. 运行与结果
+ 访问 URL：http://127.0.0.1:8081/mavenJspTest_war/TestServlet2?id=1`
+ 响应结果：id: 1`
+ 核心逻辑：通过request.getParameter()获取请求参数，通过response.getWriter().write()输出响应内容。

## 五、核心小结
1. 重点掌握HttpServletRequest的**参数获取**、**路径信息**、**属性操作**方法，以及HttpServletResponse的**响应配置**、**状态设置**、**重定向**方法。
2. 这两个对象是前后端交互的核心，前端可控数据（如请求参数、请求头）需通过HttpServletRequest获取，服务端反馈（如响应内容、Cookie）需通过HttpServletResponse设置。
3. 代码审计场景中，需重点关注前端可控的请求参数、请求头，以及响应中的敏感数据泄露风险。
