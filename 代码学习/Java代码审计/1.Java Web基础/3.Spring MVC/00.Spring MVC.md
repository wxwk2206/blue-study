## 一、什么是 Spring MVC？
**Spring MVC** 是 Spring 框架的核心模块之一，是基于 MVC 设计模式的轻量级 Java Web 开发框架，用于快速构建灵活、可扩展的 Web 应用程序。

+ 本质：Spring MVC 并非独立框架，而是 Spring 框架对 Web 层的解决方案，与 Spring 容器无缝集成（无需额外配置整合，天然兼容）。
+ 核心目标：分离 Web 应用的**数据模型（Model）、视图（View）、控制器（Controller）** ，降低模块间耦合度，提高代码可维护性和复用性。
+ 核心优势：
1. 基于 Spring 生态，支持依赖注入（DI）、AOP 等核心特性；
2. 灵活的 URL 映射、数据绑定、表单验证机制；
3. 支持多种视图技术（JSP、Thymeleaf、FreeMarker 等）；
4. 内置拦截器、异常处理器等常用功能，简化开发。

## 二、MVC 的组成部分（核心组件）
MVC（Model-View-Controller）是一种软件架构模式，Spring MVC 严格遵循该模式，核心组件分为三大类，配合其他辅助组件完成请求处理：

**1. 模型（Model）：数据与业务逻辑载体**

+ 作用：封装应用的核心数据和业务逻辑，是视图展示的数据来源，也是控制器处理请求的业务支撑。
+ 组成：
+ 业务模型（Service 层）：处理核心业务逻辑（如数据查询、计算、事务管理）；
+ 数据模型（Entity/POJO/DTO）：封装数据（如用户信息 User、订单信息 Order），用于在组件间传递数据。

**2. 视图（View）：数据展示层**

+ 作用：将 Model 中的数据以用户可见的形式渲染展示（如网页、JSON 数据、XML 等）。
+ 核心特点：仅负责数据展示，不包含业务逻辑，通过模型数据动态生成页面。
+ 常用视图技术：Thymeleaf（Spring 官方推荐）、JSP、FreeMarker、Velocity 等。

**3. 控制器（Controller）：请求处理与调度中心**

+ 作用：接收客户端（浏览器 / APP）的 HTTP 请求，协调 Model 和 View 完成处理流程，是 MVC 的核心调度者。
+ 核心职责：
1. 接收请求：通过 URL 映射（@RequestMapping/@GetMapping 等注解）绑定请求路径；
2. 处理请求：调用 Model（Service 层）的业务逻辑，获取处理结果；
3. 响应结果：将 Model 数据传递给 View，或直接返回 JSON/XML 等数据（无视图场景）。
+ 关键注解：@Controller（标识控制器）、@RestController（组合 @Controller + @ResponseBody，直接返回数据）。

**4. 辅助组件（支撑 MVC 运行）**

+ DispatcherServlet（前端控制器）：Spring MVC 的核心入口，统一接收所有请求，负责请求分发（转发给对应 Controller）、视图渲染等，是 “中央处理器”；
+ HandlerMapping（处理器映射器）：根据请求 URL 查找对应的 Controller 方法；
+ HandlerAdapter（处理器适配器）：调用 Controller 方法，处理参数绑定、数据转换等；
+ ViewResolver（视图解析器）：根据 Controller 返回的视图名称，查找对应的物理视图（如 Thymeleaf 模板文件路径）。
