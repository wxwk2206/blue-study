## 一、核心简介
1. Spring MVC 中定义控制器无需继承特定类或实现接口，核心依赖两个注解：
    - `@Controller`：标记类为控制器，使其能被 Spring MVC 识别
    - `@RequestMapping`：建立请求与控制器方法的映射关系
2. 工作流程：Spring MVC 接收请求后，通过映射关系找到对应控制器方法处理请求，使控制器可被外界访问

## 二、@Controller 注解
### （一）功能
标记在类上，将该类声明为 Spring MVC 的 Controller 类

### （二）使用要求
需在 spring-mvc 配置文件（如 springmvc.xml）中添加组件扫描配置：

1. 引入 context 命名空间：`xmlns:context="http://www.springframework.org/schema/context"`
2. 配置扫描路径：`<context:component-scan base-package="控制器目录路径（例：test.controller）"/>`

## 三、@RequestMapping 注解
### （一）基本说明
1. 必须与`@Controller`配合使用，用于处理请求地址映射
2. 可标注位置：
    - 类上：设置请求路径的初始信息（相当于 WEB 应用根目录）
    - 方法上：设置请求路径的具体信息（相当于类定义处 URL 的延伸）
3. 示例：
    - 类上标注`@RequestMapping("/user")` + 方法上标注`@RequestMapping("/hello")` → 访问 URL：`http://xxx.com/user/hello`
    - 仅方法上标注`@RequestMapping("/hello2")` → 访问 URL：`http://xxx.com/hello2`

### （二）核心属性总览
| **属性名**  | **功能描述**                            |
| -------- | ----------------------------------- |
| value    | 通过请求地址匹配映射（必填）                      |
| method   | 通过请求方式（GET/POST 等）匹配映射              |
| params   | 通过请求参数匹配映射                          |
| headers  | 通过请求头信息匹配映射                         |
| consumes | 通过请求头的 Content-Type 信息匹配映射          |
| produces | 设置返回数据的类型及编码（需与 @ResponseBody 配合使用） |


注意：属性越多，匹配条件越精确，需所有条件同时满足才会触发映射

### （三）属性详细解析
#### 1. value 属性
+ 功能：通过请求地址匹配映射，支持多个地址
+ 类型：字符串数组，至少配置一个地址
+ 示例：	

```plain
@RequestMapping(value = {"/hello", "/hello2"}) // 匹配/hello和/hello2两个地址
@ResponseBody
public String sayHello() { return "test1234"; }
```

+ 访问方式：`http://xxx.com/hello` 或 `http://xxx.com/hello2`

#### 2. value 属性（Ant 风格路径）
支持通配符实现灵活路径匹配：

| **通配符** | **含义**        | **注意事项**             |
| ------- | ------------- | -------------------- |
| ?       | 匹配任意单个字符      | 仅允许单个字符              |
| *       | 匹配任意 0 个或多个字符 | 可多个字符，不跨目录           |
| **      | 匹配任意一层或多层目录   | 仅能使用`/**/xxx`<br/>格式 |


+ 示例及访问方式：
    - `@RequestMapping("/hello?")` → 匹配`/helloa`、`/hellob`等
    - `@RequestMapping("/a*a/hello3")` → 匹配`/aa/hello3`、`/aba/hello3`等
    - `@RequestMapping("/**/hello4")` → 匹配`/hello4`、`/x/hello4`、`/x/y/hello4`等
    - `@RequestMapping("/test/**")` → 匹配`/test`、`/test/a`、`/test/a/b`等

#### 3. method 属性
+ 功能：限制请求方式，支持多种方式
+ 类型：RequestMethod 数组（可选值：GET、HEAD、POST、PUT、PATCH、DELETE、OPTIONS、TRACE）
+ 派生注解（简化写法）：
    - `@GetMapping`：等价于`method = RequestMethod.GET`
    - `@PostMapping`：等价于`method = RequestMethod.POST`
    - 同理：`@PutMapping`、`@PatchMapping`、`@DeleteMapping`
+ 示例：

```plain
@RequestMapping(value = "/hello", method = {RequestMethod.GET, RequestMethod.POST})
@ResponseBody
public String sayHello() { return "test1234"; }
```

+ 访问结果：GET/POST 请求正常，DELETE 请求返回 405（方法不允许）

#### 4. params 属性
+ 功能：通过请求参数匹配，支持四种表达式：
    - `{"param"}`：必须携带 param 参数
    - `{"!param"}`：必须不携带 param 参数
    - `{"param=value"}`：必须携带 param 参数且值为 value
    - `{"param!=value"}`：必须携带 param 参数且值不为 value
+ 示例：

```plain
@RequestMapping(value = "/hello", params = {"t1", "!t2", "t3=123456", "t4!=qwerty"})
@ResponseBody
public String sayHello() { return "test1234"; }
```

+ 访问结果：不满足参数条件时返回 400（错误的请求）

#### 5. headers 属性
+ 功能：通过请求头信息匹配，表达式格式同 params：
    - `{"header"}`：必须携带 header 头
    - `{"!header"}`：必须不携带 header 头
    - `{"header=value"}`：必须携带 header 头且值为 value
    - `{"header!=value"}`：必须携带 header 头且值不为 value
+ 示例：

```plain
@RequestMapping(value = "/hello", headers = {"h1", "!h2", "h3=123456", "h4!=qwerty"})
@ResponseBody
public String sayHello() { return "test1234"; }
```

+ 访问结果：不满足头信息条件时返回 404（未找到）

#### 6. consumes 属性
+ 功能：匹配请求头的 Content-Type 信息
+ 支持格式：
    - 精确匹配：`consumes = "text/plain"`（仅匹配 text/plain 类型）
    - 排除匹配：`consumes = {"!text/plain"}`（排除 text/plain 类型）
    - 通配匹配：`consumes = "application/*"`（匹配 application / 开头的所有类型）
+ 示例：

```plain
@RequestMapping(value = "/hello", consumes = {"text/plain", "application/*"})
@ResponseBody
public String sayHello() { return "test1234"; }
```

+ 访问结果：不满足 Content-Type 时返回 415（不支持的媒体类型）

#### 7. produces 属性
+ 功能：设置返回数据的类型及编码，需与`@ResponseBody`配合使用
+ 常用写法：
    - `produces = "text/html;charset=UTF-8"`（返回 HTML 格式，UTF-8 编码）
    - `produces = "application/json;charset=UTF-8"`（返回 JSON 格式，UTF-8 编码）
    - `produces = "application/xml;charset=UTF-8"`（返回 XML 格式，UTF-8 编码）
+ 示例：

```plain
@RequestMapping(value = "/hello", produces = "text/html;charset=UTF-8")
@ResponseBody
public String sayHello() { return "这是一段中文"; }
```

## 四、@RequestMapping 派生注解
为常用请求方式提供简化注解，无需显式指定 method 属性：

| **派生注解** | **对应请求方式** | **等价写法** |
| --- | --- | --- |
| @GetMapping | GET | @RequestMapping(method = GET) |
| @PostMapping | POST | @RequestMapping(method = POST) |
| @PutMapping | PUT | @RequestMapping(method = PUT) |
| @PatchMapping | PATCH | @RequestMapping(method = PATCH) |
| @DeleteMapping | DELETE | @RequestMapping(method = DELETE) |
