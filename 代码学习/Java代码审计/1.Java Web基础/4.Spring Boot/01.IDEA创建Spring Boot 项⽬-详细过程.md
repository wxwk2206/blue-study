##   一、项目创建
### （一）初始化项目
1. 打开 IntelliJ IDEA，点击 “创建新项目”。
2. 选择 “Spring Initializr”，JDK 默认 1.8，点击 “下一步”。
3. 完善项目信息：Group 填写公司名（示例：com.example），Artifact 填写项目名（示例：demo），选择 Maven 项目、Java 语言、Jar 打包方式，Java Version 设为 8，点击 “下一步”。
4. 选择组件：Spring Boot 版本 2.5.1，勾选 Web（Spring Web）、Template Engines（Thymeleaf）、SQL（JDBC API、MySQL Driver、MyBatis Framework），点击 “下一步”。
5. 确定项目名（示例：SpringBootDemo）和存放位置（示例：~/Desktop/SpringBootDemo），点击 “完成”。

### （三）项目分层创建
按以下四层结构创建文件夹，规范代码存放：

1. Controller 层：存放 XxxController.java，处理请求入口，创建`controller`文件夹。
2. Service 层：处理业务逻辑，创建`service`（存放 XxxService.java 接口）和`serviceImpl`（存放 XxxServiceImpl.java 实现类）文件夹。
3. Mapper 层（DAO 层）：负责数据库交互，创建`mapper`文件夹（存放 XxxMapper.java 接口），并在 resources 目录下创建`mapper`文件夹（存放 XxxMapper.xml 实现文件）。
4. Bean 层（Model/Entity 层）：存放数据库表映射实体类，创建`bean`文件夹（存放 XxxBean.java）。

最终目录结构补充后如下：

```plain
src/main/java/com.example.demo
├── bean（实体类）
├── controller（控制器）
├── mapper（数据访问接口）
├── service（业务接口）
├── serviceImpl（业务实现）
└── DemoApplication.java（启动类）
src/main/resources
├── mapper（SQL映射文件）
├── static（静态资源）
├── templates（页面模板）
└── application.yml（配置文件）
```

## 三、Spring Boot 基础用法示例
### （一）@ResponseBody 用法
+ 作用：使用 @RequestMapping 后，返回值直接写入 HTTP response body，不解析为跳转路径。
+ 示例：创建 TestController.java：

```java
package com.example.demo.controller;

import org.springframework.stereotype.Controller;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.ResponseBody;

@Controller
public class TestController {
    @RequestMapping("/hello")
    @ResponseBody
    public String hello() {
        return "Hello World!!!";
    }
}
```

+ 测试：访问[http://127.0.0.1:8081/hello](http://127.0.0.1:8081/hello)，页面显示 “Hello World!!!”。

### （二）Controller 获取 application.yml 配置值
+ 示例：创建 Test2Controller.java，通过 @Value 注解获取配置：


+ 测试：访问[http://127.0.0.1:8081/test2](http://127.0.0.1:8081/test2)，页面显示数据库用户名 “root”。

### （二）各层代码实现
1. Bean 层：创建 UserBean.java（实体类，映射 user 表）

```java
package com.example.demo.bean;

public class UserBean {
    private int id;
    private String username;
    private String password;

    // getter和setter方法
    public int getId() { return id; }
    public void setId(int id) { this.id = id; }
    public String getUsername() { return username; }
    public void setUsername(String username) { this.username = username; }
    public String getPassword() { return password; }
    public void setPassword(String password) { this.password = password; }
}
```

2. Mapper 层
+ 接口：创建 UserMapper.java

```java
package com.example.demo.mapper;

import com.example.demo.bean.UserBean;
import org.apache.ibatis.annotations.Mapper;

@Mapper
public interface UserMapper {
    UserBean getInfo(String username, String password);（查询用户信息）
}
```

+ 实现文件：在 resources/mapper 下创建 UserMapper.xml

```xml
<?xml version="1.0" encoding="utf-8" ?>
<!DOCTYPE mapper PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN" "http://mybatis.org/dtd/mybatis-3-mapper.dtd">
<mapper namespace="com.example.demo.mapper.UserMapper">
  <select id="getInfo" parameterType="String" resultType="com.example.demo.bean.UserBean">
    SELECT * FROM user WHERE username = #{username} AND password = #{password}
  </select>
</mapper>
```

3. Service 层
+ 接口：创建 UserService.java

```java
package com.example.demo.service;

import com.example.demo.bean.UserBean;

public interface UserService {
    UserBean loginIn(String username, String password);（登录业务接口）
}
```

+ 实现类：创建 UserServiceImpl.java

```java
package com.example.demo.serviceImpl;

import com.example.demo.bean.UserBean;
import com.example.demo.mapper.UserMapper;
import com.example.demo.service.UserService;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Service;

@Service
public class UserServiceImpl implements UserService {

    private UserMapper userMapper;

    @Override
    public UserBean loginIn(String username, String password) {
        return userMapper.getInfo(username, password);（调用Mapper层方法）
    }
}
```

4. 启动类添加注解：在 DemoApplication.java 添加 @MapperScan，扫描 Mapper 接口

```java

import org.mybatis.spring.annotation.MapperScan;

@SpringBootApplication

    public static void main(String[] args) {
        SpringApplication.run(DemoApplication.class, args);
    }
}
```

```java
package com.example.demo.controller;

import com.example.demo.bean.UserBean;
import com.example.demo.service.UserService;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Controller;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.RequestMethod;
import org.springframework.web.bind.annotation.ResponseBody;

@Controller
public class PublicController {
    @Autowired
    UserService userService;

    @RequestMapping("/public/login")
    public String login() {
        return "login";（跳转到登录页面）

    @RequestMapping(value = "/password/login", method = RequestMethod.POST)
    @ResponseBody
    public String passwordLogin(String username, String password) {
        UserBean userBean = userService.loginIn(username, password);
        return userBean != null ? "login_success" : "login_error";（返回登录结果）
    }
}
```

### (六)登录功能：完整逻辑梳理 + 文件作用 + 关联关系
登录功能的实现核心遵循**MVC 分层设计思想**（结合 MyBatis 的 Mapper 层），整体逻辑是「页面发起请求→控制器接收参数→业务层处理逻辑→数据层操作数据库→结果逐层返回」，各文件职责单一、通过**依赖注入**和**接口调用**实现解耦，流程环环相扣且层层传递，下面按「执行顺序 + 文件分类」拆解核心逻辑、文件作用及相互关系。

#### 一、登录功能核心执行流程
整个登录从 “用户操作” 到 “页面反馈”

1. 浏览器访问登录页面地址，**Controller 层**接收请求，跳转到登录页面（login.html）；
2. 用户在页面输入账号 / 密码，点击登录，**页面（login.html）** 向后台发起 POST 请求，携带账号、密码参数；
3. **Controller 层**（PublicController.java）接收请求和参数，调用**Service 层**的登录方法，把参数传递过去；
4. **Service 层**（UserServiceImpl.java）接收到参数，调用**Mapper 层**的数据库查询方法，继续传递参数；
5. **Mapper 层**（UserMapper.java+UserMapper.xml）根据参数执行 SQL 语句，查询数据库中是否存在「账号 + 密码」匹配的用户；
6. 数据库返回查询结果（有匹配用户则返回用户信息，无则返回 null），结果逐层向上返回（Mapper→Service→Controller）；
7. **Controller 层**判断结果，返回指定标识（匹配则返回 login_success，不匹配则返回 login_error）；
8. 浏览器接收到后台返回的标识，页面显示对应结果，登录流程结束。

**核心逻辑**：**参数从页面逐层向下传递到数据库，结果从数据库逐层向上返回给页面**，中间各层只做自己的本职工作，不跨层操作。

#### 二、按 “分层 + 配套” 分类：每个文件的作用（职责单一）
登录功能涉及的文件分为「5 大核心模块」（页面层、Controller 层、Service 层、Mapper 层、实体层）+「2 个配套文件」（启动类、配置文件），每个文件的职责固定，不做超出本层的工作，下面逐个说明：

##### （一）页面层：login.html（用户交互入口，请求发起者）
+ **核心作用**：提供登录交互界面，是**用户操作的入口**，也是**请求的发起者**；
+ **具体工作**：
    1. 展示账号、密码输入框和登录按钮，供用户输入信息；
    2. 通过`<form>`表单的`action`属性指定后台请求地址，`method`指定请求方式（POST），`name`属性绑定输入框参数（保证后台能接收）；
    3. 用户点击登录后，自动携带账号（username）、密码（password）参数，向后台发起请求，等待并展示后台返回的结果。

##### （二）Controller 层：PublicController.java（请求接收者，流程调度者）
+ **核心作用**：是**前端和后台的中间桥梁**，唯一接收前端请求的层，负责 “接收参数、调度业务层、返回结果”；
+ **具体工作**：
    1. 提供 2 个请求接口：`/public/login`（GET 请求，跳转到登录页面）、`/password/login`（POST 请求，处理登录逻辑）；
    2. 处理登录请求时，通过方法参数自动接收前端传递的 username 和 password；
    3. 注入 Service 层对象，调用其登录方法（`loginIn`），并把接收的参数传递给 Service 层；
    4. 接收 Service 层返回的结果（用户信息 / NULL），判断后返回简单标识（login_success/login_error）给前端；
+ **关键注解**：`@Controller`（标识为控制器，处理页面跳转和请求）、`@Autowired`（依赖注入 Service 层对象，实现调用）、`@ResponseBody`（将返回值直接写入响应，不跳转页面）。

##### （三）Service 层：2 个文件（业务逻辑处理层，承上启下）
Service 层采用「**接口 + 实现类**」设计，接口定义规范，实现类处理具体业务，便于后续扩展和修改。

1. **UserService.java（接口）**
    - 核心作用：**定义登录业务的方法规范**，约定方法名、参数、返回值，让 Controller 层和实现类遵循统一标准；
    - 具体工作：只声明一个方法`loginIn(String username, String password)`，返回值为 UserBean，无任何业务逻辑代码。
2. **UserServiceImpl.java（实现类）**
    - 核心作用：**处理登录的核心业务逻辑**（本案例逻辑简单，主要是 “调用 Mapper 层查询数据库”，复杂场景可在此添加逻辑：如参数校验、密码加密、登录日志记录等）；
    - 具体工作：
        * 实现 UserService 接口，重写`loginIn`方法；
        * 注入 Mapper 层对象，调用其数据库查询方法，将 Service 层接收的参数传递给 Mapper 层；
        * 接收 Mapper 层返回的查询结果，直接返回给 Controller 层（复杂场景可在此做结果处理）；
    - 关键注解：`@Service`（标识为业务层组件，让 Spring 容器管理，支持依赖注入）、`@Autowired`（依赖注入 Mapper 层对象）。

##### （四）Mapper 层（DAO 层）：2 个文件（数据访问层，直接操作数据库）
Mapper 层是 MyBatis 的核心层，负责**数据库交互**，将 Java 方法和 SQL 语句关联，实现 “通过 Java 代码执行 SQL”，同样是「接口 + XML 配置」的设计。

1. **UserMapper.java（接口）**
    - 核心作用：**定义数据库操作的方法规范**，约定查询方法的名、参数、返回值，与 XML 中的 SQL 语句一一对应；
    - 具体工作：声明方法`getInfo(String username, String password)`，返回值为 UserBean（查询到的用户信息）；
    - 关键注解：`@Mapper`（标识为 MyBatis 的映射接口，让 MyBatis 扫描并创建代理对象，支持 Service 层调用）。
2. **UserMapper.xml（SQL 配置文件）**
    - 核心作用：**编写具体的 SQL 语句**，将 Mapper 接口的方法和 SQL 语句绑定，实现 “方法调用即执行 SQL”；
    - 具体工作：
        * 通过`namespace`属性绑定对应的 Mapper 接口（com.example.demo.mapper.UserMapper），保证 SQL 和方法的关联；
        * 通过`<select>`标签编写查询 SQL，`id`属性与接口方法名一致（getInfo），`parameterType`指定参数类型，`resultType`指定 SQL 返回结果的映射实体（UserBean）；
        * 通过`#{参数名}`接收 Mapper 接口传递的参数，拼接成完整的 SQL 语句（SELECT * FROM user WHERE username = #{username} AND password = #{password}）；
        * 执行 SQL 后，自动将数据库查询结果映射为 UserBean 对象，返回给 Mapper 接口。

##### （五）实体层：UserBean.java（数据载体，各层之间的 “数据传递工具”）
+ **核心作用**：**映射数据库的 user 表**，是各层之间传递登录相关数据的**唯一载体**，相当于 “数据容器”；
+ **具体工作**：
    1. 类的属性与 user 表的字段一一对应（id、username、password），保证数据库结果能自动映射；
    2. 提供所有属性的`getter/setter`方法，让各层能读取和设置属性值（MyBatis 映射结果、各层传递数据都依赖 getter/setter）；
+ **核心意义**：各层之间不传递零散的参数（如单独的 username、password），而是通过 UserBean 对象传递，规范数据格式，便于扩展（如后续增加用户手机号、邮箱，只需在实体类添加属性即可）。

##### （六）2 个配套文件：不参与核心逻辑，但功能必须依赖
1. **DemoApplication.java（项目启动类）**
    - 核心作用：**Spring Boot 项目的入口**，启动项目时加载所有组件，同时**扫描 Mapper 层接口**，让 MyBatis 能创建代理对象；
    - 关键注解：`@MapperScan("com.example.demo.mapper")`（指定 Mapper 接口的包路径，让 Spring 扫描所有 Mapper 接口，替代每个 Mapper 接口的`@Mapper`注解，更简洁）。
2. **application.yml（全局配置文件）**
    - 核心作用：**配置项目的基础信息**，让各层能正常连接数据库、找到配置文件；
    - 关键配置：
        * 数据库连接信息（url、username、password、driver-class-name）：让 Mapper 层能正常连接 MySQL；
        * MyBatis 配置（mapper-locations、type-aliases-package）：指定 UserMapper.xml 的路径、UserBean 实体类的包路径，让 MyBatis 能找到 SQL 文件和实体类。

#### 三、核心关联关系：各文件 / 各层之间怎么 “联动”
所有文件能协同工作，核心依赖**2 个关键机制**：**Spring 的依赖注入（@Autowired）** +**MyBatis 的映射绑定（接口 + XML / 实体）**，各层 / 文件之间的联动关系清晰，无跨层依赖，下面按 “从前端到数据库” 的顺序梳理核心关联：

##### 1. 页面 ↔ Controller 层：通过「请求地址 + 参数」关联
+ login.html 的`<form action="/password/login" method="post">` 对应 PublicController.java 的`@RequestMapping(value = "/password/login", method = RequestMethod.POST)`；
+ 页面输入框的`name="username"`/`name="password"` 对应 Controller 层方法的参数`String username, String password`，Spring 会自动将请求参数赋值给方法参数。

##### 2. Controller 层 ↔ Service 层：通过「依赖注入 + 接口调用」关联
+ PublicController.java 中通过`@Autowired private UserService userService;` 实现**依赖注入**：Spring 容器会自动创建 UserService 接口的实现类（UserServiceImpl）对象，注入到 Controller 层；
+ Controller 层通过`userService.loginIn(username, password)` 调用 Service 层方法，实现参数传递和结果接收，**面向接口编程**（不直接依赖实现类，只依赖接口），降低耦合。

##### 3. Service 层 ↔ Mapper 层：通过「依赖注入 + 接口调用」关联
+ UserServiceImpl.java 中通过`@Autowired private UserMapper userMapper;` 实现依赖注入：MyBatis 会为 UserMapper 接口创建代理对象，注入到 Service 层；
+ Service 层通过`userMapper.getInfo(username, password)` 调用 Mapper 层方法，传递参数并接收数据库查询结果。

##### 4. Mapper 层 ↔ 实体层 ↔ 数据库：通过「MyBatis 映射」关联
+ Mapper 接口（UserMapper.java）的方法返回值为 UserBean，对应 UserMapper.xml 的`resultType="com.example.demo.bean.UserBean"`，MyBatis 自动将 SQL 查询结果（数据库行数据）映射为 UserBean 对象；
+ UserBean 的属性（id、username、password）与数据库 user 表的字段一一对应，保证映射无偏差；
+ Mapper 接口的方法参数（username、password），对应 UserMapper.xml 的`#{username}`/`#{password}`，MyBatis 自动将方法参数赋值给 SQL 语句的占位符。

##### 5. 所有层 ↔ 启动类 / 配置文件：通过「Spring 扫描 + 全局配置」关联
+ 启动类（DemoApplication.java）的`@SpringBootApplication` 注解会扫描当前包及子包下的所有组件（@Controller、@Service、@Mapper 等），将其加入 Spring 容器，实现依赖注入；
+ 启动类的`@MapperScan` 扫描所有 Mapper 接口，配置文件的`mybatis.mapper-locations` 找到 SQL 配置文件，两者配合让 Mapper 层能正常工作；
+ 配置文件的数据库连接信息，为整个项目提供数据库访问权限，所有涉及数据库操作的层（Mapper 层）都依赖此配置。

#### 四、核心总结
1. **流程上**：登录功能是「**页面→Controller→Service→Mapper→数据库**」的参数向下传递，和「**数据库→Mapper→Service→Controller→页面**」的结果向上返回，两层单向流程构成一个完整的闭环；
2. **职责上**：各层 / 文件各司其职 —— 页面管交互、Controller 管调度、Service 管业务、Mapper 管数据库、实体类管数据传递、启动类 / 配置文件管基础支撑；
3. **关联上**：前端与后台通过「请求地址 + 参数」关联，后台各层之间通过「Spring 依赖注入 + 接口调用」关联，Mapper 层与数据库通过「MyBatis 映射（接口 + XML + 实体类）」关联，所有组件通过「Spring 扫描 + 全局配置」实现统一管理。
