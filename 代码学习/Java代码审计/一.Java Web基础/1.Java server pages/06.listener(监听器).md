## 一、监听器基础概念
### 1. 定义
监听器是 Servlet 规范中的特殊类，用于监听`ServletContext`、`HttpSession`、`ServletRequest`三类域对象的**创建 / 销毁事件**和**属性修改事件**，可在事件发生前后执行自定义处理逻辑。

### 2. 核心特点
+ 需实现对应监听器接口并覆写方法；
+ 注册方式：web.xml 配置（Servlet2.5 及以下）或注解（Servlet3.0 及以上）；
+ 加载顺序：WEB 容器优先加载监听器 → 过滤器 → Servlet；
+ 多监听器时，按 web.xml 注册顺序启动。

## 二、监听器分类（三大类 8 个接口）

| **监听类型** | **对应接口** | **核心作用** |
| --- | --- | --- |
| 域对象创建 / 销毁 | ServletContextListener | 监听 ServletContext（应用上下文）的初始化与销毁 |
| 域对象创建 / 销毁 | HttpSessionListener | 监听 HttpSession（会话）的创建与失效 |
| 域对象创建 / 销毁 | ServletRequestListener | 监听 ServletRequest（请求）的初始化与销毁（适合做内存马） |
| 域对象属性修改 | ServletContextAttributeListener | 监听 ServletContext 域中属性的添加、修改、删除 |
| 域对象属性修改 | HttpSessionAttributeListener | 监听 HttpSession 域中属性的添加、修改、删除 |
| 域对象属性修改 | ServletRequestAttributeListener | 监听 ServletRequest 域中属性的添加、修改、删除 |
| HttpSession 对象状态 | HttpSessionBindingListener | 监听对象与 HttpSession 的绑定 / 解除绑定 |
| HttpSession 对象状态 | HttpSessionActivationListener | 监听 HttpSession 的钝化（序列化）与活化（反序列化） |


## 三、核心监听器接口详解
### 1. 域对象创建 / 销毁监听器
#### （1）ServletContextListener
+ 接口全类名：`javax.servlet.ServletContextListener`
+ 生命周期：
    - 创建：启动服务器时；
    - 销毁：关闭服务器或移除项目时。
+ 核心方法：

```java
// 初始化时调用（应用启动）
void contextInitialized(ServletContextEvent sce);
// 销毁时调用（应用停止）
void contextDestroyed(ServletContextEvent sce);
```

+ 典型用途：初始化全局配置、启动定时任务。

#### （2）HttpSessionListener
+ 接口全类名：`javax.servlet.http.HttpSessionListener`
+ 生命周期：
    - 创建：调用`getSession()`方法（一次会话仅创建一次）；
    - 销毁：超时（默认 30 分钟）、服务器非正常关闭、调用`invalidate()`方法。
+ 核心方法：

```java
// Session创建时调用
void sessionCreated(HttpSessionEvent se);
// Session销毁时调用
void sessionDestroyed(HttpSessionEvent se);
```

+ 典型用途：统计在线用户数。

#### （3）ServletRequestListener
+ 接口全类名：`javax.servlet.ServletRequestListener`
+ 生命周期：
    - 创建：客户端发送请求（访问.html/.jsp/.servlet 等）；
    - 销毁：服务器响应完成后。
+ 核心方法：

```java
// 请求初始化时调用
void requestInitialized(ServletRequestEvent sre);
// 请求销毁时调用
void requestDestroyed(ServletRequestEvent sre);
```

### 2. 域对象属性修改监听器
+ 共性：均包含`attributeAdded()`、`attributeReplaced()`、`attributeRemoved()`三个方法，分别对应属性的添加、修改、删除事件。
+ 差异：监听的域对象不同（ServletContext/HttpSession/ServletRequest），事件参数类型不同（如`ServletContextAttributeEvent`）。

### 3. HttpSession 对象状态监听器
#### （1）HttpSessionBindingListener
+ 接口全类名：`javax.servlet.http.HttpSessionBindingListener`
+ 核心特点：无需在 web.xml 注册，需实现`Serializable`接口；
+ 核心方法：

```java
// 对象绑定到Session时调用
void valueBound(HttpSessionBindingEvent event);
// 对象从Session解除绑定时调用
void valueUnbound(HttpSessionBindingEvent event);
```

#### （2）HttpSessionActivationListener
+ 接口全类名：`javax.servlet.http.HttpSessionActivationListener`
+ 核心概念：
    - 钝化：Session 从内存序列化到硬盘（.session 文件）；
    - 活化：从硬盘反序列化到内存恢复状态；
+ 核心特点：无需 web.xml 注册，需实现`Serializable`接口；
+ 核心方法：

```java
// Session钝化时调用
void sessionWillPassivate(HttpSessionEvent se);
// Session活化时调用
void sessionDidActivate(HttpSessionEvent se);
```

## 四、监听器使用步骤（web.xml 方式）
### 1. 通用流程
1. 创建类，实现对应监听器接口；
2. 覆写接口中的事件处理方法；
3. 在 web.xml 中注册监听器（部分接口无需注册）；
4. 编写测试页面（如.jsp）触发事件；
5. 启动项目，观察控制台输出验证。

### 2. 关键配置示例（web.xml）
```java
<web-app version="2.5" ...>
  <!-- 注册监听器 -->
  <listener>
    <listener-class>com.Listener.MyServletContextListener</listener-class>
  </listener>
  <!-- 全局参数（可选） -->
  <context-param>
    <param-name>name</param-name>
    <param-value>test123</param-value>
  </context-param>
</web-app>
```

### 3. 典型案例：ServletContextListener
```java
// 监听器类
package com.Listener;
import javax.servlet.ServletContextEvent;
import javax.servlet.ServletContextListener;

public class MyServletContextListener implements ServletContextListener {
  @Override
  public void contextInitialized(ServletContextEvent sce) {
    String name = sce.getServletContext().getInitParameter("name");
    System.out.println("应用启动：" + name);
  }

  @Override
  public void contextDestroyed(ServletContextEvent sce) {
    String name = sce.getServletContext().getInitParameter("name");
    System.out.println("应用停止：" + name);
  }
}
```

+ 测试结果：启动项目输出 “应用启动：test123”，停止项目输出 “应用停止：test123”。

## 五、Servlet3.0 注解方式（推荐）
### 1. 使用条件
+ JDK≥1.6；
+ 编译器级别≥6.0；
+ 依赖 Servlet3.0 及以上 jar 包；
+ web 容器支持（如 Tomcat7+）；
+ web.xml 需指定 3.0 规范：

```plain
<web-app version="3.0" xmlns="http://java.sun.com/xml/ns/javaee" ...>
```

### 2. 核心用法
+ 用`@WebListener`注解标记监听器类，无需 web.xml 注册；
+ 注解无法指定监听器启动顺序。

### 3. 案例：注解版 ServletContextListener
```java
package com.Listener;
import javax.servlet.ServletContextEvent;
import javax.servlet.ServletContextListener;
import javax.servlet.annotation.WebListener;

@WebListener // 注解注册
public class MyServletContextListener2 implements ServletContextListener {
    @Override
    public void contextInitialized(ServletContextEvent sce) {
        System.out.println("注解版监听器：应用启动");
    }

    @Override
    public void contextDestroyed(ServletContextEvent sce) {
        System.out.println("注解版监听器：应用停止");
    }
}
```

## 六、目录结构规范
```plain
src/main/webapp/
├── Listener/                # 监听器类目录
│   ├── MyServletContextListener.java
│   ├── MyHttpSessionListener.java
│   └── ...（其他监听器类）
├── WEB-INF/
│   ├── web.xml              # 2.5版本注册监听器
│   └── ...
├── META-INF/
│   └── context.xml          # Session钝化配置（可选）
├── index.jsp                # 测试页面
├── test5.jsp                # 域对象创建/销毁测试
├── servletContextOperationalAttribute.jsp  # ServletContext属性测试
└── ...（其他测试页面）
```

## 七、关键注意事项
1. 监听器优先级：web.xml 中注册的监听器按顺序执行，注解版无顺序；
2. 无需注册的监听器：`HttpSessionBindingListener`和`HttpSessionActivationListener`（需实现`Serializable`）；
3. Session 钝化配置：需在`META-INF/context.xml`中指定钝化目录和超时时间；
4. 内存马场景：`ServletRequestListener`因监听所有请求，适合用于内存马注入。
