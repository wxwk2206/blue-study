## 一、Thymeleaf 简介
1. 定位：适用于 Web 环境和独立环境的现代服务器端 Java 模板引擎。
2. 核心目标：
    - 提供优雅、高度可维护的模板创建方式。
    - 支持自然模板：在不影响模板作为设计原型的前提下，将逻辑注入模板文件，优化前端设计与开发人员的协作，减少理解偏差。
    - 原生支持 HTML5，可创建完全验证的模板。
3. 功能范围：能够处理 HTML、XML、JavaScript、CSS 及纯文本等多种类型文件。

## 二、Thymeleaf 支持的模板模式（共 6 种）
模板模式分为标记模板模式、文本模板模式和无操作模板模式三类，具体如下：

| **模板模式** | **类型归属** | **功能说明** |
| --- | --- | --- |
| HTML | 标记模板模式 | 支持 HTML5、HTML4、XHTML 等各类 HTML 输入；<br/>不执行验证或格式检查，输出时尽量保留模板原有代码 / 结构 |
| XML | 标记模板模式 | 接收 XML 输入；<br/>要求代码格式良好（无未封闭标签、无未加引号属性等），格式错误会抛出异常；<br/>不执行 DTD 或 XML 模式验证 |
| TEXT | 文本模板模式 | 适用于非标记类模板（如文本电子邮件、模板文档）；<br/>可将 HTML/XML 作为纯文本处理（不解析标记，标签、DOCTYPE、注释等均视为纯文本）；<br/>使用特殊语法 |
| JavaScript | 文本模板模式 | 用于处理应用中的 JavaScript 文件；<br/>支持在 JS 中使用模型数据，提供 JS 专属集成（如专门转义、自然脚本）；<br/>使用与 TEXT 模式相同的特殊语法 |
| CSS | 文本模板模式 | 用于处理应用中的 CSS 文件；使用 TEXT 模式的特殊处理语法 |
| RAW | 无操作模板模式 | 不处理模板本身；<br/>用于将未修改的资源（文件、URL 响应等）插入正在处理的模板中，可安全包含外部非受控 HTML 资源（其中的 Thymeleaf 代码不会执行） |
