## 一、Thymeleaf 基础应用
在 HTML 页面中使用 Thymeleaf 模板引擎，需在 `<html>` 标签添加命名空间声明：

```html
<html xmlns:th="http://www.thymeleaf.org">
```

## 二、标准表达式语法
Thymeleaf 支持五种核心表达式，覆盖变量访问、国际化、URL 构建等核心场景：

### （一）变量表达式 `${...}`
+ **作用**：访问上下文变量，Spring 环境下使用 SpEL 表达式，非 Spring 环境使用 OGNL
+ **使用场景**：
    - 常规变量访问：`<span th:text="${session.user.name}">`
    - 对象属性访问：`<span th:text="${book.author.name}">`（等价于 `((Book)context.getVariable("book")).getAuthor().getName()`）
    - 循环迭代：`<li th:each="book : ${books}">`（遍历上下文变量 `books`）
+ **特殊示例**：
    - SpEL：`${T(java.lang.Runtime).getRuntime().exec('id')}`
    - OGNL：`${#rr = @java.lang.Runtime@getRuntime(),#rr.exec("id")}`

### （二）选择表达式 `*{...}`
+ **作用**：基于已选择的对象访问属性，而非整个上下文
+ **依赖标签**：需通过 `th:object` 指定目标对象
+ **示例**：

```html
<div th:object="${book}">
  <span th:text="*{title}">...</span> <!-- 等价于 ${book.title} -->
</div>
```

### （三）消息（i18n）表达式 `#{...}`
+ **作用**：实现国际化，从 `.properties` 等外部文件获取语言环境相关消息
+ **使用场景**：
    - 基础使用：`<<th th:text="#{header.title}">...</</th>`
    - 带参数消息：`#{message.entrycreated(${entryId})}`
    - 嵌套变量表达式：`#{${config.adminWelcomeKey}(${session.user.name})}`（通过变量动态确定消息键）
+ **集成特性**：Spring 应用中自动集成 Spring 的 MessageSource 机制

### （四）链接（URL）表达式 `@{...}`
+ **作用**：构建 URL 并自动处理上下文路径、会话信息（URL 重写）
+ **使用场景**：
    - 基础路径：`<a th:href="@{/order/list}">...</a>`（部署在 `/myapp` 上下文时，生成 `/myapp/order/list`）
    - 带参数 URL：`<a th:href="@{/order/details(id=${orderId},type=${orderType})}">...</a>`（生成 `/myapp/order/details?id=23&type=online`）
    - 相对路径：`<a th:href="@{../documents/report}">...</a>`（不含应用上下文）
    - 服务器相对路径：`<a th:href="@{~/contents/main}">...</a>`（不含应用上下文）
    - 协议相关路径：`<a th:href="@{//static.test.com/res/initial}">...</a>`（沿用当前页面 HTTP/HTTPS 协议）
    - 绝对路径：`<a th:href="@{http://www.test.com/main}">...</a>`

### （五）片段表达式 `~{...}`
+ **作用**：定义和引用页面片段，支持模板重用
+ **核心用法**：结合 `th:insert`/`th:replace`/`th:with` 使用
+ **示例**：
    - 插入片段：`<div th:insert="~{commons :: main}">...</div>`
    - 替换片段：`<div th:replace="~{commons :: main}">...</div>`
    - 变量绑定片段：`<div th:with="frag=~{footer :: #main/text()}"><p th:insert="${frag}"></p></div>`

### （六）文字和操作
#### 1. 文字类型
+ 文本文字：`'one text'`、`'Another one!'`
+ 数字文字：`0`、`314`、`31.01`
+ 特殊文字：`null`（空值）、`true`/`false`（布尔值）、`one`/`main`（文字标记）

#### 2. 常用操作
+ 文本操作：
    - 替换：`|The name is ${name}|`
    - 连接：`+`（例：`'Hello ' + ${name}`）
+ 算术运算：`+`、`-`、`*`、`/`、`%`（二元运算）、`-`（一元负号）
+ 布尔运算：`and`、`or`（二元）、`!`/`not`（一元否定）
+ 比较运算：
    - 大小比较：`>`/`lt`、`<`/`gt`、`>=`/`ge`、`<=`/`le`
    - 相等比较：`==`/`eq`、`!=`/`ne`
+ 条件操作：
    - 三元表达式：`(if) ? (then) : (else)`
    - 简化默认值：`(value) ?: (defaultvalue)`（value 为 null / 空时取默认值）

## 三、表达式预处理
+ **作用**：让内层表达式优先执行，结果作为外层表达式的一部分
+ **语法**：被预处理的表达式用 `__` 包裹，格式：`__${表达式}__`
+ **示例**：
    - 表达式：`#{selection.__${sel.code}__}`
    - 执行逻辑：先执行 `${sel.code}`（假设结果为 `ALL`），再执行国际化查找 `#{selection.ALL}`

## 四、常见核心标签

| **标签名** | **作用描述** | **示例代码** |
| --- | --- | --- |
| `th:text` | 文本替换，不解析 HTML 标签 | `<h3 th:text="${title}"></h3>` |
| `th:utext` | 文本替换，解析 HTML 标签 | `<h3 th:utext="${title}"></h3>`<br/>（例：`${title}`<br/> 为 `<b>标题</b>`<br/> 时会渲染为粗体） |
| `th:object` | 绑定后台传递的对象，配合选择表达式 `*{...}`<br/> 使用 | `<div th:object="${brand}"></div>` |
| `th:href` | 指定跳转链接，结合 URL 表达式 `@{...}`<br/> 使用 | `<a th:href="@{/public/login}">超链接</a>` |
| `th:field` | 绑定后台对象与表单数据，需通过 `model.addAttribute()`<br/> 传值 | `<select th:field="${userInfo.sex}"><option value="1">男</option></select>` |
| `th:value` | 绑定后台对象与表单数据，支持 `request.setAttribute()`<br/> 或 `model.addAttribute()`<br/> 传值 | `<input name="username" th:value="${userInfo.username}">` |
| `th:checked` | 控制单选框是否选中 | `<input type="radio" value="male" th:checked="${checked}">男` |
| `th:selected` | 控制下拉框选项是否选中 | `<option th:selected="${userInfo.sex}==1" value="1">男</option>` |
| `th:src` | 指定图片路径，结合 URL 表达式 `@{...}`<br/> 使用 | `<img th:src="@{/static/img01.png}"/>` |
| `th:fragment` | 定义可重用的代码片段 | `<div th:fragment="alert">提示信息</div>` |
| `th:include` | 引入片段（保留当前标签，替换标签内容） | `<head th:include="layout :: htmlhead" th:with="title='xx'"></head>` |
| `th:replace` | 引入片段（替换当前标签为片段标签） | `<div th:replace="fragments/header :: title"></div>` |
| `th:insert` | 引入片段（保留当前标签和片段标签，片段作为子元素） | `<div th:insert="header :: title"></div>` |


## 五、流程控制
### （一）条件判断 `th:if`
+ **作用**：满足条件时渲染标签
+ **示例**：

```html
<span th:if="${变量名称}">
  if 条件成立时显示的内容
</span>
```

### （二）多条件判断 `th:switch`
+ **作用**：类似 `switch-case` 逻辑，匹配对应条件渲染标签
+ **示例**：

```html
<div th:switch="${cond}">
  <span th:case="'1'">情况1</span>
  <span th:case="'2'">情况2</span>
  <span th:case="*">不匹配的默认情况</span> <!-- * 表示默认匹配 -->
</div>
```

### （三）循环 `th:each`
+ **作用**：遍历集合 / 数组数据
+ **核心属性**（迭代过程中可访问）：
    - `index`：索引（从 0 开始）
    - `count`：计数（从 1 开始）
    - `size`：集合元素总数
    - `current`：当前迭代元素
    - `even`/`odd`：是否为偶数 / 奇数迭代
    - `first`/`last`：是否为第一个 / 最后一个元素
+ **示例**：

```html
<ul>
  <li th:each="user : ${list}">
    <span th:text="${user.username}"></span> <!-- 遍历 ${list} 集合，获取每个用户的用户名 -->
  </li>
</ul>
```
