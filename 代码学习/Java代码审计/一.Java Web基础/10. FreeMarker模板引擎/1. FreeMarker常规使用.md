## 一、模板文件组成部分
FreeMarker 模板文件核心包含 4 个部分，各部分功能与使用规则如下：

1. **文本**：直接输出的内容，无需额外标记。
2. **注释**：使用 `<#-- ... -->` 格式，注释内容不会在最终输出中显示。
3. **插值**：以 `${...}` 或 `#{...}` 格式存在，作为占位符，会被数据模型中的对应值替换后输出。
4. **FTL 指令**：与 HTML 标记类似，但名称前加 `#` 区分，用于控制模板逻辑，不会直接输出。

### 示例模板
```html
<html>
  <head>
    <title>Welcome!</title>
  </head>
  <body>
    <#-- 注释部分（不会输出） -->
      <h1>Welcome ${username} !</h1> <!-- 插值，替换username变量 -->
      <p>We have these animals:</p>
      <ul>
        <#list animals as animal> <!-- FTL指令，遍历集合 -->
          <li>${animal.name} for ${animal.price} Euros</li>
        </#list>
      </ul>
  </body>
</html>
```

## 二、核心语法使用
### （一）控制语句
1. **if-elseif-else**

```plain
<#if condition>
    满足condition时执行
<#elseif condition2>
    满足condition2时执行
<#elseif condition3>
    满足condition3时执行
<#else>
    所有条件都不满足时执行
</#if>
```

2. **switch-case**

```plain
<#switch value>
<#case refValue1>
    匹配refValue1时执行
    <#break> <!-- 跳出switch -->
<#case refValue2>
    匹配refValue2时执行
    <#break>
<#case refValueN>
    匹配refValueN时执行
    <#break>
<#default>
    无匹配值时执行
</#switch>
```

### （二）变量相关操作
1. **判断变量是否存在**：使用 `??` 运算符

```plain
<#if readonly??>
    readonly变量存在时执行
</#if>
```

2. **防止空指针报错**：变量后加 `!` 并指定默认值，变量为 null 时输出默认值

```plain
${foo!"Default"} <!-- foo为null时输出"Default" -->
```

3. **普通变量使用**：直接通过 `${变量名}` 引用，注意特殊字符（如双引号）需转义

```plain
${name} <!-- 直接引用name变量 -->
```

4. **变量声明**：使用 `assign` 指令声明变量

```plain
<#assign num=0/> <!-- 声明num变量并赋值为0 -->
```

### （三）集合遍历（List）
1. **基础遍历**：使用 `list` 指令，格式为 `<#list 集合 as 元素变量>`

```plain
<#list ["克里斯埃文斯", "斯嘉丽约翰逊", "小罗伯特唐尼"] as x>
    ${x} <!-- 输出集合中的每个元素 -->
</#list>
```

2. **特殊循环变量**：遍历过程中可使用两个内置变量
+ `item_index`：当前元素的索引值（item 为元素变量名）
+ `item_has_next`：是否存在下一个元素
2. **跳出迭代**：使用 `<#break>` 指令终止遍历

### （四）运算符
1. **算术运算符**：支持 `+`（加）、`-`（减）、`*`（乘）、`/`（除）、`%`（取余）
2. **⽐较运算符**

```html

2. !=:判断两个值是否不等.
```

### （五）模板引用指令
1. **include 指令**：类似 JSP 包含指令，用于嵌入其他模板文件
+ 语法：`<#include filename [options]>`
+ 参数说明：
    - `filename`：被包含的模板文件路径；
    - `options`（可选）：包含 `encoding`（解码集）和 `parse`（是否按 FTL 文件解析，默认 `true`）。
1. **import 指令**：导入其他模板的所有变量，存储在指定 Map 对象中
+ 语法：`<#import "模板路径" as Map名称>`
+ 示例：`<#import "/lib/common.ftl" as com>`（导入 `common.ftl` 的变量到 `com` Map 中）

### （六）macro 与复杂数据遍历
1. **功能**：用于实现自定义指令，常用来开发公共组件（如分页条）。
2. **List 中含 Map 的遍历**：支持点语法或方括号语法，可混合使用
+ 数据模型示例：

```java
Map root = new HashMap();
Author author = new Author();
Book book = new Book();
author.setName("annlee");
author.setAddress("gz");
book.setName("struts2");
book.setAuthor(author);
root.put("info","struts");
root.put("book", book);
```

+ 访问语法（获取书的作者姓名）：
    - 点语法：`book.author.name`
    - 混合语法：`book["author"].name` 或 `book.author["name"]`
    - 方括号语法：`book["author"]["name"]`
