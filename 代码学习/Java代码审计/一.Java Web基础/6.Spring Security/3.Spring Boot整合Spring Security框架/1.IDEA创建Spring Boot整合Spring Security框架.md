## 一、创建项目
### （一）打开 IDEA 并选择创建方式
1. 打开 IntelliJ IDEA（版本 2020.1.4），点击 “创建新项目”。
2. 选择 “Spring Initializr” 创建方式，JDK 默认选择 1.8，Custom 地址填写[https://start.springboot.io](https://start.springboot.io)，点击 “下一步”。

### （二）完善项目基础信息
| **配置项** | **填写内容** | **说明** |
| --- | --- | --- |
| Group | com.test | 一般填写公司名 |
| Artifact | securitydemo | 一般填写项目名 |
| Type | Maven Project | 生成基于 Maven 的项目归档 |
| Language | Java | 项目开发语言 |
| Packaging | Jar | 项目打包方式 |
| Java Version | 1.8 | 对应 JDK 版本 |
| Version | 0.0.1-SNAPSHOT | 项目版本 |
| Name | securitydemo | 项目名称 |
| Description | Demo project for Spring Boot | 项目描述 |
| Package | com.test.securitydemo | 项目包路径 |
| 填写完成后点击 “下一步”。 | | |


### （三）选择项目组件
1. Spring Boot 版本选择 2.7.4。
2. 勾选 “Spring Web” 组件（用于构建 Web 应用），点击 “下一步”。

### （四）确定项目名与存放位置
1. 项目名默认已设为 securitydemo（可按需修改）。
2. 选择项目存放位置（示例：~/Desktop/JavaTestproect/securitydemo）。
3. 点击 “完成”，首次加载需下载资源，需耐心等待。

### （五）修改 pom.xml 导入依赖
1. 找到项目根目录下的 pom.xml 文件（路径：./ 项目 /pom.xml）。
2. 在`<<dependencies></</dependencies>`标签中添加以下依赖（无该标签则手动创建）：

```xml
<!-- Spring Security核心依赖 -->
<dependency>
  <groupId>org.springframework.boot</groupId>
  <artifactId>spring-boot-starter-security</artifactId>
  <version>2.7.3</version>
</dependency>
<!-- Spring Security加密依赖 -->
<dependency>
  <groupId>org.springframework.security</groupId>
  <artifactId>spring-security-crypto</artifactId>
  <version>5.7.5</version> <!-- 需与Spring Security版本一致 -->
</dependency>
```

1. 依赖添加后若报红，点击 IDEA 右侧 Maven 按钮刷新，等待自动导入依赖文件。

### （六）修改 application 配置
1. 将项目中`src/main/resources`目录下的`application.properties`文件重命名为`application.yml`。
2. 在`application.yml`中添加以下配置：

```yaml
server:
  address: 127.0.0.1 # 访问地址
  port: 8081 # 访问端口
```

### （七）创建测试控制器
1. 在`com.test.securitydemo`包下创建`controller`子包。
2. 在`controller`包中创建`HelloController`类，代码如下：

```java
package com.test.securitydemo.controller;

import org.springframework.stereotype.Controller;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.ResponseBody;

@Controller
public class HelloController {
    @RequestMapping("/hello")
    @ResponseBody
    public String hello() {
        return "Hello World!!!";
    }
}
```

## 二、项目测试
### （一）启动项目
直接启动 Spring Boot 主程序（`SecuritydemoApplication`），启动过程中 Spring Security 会自动生成默认登录密码并输出到控制台。

### （二）访问测试
1. 打开浏览器，访问`http://127.0.0.1:8081/hello`，会自动 302 跳转到 Spring Security 默认登录页（`http://127.0.0.1:8081/login`）。
2. 登录页输入默认账号（user）和控制台输出的随机密码，点击 “Sign in” 登录。
3. 登录成功后，再次访问`http://127.0.0.1:8081/hello`，可正常显示 “Hello World!!!”，说明 Spring Security 整合成功。

## 注意事项
1. 首次创建项目时，因需下载 Maven 依赖和 Spring Boot 相关资源，加载速度可能较慢，需保证网络通畅。
2. 依赖添加后若报红，需手动刷新 Maven 依赖，等待 IDEA 完成导入。
3. 控制台生成的默认密码仅用于开发环境，生产环境需手动配置安全的登录认证信息。
4. 登录成功后，浏览器会保存登录 Cookie，后续访问无需重复登录（Cookie 有效期内）。
