## 一、核心目标
从数据库读取用户信息，基于 RBAC（角色基础访问控制）模型实现用户登录认证与动态权限管理，支持 “用户 - 角色 - 权限” 的关联控制。

#### 4. 服务层（Service）
实现`UserDetailsService`接口，用于 Spring Security 加载用户信息和权限路径：`./项目/src/main/java/com/test/securitydemo/service/RBACUserDetailsService.java`

```java
package com.test.securitydemo.service;

import com.test.securitydemo.mapper.*;
import com.test.securitydemo.pojo.*;
import org.springframework.security.core.GrantedAuthority;
import org.springframework.security.core.authority.SimpleGrantedAuthority;
import org.springframework.security.core.userdetails.UserDetails;
import org.springframework.security.core.userdetails.UserDetailsService;
import org.springframework.security.core.userdetails.UsernameNotFoundException;
import org.springframework.stereotype.Service;

import javax.annotation.Resource;
import java.util.ArrayList;
import java.util.HashMap;
import java.util.List;
import java.util.Map;

@Service
public class RBACUserDetailsService implements UserDetailsService {

    @Resource
    private UserMapper userMapper;
    @Resource
    private RoleMapper roleMapper;
    @Resource
    private UserRoleMapper userRoleMapper;
    @Resource
    private PermissionMapper permissionMapper;
    @Resource
    private RolePermissionMapper rolePermissionMapper;

    @Override
    public UserDetails loadUserByUsername(String username) throws UsernameNotFoundException {
        // 1. 根据用户名查询用户
        Map<String, Object> userMap = new HashMap<>();
        userMap.put("username", username);
        List<User> users = userMapper.selectByMap(userMap);
        if (users.isEmpty()) {
            throw new UsernameNotFoundException("用户不存在");
        }
        User user = users.get(0);

        // 2. 查询用户拥有的角色
        Map<String, Object> userRoleMap = new HashMap<>();
        userRoleMap.put("user_id", user.getId());
        List<UserRole> userRoles = userRoleMapper.selectByMap(userRoleMap);
        List<Role> roles = new ArrayList<>();
        for (UserRole ur : userRoles) {
            roles.add(roleMapper.selectById(ur.getRoleId()));
        }

        // 3. 查询角色拥有的权限
        List<Permission> permissions = new ArrayList<>();
        for (Role role : roles) {
            Map<String, Object> rolePermMap = new HashMap<>();
            rolePermMap.put("role_id", role.getId());
            List<RolePermission> rolePermissions = rolePermissionMapper.selectByMap(rolePermMap);
            for (RolePermission rp : rolePermissions) {
                permissions.add(permissionMapper.selectById(rp.getPermissionId()));
            }
        }

        // 4. 封装角色和权限（角色前缀加ROLE_区分）
        List<GrantedAuthority> authorities = new ArrayList<>();
        roles.forEach(role -> authorities.add(new SimpleGrantedAuthority("ROLE_" + role.getName())));
        permissions.forEach(perm -> authorities.add(new SimpleGrantedAuthority(perm.getName())));

        // 5. 返回Spring Security的User对象
        return new org.springframework.security.core.userdetails.User(
            user.getUsername(),
            user.getPassword(),
            authorities
        );
    }
}
```

#### 6. Spring Security 配置类
配置认证逻辑和权限规则，路径：`./项目/src/main/java/com/test/securitydemo/config/SecurityConfig.java`

```java
package com.test.securitydemo.config;

import com.test.securitydemo.mapper.PermissionMapper;
import com.test.securitydemo.pojo.Permission;
import com.test.securitydemo.service.RBACUserDetailsService;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.springframework.security.config.annotation.authentication.builders.AuthenticationManagerBuilder;
import org.springframework.security.config.annotation.web.builders.HttpSecurity;
import org.springframework.security.config.annotation.web.configuration.EnableWebSecurity;
import org.springframework.security.config.annotation.web.configuration.WebSecurityConfigurerAdapter;
import org.springframework.security.crypto.bcrypt.BCryptPasswordEncoder;
import org.springframework.security.crypto.password.PasswordEncoder;

import javax.annotation.Resource;
import java.util.List;

@Configuration
@EnableWebSecurity
public class SecurityConfig extends WebSecurityConfigurerAdapter {

    @Resource
    private RBACUserDetailsService rbacUserDetailsService;
    @Resource
    private PermissionMapper permissionMapper;

    // 配置认证：使用自定义UserDetailsService和BCrypt加密
    @Override
    protected void configure(AuthenticationManagerBuilder auth) throws Exception {
        auth.userDetailsService(rbacUserDetailsService).passwordEncoder(passwordEncoder());
    }

    // 配置权限：动态绑定URL和权限
    @Override
    protected void configure(HttpSecurity http) throws Exception {
        List<Permission> permissions = permissionMapper.selectList(null);
        http.authorizeRequests()
        .antMatchers("/").permitAll() // 首页公开访问
        .antMatchers("/hello").permitAll() // 公开接口
        .and()
        .formLogin(); // 启用默认登录页

        // 动态绑定权限与URL
        for (Permission perm : permissions) {
            http.authorizeRequests().antMatchers(perm.getUrl()).hasAnyAuthority(perm.getName());
        }
    }

    // 注入BCrypt密码加密器
    @Bean
    public PasswordEncoder passwordEncoder() {
        return new BCryptPasswordEncoder();
    }
}
```

## 三、关键说明
1. 密码加密：使用 Spring Security 的`BCryptPasswordEncoder`，不可逆加密，安全性高
2. RBAC 核心逻辑：通过 “用户→角色→权限” 的关联，动态从数据库加载权限规则，无需硬编码
3. 权限匹配：通过`hasAnyAuthority`指定 URL 所需权限，与数据库权限表动态绑定
4. 公开接口：通过`permitAll()`配置无需登录即可访问的接口
