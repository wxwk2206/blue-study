## 一、核心原理
通过编写配置类继承`WebSecurityConfigurerAdapter`，重写`configure(AuthenticationManagerBuilder auth)`方法（负责认证），在内存中配置用户名、加密后的密码，实现登录认证（`configure(HttpSecurity http)`负责授权，本文暂不涉及）。

## 二、关键依赖
需引入 Spring Security 相关依赖（Spring Boot 项目可通过 starter 快速集成）：

```xml
<!-- Spring Security Starter -->
<dependency>
  <groupId>org.springframework.boot</groupId>
  <artifactId>spring-boot-starter-security</artifactId>
</dependency>
```

## 三、实现步骤
### 1. 创建配置类
+ **目录**：`./项目/src/main/java/com/test/securitydemo/config/SecurityConfig.java`
+ **核心代码**：

```java
package com.test.securitydemo.config;

import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.springframework.security.config.annotation.authentication.builders.AuthenticationManagerBuilder;
import org.springframework.security.config.annotation.web.configuration.EnableWebSecurity;
import org.springframework.security.config.annotation.web.configuration.WebSecurityConfigurerAdapter;
import org.springframework.security.crypto.bcrypt.BCryptPasswordEncoder;
import org.springframework.security.crypto.password.PasswordEncoder;

@Configuration // 标识为配置类
@EnableWebSecurity // 启用Spring Security安全机制
public class SecurityConfig extends WebSecurityConfigurerAdapter {

    // 重写认证配置方法，配置内存中的用户信息
    @Override
    protected void configure(AuthenticationManagerBuilder auth) throws Exception {
        // 配置用户a：用户名a，密码123（加密后），无角色
        auth.inMemoryAuthentication()
        .withUser("a")
        .password(passwordEncoder().encode("123"))
        .roles();

        // 配置用户b：用户名b，密码123（加密后），无角色
        auth.inMemoryAuthentication()
        .withUser("b")
        .password(passwordEncoder().encode("123"))
        .roles();

        // 配置用户c：用户名c，密码123（加密后），无角色
        auth.inMemoryAuthentication()
        .withUser("c")
        .password(passwordEncoder().encode("123"))
        .roles();
    }

    // 注入密码加密器（Spring Security 5+强制要求密码密文存储）
    @Bean
    public PasswordEncoder passwordEncoder() {
        // 使用BCrypt加密算法
        return new BCryptPasswordEncoder();
    }
}
```

### 2. 登录访问
+ **访问地址**：浏览器输入 `http://127.0.0.1:8081/login`（默认端口 8080，若项目配置为 8081 则使用对应端口）
+ **可用账号密码**：
    - 账号 1：a，密码：123
    - 账号 2：b，密码：123
    - 账号 3：c，密码：123

## 四、重要说明
1. **密码加密要求**：Spring Security 5 版本后，强制要求密码必须通过加密器处理（密文存储），否则会报错，本文使用`BCryptPasswordEncoder`加密算法。
2. **用户存储方式**：本文采用`inMemoryAuthentication()`将用户信息存储在内存中，仅适用于测试场景，生产环境需替换为数据库存储（如整合 JPA、MyBatis 查询用户）。
3. **角色配置**：示例中`roles()`方法未指定具体角色，后续可根据需求添加角色（如`roles("ADMIN", "USER")`），结合授权配置实现权限控制。
