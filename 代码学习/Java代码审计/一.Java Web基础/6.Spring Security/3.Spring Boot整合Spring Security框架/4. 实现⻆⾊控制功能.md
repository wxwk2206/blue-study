## 一、核心目标
学习如何通过 Spring Boot 整合 Spring Security，实现**不同角色访问不同接口**的权限控制，同时配置登录验证（未登录跳转登录页）。

## 二、实现步骤
### （一）第一步：配置 Spring Security 核心类（SecurityConfig.java）
#### 1. 文件路径
`./项目/src/main/java/com/test/securitydemo/config/SecurityConfig.java`

#### 2. 核心注解
+ `@Configuration`：标识为配置类
+ `@EnableWebSecurity`：启用 Spring Security 网页安全支持
+ `@EnableGlobalMethodSecurity(prePostEnabled = true)`：启用方法级别的角色认证（关键，支持`@PreAuthorize`注解）

#### 3. 核心代码逻辑
```java
package com.test.securitydemo.config;

import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.springframework.security.config.annotation.authentication.builders.AuthenticationManagerBuilder;
import org.springframework.security.config.annotation.method.configuration.EnableGlobalMethodSecurity;
import org.springframework.security.config.annotation.web.configuration.EnableWebSecurity;
import org.springframework.security.config.annotation.web.configuration.WebSecurityConfigurerAdapter;
import org.springframework.security.crypto.bcrypt.BCryptPasswordEncoder;
import org.springframework.security.crypto.password.PasswordEncoder;

@Configuration
@EnableWebSecurity
@EnableGlobalMethodSecurity(prePostEnabled = true) // 启动角色认证
public class SecurityConfig extends WebSecurityConfigurerAdapter {

    // 配置内存用户（实际项目可替换为数据库查询）
    @Override
    protected void configure(AuthenticationManagerBuilder auth) throws Exception {
        auth.inMemoryAuthentication()
        .withUser("a") // 用户名a
        .password(passwordEncoder().encode("123")) // 密码加密（123）
        .roles("admin", "user"); // 拥有admin和user双角色

        auth.inMemoryAuthentication()
        .withUser("b") // 用户名b
        .password(passwordEncoder().encode("123"))
        .roles("user"); // 仅user角色

        auth.inMemoryAuthentication()
        .withUser("c") // 用户名c
        .password(passwordEncoder().encode("123"))
        .roles("admin"); // 仅admin角色
    }

    // 注入密码加密器（Spring Security5+强制要求密码密文存储）
    @Bean
    public PasswordEncoder passwordEncoder() {
        return new BCryptPasswordEncoder(); // 使用BCrypt加密算法
    }
}
```

#### 4. 关键说明
+ Spring Security5 版本后，**密码必须加密存储**，否则报错，此处使用`BCryptPasswordEncoder`加密类
+ 示例中用户信息存储在内存（`inMemoryAuthentication`），实际项目可替换为从数据库查询用户和角色

### （二）第二步：创建角色权限控制控制器（SecurityTest1Controller.java）
#### 1. 文件路径
`./项目/src/main/java/com/test/securitydemo/controller/SecurityTest1Controller.java`

#### 2. 核心功能
通过`@PreAuthorize`注解，限定接口的访问角色

#### 3. 核心代码
```java
package com.test.securitydemo.controller;

import org.springframework.security.access.prepost.PreAuthorize;
import org.springframework.stereotype.Controller;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.ResponseBody;

@RequestMapping("/SecurityTest1")
@Controller
public class SecurityTest1Controller {

    // 允许admin或user角色访问
    @RequestMapping("/adminOrUser")
    @ResponseBody
    @PreAuthorize(value = "hasAnyRole('admin','user')")
    public String adminORUser() {
        return "adminOrUser";
    }

    // 仅允许admin角色访问
    @RequestMapping("/admin")
    @ResponseBody
    @PreAuthorize(value = "hasAnyRole('admin')")
    public String admin() {
        return "admin";
    }

    // 仅允许user角色访问
    @RequestMapping("/user")
    @ResponseBody
    @PreAuthorize(value = "hasAnyRole('user')")
    public String user() {
        return "user";
    }
}
```

#### 4. 接口访问路径
+ `http://127.0.0.1:8081/SecurityTest1/adminOrUser`（双角色可访问）
+ `http://127.0.0.1:8081/SecurityTest1/admin`（仅 admin 可访问）
+ `http://127.0.0.1:8081/SecurityTest1/user`（仅 user 可访问）

### （三）第三步：创建无需角色、仅需登录的控制器（HelloController.java）
#### 1. 文件路径
`./项目/src/main/java/com/test/securitydemo/controller/HelloController.java`

#### 2. 核心功能
无需指定角色，**只要登录成功即可访问**；未登录时会自动跳转到 Spring Security 默认登录页

#### 3. 核心代码
```java
package com.test.securitydemo.controller;

import org.springframework.stereotype.Controller;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.ResponseBody;

@Controller
public class HelloController {

    @RequestMapping("/hello")
    @ResponseBody
    public String hello() {
        return "Hello World!!!";
    }
}
```

#### 4. 接口访问路径
`http://127.0.0.1:8081/hello`（登录后可访问）

### （四）第四步：测试不同角色的访问权限

| **登录账号** | **拥有角色** | **可访问接口** |

## 三、关键知识点总结
1. **注解作用**：
    - `@EnableGlobalMethodSecurity(prePostEnabled = true)`：开启方法级权限控制，支持`@PreAuthorize`
    - `@PreAuthorize("hasAnyRole('角色1','角色2')")`：指定接口允许访问的角色
2. **密码加密**：Spring Security5 + 强制密码密文存储，需注入`PasswordEncoder`（示例用`BCryptPasswordEncoder`）
3. **用户存储**：示例用内存存储（`inMemoryAuthentication`），实际项目可整合数据库（如 MyBatis 查询用户角色）
4. **默认登录页**：未登录访问受保护接口时，自动跳转 Spring Security 默认登录页，输入配置的用户名 / 密码（如 a/123）即可登录
5. **权限控制粒度**：支持接口级（控制器方法）的角色权限控制，灵活分配不同角色的访问范围
