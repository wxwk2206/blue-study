# 前言
```plain
Spring Security是一个功能强大且高度可定制的身份验证和访问控制框架
提供了完善的认证机制和方法级的授权功能,是一款非常优秀的权限管理框架
核心思想是通过一系列的过滤器链(filter chain)来进行拦截过滤,对用户的访问权限进行控制
```

```plain
Spring Security的两个主要核心功能是“认证”和“授权”(访问控制)
“认证”: Authentication(你是谁)
“授权”: Authorization(你能干什么)
这是个通用的概念,Spring Security只是实现了这个概念的框架之一
```

# 认证与授权(Authentication and Authorization)
```plain
一般说的应用访问安全性,是围绕认证(Authentication)和授权(Authorization)这两个核心概念来展开的
即:
第一步,首先需要确定用户身份
第二步,然后再确定这个用户是否有访问指定资源的权限

认证(Authentication)这块的主流解决方案有: CAS、SAML2、OAuth2.0
常说的单点登录方案(SSO)说的就是这块

授权(Authorization)的主流解决方案有: Spring Security、Apache Shiro

对比来说,Apache Shiro比较轻量级,Spring Security架构会复杂一些
```
