# 常用注解与常用类
## 2.1 @EnableWebSecurity 注解
```plain
@EnableWebSecurity注解在Spring Boot应用中使用Spring Security时,才会常用

@EnableWebSecurity注解,需要配合@Configuration注解一起使用

利用@EnableWebSecurity注解,去注解继承了WebSecurityConfigurer的类
利用@EnableWebSecurity注解,去注解继承了WebSecurityConfigurerAdapter的类
这样就构成了Spring Security的配置

一般来说,会选择继承WebSecurityConfigurerAdapter类,因为更简单
```

```java
// 一般情况下,会这样写
// 其中类名SecurityConfig想写成咋样都可以,不限制
@Configuration
@EnableWebSecurity
public class SecurityConfig extends WebSecurityConfigurerAdapter {
    @Override
    protected void configure(AuthenticationManagerBuilder auth) throws Exception {
        ...
    }

    @Override
    protected void configure(HttpSecurity http) throws Exception {
        ...
    }
}
```

## 2.2 WebSecurityConfigurerAdapter 抽象类 
```java
一般来说,会选择继承WebSecurityConfigurerAdapter类
因为WebSecurityConfigurerAdapter提供了一种便利的方式去创建WebSecurityConfigurer的实例
只需要重写WebSecurityConfigurerAdapter的方法,即可配置拦截什么URL、设置什么权限等安全控制
```

```java
其中有两个方法是最最常用的分别是:
WebSecurityConfigurerAdapter.configure(AuthenticationManagerBuilder auth) = 认证
WebSecurityConfigurerAdapter.configure(HttpSecurity http) = 授权
```

## 2.3 HttpSecurity 类-常用方法及说明
```java
方法: HttpSecurity.authorizeRequests()
说明: 允许基于使用HttpServletRequest限制访问
```

```java
方法: HttpSecurity.openidLogin()
说明: 用于基于 OpenId 的验证
```

```java
方法: HttpSecurity.headers()
说明: 将安全标头添加到响应
```

```java
方法: HttpSecurity.cors()
说明: 配置跨域资源共享(CORS)
```

```java
方法: HttpSecurity.sessionManagement()
说明: 允许配置会话管理
```

```java
方法: HttpSecurity.portMapper()
说明: 
允许配置一个PortMapper(HttpSecurity#(getSharedObject(class)))
其它提供SecurityConfigurer的对象使用PortMapper从HTTP重定向到HTTPS或者HTTPS重定向到HTTP
默认情况下,使用一个PortMapperImpl映射HTTP端口8080到HTTPS端口8443,HTTP端口80到HTTPS端口443
```

```java
方法: HttpSecurity.jee()
说明: 配置基于容器的预认证,在这种情况下,认证由Servlet容器管理
```

```java
方法: HttpSecurity.x509()
说明: 配置基于x509的认证
```

```java
方法: HttpSecurity.rememberMe()
说明: 允许配置“记住我”的验证
```

```java
方法: HttpSecurity.requestCache()
说明: 允许配置请求缓存
```

```java
方法: HttpSecurity.exceptionHandling()
说明: 允许配置错误处理
```

```java
方法: HttpSecurity.securityContext()
说明: 
在HttpServletRequests之间的SecurityContextHolder上设置SecurityContext的管理
当使用WebSecurityConfigurerAdapter时,这将自动应用
```

```java
方法: HttpSecurity.servletApi()
说明: 
将HttpServletRequest方法与在其上找到的值集成到SecurityContext中
当使用WebSecurityConfigurerAdapter时,这将自动应用
```

```java
方法: HttpSecurity.csrf()
说明: 添加CSRF支持,使用WebSecurityConfigurerAdapter时,默认启用
```

```java
方法: HttpSecurity.logout()
说明: 
添加退出登录支持
当使用WebSecurityConfigurerAdapter时,这将自动应用
默认情况是,访问URL“/ logout”
使HTTP Session无效来清除用户
清除已配置的任何HttpSecurity.rememberMe()身份验证
清除SecurityContextHolder
然后重定向到“/login?success”
```

```java
方法: HttpSecurity.anonymous()
说明: 
允许配置匿名用户的表示方法
当与WebSecurityConfigurerAdapter结合使用时,这将自动应用
默认情况下,匿名用户将使用AnonymousAuthenticationToken对象,并包含角色“ROLE_ANONYMOUS”
```

```java
方法: HttpSecurity.formLogin()
说明:
指定支持基于表单的身份验证
如果未指定HttpSecurity.loginPage(String),则将生成默认登录页面
```

```java
方法: HttpSecurity.oauth2Login()
说明: 根据外部OAuth 2.0或OpenID Connect 1.0提供程序配置身份验证
```

```java
方法: HttpSecurity.requiresChannel()
说明: 
配置通道安全
为了使该配置有用,必须提供至少一个到所需信道的映射
```

```java
方法: HttpSecurity.httpBasic()
说明: 配置Http Basic验证
```

```java
方法: HttpSecurity.addFilterAt()
说明: 在指定的Filter类的位置添加过滤器
```

## 2.4 HttpSecurity 类-常用链式方法说明
```java
方法: HttpSecurity.authorizeRequests().antMatchers(String...);
说明: 访问的路径需要进行权限的限制
```

```java
方法-1: HttpSecurity.authorizeRequests().antMatchers().permitAll();
方法-2: HttpSecurity.formLogin().permitAll()
说明: 所有用户都可以访问的页面
```

```java
方法: HttpSecurity.authorizeRequests().antMatchers().hasRole(String);
说明: 如果用户具备给定角色就允许访问
```

```java
方法: HttpSecurity.authorizeRequests().antMatchers().hasAuthority(String);
说明: 判断用户是否具有特定的权限,用户的权限是在自定义登录逻辑中创建User对象时指定的
```

```java
方法: HttpSecurity.authorizeRequests().antMatchers().hasAnyAuthority(String...);
说明: 如果用户具备给定权限中某一个,就允许访问
```

```java
方法: HttpSecurity.authorizeRequests().antMatchers().hasAnyRole(String...);
说明: 如果用户具备给定角色的任意一个,就允许被访问
```

```java
方法: HttpSecurity.authorizeRequests().antMatchers().hasIpAddress(String);
说明: 
如果请求是指定的IP就运行访问,可以通过request.getRemoteAddr()获取ip地址
需要注意的是在本机进行测试时localhost和127.0.0.1输出的ip地址是不一样的
```

```java
方法: and();
说明: 在末端返回一个getBuilder(),因此,可以使用该“and()”方法链接其他配置器

使用方法例子:
HttpSecurity.authorizeRequests()
            .anyRequest().authenticated()
            .and()
            .formLogin()
            .loginProcessingUrl("/doLogin")
            .permitAll()
            .and()
            .logout()
            .logoutUrl("/logout")
            .permitAll()
            .and()
            .csrf().disable();
```

```java
注意: 
hasRole和hasAuthority的不同之处在于
hasRole是判断用户是否有这个角色,会自动加上“ROLE_”前缀
hasAuthority则是判断用户是否有权限,不会自动加上“ROLE_”前缀
```
