## 一、核心前提
### 1. 通讯基础
+ 服务端与客户端通讯必须绑定处理器（Handler）：
    - 服务端：通过 `ServerBootstrap.childHandler()` 绑定
    - 客户端：通过 `Bootstrap.handler()` 绑定
+ 绑定后监听通道（Channel），数据触发对应事件后执行后续业务处理

### 2. 学习建议
+ 入门阶段无需深入消化细节，先建立核心类与方法的认知框架
+ 重点关注事件触发场景与方法核心作用

## 二、核心类：IdleStateHandler（空闲状态处理器）
### 1. 基础信息
+ 所在包：`io.netty.handler.timeout`
+ 核心功能：监控 Channel 空闲状态，指定时间内无读 / 写操作时触发 `IdleStateEvent` 事件
+ 关联枚举：`IdleState`（定义 3 种空闲状态）

### 2. 构造方法与参数解析
构造方法：`IdleStateHandler(long readerIdleTime, long writerIdleTime, long allIdleTime, TimeUnit unit)`

| **参数名** | **触发条件** | **对应状态** | **禁用方式** |
| --- | --- | --- | --- |
| readerIdleTime | 指定时间内无读操作 | `IdleState.READER_IDLE` | 设置为 0 |
| writerIdleTime | 指定时间内无写操作 | `IdleState.WRITER_IDLE` | 设置为 0 |
| allIdleTime | 指定时间内无读 / 写操作 | `IdleState.ALL_IDLE` | 设置为 0 |


## 三、核心类：ChannelInboundHandlerAdapter（入站处理器适配器）
### 1. 辅助知识：ChannelHandlerContext（通道上下文）
#### （1）核心作用
+ 通道的程序上下文对象，用于通道事件传递与数据交互

#### （2）关键方法

| **方法** | **作用** | **业务场景** |
| --- | --- | --- |
| `fireChannelRead(xxx)` | 提交事件 / 数据给下一个 Channel 处理 | 业务未结束，需后续处理器继续处理 |
| `writeAndFlush(xxx)` | 出站操作：先写 Netty 缓冲区，再刷到 Socket 缓冲区 | 响应客户端数据，本次业务结束（不传递给下一个 Channel） |


### 2. 事件传递规则
+ 类中方法执行 `super.xxxx(xx,...)` 可将事件传递给下一个 Channel
+ 若需传递事件，`ChannelHandlerContext.fireChannelRead(xxx)` 与 `super.xxxx(xx,...)` 二选一（避免通道断开）
+ 示例代码：

```java
public class XxxHandler extends ChannelInboundHandlerAdapter {
    @Override
    public void channelRegistered(ChannelHandlerContext ctx) throws Exception {
        super.channelRegistered(ctx); // 事件传递给下一个 Channel
    }

    @Override
    public void channelRead(ChannelHandlerContext ctx, Object msg) throws Exception {
        super.channelRead(ctx, msg); // 数据传递给下一个 Channel
    }
}
```

### 3. 核心方法清单（按功能分类）
#### （1）通道生命周期相关

| **方法** | **触发场景** | **核心作用** |
| --- | --- | --- |
| `channelRegistered(ChannelHandlerContext ctx)` | Channel 注册时 | 通道初始化注册回调 |
| `channelUnregistered(ChannelHandlerContext ctx)` | Channel 取消注册时 | 通道注销回调 |
| `channelActive(ChannelHandlerContext ctx)`<br/>（常用） | Channel 激活时（连接建立成功） | 连接建立后的初始化操作（如发送问候消息） |
| `channelInactive(ChannelHandlerContext ctx)` | 1. 服务端调用 `ctx.close()`<br/>；2. 客户端发送关闭帧；3. 客户端进程结束 | 连接断开后的资源释放操作 |


#### （2）数据读写相关

| **方法** | **触发场景** | **核心作用** |
| --- | --- | --- |
| `channelRead(ChannelHandlerContext ctx, Object msg)`<br/>（常用） | 收到客户端 / 服务端发送的数据时 | 核心数据处理逻辑（如解析请求参数） |
| `channelReadComplete(ChannelHandlerContext ctx)` | 数据读取完毕时 | 读取完成后的收尾操作（如批量提交数据） |
| `channelWritabilityChanged(ChannelHandlerContext ctx)` | Channel 读写能力变化时（如缓冲区满 / 空） | 动态调整读写策略（如暂停写操作避免溢出） |


#### （3）特殊事件与异常处理

| **方法** | **触发场景** | **核心作用** |
| --- | --- | --- |
| `userEventTriggered(ChannelHandlerContext ctx, Object evt)` | 触发自定义事件（如 IdleStateEvent） | 常用于心跳检测（检测空闲连接） |
| `exceptionCaught(ChannelHandlerContext ctx, Throwable cause)`<br/>（常用） | Channel 捕获到异常时（如网络中断、数据格式错误） | 异常处理（如打印日志、关闭连接、返回错误响应） |


### 4. 常用方法伪代码示例
#### （1）channelActive（连接激活）
```java
@Override
public void channelActive(ChannelHandlerContext ctx) throws Exception {
    System.out.println("连接建立成功，channelActive() 触发");
    super.channelActive(ctx);
    // 可选：连接建立后发送初始化消息
    ctx.writeAndFlush("连接已建立，欢迎使用！");
}
```

#### （2）channelRead（数据接收）
```java
@Override
public void channelRead(ChannelHandlerContext ctx, Object msg) throws Exception {
    System.out.println("收到数据：" + msg);
    super.channelRead(ctx, msg);
    // 业务处理：如解析数据、调用服务、构造响应
    String response = "已收到数据：" + msg;
    ctx.writeAndFlush(response); // 响应客户端
}
```

#### （3）exceptionCaught（异常捕获）
```java
@Override
public void exceptionCaught(ChannelHandlerContext ctx, Throwable cause) throws Exception {
    System.out.println("通道发生异常：" + cause.getMessage());
    super.exceptionCaught(ctx, cause);
    ctx.close(); // 关闭通道释放资源
}
```

#### （4）userEventTriggered（心跳检测）
```java
@Override
public void userEventTriggered(ChannelHandlerContext ctx, Object evt) throws Exception {
    super.userEventTriggered(ctx, evt);
    if (evt instanceof IdleStateEvent) {
        IdleStateEvent event = (IdleStateEvent) evt;
        if (event.state() == IdleState.READER_IDLE) {
            System.out.println("读空闲超时，关闭连接");
            ctx.close(); // 关闭空闲连接
        }
    }
}
```
