#### 1. 自定义处理器 1：NettyServer1Handler.java
+ 路径：`./项目/src/main/java/Netty/Server/Handler/NettyServer1Handler.java`

```java
// 第一步
// 创建测试自定义Handler1
// 路径: ./项目/src/main/java/Netty/Server/Handler/NettyServer1Handler.java
package Netty.Server.Handler;

import io.netty.buffer.ByteBuf;
import io.netty.channel.ChannelHandlerContext;
import io.netty.channel.ChannelInboundHandlerAdapter;
import io.netty.util.CharsetUtil;

public class NettyServer1Handler extends ChannelInboundHandlerAdapter {
    /**
     * Channel(频道)激活时触发
     * 简单理解就是当客户端去和服务端连接成功时触发
     * 注意:
     * 在不执行super.xxxx(xx,...)的情况下,该方法只有在连接成功后会执行一次
     *
     * @param ctx
     * @throws Exception
     */
    @Override
    public void channelActive(ChannelHandlerContext ctx) throws Exception {
        System.out.println("NettyServer1Handler类的channelActive()方法触发了");
    }

    /**
     * 客户端/服务端,发送数据过来时触发
     * 注意:
     * 在该方法如果执行了,ChannelHandlerContext.fireChannelRead(),那么说明业务还没结束,就可以传递给下一个Channel(频道)继续执行业务
     * 在该方法如果执行了,ChannelHandlerContext.writeAndFlush(),表示响应给客户端的数据,那么就认为业务结束了,就不在传递给下一个Channel(频道)了
     *
     * @param ctx
     * @param msg
     * @throws Exception
     */
    @Override
    public void channelRead(ChannelHandlerContext ctx, Object msg) throws Exception {
        System.out.println("NettyServer1Handler类的channelRead()方法触发了");

        ByteBuf buf = (ByteBuf) msg;
        String body = buf.toString(CharsetUtil.UTF_8);
        System.out.println("NettyServer1Handler类的channelRead()方法,接收到客户端发送过来的消息是:" + body);

        ctx.fireChannelRead(msg);
    }

    /**
     * 数据读取完毕时的处理方法
     * 注意-1:
     * 在不执行super.xxxx(xx,...)的情况下,该方法只会执行一次
     * 注意-2:
     * 在该方法如果执行了,ChannelHandlerContext.fireChannelRead(),那么说明业务还没结束,就可以传递给下一个Channel(频道)继续执行业务
     * 在该方法如果执行了,ChannelHandlerContext.writeAndFlush(),表示响应给客户端的数据,那么就认为业务结束了,就不在传递给下一个Channel(频道)了
     *
     * @param ctx
     * @throws Exception
     */
    @Override
    public void channelReadComplete(ChannelHandlerContext ctx) throws Exception {
        System.out.println("NettyServer1Handler类的channelReadComplete()方法触发了");
    }

    /**
     * 用户超时长时间未操作时则会触发,常用于心跳检测
     *
     * @param ctx
     * @param evt
     * @throws Exception
     */
    @Override
    public void userEventTriggered(ChannelHandlerContext ctx, Object evt) throws Exception {
        System.out.println("NettyServer1Handler类的userEventTriggered()方法触发了,因为已经5秒未收到客户端的消息了!");

        // 将当前频道关闭
        // 会导致客户端,关闭连接
        // ctx.channel().close();
    }

    /**
     * Channel(频道)捕获到异常时触发
     *
     * @param ctx
     * @param cause
     * @throws Exception
     */
    @Override
    public void exceptionCaught(ChannelHandlerContext ctx, Throwable cause) throws Exception {
        System.out.println("NettyServer1Handler类的exceptionCaught()方法触发了");
        cause.printStackTrace();
        ctx.close();
    }
}
```

+ 核心方法及说明：

| **方法名** | **触发时机** | **关键逻辑** |
| --- | --- | --- |
| `channelActive` | 客户端与服务端连接成功时 | 仅执行一次，打印连接成功日志 |
| `channelRead` | 接收客户端 / 服务端发送的数据时 | 1. 解析接收的 ByteBuf 数据为 UTF-8 字符串；2. 打印接收的消息；3. 通过 `ctx.fireChannelRead(msg)`<br/> 传递数据给下一个处理器 |
| `channelReadComplete` | 数据读取完毕时 | 仅执行一次，可选择传递业务或直接响应客户端 |
| `userEventTriggered` | 超时未操作时（心跳检测） | 服务端 5 秒未收消息触发，打印日志（默认不关闭连接） |
| `exceptionCaught` | 捕获到异常时 | 打印异常堆栈，关闭通道释放资源 |

import io.netty.buffer.Unpooled;
import io.netty.channel.ChannelHandlerContext;
import io.netty.channel.ChannelInboundHandlerAdapter;
import io.netty.util.CharsetUtil;

public class NettyServer2Handler extends ChannelInboundHandlerAdapter {
    @Override
    public void channelRead(ChannelHandlerContext ctx, Object msg) throws Exception {

        data += "NettyServer2Handler类的channelRead()方法触发了,";
        data += "输出给客户端:S-456";
        ByteBuf buf = Unpooled.copiedBuffer(data.getBytes(CharsetUtil.UTF_8));
        ctx.writeAndFlush(buf);

    @Override
    public void exceptionCaught(ChannelHandlerContext ctx, Throwable cause) throws Exception {
        System.out.println("NettyServer2Handler类的exceptionCaught()方法触发了");
        cause.printStackTrace();
        ctx.close();
    }
}

    - `exceptionCaught`：捕获异常，打印日志并关闭通道。

#### 3. 服务端启动类：Server.java
+ 路径：`./项目/src/main/java/Netty/Server/Server.java`

```java
// 第三步
// 创建测试服务端
// 路径: ./项目/src/main/java/Netty/Server/Server.java
package Netty.Server;

import io.netty.bootstrap.ServerBootstrap;
import io.netty.channel.ChannelFuture;
import io.netty.channel.ChannelInitializer;
import io.netty.channel.ChannelOption;
import io.netty.channel.EventLoopGroup;
import io.netty.channel.nio.NioEventLoopGroup;
import io.netty.channel.socket.SocketChannel;
import io.netty.channel.socket.nio.NioServerSocketChannel;

import Netty.Server.Handler.NettyServer1Handler;
import Netty.Server.Handler.NettyServer2Handler;
import io.netty.handler.timeout.IdleStateHandler;

import java.util.concurrent.TimeUnit;

public class Server {
    public static void main(String[] args) throws InterruptedException {
        System.out.println("服务端启动中...");

        // 用户接收客户端连接的线程工作组
        EventLoopGroup bossGroup = new NioEventLoopGroup();
        // 用于接收客户端连接读写操作的线程组
        EventLoopGroup workerGroup = new NioEventLoopGroup();
        // 启动器,负责装配netty组件,启动服务器
        ServerBootstrap serverBootstrap = new ServerBootstrap();

        try {
            serverBootstrap.group(bossGroup, workerGroup)
                    // 选择服务器的 ServerSocketChannel 实现
                    .channel(NioServerSocketChannel.class)
                    // option()针对于服务端配置, childOption()针对于客户端连接通道配置
                    // 设置服务端的tcp缓冲区大小
                    .option(ChannelOption.SO_BACKLOG, 1024)
                    // ChannelInitializer()处理器(仅执行一次)
                    // 它的作用是待客户端SocketChannel建立连接后,执行initChannel以便添加更多的处理器
                    .childHandler(new ChannelInitializer<SocketChannel>() {
                        @Override
                        protected void initChannel(SocketChannel sc) throws Exception {
                            // 设定IdleStateHandler心跳检测每五秒进行一次读检测
                            sc.pipeline().addLast(new IdleStateHandler(5, 0, 0, TimeUnit.SECONDS));
                            // SocketChannel的处理器,业务处理,数据传输过来的时候会进行拦截和执行(可以有多个拦截器)
                            sc.pipeline().addLast(new NettyServer1Handler());
                            sc.pipeline().addLast(new NettyServer2Handler());
                        }
                    });

            // 监听频道
            ChannelFuture cf = serverBootstrap.bind(8765).sync();

            System.out.println("服务端启动成功");

            // 执行完,使主线程变为wait状态,然后子线程去监听和接受请求
            cf.channel().closeFuture().sync();
        } finally {
            // 资源优雅释放
            workerGroup.shutdownGracefully();
            bossGroup.shutdownGracefully();
        }
    }
}
```

+ 核心逻辑：
    1. 创建两个 EventLoopGroup 线程组：
        * `bossGroup`：负责接收客户端连接。
        * `workerGroup`：负责处理客户端连接的读写操作。
    2. 通过 `ServerBootstrap` 装配 Netty 组件：
        * 指定服务端通道实现 `NioServerSocketChannel`。
        * 配置 TCP 缓冲区大小 `SO_BACKLOG=1024`。
        * 添加处理器链：
            + `IdleStateHandler(5, 0, 0, TimeUnit.SECONDS)`：5 秒读超时检测（心跳）。
            + 自定义处理器 `NettyServer1Handler` 和 `NettyServer2Handler`。
    3. 绑定端口 8765，同步启动服务端。
    4. 优雅关闭资源：服务端停止时，关闭 `bossGroup` 和 `workerGroup`。

```java
// 第三步
// 创建测试客户端
// 路径: ./项目/src/main/java/Netty/Client/Client.java
package Netty.Client;

import io.netty.bootstrap.Bootstrap;
import io.netty.channel.ChannelFuture;
import io.netty.channel.ChannelInitializer;
import io.netty.channel.EventLoopGroup;
import io.netty.channel.nio.NioEventLoopGroup;
import io.netty.channel.socket.SocketChannel;
import io.netty.channel.socket.nio.NioSocketChannel;

import Netty.Client.Handler.NettyClient1Handler;
import Netty.Client.Handler.NettyClient2Handler;
import io.netty.handler.timeout.IdleStateHandler;

import java.util.concurrent.TimeUnit;

public class Client {
    public static void main(String[] args) throws InterruptedException {
        // 用于接收客户端连接读写操作的线程组
        EventLoopGroup workerGroup = new NioEventLoopGroup();
        try {
            // 辅助类,帮助创建netty服务
            Bootstrap bootstrap = new Bootstrap();
            bootstrap.group(workerGroup)
                    // 选择客户端Socket实现类,NioSocketChannel表示基于NIO的客户端实现
                    .channel(NioSocketChannel.class)
                    // ChannelInitializer()处理器(仅执行一次)
                    // 它的作用是待客户端SocketChannel建立连接后,执行initChannel以便添加更多的处理器
                    .handler(new ChannelInitializer<SocketChannel>() {
                        @Override
                        protected void initChannel(SocketChannel sc) throws Exception {
                            // 设定IdleStateHandler心跳检测每八秒进行一次写检测
                            sc.pipeline().addLast(new IdleStateHandler(0, 8, 0, TimeUnit.SECONDS));
                            // SocketChannel的处理器,业务处理,数据传输过来的时候会进行拦截和执行(可以有多个拦截器)
                            sc.pipeline().addLast(new NettyClient1Handler());
                            sc.pipeline().addLast(new NettyClient2Handler());
                        }
                    });

            // 连接的服务器和端口
            ChannelFuture cf = bootstrap.connect("127.0.0.1", 8765).sync();

            // 执行完,使主线程变为wait状态,然后子线程去监听和接受请求
            cf.channel().closeFuture().sync();
        } finally {
            workerGroup.shutdownGracefully();
        }
    }
}
```

+ 核心逻辑：
    1. 创建 `workerGroup` 线程组（客户端仅需处理读写操作）。
    2. 通过 `Bootstrap` 装配 Netty 组件：
        * 指定客户端通道实现 `NioSocketChannel`。
        * 添加处理器链：
            + `IdleStateHandler(0, 8, 0, TimeUnit.SECONDS)`：8 秒写超时检测（心跳）。
            + 自定义处理器 `NettyClient1Handler` 和 `NettyClient2Handler`。
    3. 连接服务端地址 `127.0.0.1:8765`，同步建立连接。
    4. 优雅关闭资源：客户端停止时，关闭 `workerGroup`。

## 六、核心要点总结
1. **处理器链机制**：Netty 通过 `ChannelPipeline` 管理处理器链，`ctx.fireChannelRead(msg)` 可将数据传递给下一个处理器，`ctx.writeAndFlush()` 直接响应数据并终止传递。
2. **心跳检测**：通过 `IdleStateHandler` 实现超时检测，服务端检测读超时（5 秒），客户端检测写超时（8 秒），用于处理空闲连接。
3. **资源管理**：使用 `EventLoopGroup` 管理线程，启动后需通过 `shutdownGracefully()` 优雅关闭，避免资源泄露。
4. **数据传输**：Netty 中数据以 `ByteBuf` 形式传输，需通过 `CharsetUtil.UTF_8` 手动解析为字符串。
