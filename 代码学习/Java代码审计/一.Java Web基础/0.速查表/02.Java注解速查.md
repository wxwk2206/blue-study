# Java注解速查（按来源分类）

> [!abstract] 概述
> 按来源包/框架分类，梳理 Java Web 开发与代码审计中高频注解。**注解本身不执行逻辑**，靠运行时反射或编译期处理器（APT，Annotation Processing Tool）解析生效，审计时重点关注注解如何改变程序行为。

---

## 一、注解基础

### 1.1 注解是什么？

**概括**：注解（Annotation）是 Java 5 引入的元数据（Metadata）标记机制，用 `@` 符号声明，附加到类/方法/字段/参数上，不直接影响代码执行。

**比喻**：注解就像贴在快递箱上的标签纸——"易碎品""向上放置""冷藏运输"。箱子里的东西（代码逻辑）不受标签影响，但分拣员（编译器/框架）看到标签会据此改变处理方式。

**细节**：

```java
// 注解本质上是继承了 java.lang.annotation.Annotation 的接口
@Override                 // 告诉编译器：我覆盖了父类方法
public String toString() {
    return "hello";
}
```

### 1.2 注解三要素

| 要素 | 说明 | 决定权 |
|------|------|------|
| **Target**（目标） | 能贴在哪儿：类/方法/字段/参数... | `@Target` 元注解 |
| **Retention**（生命周期） | 保留到哪一层：源码/字节码/运行时 | `@Retention` 元注解 |
| **处理器**（Processor） | 谁来解析执行：编译器/APT/运行时反射 | 框架实现 |

> [!important] 审计要点
> `@Retention(RUNTIME)` 的注解在运行时可通过反射读取，是 Spring、Hibernate 等框架实现 AOP/DI 的基石，也是绕过安全检测的潜在路径——某些配置可能被注解覆盖而隐蔽生效。

## 二、Java 内置注解

### 2.1 元注解（java.lang.annotation）

> 元注解（Meta-Annotation）：**注解的注解**，用来定义注解的行为规则。

| 元注解 | 作用 | 参数 |
|------|------|------|
| `@Target` | 指定注解能用于哪些元素 | `ElementType.TYPE`（类）、`FIELD`（字段）、`METHOD`（方法）、`PARAMETER`（参数）、`CONSTRUCTOR`（构造）、`ANNOTATION_TYPE`（注解）、`TYPE_USE`（任何类型使用处，Java 8+） |
| `@Retention` | 指定注解保留到哪个阶段 | `SOURCE`（源码，编译后丢弃）、`CLASS`（字节码，JVM 不加载）、⭐ `RUNTIME`（运行时可通过反射读取） |
| `@Documented` | 标记后会被 javadoc 文档化 | 无参数 |
| `@Inherited` | 子类自动继承父类的该注解 | 无参数 |
| `@Repeatable`（Java 8+） | 同一位置可重复标注 | 指定容器注解 Class |

```java
// ⭐ 一个典型自定义注解的完整声明
@Target({ElementType.TYPE, ElementType.METHOD})   // 可用于类和方法
@Retention(RetentionPolicy.RUNTIME)                // 运行时保留 → 可被反射读取
@Documented                                        // 进 javadoc
public @interface MyAnnotation {
    String value() default "";                     // 属性
    int priority() default 0;
}
```

### 2.2 编译器注解
（java.lang）

| 注解 | 作用 | 审计关注 |
|------|------|------|
| `@Override` | 声明方法覆盖父类/接口方法 | 编译检查，避免拼写错误导致"伪重写" |
| `@Deprecated` | 标记已过时的 API | 调用过时方法可能存在已知安全问题 |
| `@SuppressWarnings` | 抑制编译器警告 | `"unchecked"` / `"deprecation"` 可能掩盖潜在问题 ⭐ |
| `@SafeVarargs`（Java 7+） | 声明可变参数方法线程安全 | 抑制堆污染警告 |
| `@FunctionalInterface`（Java 8+） | 标记函数式接口（只有一个抽象方法） | 仅编译期校验 |

 ## 四、Servlet 3.0+ Web 注解
（javax.servlet.annotation / jakarta.servlet.annotation）

> [!tip]
> Servlet 3.0 起可以用注解替代 web.xml 配置。审计时关注：**注解注册的 Servlet/Filter 可能不容易被发现**。

| 注解 | 作用 | 关键属性 |
|------|------|------|
| `@WebServlet` | ⭐ 声明 Servlet（替代 web.xml `<servlet>`） | `name`、`urlPatterns`、`loadOnStartup`、`initParams` |
| `@WebFilter` | ⭐ 声明过滤器 | `filterName`、`urlPatterns`、`dispatcherTypes` |
| `@WebListener` | 声明监听器 | 实现 `ServletContextListener` 等接口 |
| `@WebInitParam` | 初始化参数（配合上面三个） | `name`、`value` |
| `@MultipartConfig` | 标记 Servlet 支持文件上传 | `maxFileSize`、`maxRequestSize`、`location` |
| `@HandlesTypes` | 声明 ServletContainerInitializer 处理的类型 | `value = {MyInterface.class}` |
| `@HttpConstraint` | 安全约束 | `rolesAllowed`、`transportGuarantee` |
| `@ServletSecurity` | Servlet 安全注解 | ⭐ 声明角色权限 |

```java
// ⭐ @WebServlet 示例 —— 无 web.xml 也能注册 Servlet
@WebServlet(
    name = "loginServlet",
    urlPatterns = {"/login", "/auth"},
    loadOnStartup = 1,
    initParams = {
        @WebInitParam(name = "encoding", value = "UTF-8")
    }
)
public class LoginServlet extends HttpServlet { ... }
```

## 五、Spring Framework 核心注解
（org.springframework）

### 5.1 组件注册与 IOC

| 注解 | 作用 | 等价 XML 配置 |

| `@Import` | 导入其他配置类 | `<import resource="...">` |
| `@Scope` | Bean 作用域 | `singleton`（默认单例）、`prototype`（每次新建） |

| `@Primary` | 同类型多 Bean 时的首选 | 解决 `@Autowired` 歧义 |

| `@PreDestroy` | Bean 销毁前回调 | JSR 250，等价 `destroy-method` |

### 5.2 依赖注入（DI）

| 注解 | 注入方式 | 说明 |
|------|------|------|
| `@Autowired` | 按类型（byType） | ⭐ Spring 原生，默认必须注入（可 `required = false`） |
| `@Qualifier` | 按名称（byName） | 配合 `@Autowired` 解决同类型多 Bean 歧义 |
| `@Resource` | 先按名称，再按类型 | JSR 250 标准，非 Spring 专属；`name` 属性指定 Bean 名 |
| `@Inject` | 按类型 | JSR 330 标准，需额外引入 javax.inject 包 |
| `@Value` | 注入配置值 | `@Value("${server.port}")`，⭐ 支持 SpEL 表达式（可执行代码） |
| `@PropertySource` | 加载 .properties 文件 | `@PropertySource("classpath:db.properties")` |

```java
// ⭐ 三种注入方式对比
@Service
public class UserService {
    // 方式一：字段注入（最常见，但反射破坏封装）
    @Autowired
    private UserDao userDao;

    // 方式二：构造器注入（推荐，不可变 + 易测试）
    private final RoleDao roleDao;
    public UserService(RoleDao roleDao) {
        this.roleDao = roleDao;
    }

    // 方式三：Setter 注入
    private LogDao logDao;
    @Autowired
    public void setLogDao(LogDao logDao) {
        this.logDao = logDao;
    }
}
```

### 5.3 AOP 切面
（org.aspectj.lang.annotation）

| 注解 | 作用 | 说明 |
|------|------|------|
| `@Aspect` | 声明切面类 | 配合 `@Component` 注册 |
| `@Pointcut` | 定义切入点表达式 | `@Pointcut("execution(* com.xxx.*.*(..))")` |
| `@Before` | 前置通知 | ⭐ 目标方法执行前 |
| `@After` | 后置通知（无论成功失败） | 类似 finally |

```java
    }
}
```

### 5.4 事务管理
（org.springframework.transaction.annotation）

| 注解 | 作用 | 审计关注 |
|------|------|------|
| `@Transactional` | ⭐ 声明式事务 | ⭐ 审计重点：`propagation` 传播行为可能导致事务失效 |
| `@EnableTransactionManagement` | 开启注解驱动事务 | 通常 Spring Boot 自动配置 |

`@Transactional` 关键属性：

| 属性 | 值 | 说明 |
|------|------|------|
| `propagation` | `REQUIRED`（默认）、`REQUIRES_NEW`、`NESTED`、`SUPPORTS` | ⭐ 传播行为 |
| `isolation` | `DEFAULT`、`READ_UNCOMMITTED`、`READ_COMMITTED`、`REPEATABLE_READ`、`SERIALIZABLE` | ⭐ 隔离级别（`READ_UNCOMMITTED` 有脏读风险） |
| `readOnly` | `true` / `false`（默认） | 只读事务（`true` 时不能写操作） |
| `timeout` | 秒数 | 超时自动回滚 |
| `rollbackFor` | Exception 类 | ⭐ 指定哪些异常触发回滚（默认只回滚 RuntimeException） |

> [!warning] 事务失效的常见场景
> 1. `@Transactional` 加在非 public 方法上 → AOP 无法代理
> 2. 同一个类中内部方法调用 → 不经过代理对象
> 3. 异常被 catch 吞掉 → 没有抛出到代理层
> 4. `rollbackFor` 不匹配 → 抛了受检异常但没指定回滚

### 6.2 参数绑定（外部输入入口）

| 注解 | 绑定来源 | 审计关注 |
|------|------|------|
| `@RequestParam` | URL 查询参数 / POST 表单 | ⭐ 基本输入入口：`name`、`required`、`defaultValue` |
| `@PathVariable` | URL 路径变量（RESTful） | ⭐ `/user/{id}` → `@PathVariable Long id` |
| `@RequestBody` | HTTP 请求体（JSON/XML） | ⭐ POST JSON 注入、反序列化攻击 |
| `@RequestHeader` | 请求头 | ⭐ Host、X-Forwarded-For 等可能伪造 |
| `@CookieValue` | Cookie 值 | ⭐ Cookie 注入 |
| `@ModelAttribute` | 表单/查询参数 → JavaBean | ⭐ 自动绑定可能触发"超量绑定"（Mass Assignment） |
| `@SessionAttribute` | HttpSession 属性 | 从 Session 中取值 |
| `@RequestAttribute` | Request 属性 | 从 `request.setAttribute()` 取值 |
| `@MatrixVariable` | URL 矩阵变量 | `/cars;color=red;year=2012` |

```java
// ⭐ 审计关注：参数不校验 = 漏洞入口
@PostMapping("/user/create")
public String createUser(
    @RequestParam("name") String name,          // ← 外部输入，需校验
    @RequestBody UserDTO dto,                    // ← 反序列化入口，需校验
    @RequestHeader("X-Forwarded-For") String ip, // ← 可能伪造
    @CookieValue("JSESSIONID") String sessionId  // ← Cookie 输入
) { ... }
```

## 八、Bean Validation 数据校验
（javax.validation.constraints / jakarta.validation.constraints）

> [!important] 
> 校验注解是防 XSS/SQL 注入的第一道防线——但如果校验被禁用或遗漏，输入就直达业务代码。

| 注解 | 校验规则 | 示例 |
|------|------|------|
| `@NotNull` | 不为 null | `@NotNull String name` |
| `@NotEmpty` | 不为 null 且不为空串（`""`） | 字符串/集合/Map |
| `@NotBlank` | 不为空且至少含一个非空白字符 | `@NotBlank String username` |
| `@Null` | 必须为 null | 少见，常用于分组校验 |
| `@Size(min, max)` | 字符串长度/集合大小 | `@Size(min=2, max=30) String name` |
| `@Min(value)` / `@Max(value)` | 数值最小/最大值 | `@Min(0) int age` |
| `@DecimalMin` / `@DecimalMax` | 精确数值范围 | `@DecimalMin("0.01") BigDecimal price` |
| `@Positive` / `@Negative` | 正/负数 | `@Positive Long id` |
| `@PositiveOrZero` / `@NegativeOrZero` | ≥0 / ≤0 | — |
| `@Email` | 邮箱格式 | `@Email String email` |
| `@Pattern(regexp)` | ⭐ 正则匹配 | `@Pattern(regexp = "^[a-zA-Z0-9]+$")` |
| `@Digits(integer, fraction)` | 数字整数+小数位数 | `@Digits(integer=5, fraction=2)` |
| `@Past` / `@Future` | 日期是过去/未来 | `@Past LocalDate birthday` |
| `@AssertTrue` / `@AssertFalse` | 布尔断言 | 自定义复杂校验 |
| `@Valid` | ⭐ 级联校验（嵌套对象也要校验） | `@Valid @RequestBody UserDTO dto` |
| `@Validated`（Spring） | Spring 扩展，支持分组校验 | `@Validated(GroupA.class)` |

```java
// ⭐ 审计关注：Controller 是否加了 @Valid / @Validated
@PostMapping("/register")
public Result register(@Valid @RequestBody UserDTO dto) {
    // @Valid 确保 dto 内部校验注解生效
    // 漏写 → 所有 @NotNull / @Size 等全部不生效！
}
```

| `@JsonCreator` | 指定反序列化时用的构造器/工厂方法 | ⭐ 构造函数注入配合 |
| `@JsonTypeInfo` / `@JsonSubTypes` | 多态反序列化 | ⭐ 审计高危！多态反序列化是 Jackson RCE 的根源 |

## 十一、Lombok 注解（lombok）

> 编译时生成代码，减少样板代码。

| 注解 | 生成内容 | 等效手写代码 |

## 十二、Spring Security 注解
（org.springframework.security.access.prepost）

| 注解                            | 作用                 | 说明                                                        |
| ----------------------------- | ------------------ | --------------------------------------------------------- |
| `@EnableWebSecurity`          | 开启 Spring Security | 配合 `@Configuration`                                       |
| `@EnableGlobalMethodSecurity` | ⭐ 开启方法级安全注解        | `prePostEnabled = true` 启用 `@PreAuthorize`                |
| `@PreAuthorize`               | 方法执行前权限校验          | ⭐ `@PreAuthorize("hasRole('ADMIN')")`                     |
| `@PostAuthorize`              | 方法执行后权限校验          | 返回值校验                                                     |
| `@PreFilter`                  | 方法执行前过滤集合参数        | `@PreFilter("filterObject.owner == authentication.name")` |
| `@PostFilter`                 | 方法执行后过滤返回集合        | —                                                         |
| `@Secured`                    | 角色检查               | `@Secured("ROLE_ADMIN")`（不如 `@PreAuthorize` 灵活）           |
| `@RolesAllowed`               | JDK 标准角色注解         | JSR 250                                                   |

```java
}
```

## 十三、自定义注解

### 13.1 定义语法

```java
import java.lang.annotation.*;
import java.lang.reflect.Method;

// ⭐ 自定义注解完整声明
@Target({ElementType.METHOD, ElementType.TYPE})   // 可用于方法、类
@Retention(RetentionPolicy.RUNTIME)                // 运行时保留
@Documented
public @interface AuditLog {
    String value() default "";                     // 操作描述
    String module() default "default";             // 所属模块
    boolean recordParams() default true;           // 是否记录参数
}
```

### 13.2 注解属性类型限制

| 允许的类型 | 不允许的类型 |
|------|------|
| 基本类型（int、boolean...） | 自定义对象 |
| String | `List<E>`（只能用数组） |
| Class | `Map<K,V>` |
| 枚举 | — |
| 注解（嵌套注解） | — |
| 以上类型的数组 | — |

### 13.3 运行时读取注解

```java
// 方法一：通过 Class 获取类上注解
AuditLog log = UserController.class.getAnnotation(AuditLog.class);

// 方法二：通过 Method 获取方法上注解
Method method = UserController.class.getMethod("createUser", UserDTO.class);
if (method.isAnnotationPresent(AuditLog.class)) {
    AuditLog audit = method.getAnnotation(AuditLog.class);
    System.out.println(audit.value());  // 获取属性值
}

// 方法三：获取所有注解
Annotation[] annotations = method.getAnnotations();
```

## 代码审计中的注解关注清单

| 关注点 | 相关注解 | 风险 |
|------|------|------|
| 外部输入入口 | `@RequestParam` `@PathVariable` `@RequestBody` `@RequestHeader` `@CookieValue` | 未经校验的参数直接进入业务逻辑 |
| 参数校验缺失 | `@Valid` / `@Validated` 漏写 | 校验全部失效 |
| 跨域配置 | `@CrossOrigin(origins = "*")` | 任意来源跨域 |
| Spring Security 权限 | `@PreAuthorize` / `@PostAuthorize` 配置错误或遗漏 | 越权 |
| 声明式事务失效 | `@Transactional` 非 public 方法 / 同类内部调用 | 数据不一致 |
| SpEL 表达式 | `@Value(SpEL)` / `@PreAuthorize(SpEL)` / `@ConditionalOnExpression(SpEL)` | SpEL 注入 → RCE |
| Jackson 多态 | `@JsonTypeInfo` + `@JsonSubTypes` | 反序列化 RCE |
| 敏感字段暴露 | `@JsonIgnore` / `@ToString.Exclude` 遗漏 | 密码/密钥泄露 |
| 隐蔽的 Servlet | `@WebServlet` / `@WebFilter` 动态注册 | 未在 web.xml 体现，安全审计死角 |
| URL 跳转 | `redirect:` / `return "redirect:" + userInput` | 开放重定向 |
