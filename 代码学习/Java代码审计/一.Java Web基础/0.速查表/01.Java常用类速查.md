## 一、java.lang —— 核心基础包（自动导入）

### 10. 异常体系

```plain
Throwable
├── Error          // 致命错误 → 不处理（OutOfMemoryError、StackOverflowError）
└── Exception
    ├── RuntimeException  // 运行时异常 → 不强制捕获（NullPointerException、IndexOutOfBoundsException）
    │   ├── NullPointerException
    │   ├── IllegalArgumentException
    │   ├── ClassCastException
    │   ├── ArithmeticException
    │   └── SecurityException    ⭐ 审计相关
    └── IOException             // 受检异常 → 必须处理
        ├── FileNotFoundException
        └── EOFException
```

> [!tip] 
> `Throwable.getMessage()` 获取异常信息，`printStackTrace()` 打印调用栈。注意生产环境不应暴露 `printStackTrace()` 输出版本细节。

## 二、java.util —— 集合框架与工具

### 1. 集合接口层次

```plain
Collection<E>
├── List<E>           // 有序、可重复
│   ├── ArrayList      // 数组实现，随机访问快
│   ├── LinkedList     // 双向链表，增删快
│   └── Vector         // 线程安全（已过时）
│       └── Stack      // 栈（LIFO）
│
├── Set<E>            // 无序、不可重复
│   ├── HashSet        // 哈希表，O(1) 查找
│   ├── LinkedHashSet  // 链表+哈希，保持插入顺序
│   └── TreeSet        // 红黑树，自动排序
│
└── Queue<E> / Deque<E>
    ├── PriorityQueue  // 优先队列
    └── ArrayDeque     // 双端队列（代替 Stack）
```

| 接口 | 关键方法 |
|------|------|
| `Collection` | `add(E)`、`remove(Object)`、`contains(Object)`、`size()`、`isEmpty()`、`clear()`、`iterator()` |
| `List` | `get(int)`、`set(int, E)`、`indexOf(Object)`、`subList(int, int)` |
| `Set` | （继承 Collection） |
| `Queue` | `offer(E)`、`poll()`、`peek()` |
| `Deque` | `addFirst(E)`、`addLast(E)`、`removeFirst()`、`removeLast()` |

### 6. Java 8 Stream API（java.util.stream）

> [!tip]
> Stream 不存储数据，只对数据源进行计算。惰性求值，终端操作触发执行。

```java
// 典型链式操作
list.stream()
    .filter(x -> x > 0)        // 中间操作：过滤
    .map(String::valueOf)       // 中间操作：转换
    .sorted()                   // 中间操作：排序
    .collect(Collectors.toList()); // 终端操作：收集
```

| 中间操作 | 说明 |
|------|------|
| `filter(Predicate)` | 过滤 |
| `map(Function)` | 转换每个元素 |
| `flatMap(Function)` | 将嵌套流扁平化 |
| `distinct()` | 去重 |
| `sorted()` / `sorted(Comparator)` | 排序 |
| `limit(long n)` / `skip(long n)` | 截取/跳过 |
| `peek(Consumer)` | 查看元素（调试用） |

| 终端操作 | 说明 |
|------|------|
| `collect(Collector)` | 收集为集合（`Collectors.toList()`、`toSet()`、`toMap()`、`joining()`） |
| `forEach(Consumer)` | 遍历 |
| `reduce(identity, BinaryOperator)` | 归约 |
| `count()` | 计数 |
| `anyMatch/allMatch/noneMatch(Predicate)` | 匹配判断 |
| `findFirst()` / `findAny()` | 查找第一个/任意 |
| `max(Comparator)` / `min(Comparator)` | 最大/最小 |

> [!tip] try-with-resources（Java 7+）
> ```java
> // ⭐ 自动关闭资源，无需手动 finally

>         System.out.println(line);
>     }
> }  // br 会自动调用 close()
> ```

| 类 | 关键方法 | 说明 |

| | `openConnection()` | 返回 URLConnection |

## 七、java.lang.reflect —— 反射机制（审计核心）

> [!danger] 审计高危
> 反射是动态加载类、调用私有方法、绕过访问控制的核心手段，绝大部分 RCE 利用链都依赖反射。

| 类 | 关键方法 | 说明 |
|------|------|------|
| `Class` | 见 java.lang.Class | ⭐ 反射入口 |
| `Field` | `get(Object obj)` | 获取字段值 |
| | `set(Object obj, Object value)` | ⭐ 设置字段值（可修改 final 字段） |
| | `setAccessible(boolean flag)` | ⭐ 绕过访问控制（访问私有成员） |
| | `getName()` / `getType()` | 获取字段名/类型 |
| | `getModifiers()` | 获取修饰符（public/static/final） |
| `Method` | `invoke(Object obj, Object... args)` | ⭐ 调用方法（RCE利用关键） |
| | `getName()` / `getReturnType()` / `getParameterTypes()` | 方法名/返回类型/参数类型 |
| | `setAccessible(boolean flag)` | ⭐ 绕过访问控制 |
| | `getModifiers()` | 获取修饰符 |
| `Constructor` | `newInstance(Object... initargs)` | ⭐ 创建实例（可调用私有构造） |
| | `setAccessible(boolean flag)` | ⭐ 绕过访问控制 |
| | `getParameterTypes()` | 获取参数类型 |
| `Modifier` | `isPublic(int)` / `isPrivate(int)` / `isStatic(int)` / `isFinal(int)` | 判断修饰符 |
| `Array` | `newInstance(Class<?> type, int length)` | 动态创建数组 |
| | `get(Object array, int index)` / `set(Object array, int index, Object value)` | 操作数组元素 |

```java
// ⭐ 典型反射调用 → Runtime 命令执行（审计中反复出现）
Class<?> clazz = Class.forName("java.lang.Runtime");
Method method = clazz.getDeclaredMethod("exec", String.class);  // 获取 exec 方法
method.setAccessible(true);                                      // 绕过访问控制
method.invoke(clazz.getMethod("getRuntime").invoke(null), "calc.exe"); // 执行
```
