# 1.介绍
```plain
AOP为Aspect Oriented Programming的缩写,意思为⾯向切⾯编程,是通过预编译⽅式和运⾏期动态代理
实现程序功能的统⼀维护的⼀种技术
AOP是OOP的延续,是软件开发中的⼀个热点,也是Spring框架中的⼀个重要内容,是函数式编程的⼀种衍
⽣范型
利⽤AOP可以对业务逻辑的各个部分进⾏隔离,从⽽使得业务逻辑各部分之间的耦合度降低,提⾼程序的可
重⽤性,同时提⾼了开发的效率
```

# 2.作用
AOP能够将那些与业务无关,却为业务模块所共同调用的逻辑或责任

例如: 事务处理、日志管理、权限控制、性能统计、安全控制、事务处理、异常处理等等,封装起来,便于减少系统的重复代码,降低模块间的耦合度,并有利于未来的可拓展性和可维护性

# 3.AOP相关概念
Spring AOP模块提供拦截器来拦截一个应用程序

通过这些拦截器,我们只需要对需要关注的部分进行代码编写,并通过配置的方式完成指定目标的方法增强



在我们开始学习使用AOP之前,让我们熟悉一下AOP概念和术语,这些术语并不特定于Spring,而是与AOP有关的

| 项 | 描述 |
| --- | --- |
| Aspect | 一个模块具有一组提供横切需求的APIs<br/>例如:一个日志模块为了记录日志将被AOP方面调用<br/>应用程序可以拥有任意数量的方面,这取决于需求 |
| Join point | 在你的应用程序中它代表一个点,你可以在插件 AOP 方面<br/>你也能说,它是在实际的应用程序中,其中一个操作将使用Spring AOP框架 |
| Advice | 这是实际行动之前或之后执行的方法<br/>这是在程序执行期间通过Spring AOP框架实际被调用的代码 |
| Pointcut | 这是一组一个或多个连接点,通知应该被执行<br/>你可以使用表达式或模式指定切入点正如我们将在AOP的例子中看到的 |
| Introduction | 引用允许你添加新方法或属性到现有的类中 |
| Target object | 被一个或多个方面所通知的对象,该对象永远是一个被代理对象,也称为被通知对象 |
| Weaving | Weaving把方面连接到其它的应用程序类型或者对象上,并创建一个被通知的对象<br/>这些可以在编译时,类加载时和运行时完成 |


# 4.通知的类型

| Spring方面可以使用下面提到的五种通知工作 |                               |
| ----------------------- | ----------------------------- |
| 通知                      | 描述                            |
| 前置通知                    | 在一个方法执行之前,执行通知                |
| 后置通知                    | 在一个方法执行之后,不考虑其结果,执行通知         |
| 返回后通知                   | 在一个方法执行之后,只有在方法成功完成时,才能执行通知   |
| 抛出异常后通知                 | 在一个方法执行之后,只有在方法退出抛出异常时,才能执行通知 |
| 环绕通知                    | 在建议方法调用之前和之后,执行通知             |

# 6.XML配置AOP
## 6.1 XML配置中的AOP元素
| AOP 配置元素 | 用途 | 备注 |
| --- | --- | --- |
| aop:config | 顶层的 AOP 配置元素 | AOP 的配置是以它为开始的 |
| aop:aspect | 定义一个切面 |  |
| aop:pointcut | 定义切点 |  |
| aop:before | 定义前置通知 |  |
| aop:after | 定义后置通知 |  |
| aop:after-returning | 定义返回通知 |  |
| aop:after-throwing | 定义异常通知 |  |
| aop:around | 定义环绕通知 |  |


```xml
XML配置中的AOP元素语法:

<aop:config>

<aop:aspect id="切面名称" ref="切面Bean名称">

<aop:pointcut id="切点名称" expression="切点表达式"/>

<aop:before pointcut-ref="切点引入" method="切面对象中的前置通知方法名"/>

<aop:after pointcut-ref="切点引入" method="切面对象中的后置通知方法名"/>

<aop:after-returning pointcut-ref="切点引入" method="切面对象中的返回通知方法名" returning="形参名"/>

<aop:after-throwing pointcut-ref="切点引入" method="切面对象中的异常通知方法名" throwing="形参名"/>

<aop:around pointcut-ref="切点引入" method="切面对象中的环绕通知方法名"/>
```

## 6.3 基础例子
该例子的作用主要是通过XML配置来认识AOP元素的各种触发点

```java
// 第一步
// 目录: ./SpringDemo/src/main/java/com/test/AOP/XML例一/Enhance.java
// 文件名: Enhance.java
package com.test.AOP.XML例一;

import org.aspectj.lang.ProceedingJoinPoint;

public class Enhance {
    /**
     * 在所选方法执行之前
     */
    public void beforeAdvice() {
        System.out.println("Enhance类-beforeAdvice()方法-执行,前置通知-执行");
    }
    
    /**
     * 在选定的方法执行之后
     */
    public void afterAdvice() {
        System.out.println("Enhance类-afterAdvice()方法-执行,后置通知-执行");
    }
    
    /**
     * 当选定的方法返回时
     *
     * @param retVal
     */
    public void afterReturningAdvice(Object retVal) {
        System.out.println("Enhance类-afterReturningAdvice()方法-执行,返回通知-执行");
    }

    /**
     * 如果出现异常
     *
     * @param ex
     */
    public void afterThrowingAdvice(IllegalArgumentException ex) {
        System.out.println("Enhance类-afterThrowingAdvice()方法-执行,异常通知-执行,捕获到一个异常: " + ex.toString());
    }

    /**
     * 环绕通知
     *
     * @param pjp
     */
    public void arround(ProceedingJoinPoint pjp) {
        Object[] args = pjp.getArgs();
        try {
            System.out.println("Enhance类-arround()方法-执行,环绕通知-实现-前置通知-功能");
            pjp.proceed(args);
            System.out.println("Enhance类-arround()方法-执行,环绕通知-实现-后置通知-功能");
        } catch (Throwable e) {
            System.out.println("Enhance类-arround()方法-执行,环绕通知-实现-异常通知-功能");
            e.printStackTrace();
        } finally {
            System.out.println("Enhance类-arround()方法-执行,环绕通知-实现-返回通知-功能");
        }
    }
}
```

// 目录: ./SpringDemo/src/main/resources/

<?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
  xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
  xmlns:aop="http://www.springframework.org/schema/aop"
  xsi:schemaLocation="http://www.springframework.org/schema/beans
  https://www.springframework.org/schema/beans/spring-beans.xsd
  http://www.springframework.org/schema/aop
  http://www.springframework.org/schema/aop/spring-aop.xsd">

  <!-- 装配 Bean-->
  <bean id="student" class="com.test.AOP.XML例一.Student">
    <property name="name" value="小王"/>
    <property name="age" value="18"/>
  </bean>
  <bean id="enhance" class="com.test.AOP.XML例一.Enhance"/>

  <!-- 配置AOP -->
  <aop:config>
    <!-- where: 在哪些地方(包.类.方法)做增强 -->
    <!-- execution: 代表执行方法的时候会触发 -->
    <!-- *: 代表任意返回类型的方法 -->
    <!-- com.test.AOP.XML例一.Student: 代表类的全限定名 -->
    <!-- getName(): 被拦截的方法名称 -->
    <aop:pointcut id="studentTest" expression="execution(* com.test.AOP.XML例一.Student.getName())"/>
    <!-- what: 做什么增强 -->
    <aop:aspect id="enhanceTest" ref="enhance">
      <!-- when: 在什么时机(方法前/后/返回/异常/环绕)进行增强 -->
      <aop:before pointcut-ref="studentTest" method="beforeAdvice"/>
      <aop:after pointcut-ref="studentTest" method="afterAdvice"/>
      <aop:after-returning pointcut-ref="studentTest" method="afterReturningAdvice" returning="retVal"/>
      <aop:after-throwing pointcut-ref="studentTest" method="afterThrowingAdvice" throwing="ex"/>
      <aop:around pointcut-ref="studentTest" method="arround"/>
    </aop:aspect>
  </aop:config>
</beans>

# 7.注解配置AOP
## 7.1 注解配置中的AOP元素
| 注解 | 说明 |
| --- | --- |
| @Pointcut | 定义切点 |
| @Before | 前置通知,在连接点方法前调用 |
| @After | 后置通知,在连接点方法后调用 |
| @AfterReturning | 返回通知,在连接点方法执行并正常返回后调用 |
| @AfterThrowing | 异常通知,当连接点方法异常时调用 |
| @Around | 环绕通知,它将覆盖原有方法,但是允许你通过反射调用原有方法 |


## 7.2 基础例子
该例子的作用主要是通过注解的方式来认识AOP元素的各种触发点

```java
// 第一步
// 目录: ./SpringDemo/src/main/java/com/test/AOP/注解例一/Enhance2.java
// 文件名: Enhance2.java
package com.test.AOP.注解例一;

import org.aspectj.lang.ProceedingJoinPoint;
import org.aspectj.lang.annotation.*;
import org.springframework.stereotype.Component;

@Component("enhance2")
@Aspect
public class Enhance2 {
    /**
     * execution: 代表执行方法的时候会触发
     * *: 代表任意返回类型的方法
     * com.test.AOP.注解例一.Student2: 代表类的全限定名
     * getName(): 被拦截的方法名称
     */
    @Pointcut(value = "execution(* com.test.AOP.注解例一.Student2.getName())")
    private void pointCutTest() {
    }

    /**
     * 在所选方法执行之前
     */
    @Before(value = "pointCutTest()")
    public void beforeAdvice() {
        System.out.println("Enhance2类-beforeAdvice()方法-执行,前置通知-执行");
    }

    /**
     * 在选定的方法执行之后
     */
    @After(value = "pointCutTest()")
    public void afterAdvice() {
        System.out.println("Enhance2类-afterAdvice()方法-执行,后置通知-执行");
    }

    /**
     * 当选定的方法返回时
     *
     * @param retVal
     */
    @AfterReturning(value = "pointCutTest()", returning = "retVal")
    public void afterReturningAdvice(Object retVal) {
        System.out.println("Enhance2类-afterReturningAdvice()方法-执行,返回通知-执行");
    }

    /**
     * 如果出现异常
     *
     * @param ex
     */
    @AfterThrowing(value = "pointCutTest()", throwing = "ex")
    public void afterThrowingAdvice(IllegalArgumentException ex) {
        System.out.println("Enhance2类-afterThrowingAdvice()方法-执行,异常通知-执行,捕获到一个异常: " + ex.toString());
    }

    /**
     * 环绕通知
     *
     * @param pjp
     */
    @Around(value = "pointCutTest()")
    public void arround(ProceedingJoinPoint pjp) {
        Object[] args = pjp.getArgs();
        try {
            System.out.println("Enhance2类-arround()方法-执行,环绕通知-实现-前置通知-功能");
            pjp.proceed(args);
            System.out.println("Enhance2类-arround()方法-执行,环绕通知-实现-后置通知-功能");
        } catch (Throwable e) {
            System.out.println("Enhance2类-arround()方法-执行,环绕通知-实现-异常通知-功能");
            e.printStackTrace();
        } finally {
            System.out.println("Enhance2类-arround()方法-执行,环绕通知-实现-返回通知-功能");
        }
    }
}
```
