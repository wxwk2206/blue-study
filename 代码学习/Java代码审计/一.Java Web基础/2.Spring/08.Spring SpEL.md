 SpEL（Spring Expression Language）是 Spring 框架提供的**强大表达式语言**，核心定位是 “运行时动态查询、操作对象及解析规则”，功能覆盖取值、运算、方法调用、对象创建等，且无缝集成 Spring 生态（注解、XML 配置、代码逻辑等），比 JSP 的 EL 表达式功能更全面、灵活性更高。  

## 一、核心：语法与定界符
### 1. 定界符规则
SpEL 统一使用 `#{...}` 作为表达式定界符，所有表达式逻辑需包裹在其中，未被包裹的内容会被视为普通字面量。

+ 示例：`#{1+2}`（表达式，运行时计算结果 3）、`'普通字符串'`（字面量，直接返回字符串本身）

### 2. 核心变量
SpEL 内置两个关键变量，用于上下文导航：

+ `#this`：引用**当前评估对象**（比如集合筛选时，指代集合中的每个元素）；
+ `#root`：引用**根上下文对象**（整个表达式的顶层对象，不受当前评估对象影响）。

## 二、核心功能与语法详解（附示例）
### 1. 属性引用
支持直接引用对象、数组、集合、Map 的属性 / 元素，语法简洁直观：

+ 引用对象及属性：`#{对象名.属性名}`，例 `#{user.name}`（引用名为 `user` 的 Bean 的 `name` 属性）；
+ 数组 / 集合取值：`#{集合名[index]}`，例 `#{members[0].inventions[6]}`（取 `members` 集合第 1 个元素的 `inventions` 数组第 7 个元素）；
+ Map 取值：`#{map名['key']}`，例 `#{officers['advisors'][0].placeOfBirth.country}`（取 `officers` Map 中 `advisors` 键对应的集合第 1 个元素的出生地国家）；
+ 静态类属性 / 方法：通过 `T(全类名)` 引用，例 `#{T(java.lang.Math).PI}`（获取 Math 类的静态常量 π）、`#{T(java.lang.Math).random()}`（调用 Math 类的静态方法生成随机数）。

### 2. 方法调用
支持调用对象方法、静态方法，甚至直接调用字符串、集合等内置对象的方法：

+ 普通对象方法：`#{对象名.方法名(参数)}`，例 `#{user.toString()}`（调用 `user` 对象的 `toString` 方法）；
+ 字面量方法：直接对字面量调用方法，例 `#{'abcde'.substring(1, 3)}`（调用字符串的 `substring` 方法，结果为 `bc`）；
+ 静态方法：通过 `T(全类名)` 调用，例 `#{T(String).valueOf(123)}`（调用 String 类的静态方法 `valueOf`，结果为字符串 `"123"`）；
+ 复杂方法链：`#{'Hello World'.concat('!').toUpperCase()}`（先拼接字符串，再转大写，结果为 `HELLO WORLD!`）。

### 3. 运算符（全面支持多类运算）
SpEL 支持数学、关系、逻辑、赋值、三元等多种运算符，优先级遵循标准规则：

#### （1）数学运算符
+ 支持符号：`+`（加 / 拼接）、`-`（减）、`*`（乘）、`/`（除）、`%`（取模）、`^`（指数幂）；
+ 适用场景：数字运算、字符串拼接；
+ 示例：
    - 数字运算：`#{10 + 20 * 3}`（结果 70）、`#{8 ^ 2}`（结果 64）、`#{10 % 3}`（结果 1）；
    - 字符串拼接：`#{'test' + ' ' + 'string'}`（结果为 `"test string"`）。

#### （2）关系运算符
+ 支持符号（两种写法等价）：
    - 等于：`==` / `eq`；不等于：`!=` / `ne`；
    - 小于：`<` / `lt`；小于等于：`<=` / `le`；
    - 大于：`>` / `gt`；大于等于：`>=` / `ge`；
    - 类型判断：`instanceOf`；正则匹配：`matches`；
+ 示例：
    - `#{2 == 2}`（结果 `true`）、`#{5 > 3}`（结果 `true`）；
    - 类型判断：`#{'xyz' instanceof T(Integer)}`（结果 `false`，字符串不是 Integer 类型）；
    - 正则匹配：`#{'5.00' matches '^-?\\d+(\\.\\d{2})?$'}`（结果 `true`，匹配两位小数的数字）。

#### （3）逻辑运算符
+ 支持符号（两种写法等价）：`&&` / `and`（与）、`||` / `or`（或）、`!` / `not`（非）；
+ 示例：`#{(2 > 1) and (3 != 4)}`（结果 `true`）、`#{not (1 == 2)}`（结果 `true`）。

#### （4）赋值运算符
+ 符号：`=`；
+ 用途：通过 `Expression.setValue()` 或 `getValue()` 方法动态设置对象属性值；
+ 示例：`parser.parseExpression("user.age = 25").getValue(context)`（将 `user` 对象的 `age` 属性设为 25）。

#### （5）三元运算符
+ 语法：`#{条件 ? 成立结果 : 不成立结果}`；
+ 示例：`#{score >= 60 ? '及格' : '不及格'}`、`#{environment['app.user.name'] ?: '小王'}`（配置中取值，无则用默认值 `小王`）。

### 4. 实例创建与调用构造方法
通过 `new` 关键字直接调用构造方法创建对象，支持所有合法的 Java 构造器：

+ 语法：`#{new 全类名(参数)}`；
+ 示例：
    - 简单对象：`#{new String("abc")}`（创建字符串对象，结果为 `"abc"`）；
    - 复杂对象：`#{new java.util.Date()}`（创建 Date 对象，返回当前时间）。

### 5. 表达式模板
当需要将表达式嵌入普通文本时，使用**表达式模板**，需配合 `TemplateParserContext` 解析：

+ 语法：`普通文本#{表达式}普通文本`；
+ 作用：同时保留静态文本和动态表达式结果，无需手动拼接；
+ 示例：java

```java
// 解析模板表达式，获取随机数并拼接文本
String expressionString = "随机数字是: #{T(java.lang.Math).random()}";
Expression expression = parser.parseExpression(expressionString, new TemplateParserContext());
String result = expression.getValue(String.class); // 结果示例："随机数字是: 0.0430254651283285"
```

## 三、SpEL 的典型使用场景
SpEL 无缝集成 Spring 全生态，核心使用场景分为 3 类：

### 1. 注解中动态注入（@Value）
最常用场景，通过 `@Value` 注解将表达式结果注入到类属性中，支持配置取值、默认值、运算等：

```java
public class User {
    // 1. 注入字面量（等价于直接赋值 "小王"）
    @Value("#{'小王'}")
    private String fixedName;

    // 2. 从环境配置中取值，无则用默认值 "小李"
    @Value("#{environment['app.user.name'] ?: '小李'}")
    private String configName;

    // 3. 注入运算结果（20+5=25）
    @Value("#{20 + 5}")
    private int age;

    // 4. 注入 Bean 的属性（引用名为 userService 的 Bean 的 appName 属性）
    @Value("#{userService.appName}")
    private String appName;
}
```

### 2. XML 配置中动态赋值
在 Spring XML 配置文件中，为 Bean 的属性动态设置表达式结果：

```xml
<!-- 为 xxx Bean 的 arg 属性注入 10*3 的结果（30） -->
<bean id="xxx" class="com.xx.xx.Xxx">
  <property name="arg" value="#{10 * 3}"/>
</bean>

<!-- 为 user Bean 的 nickname 属性注入 systemConfig Bean 的 nickname 属性 -->
<bean id="user" class="com.xx.xx.User">
  <property name="nickname" value="#{systemConfig.nickname}"/>
</bean>
```

### 3. 代码中直接解析表达式
通过 `SpelExpressionParser` 手动解析表达式，适用于复杂动态逻辑场景：

```java
public class SpELCodeDemo {
    public static void main(String[] args) {
        // 1. 创建 SpEL 解析器
        SpelExpressionParser parser = new SpelExpressionParser();

        // 2. 解析简单表达式（字符串输出）
        Expression exp1 = parser.parseExpression("'Hello World'");
        String strResult = (String) exp1.getValue(); // 结果："Hello World"

        // 3. 解析运算表达式（数学运算）
        Expression exp2 = parser.parseExpression("100 + 20 * 5");
        int numResult = exp2.getValue(Integer.class); // 结果：200

        // 4. 解析构造方法表达式（创建对象）
        Expression exp3 = parser.parseExpression("new String('abc')");
        String objResult = (String) exp3.getValue(); // 结果："abc"
    }
}
```

## 四、实例
### 1 纯字面意义的字符串输出 
```java
package com.test.SpEL;
import org.springframework.expression.Expression;
import org.springframework.expression.spel.standard.SpelExpressionParser;

public class SpELTest {
    public static void main(String[] args) {
        // 1. 定义解析器
        SpelExpressionParser parser = new SpelExpressionParser();
        
        // 2. 使用解析器解析表达式
        Expression exp = parser.parseExpression("'Hello World'");
        
        // 3. 获取解析结果
        String value = (String) exp.getValue();
        System.out.println(value);
    }
}
```

**解释**：

+ 创建SpEL解析器
+ 解析字符串字面量`'Hello World'`（单引号表示字符串）
+ 获取解析结果为String类型
+ 输出：`Hello World`
+ **作用**：最基本的字符串常量解析

### 2 字符串方法的字面调用 
```java
package com.test.SpEL;
import org.springframework.expression.Expression;
import org.springframework.expression.spel.standard.SpelExpressionParser;
import java.util.Arrays;

public class SpELTest2 {
    public static void main(String[] args) {
        SpelExpressionParser parser = new SpelExpressionParser();
        
        // 字符串连接
        Expression exp1 = parser.parseExpression("'Hello World'.concat('!')");
        String value = (String) exp1.getValue();
        System.out.println(value);  // Hello World!
        
        // 获取字符串bytes
        Expression exp2 = parser.parseExpression("'Hello World'.bytes");
        byte[] bytes = (byte[]) exp2.getValue();
        System.out.println(Arrays.toString(bytes));
        // [72, 101, 108, 108, 111, 32, 87, 111, 114, 108, 100]
        
        // 获取字符串长度
        Expression exp3 = parser.parseExpression("'Hello World'.bytes.length");
        int length = (Integer) exp3.getValue();
        System.out.println("长度为: " + length);  // 11
        
        // 字符串小写字符转换为大写
        Expression exp4 = parser.parseExpression("new String('hello world').toUpperCase()");
        System.out.println("大写: " + exp4.getValue());  // HELLO WORLD
    }
}
```

**解释**：

1. **concat()**：字符串连接，相当于Java的`"Hello World" + "!"`
2. **.bytes**：获取字符串的字节数组
3. **.bytes.length**：获取字节数组长度（字符数）
4. **new String()**：创建新字符串对象
5. **toUpperCase()**：转大写
    - **作用**：展示SpEL调用Java字符串方法

### 3 关系运算符 
```java
package com.test.SpEL;
import org.springframework.expression.spel.standard.SpelExpressionParser;
import org.springframework.expression.spel.support.SimpleEvaluationContext;

public class SpELTest3 {
    public static void main(String[] args) {
        SpelExpressionParser parser = new SpelExpressionParser();
        SimpleEvaluationContext context = SimpleEvaluationContext.forReadOnlyDataBinding().build();
        
        Boolean value = parser.parseExpression("1+1!=2").getValue(context, Boolean.class);
        System.out.println(value);  // false
    }
}
```

**解释**：

+ 创建评估上下文`SimpleEvaluationContext`
+ 表达式`1+1!=2`：计算1+1是否不等于2
+ 结果为`false`，因为1+1等于2
+ **作用**：展示SpEL的关系运算和布尔表达式

### 4 方法调用 
```java
package com.test.SpEL;
import org.springframework.expression.spel.standard.SpelExpressionParser;

public class SpELTest4 {
    public static void main(String[] args) {
        SpelExpressionParser parser = new SpelExpressionParser();
        String v = parser.parseExpression("'abc'.substring(1, 3)").getValue(String.class);
        System.out.println(v);  // bc
    }
}
```

**解释**：

+ 调用字符串的`substring(1, 3)`方法
+ 从索引1（包含）到3（不包含）截取子串
+ "abc" → 索引1是'b'，到索引3是'c' → "bc"
+ **作用**：展示方法调用和参数传递

### 5 调用构造方法

### 6 T() 使用 
```java
package com.test.SpEL;
import org.springframework.expression.Expression;
import org.springframework.expression.spel.standard.SpelExpressionParser;

public class SpELTest6 {
    public static void main(String[] args) {
        SpelExpressionParser parser = new SpelExpressionParser();
        
        // 1. 获取类的Class
        Class value = parser.parseExpression("T(String)").getValue(Class.class);
        System.out.println(value);  // class java.lang.String
        
        // 2. 获取类中的静态变量、静态方法
        boolean trueValue = parser.parseExpression("T(java.math.RoundingMode).CEILING < T(java.math.RoundingMode).FLOOR").getValue(Boolean.class);
        System.out.println(trueValue);  // true
        
        Long longValue = parser.parseExpression("T(Long).parseLong('9999')").getValue(Long.class);
        System.out.println(longValue);  // 9999
        
        // 3. 通过T(xxx)命令执行（⚠️ 安全风险！）
        String input = "new java.util.Scanner(T(java.lang.Runtime).getRuntime().exec(\"id\").getInputStream()).next()";
        Expression expression = parser.parseExpression(input);
        System.out.println(expression.getValue());  // 执行系统命令
    }
}
```

**解释**：

#### 1. T() 操作符的作用 
+ `**T()**`**表示类型操作符**，用于访问类的静态成员
+ 格式：`T(完整类名)`

#### 2. 三种用法示例： 
**用法1：获取类的Class对象**

```plain
T(String)  // 等价于 String.class
```

**用法2：访问静态成员和比较**

```plain
T(java.math.RoundingMode).CEILING  // 访问枚举值
T(Long).parseLong('9999')  // 调用静态方法
```

**用法3：执行系统命令（***⚠️****危险！）

```java
// 分解这个危险表达式：
new java.util.Scanner(                  // 创建Scanner
    T(java.lang.Runtime)               // 获取Runtime类
    .getRuntime()                      // 获取Runtime实例
    .exec("id")                        // 执行系统命令"id"
    .getInputStream()                  // 获取命令输出流
).next()                               // 读取输出

// 等价Java代码：
Runtime runtime = Runtime.getRuntime();
Process process = runtime.exec("id");
Scanner scanner = new Scanner(process.getInputStream());
String result = scanner.next();
```

### 7 表达式模板 #{} 使用 
```java
package com.test.SpEL;
import org.springframework.expression.Expression;
import org.springframework.expression.common.TemplateParserContext;
import org.springframework.expression.spel.standard.SpelExpressionParser;

public class SpELTest7 {
    public static void main(String[] args) {
        SpelExpressionParser parser = new SpelExpressionParser();
        
        // 获取随机数字返回
        String expressionString = "随机数字是: #{T(java.lang.Math).random()}";
        Expression expression = parser.parseExpression(expressionString, 
                          new TemplateParserContext());
        String value = expression.getValue(String.class);
        System.out.println(value);  // 随机数字是: 0.0430254651283285
        
        // 命令执行（⚠️ 安全风险！）
        String expressionString2 = "命令执行: #{new java.util.Scanner(T(java.lang.Runtime).getRuntime().exec(\"id\").getInputStream()).next()}";
        Expression expression2 = parser.parseExpression(expressionString2, 
                          new TemplateParserContext());
        String value2 = expression2.getValue(String.class);
        System.out.println(value2);  
    }
}
```

**解释**：

#### 1. 模板表达式`#{}`
+ `**#{表达式}**` 表示模板中的动态部分
+ 需要配合`TemplateParserContext`使用
+ 模板中可以混合静态文本和动态表达式

#### 2. 示例解析： 
**安全示例**：

```plain
"随机数字是: #{T(java.lang.Math).random()}"
// 静态文本："随机数字是: "
// 动态部分：#{T(java.lang.Math).random()}
// 结果："随机数字是: 0.0430254651283285"
```

**危险示例**（命令注入）：

```plain
"命令执行: #{new java.util.Scanner(T(java.lang.Runtime).getRuntime().exec(\"id\").getInputStream()).next()}"
// 静态文本："命令执行: "
// 动态部分：执行系统命令"id"
```

## 五、关键注意事项
1. **属性访问要求**：SpEL 访问对象属性时，需确保对象提供**getter 方法**（否则无法解析）；
2. **静态类引用**：`T(全类名)` 中，`java.lang` 包下的类可省略包名（如 `T(String)`），其他包必须写全类名（如 `T(java.math.RoundingMode)`）；
3. **安全性**：避免在表达式中执行未授权的命令（如 `T(java.lang.Runtime).getRuntime().exec("rm -rf /")`），可能导致安全漏洞；
4. **上下文作用域**：`#this` 仅在当前评估上下文有效（如集合筛选），`#root` 始终指向顶层对象，不受局部上下文影响。
