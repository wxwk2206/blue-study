### Spring 核心注解分类及详解
Spring 注解主要围绕**组件管理**、**依赖注入**、**配置**、**AOP**、**事务**等核心功能，下面按使用频率和重要性逐一介绍：

#### 一、核心容器注解（最基础）
这类注解主要用于定义 Bean、管理依赖，是 Spring 最核心的注解。

##### 1. 组件扫描与 Bean 定义

| **注解** | **用途** | **使用场景** |
| --- | --- | --- |
| `@Component` | 通用注解，标记类为 Spring 管理的组件（Bean） | 所有需要被 Spring 容器管理的普通类 |
| `@Controller` | 特殊的 @Component，标记控制器层（MVC） | Spring MVC/ Spring Boot 的 Controller 类 |
| `@Service` | 特殊的 @Component，标记业务逻辑层 | 服务层（Service）类，处理业务逻辑 |
| `@Repository` | 特殊的 @Component，标记数据访问层 | 持久层（DAO/Mapper）类，操作数据库 |
| `@Configuration` | 标记类为配置类，替代 XML 配置文件 | 定义 Bean、配置第三方组件（如数据源） |
| `@Bean` | 标注在方法上，返回值作为 Spring Bean | 在 @Configuration 类中定义第三方 Bean（如 RedisTemplate） |


**示例代码**：

```java
// 1. 组件类定义
@Service // 标记为服务层Bean
public class UserService {
    // 业务逻辑
}

// 2. 配置类定义Bean
@Configuration // 配置类
public class AppConfig {
    // 定义一个数据源Bean，方法名默认是Bean的名称
    @Bean
    public DataSource dataSource() {
        DruidDataSource ds = new DruidDataSource();
        ds.setUrl("jdbc:mysql://localhost:3306/test");
        ds.setUsername("root");
        ds.setPassword("123456");
        return ds;
    }
}
```

##### 2. 依赖注入

| **注解** | **用途** | **注意事项** |
| --- | --- | --- |
| `@Autowired` | 自动注入依赖的 Bean（按类型匹配） | 默认要求 Bean 必须存在，找不到会报错 |
| `@Qualifier` | 配合 @Autowired，按 Bean 名称注入 | 解决同类型多个 Bean 的注入冲突 |
| `@Resource` | JDK 自带注解，按名称注入（也可按类型） | 比 @Autowired 更灵活，非 Spring 原生 |
| `@Value` | 注入配置文件中的属性值（如 application.properties） | 支持 SpEL 表达式（如 ${user.name}） |


**示例代码**：

```java
@Service
public class UserService {
    // 按类型注入UserMapper
    @Autowired
    // 若有多个UserMapper类型的Bean，指定名称为userMapperImpl
    @Qualifier("userMapperImpl")
    private UserMapper userMapper;

    // 注入配置文件中的属性
    @Value("${app.name}")
    private String appName;

    // 按名称注入数据源（name属性指定Bean名称）
    @Resource(name = "dataSource")
    private DataSource dataSource;
}
```

#### 二、Spring MVC 注解
这类注解用于 Web 层，处理 HTTP 请求：

| **注解** | **用途** |
| --- | --- |
| `@Controller` | 标记为 MVC 控制器 |
| `@RestController` | **组合注解（@Controller + @ResponseBody）**，返回 JSON/XML |
| `@RequestMapping` | 映射 HTTP 请求（指定 URL、请求方法） |
| `@GetMapping/@PostMapping` | 简化 @RequestMapping，分别对应 GET/POST 请求 |
| `@RequestParam`<br/>parameter：参数 | 获取请求参数（如 URL 中的？name = 张三） |
| `@PathVariable` | 获取 URL 路径中的参数（如 /user/{id}） |
| `@RequestBody` | 接收请求体中的 JSON 数据（绑定到 Java 对象） |


**示例代码**：

```java
// 返回JSON数据，无需额外加@ResponseBody
@RestController
@RequestMapping("/user")
public class UserController {
    // 处理GET请求：/user/{id}
    @GetMapping("/{id}")
    public User getUser(
        // 获取路径中的id参数
        @PathVariable Long id,
        // 获取请求参数name（required=false表示非必传）
        @RequestParam(required = false) String name) {
        return userService.getUserById(id);
    }

    // 处理POST请求：/user，接收JSON请求体
    @PostMapping
    public User addUser(@RequestBody User user) {
        return userService.addUser(user);
    }
}
```

#### 三、事务管理注解

| **注解** | **用途** | **关键属性** |
| --- | --- | --- |
| `@Transactional` | 标记方法 / 类支持事务 | `rollbackFor`：指定触发回滚的异常类型<br/>`propagation`：事务传播行为（如REQUIRED、REQUIRES_NEW） |


**示例代码**：

```java
@Service
public class UserService {
    // 该方法开启事务，发生RuntimeException时回滚
    @Transactional(rollbackFor = RuntimeException.class)
    public void transferMoney(Long fromId, Long toId, Double amount) {
        // 转账业务逻辑（数据库操作）
        userDao.decreaseMoney(fromId, amount);
        userDao.increaseMoney(toId, amount);
    }
}
```

#### 四、AOP 相关注解

| **注解** | **用途** |
| --- | --- |
| `@Aspect` | 标记类为切面类 |
| `@Before` | 前置通知（方法执行前执行） |
| `@After` | 后置通知（方法执行后执行，无论是否异常） |
| `@AfterReturning` | 返回通知（方法正常返回后执行） |
| `@AfterThrowing` | 异常通知（方法抛出异常时执行） |
| `@Around` | 环绕通知（包裹方法执行，最强大） |
| `@Pointcut` | 定义切入点（指定要拦截的方法） |


**示例代码**：

```java
// 切面类
@Aspect
@Component
public class LogAspect {
    // 定义切入点：拦截com.example.service包下所有类的所有方法
    @Pointcut("execution(* com.example.service.*.*(..))")
    public void servicePointcut() {}

    // 前置通知：切入点执行前打印日志
    @Before("servicePointcut()")
    public void beforeMethod(JoinPoint joinPoint) {
        System.out.println("方法" + joinPoint.getSignature().getName() + "开始执行");
    }

    // 环绕通知：统计方法执行时间
    @Around("servicePointcut()")
    public Object aroundMethod(ProceedingJoinPoint pjp) throws Throwable {
        long start = System.currentTimeMillis();
        Object result = pjp.proceed(); // 执行原方法
        long end = System.currentTimeMillis();
        System.out.println("方法执行耗时：" + (end - start) + "ms");
        return result;
    }
}
```

#### 五、其他常用注解

| **注解** | **用途** |
| --- | --- |
| `@Scope` | 指定 Bean 的作用域（singleton 单例 /prototype 多例 /request/ session） |
| `@Lazy` | 延迟初始化 Bean（第一次使用时才创建） |
| `@PostConstruct` | Bean 初始化完成后执行（替代 init-method） |
| `@PreDestroy` | Bean 销毁前执行（替代 destroy-method） |


### 总结
1. **核心基础**：`@Component/@Controller/@Service/@Repository` 定义 Bean，`@Autowired/@Resource` 实现依赖注入，`@Configuration/@Bean` 替代 XML 配置。
2. **Web 开发**：`@RestController` 替代`@Controller + @ResponseBody`，`@GetMapping/@PostMapping` 简化请求映射，`@RequestBody/@PathVariable` 处理请求参数。
3. **进阶功能**：`@Transactional` 管理事务，`@Aspect` 实现 AOP 切面编程，是 Spring 企业级开发的核心。
