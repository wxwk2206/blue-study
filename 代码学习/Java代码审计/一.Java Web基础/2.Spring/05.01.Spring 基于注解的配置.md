##  3.⽤于创建对象的注解标签-⽤法与例⼦  
###  3.1描述  
```plain
@Component ⽤于其他
@Controller 标注⼀个控制层的组件类,⽤在表现层
@Service 标注⼀个业务逻辑组件类,⽤在业务层
@Repository 标注⼀个DAO组件类,⽤在持久层
⽤于把当前对象存⼊到Spring容器中,注解标签中含有属性value,⽤于指定Bean的id
当只有⼀个属性时可以省略,Bean名称默认是该类的⾸字⺟⼩写,其它不变
平时我们没有其他要求的时候,都是直接⽤省略写法的
⽤于创建对象的注解除了@Component还有@Controller、@Service、@Repository
它们的作⽤和属性与@Component完全⼀样,⽬的是使创建的对象更加清晰,⽤法⼀摸⼀样,只是改了个名字
使⽤⽅法:@Component("userTest")对标XML配置<bean id="userTest" class="com.test.User">
```

### 3.2例子
```plain
因为@Component、@Controller、@Service、@Repository用法都是一模一样的
所以这里的例子就使用@Component进行演示了
```

```java

// 引入注解来替换<context:component-scan>的xml配置,进行自动扫描,给予部分例子的配置

import org.springframework.context.annotation.Configuration;

@Configuration
@ComponentScan(basePackages = "com.test.基于注解的配置")
public class AppContextConfig {
}
```

```java
// 第二步
// 通过注解创建一个Bean
// 目录: ./SpringDemo/src/main/java/com/test/基于注解的配置/创建对象的注解标签/
// 文件名: Person.java
package com.test.基于注解的配置.创建对象的注解标签;

import org.springframework.stereotype.Component;

// @Component // 直接省略value属性,默认名称为类名首字母小写,也就是Person转成person
// @Component("personTest")	// 省略value的写法
```

### 4.4 @Qualifier
#### 4.4.1 描述

### 5.6 @Lazy
#### 5.6.1 描述
```plain
@Lazy用于配置Bean的懒加载

注解的@Lazy(value=true):
懒加载,指Spring延迟加载Bean,true=延迟加载,false=不延迟加载

注: 如果使用这个注解不设置属性,那么就默认为true, 也就是默认延迟加载
例如: 直接在类中使用@Lazy那表示为true, @Lazy(value=false)表示为false

@Lazy(value = false)对标XML配置<bean ... lazy-init="true" />
```

```plain
注意一: 懒加载只对scope属性为singleton才有效
注意二: 如果scope属性为prototype,无论懒加载设置的属性值是什么,都只在通过GetBean时进行实例化
```

### 5.7 @PostConstruct与@PreDestroy
#### 5.7.1 描述
```plain
@PostConstruct用于指定初始化方法
@PreDestroy指定销毁方法
```

#### 5.7.2 例子
```java
// 第一步
// 目录: ./SpringDemo/src/main/java/com/test/基于注解的配置/其它设置的注解标签/PostConstruct与PreDestroy/
// 文件名: Software.java
package com.test.基于注解的配置.其它设置的注解标签.PostConstruct与PreDestroy;

import org.springframework.stereotype.Component;

import javax.annotation.PostConstruct;
import javax.annotation.PreDestroy;

@Component("softwareTest")
public class Software {
    public Software() {
        System.out.println("Software类的构造方法被执行了");
    }

    @PostConstruct
    public void init() {
        System.out.println("初始化方法init");
    }

    @PreDestroy
    public void close() {
        System.out.println("销毁前执行方法close");
    }

    public void run() {
        System.out.println("程序正在运行");
    }
}
```

```java
// 第二步
// 目录: ./SpringDemo/src/main/java/com/test/基于注解的配置/其它设置的注解标签/PostConstruct与PreDestroy/
// 文件名: SpringConfiguration8.java
package com.test.基于注解的配置.其它设置的注解标签.PostConstruct与PreDestroy;

import org.springframework.context.annotation.ComponentScan;
import org.springframework.context.annotation.Configuration;

@Configuration
@ComponentScan(basePackages = "com.test.基于注解的配置.其它设置的注解标签.PostConstruct与PreDestroy")
public class SpringConfiguration8 {
}
```

```java
// 第三步
// 创建一个测试类Test41进行测试
// 目录: ./SpringDemo/src/test/java/
// 文件名: Test41.java
import com.test.基于注解的配置.其它设置的注解标签.PostConstruct与PreDestroy.Software;
import com.test.基于注解的配置.其它设置的注解标签.PostConstruct与PreDestroy.SpringConfiguration8;

import org.springframework.context.annotation.AnnotationConfigApplicationContext;

public class Test41 {
    public static void main(String[] args) {
        AnnotationConfigApplicationContext context = new AnnotationConfigApplicationContext(SpringConfiguration8.class);
        Software obj = (Software) context.getBean("softwareTest");
        obj.run();

        // 必须加,用来触发@PreDestroy注解
        context.close();
```
