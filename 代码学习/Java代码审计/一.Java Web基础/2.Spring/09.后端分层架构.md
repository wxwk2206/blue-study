### 1. 核心分层与职责（从左到右，请求流向）
+ **用户**：发起请求的终端使用者（如手机、电脑端操作）。
+ **浏览器侧**：接收用户操作，发送 HTTP 请求到后端（如浏览器、APP 客户端）。
+ **lis（Listener，监听器）**：Web 应用启动 / 销毁、会话创建 / 销毁等事件的监听组件，做初始化、资源清理等前置 / 后置处理。
+ **filter（过滤器）**：请求进入 Controller 前的拦截器，做**统一处理**（如登录校验、编码设置、权限过滤、日志记录）。
+ **controller（控制器）**：请求入口，接收浏览器的 HTTP 请求，解析参数，调用 Service 层，返回响应结果（如 JSON、页面）。
+ **service（服务层）**：**核心业务逻辑层**，处理复杂业务（如事务控制、多 Dao 组合操作、业务规则校验），是 Controller 和 Dao 的中间层。
+ **Dao（数据访问对象）**：直接与数据库交互，执行 CRUD（增删改查）操作，封装 JDBC/MyBatis 等数据库操作细节。
+ **数据库**：持久化存储数据（如 MySQL、Oracle），接收 Dao 层的 SQL 指令并返回数据。
+ **pojo（Plain Old Java Object，普通 Java 对象）**：**数据载体**，贯穿全层，封装业务数据（如 User、Order 实体类），在 Controller、Service、Dao 之间传递数据，不依赖任何框架。**核心**：纯 Java 原生对象，无框架侵入、无强制继承 / 实现，只关注数据 / 业务逻辑  



### 2. 完整请求流程
1. 用户在浏览器侧发起请求 → 2. 过滤器（filter）拦截并预处理 → 3. 控制器（controller）接收并解析 → 4. 调用服务层（service）执行业务逻辑 → 5. 服务层调用数据访问层（Dao） → 6. Dao 层操作数据库（增删改查） → 7. 数据库返回数据给 Dao → 8. 数据经 Service、Controller 封装后，通过 pojo 传递 → 9. 最终返回响应给浏览器侧，展示给用户。
