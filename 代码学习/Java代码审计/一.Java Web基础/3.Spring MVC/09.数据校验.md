## 一、核心概念
### 1. 数据校验的必要性
用户输入具有随意性，为保证系统接收数据的合法性，数据验证是所有 Web 应用的必备环节。

### 2. 核心技术栈
+ **JSR 303**：Java 为 Bean 数据合法性校验提供的标准框架（规范）。
+ **Hibernate Validator**：JSR 303 的参考实现，直接引入依赖即可使用，无需额外适配。

## 二、JSR 303 核心校验注解
| **注解** | **适用场景** | **核心说明** |
| --- | --- | --- |
| @Null | 所有类型 | 被标注元素必须为 null |
| @NotNull | 所有类型 | 被标注元素必须不为 null（仅判断非 null，不处理空字符串 / 空格） |
| @NotEmpty | 字符串、数组、集合等 | 元素不能为空：不允许 null、空字符串 ""、未传参；允许含空格的字符串" " |
| @NotBlank | 字符串 | 元素不能为空：不允许 null、空字符串 ""、未传参、含空格的空字符串" "（严格校验） |
| @AssertTrue | 布尔类型 | 被标注元素必须为 true |
| @AssertFalse | 布尔类型 | 被标注元素必须为 false |
| @Min(value) | 数字类型（int、long 等） | 数值必须大于等于指定最小值 |
| @Max(value) | 数字类型 | 数值必须小于等于指定最大值 |
| @DecimalMax(value) | 数字类型（支持高精度） | 数值小于等于给定的 BigDecimal 值 |
| @DecimalMin(value) | 数字类型（支持高精度） | 数值大于等于给定的 BigDecimal 值 |
| @Size(min, max) | 数组、集合、Map、字符串 | 长度在 [min, max] 范围内 |
| @Length(min, max) | 字符串 | 长度在 [min, max] 范围内（专门用于字符串） |
| @Digits(integer, fraction) | 数字类型 | 整数部分精度 ≤ integer，小数部分精度 ≤ fraction |
| @Past | 日期类型 | 必须是过去的日期 |
| @Future | 日期类型 | 必须是将来的日期 |
| @Pattern(value) | 字符串 | 必须符合指定正则表达式（建议与 @NotNull 并用，确保参数必传且合规） |
| @Email | 字符串 | 必须符合邮箱格式 |

## 五、实操步骤（含示例代码）
### 第一步：创建带校验注解的实体类
路径：`src/main/com/test/model/UserInfo.java`

```plain
package test.model;

import org.hibernate.validator.constraints.Length;
import javax.validation.constraints.Email;
import javax.validation.constraints.NotNull;

public class UserInfo {
    // id 不能为空
    @NotNull(message = "id不能为空")
    private Integer id;

    // 用户名：非空 + 长度2-8位
    @NotNull(message = "name不能为空")
    @Length(min = 2, max = 8, message = "用户名不能少于2位大于8位")
    private String name;

    // 邮箱：非空 + 格式正确
    @NotNull(message = "email不能为空")
    @Email(message = "邮箱格式不正确")
    private String email;

    // 手机号不能为空
    @NotNull(message = "phone不能为空")
    private String phone;

    // Getter 和 Setter 方法（必须提供，否则校验无法获取属性值）
    public Integer getId() { return id; }
    public void setId(Integer id) { this.id = id; }
    public String getName() { return name; }
    public void setName(String name) { this.name = name; }
    public String getEmail() { return email; }
    public void setEmail(String email) { this.email = email; }
    public String getPhone() { return phone; }
    public void setPhone(String phone) { this.phone = phone; }
}
```

### 第二步：创建测试控制器（使用校验）
路径：`src/main/com/test/controller/PublicController.java`

```plain
package test.controller;

import org.springframework.validation.BindingResult;
import org.springframework.validation.ObjectError;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.RestController;
import test.model.UserInfo;
import javax.validation.Valid;
import java.util.List;

@RestController
public class PublicController {

    // 数据校验接口：@Valid 触发校验，BindingResult 接收校验结果
    @RequestMapping(value = "/userDataVerify", produces = "text/html;charset=UTF-8")
    public String userDataVerify(
            @Valid UserInfo userInfo,  // @Valid 注解：告诉Spring对该对象进行数据校验
            BindingResult result       // 接收校验结果（必须紧跟在@Valid标注的对象后）
    ) {
        // 判断是否存在校验失败的错误
        if (result.hasErrors()) {
            StringBuilder errorInfo = new StringBuilder();
            // 获取所有校验错误信息
            List<ObjectError> errors = result.getAllErrors();
            // 拼接错误信息（换行展示）
            for (ObjectError error : errors) {
                errorInfo.append(error.getDefaultMessage()).append("</br>");
            }
            // 返回错误信息给前端
            return errorInfo.toString();
        }

        // 校验通过：返回正确的用户数据
        return "id:" + userInfo.getId() + "</br>" +
               "name:" + userInfo.getName() + "</br>" +
               "email:" + userInfo.getEmail() + "</br>" +
               "phone:" + userInfo.getPhone() + "</br>";
    }
}
```

## 六、测试结果示例
### 1. 校验不通过（未传任何参数）
访问地址：`http://xxx.com/userDataVerify`返回结果：

```plain
email不能为空
id不能为空
phone不能为空
name不能为空
```

### 2. 校验不通过（参数不全 / 格式错误）
访问地址：`http://xxx.com/userDataVerify?email=123`返回结果：

```plain
name不能为空
邮箱格式不正确
phone不能为空
id不能为空
```

### 3. 校验通过（参数完整且合规）
访问地址：`http://xxx.com/userDataVerify?id=1&name=test&email=123@qq.com&phone=13666666666`返回结果：

```plain
id:1
name:test
email:123@qq.com
phone:13666666666
```

## 七、关键注意事项
1. **@Valid 注解**：必须加在需要校验的实体类参数前，否则 Spring 不会触发校验。
2. **BindingResult 位置**：必须紧跟在 @Valid 标注的对象之后，否则无法接收校验结果。
3. **Getter/Setter 方法**：实体类必须提供属性的 Getter 和 Setter 方法，校验框架通过反射获取属性值进行校验。
4. **错误信息配置**：注解的 `message` 属性用于自定义错误提示，建议明确、友好（方便前端展示给用户）。
5. **组合校验**：复杂场景可组合多个注解（如 `@NotNull + @Length`），实现多规则校验。
