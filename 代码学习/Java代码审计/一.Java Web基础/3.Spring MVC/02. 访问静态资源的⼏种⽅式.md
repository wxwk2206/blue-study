##  前⾔  
```xml
如果在web.xml中配置DispatcherServlet时配置的url-pattern为“/”时
代表所有请求都交由Spring容器管理,这时会发现,静态资源,⽐如图⽚资源,js资源,html资源等
都会变的访问不到,成为404,因此本⽂提供⼀些常⽤的解决⽅案
```

##  解决⽅案  
###  ⽅案⼀: 使⽤<mvc:default-servlet-handler/>
```xml
找到你项⽬的SpringMVC配置⽂件,⼀般在resources⽬录,叫springmvc.xml
在<beans></beans>标签中添加如下内容:
<mvc:default-servlet-handler/>
<mvc:annotation-driven/>
```

###  ⽅案⼆: 使⽤<mvc:resources>
```xml
找到你项⽬的SpringMVC配置⽂件,打开它,⼀般在resources目录,叫springmvc.xml
在<beans></beans>标签中添加如下内容:
⽅式⼀: 
<mvc:resources mapping="/**" location="/"></mvc:resources>
<mvc:annotation-driven/>
⽅式⼆: 
<mvc:resources mapping="/resource/**" location="/resource/"></mvc:resources>
<mvc:annotation-driven/>
注意-1:location="/" 代表在webapp⽬录下就⽣效
可访问的⽅式: http://xx.com/1.txt

注意-2:location="/resource/" 代表在
webapp/resource/⽬录的都⽣效
可访问的⽅式⼀: http://xx.com/resource/1.txt
可访问的⽅式⼆: http://xx.com/resource/img/1.png

语法格式:mapping = 映射
location = 本地资源路径,注意必须是webapp根⽬录下的路径。
两个*, 它表示映射/下全部的URL,包括⼦路径
```

###  ⽅案三: 使⽤Tomcat的defaultServlet来处理静态⽂件  
```xml
第⼀步:找到你项⽬的SpringMVC配置⽂件,打开它,⼀般在resources⽬录,叫springmvc.xml
在<beans></beans>标签中添加如下内容:<mvc:annotation-driven/>
```

```xml
第⼆步:找到web.xml配置⽂件,打开它,⼀般在/webapp/WEB-INF/⽬录,叫web.xml
在<web-app></web-app>标签中添加如下内容:
注: 以下配置最好写在DispatcherServlet的配置之前
<servlet-mapping>
<servlet-name>default</servlet-name>
<url-pattern>*.html</url-pattern>
</servlet-mapping>
<servlet-mapping>
<servlet-name>default</servlet-name>
<url-pattern>*.jpg</url-pattern>
</servlet-mapping>
<servlet-mapping>
<servlet-name>default</servlet-name>
<url-pattern>*.js</url-pattern>
</servlet-mapping>
<servlet-mapping>
<servlet-name>default</servlet-name>
<url-pattern>*.css</url-pattern>
</servlet-mapping>
```
