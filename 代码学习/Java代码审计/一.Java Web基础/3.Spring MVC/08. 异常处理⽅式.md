## 一、核心异常处理方式总览
Spring MVC 提供三种核心异常处理方式，按作用范围可分为全局和局部两类：

| **处理方式** | **作用范围** | **核心特点** |
| --- | --- | --- |
| SimpleMappingExceptionResolver | 全局 | Spring 内置处理器，配置简单，无需编码 |
| 实现 HandlerExceptionResolver 接口 | 全局 | 自定义程度高，需编写实现类 |
| @ExceptionHandler 注解 | 局部 / 全局 | 注解驱动，灵活度高；配合 @ControllerAdvice 可升级为全局 |


**关键说明**：

+ 全局异常处理器：可捕获所有 Controller 中的异常
+ 局部异常处理器：仅能处理指定 Controller 中的异常
+ @ControllerAdvice + @ExceptionHandler：将局部处理器升级为全局处理器

## 二、项目目录结构
```plain
├── src
│   ├── main
│   │   ├── com/test
│   │   │   ├── controller       # 控制器（含异常处理控制器）
│   │   │   ├── exception         # 自定义异常处理器
│   │   │   └── ...（其他包：model/converter等）
│   │   ├── resources
│   │   │   └── springmvc.xml     # Spring MVC 核心配置文件
│   │   └── webapp
│   │       ├── WEB-INF
│   │       │   ├── view
│   │       │   │   └── error.jsp # 异常处理页面
│   │       │   └── web.xml       # Web 配置文件
│   │       └── index.jsp
│   └── pom.xml                   # 依赖配置（Maven）
```

## 三、前置操作（通用准备）
所有异常处理方式都需先完成以下基础配置：

### 1. 创建异常处理页面
+ 路径：`./src/main/webapp/WEB-INF/view/error.jsp`
+ 代码：

```plain
<%@ page contentType="text/html;charset=UTF-8" language="java" %>
<%@ page isELIgnored="false" %>
<html>
<head>
    <title>异常捕获测试</title>
</head>
<body>
    异常捕获测试-${ex} <!-- ex 为异常信息变量名，可自定义 -->
</body>
</html>
```

### 2. 创建测试控制器
+ 路径：`./src/main/com/test/controller/TestExceptionHandle.java`
+ 代码：

```java
package test.controller;

import org.springframework.web.bind.annotation.RestController;
import org.springframework.web.bind.annotation.RequestParam;
import org.springframework.web.bind.annotation.RequestMapping;

@RestController
public class TestExceptionHandle {
    @RequestMapping("/testException")
    public String testException(@RequestParam("i") Integer i) {
        // 故意制造除零异常，用于测试
        return "10/" + i + "=" + (10 / i);
    }
}
```

### 3. 访问测试地址
+ 正常访问：`http://xxx.com/testException?i=1` → 返回 `10/1=10`
+ 异常访问：`http://xxx.com/testException?i=0` → 触发除零异常，进入异常处理流程

## 四、具体异常处理实现
### 方式一：SimpleMappingExceptionResolver（内置全局处理器）
#### 核心原理
通过 Spring MVC 配置文件配置内置处理器，映射异常类型到指定页面，无需编写 Java 代码。

#### 配置步骤
1. 找到 Spring MVC 配置文件 `springmvc.xml`（路径：`./src/main/resources/`）
2. 在 `<beans>` 标签内添加如下配置：

```xml
<!-- 配置简单异常处理器 -->
<bean class="org.springframework.web.servlet.handler.SimpleMappingExceptionResolver">
  <!-- 1. 映射异常类型到处理页面（key：异常全类名，value：页面名） -->
  <property name="exceptionMappings">
    <props>
      <!-- 所有异常都映射到 error.jsp（视图解析器自动拼接路径） -->
      <prop key="java.lang.Exception">error</prop>
    </props>
  </property>
  <!-- 2. 定义异常信息变量名（默认是 exception，这里改为 ex） -->
  <property name="exceptionAttribute" value="ex"></property>
</bean>
```

#### 效果验证
访问异常地址 `http://xxx.com/testException?i=0`，页面显示：`异常捕获测试-java.lang.ArithmeticException: / by zero`

### 方式二：实现 HandlerExceptionResolver（自定义全局处理器）
#### 核心原理
通过实现 Spring 提供的 `HandlerExceptionResolver` 接口，编写自定义异常处理逻辑，灵活性更高。

#### 实现步骤
1. 创建自定义异常处理器类
    - 路径：`./src/main/com/test/exception/MyExceptionMappingResolver.java`
    - 代码：

```java
package test.exception;

import java.util.HashMap;
import java.util.Map;
import javax.servlet.http.HttpServletRequest;
import javax.servlet.http.HttpServletResponse;
import org.springframework.web.servlet.HandlerExceptionResolver;
import org.springframework.web.servlet.ModelAndView;

public class MyExceptionMappingResolver implements HandlerExceptionResolver {
    @Override
    public ModelAndView resolveException(
        HttpServletRequest request,
        HttpServletResponse response,
        Object handler,
        Exception ex
    ) {
        // 存储异常信息
        Map<String, Object> map = new HashMap<>();
        map.put("ex", ex);
        // 返回 ModelAndView：参数1=页面名，参数2=异常信息Map
        return new ModelAndView("error", map);
    }
}
```

1. 配置自定义处理器到 `springmvc.xml`

```xml
<!-- 注册自定义异常处理器 -->
<bean id="exceptionhandler" class="test.exception.MyExceptionMappingResolver"/>
```

#### 效果验证
访问异常地址，页面显示与方式一一致。

### 方式三：@ExceptionHandler 注解（局部 / 全局处理器）
#### 子方式 1：全局异常处理器（@ControllerAdvice + @ExceptionHandler）
##### 核心原理
用 `@ControllerAdvice` 注解标识全局异常处理类，配合 `@ExceptionHandler` 捕获所有 Controller 的异常。

##### 实现步骤
1. 创建全局异常处理控制器
    - 路径：`./src/main/com/test/controller/BaseExceptionHandlerController.java`
    - 代码：

```java
package test.controller;

import org.springframework.ui.Model;
import org.springframework.web.bind.annotation.ControllerAdvice;
import org.springframework.web.bind.annotation.ExceptionHandler;

// 该注解标识为全局异常处理器
@ControllerAdvice
public class BaseExceptionHandlerController {
    // @ExceptionHandler：标识异常处理方法
    @ExceptionHandler
    public String execute(Model model, Exception ex) {
        // 向页面传递异常信息
        model.addAttribute("ex", ex);
        // 返回异常处理页面
        return "error";
    }
}
```

##### 效果验证
访问异常地址，页面显示与前两种方式一致。

#### 子方式 2：局部异常处理器（仅处理指定 Controller 异常）
##### 核心原理
将 `@ExceptionHandler` 注解添加到基础控制器类中，目标 Controller 继承该基础类，仅捕获该 Controller 的异常。

##### 实现步骤
1. 创建基础异常处理类（局部）
    - 路径：`./src/main/com/test/controller/Base2ExceptionHandlerController.java`
    - 代码：

```java
package test.controller;

import org.springframework.web.bind.annotation.ExceptionHandler;

public class Base2ExceptionHandlerController {
    // 仅捕获 ArithmeticException 类型异常（可指定多个异常类）
    @ExceptionHandler({ArithmeticException.class})
    public String execute(Exception ex) {
        // 返回字符串（因目标 Controller 用 @RestController，不返回视图）
        return "error-" + ex;
    }
}
```

2. 目标 Controller 继承基础异常处理类
    - 修改 `TestExceptionHandle.java`：

```java
package test.controller;

import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.RestController;
import org.springframework.web.bind.annotation.RequestParam;

@RestController
// 继承基础异常处理类，获得局部异常处理能力
public class TestExceptionHandle extends Base2ExceptionHandlerController {
    @RequestMapping("/testException")
    public String testException(@RequestParam("i") Integer i) {
        return "10/" + i + "=" + (10 / i);
    }
}
```

##### 关键说明
+ 若目标 Controller 用 `@RestController` 或方法用 `@ResponseBody`，异常处理方法返回字符串而非视图
+ `@ExceptionHandler({异常类1.class, 异常类2.class})` 可指定捕获的异常类型

##### 效果验证
访问异常地址 `http://xxx.com/testException?i=0`，返回字符串：`error-java.lang.ArithmeticException: / by zero`

## 五、关键注意事项
1. 视图解析器配置：异常处理页面的路径由视图解析器（`InternalResourceViewResolver`）拼接，需确保配置一致
    - 示例配置：路径前缀 `/WEB-INF/view/` + 后缀 `.jsp` → 页面名 `error` 对应路径 `./WEB-INF/view/error.jsp`
2. 异常信息变量名：默认是 `exception`，可通过配置（方式一）或代码（方式二、三）自定义为 `ex` 等
3. 异常捕获优先级：局部异常处理器（方式三.2）优先级高于全局处理器（方式一、二、三.1）
4. 注解冲突：`@RestController` 会导致异常处理方法返回字符串而非视图，需根据需求选择注解
