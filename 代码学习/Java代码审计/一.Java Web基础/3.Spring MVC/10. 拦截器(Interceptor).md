## 一、前言
Spring MVC 的拦截器（Interceptor）主要用于拦截用户请求并进行相应处理，常见应用场景包括权限验证、记录请求日志、判断用户是否登录等。

## 二、拦截器定义方式
Spring MVC 中定义拦截器有两种核心方式，均需先定义实现类，再进行配置。

### （一）HandlerInterceptor 接口实现（常用）
#### 1. 实现要求
创建类实现 `HandlerInterceptor` 接口，并实现接口中的三个抽象方法。

#### 2. 核心方法说明
| **方法名** | **执行时机** | **返回值** | **核心作用** |
| --- | --- | --- | --- |
| `preHandle(HttpServletRequest request, HttpServletResponse response, Object handler)` | 控制器处理请求方法**之前**执行 | boolean | 返回 `true`<br/> 表示继续后续操作，返回 `false`<br/> 表示中断后续操作（可用于请求阻断，如未登录拦截） |
| `postHandle(HttpServletRequest request, HttpServletResponse response, Object handler, ModelAndView modelAndView)` | 控制器处理请求方法调用**之后**、视图解析**之前**执行 | void | 可修改请求域中的模型（Model）和视图（View） |
| `afterCompletion(HttpServletRequest request, HttpServletResponse response, Object handler, Exception ex)` | 控制器处理请求方法执行**完成后**（视图渲染结束后） | void | 用于资源清理、记录日志等 |


#### 3. 代码示例
```java
package test.interceptor;
import org.springframework.web.servlet.HandlerInterceptor;
import org.springframework.web.servlet.ModelAndView;
import javax.servlet.http.HttpServletRequest;
import javax.servlet.http.HttpServletResponse;

public class TestAllHandlerInterceptor implements HandlerInterceptor {
    @Override
    public boolean preHandle(HttpServletRequest request, HttpServletResponse response, Object handler) throws Exception {
        System.out.println("TestAllHandlerInterceptor类的preHandle方法在控制器的处理请求方法前执行");
        return true;
    }

    @Override
    public void postHandle(HttpServletRequest request, HttpServletResponse response, Object handler, ModelAndView modelAndView) throws Exception {
        System.out.println("TestAllHandlerInterceptor类的postHandle方法在控制器的处理请求方法调用之后，解析视图之前执行");
    }

    @Override
    public void afterCompletion(HttpServletRequest request, HttpServletResponse response, Object handler, Exception ex) throws Exception {
        System.out.println("TestAllHandlerInterceptor类的afterCompletion方法在控制器的处理请求方法执行完成后执行，即视图渲染结束之后执行");
    }
}
```

### （二）WebRequestInterceptor 接口实现
#### 1. 实现要求
创建类实现 `WebRequestInterceptor` 接口，并实现接口中的三个抽象方法。

#### 2. 核心特性
+ 方法参数为 `WebRequest`（Spring 对 `HttpServletRequest` 的封装），所有操作同步到 `HttpServletRequest`，并在当前请求中传递。
+ `WebRequest` 的 `getAttribute()`/`setAttribute()` 方法多了 `scope` 参数，支持三种作用域：
    - `SCOPE_REQUEST`（0）：请求作用域
    - `SCOPE_SESSION`（1）：会话作用域
    - `SCOPE_GLOBAL_SESSION`（2）：全局会话作用域（ServletContext 上下文）

#### 3. 核心方法说明
| **方法名** | **执行时机** | **返回值** | **核心作用** |
| --- | --- | --- | --- |
| `preHandle(WebRequest request)` | 控制器处理请求方法**之前**执行 | void | 不可用于请求阻断，仅用于资源准备（如设置请求属性） |
| `postHandle(WebRequest request, ModelMap model)` | 控制器处理请求方法调用**之后**、视图解析**之前**执行 | void | 可修改 `ModelMap`<br/>（控制器返回的模型对象），改变视图渲染效果 |
| `afterCompletion(WebRequest request, Exception ex)` | 控制器处理请求方法执行**完成后**（视图渲染结束后） | void | 处理控制器抛出的异常（若已被 Spring 异常处理器处理，`ex`<br/> 为 null） |


#### 4. 代码示例
```java
package test.interceptor;
import org.springframework.ui.ModelMap;
import org.springframework.web.context.request.WebRequest;
import org.springframework.web.context.request.WebRequestInterceptor;

public class TestAllWebRequestInterceptor implements WebRequestInterceptor {
    @Override
    public void preHandle(WebRequest request) throws Exception {
        request.setAttribute("preHandle-testkey1", "testvalue1", WebRequest.SCOPE_REQUEST);
        System.out.println("TestAllWebRequestInterceptor类的preHandle方法在控制器的处理请求方法前执行");
    }

    @Override
    public void postHandle(WebRequest request, ModelMap model) throws Exception {
        request.setAttribute("postHandle-testkey", "testvalue2", WebRequest.SCOPE_REQUEST);
        System.out.println("TestAllWebRequestInterceptor类的postHandle方法在控制器的处理请求方法调用之后，解析视图之前执行");
    }

    @Override
    public void afterCompletion(WebRequest request, Exception ex) throws Exception {
        request.setAttribute("afterCompletion-testkey", "testvalue3", WebRequest.SCOPE_REQUEST);
        System.out.println("TestAllWebRequestInterceptor类的afterCompletion方法在控制器的处理请求方法执行完成后执行，即视图渲染结束之后执行");
    }
}
```

## 三、拦截器配置（springmvc.xml）
### （一）配置位置
在项目 `src/main/resources/springmvc.xml` 的 `<beans></beans>` 标签内添加配置。

### （二）配置类型及语法
#### 1. 全局拦截器（拦截所有请求）
```xml
<mvc:interceptors>
  <!-- 全局拦截器：所有 Spring MVC 请求都会经过 -->
  <bean class="test.interceptor.TestAllHandlerInterceptor"/>
  <bean class="test.interceptor.TestAllWebRequestInterceptor"/>
</mvc:interceptors>
```

#### 2. 路径指定拦截器（仅拦截特定路径）
```xml
<mvc:interceptors>
  <mvc:interceptor>
    <!-- 拦截路径：/** 表示所有路径，/gotoTest 表示特定路径 -->
    <mvc:mapping path="/gotoTest"/>
    <!-- 排除路径：不拦截的 URL -->
    <mvc:exclude-mapping path="/user/login"/>
    <mvc:exclude-mapping path="/user/logout"/>
    <mvc:exclude-mapping path="/**.ico"/>
    <mvc:exclude-mapping path="/**.js"/>
    <mvc:exclude-mapping path="/**.jpg"/>
    <mvc:exclude-mapping path="/**.gif"/>
    <mvc:exclude-mapping path="/**.css"/>
    <!-- 拦截器实现类 -->
    <bean class="test.interceptor.Test2HandlerInterceptor"/>
  </mvc:interceptor>
</mvc:interceptors>
```

### （三）配置规则
1. `<mvc:interceptor>` 子元素顺序必须为：`mapping` → `exclude-mapping` → `bean`。
2. 路径通配符说明：
    - `?`：匹配任意单个字符（如 `/hello?` 匹配 `/helloa`、`/hellob`，不匹配 `/hello`、`/helloab`）。
    - `*`：匹配任意一层目录（如 `/admin/*` 匹配 `/admin/add`、`/admin/save`，不匹配 `/admin/add/user`）。
    - `**`：匹配任意一层或多层目录（如 `/admin/**` 匹配 `/admin/add`、`/admin/add/user`；`/**.js` 匹配所有 `.js` 结尾的文件）。
3. `<mvc:exclude-mapping>`：指定不被拦截的路径（如登录、注销接口，静态资源等）。

### （四）多拦截器执行顺序
+ `preHandle` 方法：按配置文件中拦截器的**配置顺序**执行。
+ `postHandle` 和 `afterCompletion` 方法：按配置顺序的**反序**执行。

示例：配置顺序为 `TestAllHandlerInterceptor` → `TestAllWebRequestInterceptor`

+ 执行顺序：
    1. `TestAllHandlerInterceptor.preHandle`
    2. `TestAllWebRequestInterceptor.preHandle`
    3. `TestAllWebRequestInterceptor.postHandle`
    4. `TestAllHandlerInterceptor.postHandle`
    5. `TestAllWebRequestInterceptor.afterCompletion`
    6. `TestAllHandlerInterceptor.afterCompletion`

## 四、示例演示
### （一）项目目录结构
```plain
├── src
│   ├── main
│   │   ├── java
│   │   │   └── test
│   │   │       ├── controller  // 控制器
│   │   │       ├── interceptor  // 拦截器实现类
│   │   │       └── ...
│   │   ├── resources
│   │   │   └── springmvc.xml  // 拦截器配置文件
│   │   └── webapp
│   │       ├── WEB-INF
│   │       │   ├── web.xml
│   │       │   └── index.jsp
│   └── pom.xml
```

### （二）前置操作：创建测试控制器
```java
package test.controller;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.RestController;

@RestController
public class TController {
    @RequestMapping(value = "/tTest")
    public String tTest() {
        return "helloTest";
    }

    @RequestMapping(value = "/gotoTest")
    public String gotoTest() {
        return "gotoTest";
    }
}
```

### （三）HandlerInterceptor 示例结果
1. 配置：全局拦截器 `TestAllHandlerInterceptor` + 路径拦截器 `Test2HandlerInterceptor`（拦截 `/gotoTest`）。
2. 访问 `http://xxx.com/tTest`（仅触发全局拦截器）：

```plain
TestAllHandlerInterceptor类的preHandle方法在控制器的处理请求方法前执行
TestAllHandlerInterceptor类的postHandle方法在控制器的处理请求方法调用之后，解析视图之前执行
TestAllHandlerInterceptor类的afterCompletion方法在控制器的处理请求方法执行完成后执行，即视图渲染结束之后执行
```

3. 访问 `http://xxx.com/gotoTest`（触发全局 + 路径拦截器）：

```plain
TestAllHandlerInterceptor类的preHandle方法在控制器的处理请求方法前执行
Test2HandlerInterceptor类的preHandle方法在控制器的处理请求方法前执行
Test2HandlerInterceptor类的postHandle方法在控制器的处理请求方法调用之后，解析视图之前执行
TestAllHandlerInterceptor类的postHandle方法在控制器的处理请求方法调用之后，解析视图之前执行
Test2HandlerInterceptor类的afterCompletion方法在控制器的处理请求方法执行完成后执行，即视图渲染结束之后执行
TestAllHandlerInterceptor类的afterCompletion方法在控制器的处理请求方法执行完成后执行，即视图渲染结束之后执行
```

### （四）WebRequestInterceptor 示例结果

2. 访问 `http://xxx.com/tTest`：

TestAllWebRequestInterceptor类的postHandle方法在控制器的处理请求方法调用之后，解析视图之前执行
TestAllWebRequestInterceptor类的afterCompletion方法在控制器的处理请求方法执行完成后执行，即视图渲染结束之后执行
