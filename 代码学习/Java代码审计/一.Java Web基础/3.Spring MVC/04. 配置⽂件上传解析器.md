##  前⾔  
```plain
上传时,爆错: 由于没有提供multi-part配置,⽆法处理parts
这是因为没配置⽂件上传解析器导致的,这⾥列出解决⽅案
```

##  前置操作  
###  导⼊commons-fileupload包  
```xml
为了防⽌爆下⾯这个错误
Error creating bean with name 'multipartResolver': Lookup method resolution failed
我们需要引⼊commons-fileupload包
在建好的项⽬下找到pom.xml⽂件并打开路径: ./项⽬/pom.xml添加commons-fileupload依赖
注: 在<dependencies></dependencies> 标签中添加如下数据,没有这个标签就⾃⼰创建
<!-- https://mvnrepository.com/artifact/commons-fileupload/commons-fileupload -->
<dependency>
  <groupId>commons-fileupload</groupId>
  <artifactId>commons-fileupload</artifactId>
  <version>1.4</version>
</dependency>
```

##  解决⽅案  
###  ⽅案⼀: springmvc.xml配置⽂件中配置⽂件上传解析器  
```xml
在建好的项⽬下找到springmvc.xml⽂件并打开路径: ./项⽬/src/main/resources/springmvc.xml
在<beans></beans>标签中添加如下内容:
<bean id="multipartResolver" class="org.springframework.web.multipart.commons.CommonsMultipartResolver">
<property name="maxUploadSize">
<value>5242880</value>
</property>
</bean>
```

###  ⽅案⼆: web.xml配置⽂件中配置⽂件上传解析器  
```xml
在建好的项⽬下找到web.xml⽂件并打开
路径: ./项⽬/src/main/resources/web.xml
修改你的DispatcherServlet配置⽂件上传解析器
在<web-app></web-app>标签中添加如下内容:
<servlet>
<!-- 要注册servlet的名字 -->
<servlet-name>ServletTestSpringMVC</servlet-name>
<!-- 要注册的servlet的类地址 -->
<servlet-class>org.springframework.web.servlet.DispatcherServlet</servlet-class>
<!-- 初始化参数 -->
<init-param>
<param-name>contextConfigLocation</param-name>
<param-value>classpath:springmvc.xml</param-value>
</init-param>
<!--       1. <load-on-startup>元素,标记容器是否在启动的时候就加载这个servlet
  (实例化并调⽤其init()⽅法)
           2. 它的值必须是⼀个整数,表示servlet应该被载⼊的顺序
           3.当值为0或者⼤于0时,表示容器在应⽤启动时就加载并初始化这个servlet
           4.当值⼩于0或者没有指定时,则表示容器在该servlet被选择时才会去加载
          5.正数的值越⼩,该servlet的优先级越⾼,应⽤启动时就越先加载
          6.当值相同时,容器就会⾃⼰选择顺序来加载 -->
<load-on-startup>1</load-on-startup>
<!-- 配置⽂件上传解析器 -->
<multipart-config>
<max-file-size>20848820</max-file-size>
<max-request-size>418018841</max-request-size>
<file-size-threshold>1048576</file-size-threshold>
</multipart-config>
</servlet>
<servlet-mapping>
<servlet-name>ServletTestSpringMVC</servlet-name>
<url-pattern>/</url-pattern>
</servlet-mapping>
```
