## 一、应用场景
Velocity 常用于模板文件中，可实现变量替换、逻辑控制、模板引入等功能，适用于动态页面生成、配置模板化等场景，所有操作需在模板文件中测试使用。

## 二、核心基础
### （一）语句标识符
Velocity 关键字均以 `#` 开头，常用关键字包括 `#set`、`#if`、`#else`、`#end`、`#foreach`、`#include`、`#parse`、`#macro` 等。

### （二）变量相关
1. **替换变量**：通过 `$变量名` 或 `${变量名}` 标识变量，模板中可直接引用上下文传递的变量。示例：模板文件中写 `Hello $a`，可获取上下文传递的 `$a` 值。
2. **变量定义**：使用 `#set` 指令定义变量，语法为 `#set($变量名 = 值)`。示例：

```plain
#set($root = "www") // 定义 root 变量值为 www
#set($name = "index") // 定义 name 变量值为 index
#set($template = "$root/$name") // 定义 template 变量值为 www/index
hello $template // 输出结果：hello www/index
```

3. **变量赋值**
    - 赋值规则：赋值左边必须是变量或属性引用；右边可支持变量引用、字面字符串、属性引用、方法引用、字面数字、数组。
    - 示例：

```plain
#set($name = $bill) // 变量引用
#set($name.pre = "monica") // 字面字符串
#set($name.last = $address.num) // 属性引用
#set($name.mid = $hotel.find($web)) // 方法引用
#set($name.num = 123) // 字面数字
#set($name.say = ["yes",$my,"yes"]) // 数组
```

+ 特殊说明：Velocity 会将属性解释为对应的 get 方法，例如 `$foo.Bar` 等同于 `$foo.getBar()`，`$foo.User("join")` 等同于 `$foo.getUser("join")`，`$foo.Request.ServerName` 等同于 `$foo.getRequest().getServerName()`。

## 三、核心功能
### （一）调用对象方法
可直接调用变量对应的对象方法，支持方法返回值的后续操作（如数组索引取值）。示例：

```plain
#set($name = "abc def")
$name.split(" ")[0] // 结果是 abc
```

### （二）条件控制（if-else）
通过 `#if`、`#elseif`、`#else`、`#end` 实现条件判断，支持逻辑运算符组合条件。语法：

```plain
#if($prop.name!='username' && $prop.name!='password')
    // 满足条件的逻辑
#elseif($prop.name=='id')
    // 满足分支条件的逻辑
#else
    // 其他情况的逻辑
#end
```

### （三）循环遍历（foreach）
通过 `#foreach`、`#end` 遍历集合，支持 `#break` 终止循环，常用于遍历 List 等集合。语法：

```velocity
#foreach($prop in $struct.cppPropList)
    $prop // 遍历输出集合中的元素
    #break // 可选，用于终止循环
#end
```

### （四）模板引入（include、parse）
两种引入外部模板的方式，核心区别在于是否解析模板内容：

| **指令** | **功能说明** | **特点** | **示例** |
| --- | --- | --- | --- |
| #include | 包含外部模板文件 | 仅引入内容，不进行 Velocity 语法解析 | `#include("/template/utils/compare.vm")` |
| #parse | 解析并引入外部模板 | 会解析模板中的 Velocity 语法 | `#parse("/template/component/menu.vm")` |


+ 注意：引入路径基于 VelocityEngine 中设置的读取基本路径（类路径）。

### （五）公共模块定义（define）
通过 `#define` 定义可复用的公共模块，后续可直接通过模块名调用，调用时会使用当前上下文的变量值。示例：

```velocity
#set($name = "小白")
#define($common)
    hello,$name,你好呀!
#end

#set($name = "王尼玛")
$common // 输出：hello,王尼玛,你好呀!

#set($name = "张三")
$common // 输出：hello,张三,你好呀!
```

### （六）动态解析（evaluate）
支持解析后端传入的 Velocity 语句字符串，实现动态执行语法逻辑。示例：

1. Java 后端传入语句：`context.put("v","#set($name = \"王尼玛\")");`
2. 模板中解析执行：

```velocity
#evaluate($v) // 执行传入的 Velocity 语句
$name // 解析结果：王尼玛
```

## 四、运算符与注释
### （一）关系和逻辑操作符

| **运算符** | **功能** | **示例** |
| --- | --- | --- |
| && | 逻辑与（并且） | `$a && $b` |
|  ||   | 逻辑或（或者） | `$a || $b` |
| ! | 逻辑非（取反） | `!$a` |
| == | 等于 | `$a == $b` |
| != | 不等于 | `$a != $b` |


### （二）注释
1. 单行注释：使用 `##` 开头，注释内容仅占一行。示例：`## 这是单行注释`
2. 多行注释：使用 `#*` 开头、`*#` 结尾，支持多行注释内容。示例：

```velocity
#*
    这是多行注释
    可以换行书写
    注释内容不会被解析执行
*#
```
