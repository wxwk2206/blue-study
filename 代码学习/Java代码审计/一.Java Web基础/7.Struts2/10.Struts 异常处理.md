## 一、核心概述
+ Struts2 提供简化的未捕获异常处理机制，可为不同异常配置专属错误页面
+ 核心依赖 `exception` 拦截器，该拦截器已集成在默认拦截器堆栈中，无需额外配置，开箱即用

## 二、全局异常处理案例（完整步骤）
### 1. 步骤 1：创建含异常的 Action 类
+ 路径：`./项目/src/main/webapp/action/Test6Action.java`
+ 功能：主动触发 `NullPointerException` 空指针异常

```java
package action;
import com.opensymphony.xwork2.ActionSupport;

public class Test6Action extends ActionSupport {
    @Override
    public String execute() throws Exception {
        String x = null;
        x = x.substring(0); // 触发空指针异常
        return "index";
    }
}
```

### 2. 步骤 2：创建正常页面（index.jsp）
+ 路径：`./项目/src/main/webapp/index.jsp`
+ 作用：Action 无异常时跳转的页面

```html
<html>
  <body>
    <h2>Hello World!</h2>
  </body>
</html>
```

### 3. 步骤 3：创建自定义错误页面（error2.jsp）
+ 路径：`./项目/src/main/webapp/error2.jsp`
+ 作用：捕获指定异常后显示的自定义错误页面

```html
<%@ page contentType="text/html;charset=UTF-8" language="java" %>
  <html>
    <head>
      <title>error2</title>
    </head>
    <body>
      这是我的自定义错误页面
    </body>
  </html>
```

### 4. 步骤 4：配置 struts.xml（核心配置）
+ 路径：`./项目/src/main/webapp/WEB-INF/classes/struts.xml`
+ 核心配置：定义全局结果集、全局异常映射、Action 映射

```xml
<struts>
  <package name="test6Demo" namespace="/" extends="struts-default">
    <!-- 1. 全局结果集：映射错误页面 -->
    <global-results>
      <result name="error2">/error2.jsp</result>
    </global-results>

    <!-- 2. 全局异常映射：关联异常类型与结果名称 -->
    <global-exception-mappings>
      <!-- 捕获空指针异常，对应结果名称error2 -->
      <exception-mapping exception="java.lang.NullPointerException" result="error2"/>
    </global-exception-mappings>

    <!-- 3. Action配置：绑定Action类与正常页面 -->
    <action name="test6" class="action.Test6Action">
      <result name="index">/index.jsp</result>
    </action>
  </package>
</struts>
```

### 5. 步骤 5：测试验证
+ 访问地址：`http://127.0.0.1:8081/Struts2Test_war/test6.action`
+ 预期结果：页面显示 "这是我的自定义错误页面"（成功捕获空指针异常）

## 三、关键说明
1. 全局异常作用范围：当前 `package` 下的所有 Action 均可复用该异常配置
2. 异常拦截器：无需手动注册，因 `struts-default` 包已包含默认拦截器堆栈
3. 配置逻辑：异常发生时，`exception` 拦截器捕获异常，匹配 `exception-mapping` 中的异常类型，跳转至对应 `result` 配置的页面
