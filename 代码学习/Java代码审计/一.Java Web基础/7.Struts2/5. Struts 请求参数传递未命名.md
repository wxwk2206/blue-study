## 一、前言
在 Struts2 项目开发中，前台与后台之间的参数传递是核心基础操作，本文整理了三种最常用的参数传递方式，包含详细实现步骤、代码示例及测试结果。

## 二、项目目录结构
```plain
├── src
│   └── main
│       └── webapp
│           ├── action（存放 Action 类）
│           ├── model（存放模型类，如 User）
│           ├── WEB-INF
│           │   ├── classes（存放配置文件 struts.xml）
│           │   └── web.xml（Web 配置文件）
│           ├── index.jsp（首页）
│           └── 各类视图.jsp（如 test2View.jsp、test3View.jsp 等）
└── pom.xml（Maven 依赖配置）
```

## 三、参数传递方式
### （一）属性驱动 - 通过成员 setter 封装
#### 1. 实现步骤
1. 创建 Action 类，定义成员变量，提供 setter 和 getter 方法，重写 execute 方法处理业务逻辑。
2. 创建视图 JSP，通过 Struts 标签展示参数。
3. 在 struts.xml 中配置 Action 映射关系。
4. 访问测试。

#### 2. 代码示例
+ Action 类（路径：./src/main/webapp/action/Test2Action.java）

```java
package action;
import com.opensymphony.xwork2.ActionSupport;

public class Test2Action extends ActionSupport {
    // 定义接收参数的成员变量
    private String name;

    @Override
    public String execute() throws Exception {
        System.out.println("name=" + this.getName());
        return "test2View"; // 对应视图名称
    }

    // setter 方法（关键：Struts2 通过 setter 注入参数）
    public void setName(String name) {
        this.name = name;
    }

    // getter 方法（供视图获取参数）
    public String getName() {
        return this.name;
    }
}
```

+ 视图 JSP（路径：./src/main/webapp/test2View.jsp）

```plain
<%@ page contentType="text/html;charset=UTF-8" language="java" %>
<%@ taglib prefix="s" uri="/struts-tags" %> <!-- 引入 Struts 标签库 -->
<s:property value="name"/> <!-- 展示接收的参数 -->
```

+ Struts 配置（路径：./src/main/webapp/WEB-INF/classes/struts.xml）

```xml
<struts>
  <package name="test2Demo" namespace="/" extends="struts-default">
    <!-- action 名称为 test2，对应 Action 类路径 -->
    <action name="test2" class="action.Test2Action">
      <!-- 结果集：返回 test2View 时跳转至 test2View.jsp -->
      <result name="test2View">/test2View.jsp</result>
    </action>
  </package>
</struts>
```

#### 3. 测试
+ 访问地址：`http://localhost:8080/struts_war/test2.action?name=test1234`
+ 控制台输出：`name=test1234`
+ 页面输出：`test1234`

### （二）调用 ServletActionContext 获得 Servlet API
#### 1. 核心说明
通过 `ServletActionContext.getRequest()` 方法获取 HttpServletRequest 对象，直接操作 Servlet API 接收参数，适用于熟悉 JSP/Servlet 开发的场景。

#### 2. 依赖解决（IDEA 环境）
若 IDEA 提示 `javax.servlet.http.HttpServletRequest` 类未找到，按以下步骤添加 Tomcat 依赖：

1. 打开 `File -> Project Structure`；
2. 进入 `Modules -> Dependencies` 选项卡；
3. 点击 `+` 选择 `Library -> Application Server Libraries`；
4. 选择当前项目使用的 Tomcat 版本（如 Tomcat 9.0.48）；
5. 点击 `Add Selected`，应用并确定配置。

#### 3. 代码示例
+ Action 类（路径：./src/main/webapp/action/Test3Action.java）

```java
package action;
import com.opensymphony.xwork2.ActionSupport;
import org.apache.struts2.ServletActionContext;
import javax.servlet.http.HttpServletRequest;

public class Test3Action extends ActionSupport {
    @Override
    public String execute() throws Exception {
        // 获取 HttpServletRequest 对象
        HttpServletRequest request = ServletActionContext.getRequest();
        // 接收请求参数
        String value1 = request.getParameter("value1");
        String value2 = request.getParameter("value2");
        // 存储参数到 request 域（供视图获取）
        request.setAttribute("value1", value1);
        request.setAttribute("value2", value2);
        // 控制台输出参数
        System.out.println("value1:" + value1 + " " + "value2:" + value2);
        return "test3View";
    }
}
```

+ 视图 JSP（路径：./src/main/webapp/test3View.jsp）

```plain
<%@ page contentType="text/html;charset=UTF-8" language="java" %>
<html>
<head>
    <title>test3</title>
</head>
<body>
    <!-- 两种方式获取参数：EL 表达式或 Servlet API -->
    <div>value1:<%=request.getParameter("value1")%></div>
    <div>value2:${value2}</div>
</body>
</html>
```

+ Struts 配置（路径：./src/main/webapp/WEB-INF/classes/struts.xml）

```xml
<struts>
  <package name="test3Demo" namespace="/" extends="struts-default">
    <action name="test3" class="action.Test3Action">
      <result name="test3View">/test3View.jsp</result>
    </action>
  </package>
</struts>
```

#### 4. 测试
+ 访问地址：`http://127.0.0.1:8081/Struts2Test_war/test3.action?value1=123&value2=456`
+ 控制台输出：`value1:123 value2:456`
+ 页面输出：

```plain
value1:123
value2:456
```

### （三）属性驱动 - 通过 ModelDriven
#### 1. 核心说明
实现 ModelDriven 接口，直接操作模型对象（如 User）接收参数，适用于参数较多的场景，简化代码结构。

#### 2. 代码示例
+ 模型类（路径：./src/main/webapp/model/User.java，需自行创建）

```java
package model;
// 模型类：封装用户相关参数
public class User {
    private String username;
    private String password;

    // setter 和 getter 方法
    public String getUsername() {
        return username;
    }

    public void setUsername(String username) {
        this.username = username;
    }

    public String getPassword() {
        return password;
    }

    public void setPassword(String password) {
        this.password = password;
    }
}
```

+ Action 类（路径：./src/main/webapp/action/Test4Action.java）

```java
package action;
import com.opensymphony.xwork2.ActionSupport;
import com.opensymphony.xwork2.ModelDriven;
import model.User;
import org.apache.struts2.ServletActionContext;
import javax.servlet.http.HttpServletRequest;

// 实现 ModelDriven 接口，指定模型类型为 User
public class Test4Action extends ActionSupport implements ModelDriven<User> {
    // 必须实例化模型对象（关键）
    private User user = new User();

    @Override
    public String execute() throws Exception {
        HttpServletRequest request = ServletActionContext.getRequest();
        // 直接通过模型对象获取参数
        System.out.println("user.username:" + user.getUsername() + " user.password:" + user.getPassword());
        // 存储模型对象到 request 域
        request.setAttribute("user", user);
        return "test4View";
    }

    // 实现 ModelDriven 接口的 getModel 方法，返回模型对象
    @Override
    public User getModel() {
        return user;
    }
}
```

+ 视图 JSP（路径：./src/main/webapp/test4View.jsp）

```plain
<%@ page contentType="text/html;charset=UTF-8" language="java" %>
<html>
<head>
    <title>test4</title>
</head>
<body>
    <!-- 通过 EL 表达式获取模型对象的属性 -->
    <div>user.username:${user.username}</div>
    <div>user.password:${user.password}</div>
</body>
</html>
```

+ Struts 配置（路径：./src/main/webapp/WEB-INF/classes/struts.xml）

```xml
<struts>
  <package name="test4Demo" namespace="/" extends="struts-default">
    <action name="test4" class="action.Test4Action">
      <result name="test4View">/test4View.jsp</result>
    </action>
  </package>
</struts>
```

#### 3. 测试
+ 访问地址：`http://127.0.0.1:8081/Struts2Test_war/test4.action?username=123&password=456`
+ 控制台输出：`user.username:123 user.password:456`
+ 页面输出：

```plain
user.username:123
user.password:456
```

## 四、三种方式对比
| **传递方式** | **核心特点** | **适用场景** |
| --- | --- | --- |
| 成员 setter 封装 | 直接定义成员变量，通过 setter 注入，简单直观 | 参数较少（1-2 个）的场景 |
| Servlet API 直接获取 | 依赖 HttpServletRequest，兼容 JSP/Servlet 开发习惯 | 需要操作 Servlet 域对象（request/session）的场景 |
| ModelDriven 接口 | 基于模型对象封装参数，代码整洁，可复用 | 参数较多（3 个及以上）， 需封装为实体类的场景   |
