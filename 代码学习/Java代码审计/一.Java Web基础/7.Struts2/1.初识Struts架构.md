## 一、概述
### （一）核心设计模式
Struts2 框架基于**MVC 设计模式**，即 Model（模型）、View（视图）、Controller（控制器），三者分工明确、协同工作，实现 Web 应用的分层开发。

### （二）MVC 各组件职责

| **组件** | **核心职责** | **实现方式** |
| --- | --- | --- |
| Model（模型） | 管理应用程序数据，实现业务逻辑，响应视图请求和控制器指令进行数据更新 | JavaBean 或 EJB |
| View（视图） | 作为用户交互界面，接收用户输入，显示处理后的数据（不负责数据实际处理） | 1. 主要组件：JSP 页面（承担信息显示、控制器处理结果展示功能）；2. 辅助组件：ActionForm（封装用户表单输入，仅含属性的 getter/setter 方法，无业务逻辑） |
| Controller（控制器） | 接收用户请求和数据，调度对应模型处理，指定处理结果的展示视图 | 核心控制器：ActionServlet（处理所有用户请求，通过 struts.xml 配置文件匹配 URL，分发请求） |


## 二、Struts2 工作流程核心组件
Struts2 通过以下**五大核心部分**实现 MVC 模式的落地，支撑整体工作流程：

1. **Actions（动作）**：作为模型与控制器的衔接核心，封装业务处理逻辑，提供 execute () 方法执行核心操作，通过 setXxx ()/getXxx () 方法处理数据传递；
2. **Interceptors（拦截器）**：在 Action 执行前后进行拦截处理，实现日志记录、权限验证、数据校验等通用功能；
3. **Value Stack（值栈）/OGNL**：负责 Action 与视图之间的数据传递和共享，OGNL（对象图导航语言）用于访问值栈数据；
4. **Result Types（结果类型）**：定义 Action 执行后的跳转逻辑，指定结果展示的方式（如跳转 JSP、FreeMarker 模板等）；
5. **View（视图技术）**：除核心 JSP 外，支持 FreeMarker 等模板引擎，实现灵活的页面渲染。

## 三、Struts2 核心工作流程示意
![[Pasted image 20260713234427.png]]

1. 用户发送 HTTP 请求；
2. 请求经 Filter（过滤器）拦截后，传递至核心控制器 ActionServlet；
3. ActionServlet 通过 struts.xml 配置文件匹配请求 URL，找到对应的 Action；
4. 拦截器（Interceptors）对 Action 进行预处理（如数据校验、权限检查）；
5. Action 调用 Model（JavaBean/EJB）处理业务逻辑，更新数据；
6. Action 执行完成后，通过 Result Types 指定跳转视图；
7. 借助 Value Stack/OGNL 将处理结果数据传递给视图（JSP/FreeMarker 等）；
8. 视图渲染数据并展示给用户。
