## 一、前言
### 1. 核心作用
在 Struts2 MVC 框架中，`<results>` 标签承担视图角色。Action 执行完业务逻辑后，会返回一个 String 类型值，Struts2 依据该值匹配对应的结果，进而渲染视图。

### 2. `<results>` 标签属性

| **属性** | **说明** |
| --- | --- |
| name | 结果名称，需与 Action 方法返回值匹配，默认值为 `success` |
| type | 响应结果类型，默认值为 `dispatcher` |


### 3. 内置结果类型来源
Struts2 核心包 `struts2-core-x.x.xx.jar` 中的 `struts-default.xml` 定义了默认结果类型，包含 `dispatcher`、`redirect`、`chain` 等十余种，满足不同场景需求。

## 二、常用结果类型详解
### 1. dispatcher（请求转发）
#### （1）概述
+ 最常用的结果类型，也是 Struts2 默认类型。
+ 功能：将请求转发到指定的 JSP 或 HTML 页面，共享同一请求上下文。
+ 核心参数：`location`（视图资源地址，指定目标页面路径）。

#### （2）配置方式
+ 简化配置（默认 type=dispatcher）：

```xml
<action name="login" class="...">
  <result name="success">/success.jsp</result>
</action>
```

+ 显式指定 type：

```xml
<action name="login" class="...">
  <result name="success" type="dispatcher">/success.jsp</result>
</action>
```

+ 通过 param 标签配置 location：

```xml
<action name="login" class="...">
  <result name="success" type="dispatcher">
    <param name="location">/success.jsp</param>
  </result>
</action>
```

#### （3）注意事项
+ 未指定 `name` 属性时，默认值为 `success`。
+ 未指定 `location` 参数时，将 `<result>` 标签中间的字符串作为视图资源路径。
+ 未指定 `type` 属性时，默认使用 `dispatcher`。

### 2. redirect（重定向到页面）
#### （1）概述
+ 功能：将响应重定向到另一个页面（JSP/HTML 或外部 URL），会创建新的 HTTP 请求。
+ 核心参数：
    - `location`：重定向目的地地址。
    - `parse`：是否将 `location` 视为 OGNL 表达式，默认值为 `true`。

#### （2）配置方式
+ 不带参数：

```xml
<action name="login" class="..." method="myExecute">
  <result name="success" type="redirect">/result.jsp</result>
</action>
```

+ 带参数（通过 OGNL 表达式传递 Action 属性）：

```xml
<action name="login" class="...">
  <result name="success" type="redirect">
    <param name="location">/action.jsp</param>
    <param name="username">${username}</param>
    <param name="password">${password}</param>
  </result>
</action>
```

#### （3）注意事项
+ 重定向会生成新请求，原请求的参数、属性、Action 实例及封装的属性会全部丢失。

### 3. redirectAction（重定向到 Action）
#### （1）概述
+ 功能：将响应重定向到另一个 Action，生成新的 HTTP 请求。
+ 核心参数：
    - `actionName`：目标 Action 的名称（默认属性）。
    - `namespace`：目标 Action 的命名空间，未指定时默认使用当前 Action 的命名空间。

#### （2）配置方式
+ 不带参数：

```xml
<result name="success" type="redirectAction">action2</result>
```

+ 带参数：

```xml
<result name="success" type="redirectAction">
  <param name="actionName">action2</param>
  <param name="username">${username}</param>
  <param name="password">${password}</param>
</result>
```

### 4. chain（Action 转发）
#### （1）概述
+ 功能：一个 Action 处理完成后，将请求转发到另一个 Action，共享同一请求上下文。
+ 核心优势：解决重定向导致的 request 作用域数据丢失问题，请求参数、属性可传递。

#### （2）配置方式
+ 不带参数：

```xml
<result name="success" type="chain">action2</result>
```

+ 带参数：

```xml
<result name="success" type="chain">
  <param name="actionName">action2</param>
  <param name="username">${username}</param>
  <param name="password">${password}</param>
</result>
```

## 三、其他内置结果类型（了解）

| **结果类型** | **类路径** | **功能描述** |
| --- | --- | --- |
| freemarker | org.apache.struts2.views.freemarker.FreemarkerResult | 整合 FreeMarker 模板引擎 |
| httpheader | org.apache.struts2.dispatcher.HttpHeaderResult | 控制 HTTP 响应头（如状态码、跳转） |
| stream | org.apache.struts2.dispatcher.StreamResult | 向浏览器返回 InputStream（如下载文件） |
| velocity | org.apache.struts2.dispatcher.VelocityResult | 整合 Velocity 模板引擎 |
| xslt | org.apache.struts2.views.xslt.XSLTResult | 整合 XML/XSLT 转换 |
| plainText | org.apache.struts2.dispatcher.PlainTextResult | 显示页面原始代码（不解析标签） |
| postback | org.apache.struts2.dispatcher.PostbackResult | 以 Form 表单形式提交请求参数到指定地址 |


## 四、实战示例（redirectAction 用法）
### 1. 需求
通过 `redirectAction` 实现 Action 间重定向，并传递 `username` 参数。

### 2. 步骤
#### （1）创建视图文件（index.jsp）
```html
<html>
  <body>
    <h2>Hello World!</h2>
  </body>
</html>
```

#### （2）创建源 Action（ResultTypeAction.java）
```java
package action;

import com.opensymphony.xwork2.ActionContext;
import com.opensymphony.xwork2.ActionSupport;

public class ResultTypeAction extends ActionSupport {
    @Override
    public String execute() throws Exception {
        System.out.println("欢迎访问 ResultTypeAction 中的 execute() 方法!");
        // 向 request 作用域存入参数
        ActionContext.getContext().put("username", "request_username");
        return SUCCESS;
    }
}
```

#### （3）创建目标 Action（HelloWorldAction.java）
```java
package action;

import com.opensymphony.xwork2.ActionSupport;
import org.apache.struts2.ServletActionContext;

public class HelloWorldAction extends ActionSupport {
    @Override
    public String execute() throws Exception {
        System.out.println("欢迎访问 HelloWorldAction 中的 execute() 方法!");
        // 获取重定向传递的参数
        String username = ServletActionContext.getRequest().getParameter("username");
        System.out.println("跨 Action 获取: " + username);
        return SUCCESS;
    }
}
```

#### （4）配置 struts.xml
```xml
<struts>
  <!-- 源 Action 配置 -->
  <package name="resultTypeDemo" namespace="/" extends="struts-default">
    <action name="resultType" class="action.ResultTypeAction">
      <result name="success" type="redirectAction">
        <!-- 重定向到目标 Action，传递 username 参数 -->
        <param name="actionName">helloWorld</param>
        <param name="username">${#request.username}</param>
      </result>
    </action>
  </package>

  <!-- 目标 Action 配置 -->
  <package name="helloWorldDemo" namespace="/" extends="struts-default">
    <action name="helloWorld" class="action.HelloWorldAction">
      <result name="success">/index.jsp</result>
    </action>
  </package>
</struts>
```

#### （5）测试
+ 访问地址：`http://localhost:8080/struts_war/resultType.action`
+ 现象：浏览器 302 重定向到 `helloWorld.action?username=request_username`

## 五、核心总结

| **结果类型** | **核心场景** | **数据传递** | **底层原理** |
| --- | --- | --- | --- |
| dispatcher | 跳转 JSP/HTML | 共享 request 数据 | 请求转发（服务器内部跳转） |
| redirect | 跳转外部页面 / URL | 丢失原请求数据 | 客户端重定向（新 HTTP 请求） |
| redirectAction | 跳转另一个 Action | 需显式传递参数 | 客户端重定向（新 HTTP 请求） |
| chain | Action 间转发 | 共享 request 数据 | 请求转发（服务器内部跳转） |
