## 一、前言
Struts2 验证框架是 Struts 核心组件之一，作用是在执行 Action 方法之前，按照预设规则对数据进行验证，确保输入数据的合法性，减少业务逻辑中的数据校验冗余。

## 二、前置操作（基础页面准备）
需提前创建两个 JSP 页面，分别对应验证成功和失败的跳转目标：

### 1. 验证成功页面（employeeView.jsp）
+ 路径：`./项目/src/main/webapp/employeeView.jsp`
+ 功能：展示验证通过后的用户数据（姓名、年龄）
+ 核心代码：

```html
<%@ page contentType="text/html;charset=UTF-8" language="java" %>
  <%@ taglib prefix="s" uri="/struts-tags" %>
    <html>
      <head>
        <title>employeeView</title>
      </head>
      <body>
        <div>name: <s:property value="name"/></div>
        <div>age: <s:property value="age"/></div>
      </body>
    </html>
```

### 2. 验证失败页面（verifyView.jsp）
+ 路径：`./项目/src/main/webapp/verifyView.jsp`
+ 功能：显示验证失败的错误信息
+ 核心代码：

```html
<%@ page contentType="text/html;charset=UTF-8" language="java" %>
  <%@ taglib prefix="s" uri="/struts-tags" %>
    <html>
      <head>
        <title>verifyView</title>
      </head>
      <body>
        <s:fielderror/> <!-- 输出字段级错误信息 -->
      </body>
    </html>
```

## 三、验证框架通用说明
### 1. 错误信息输出标签

| **标签** | **功能** |
| --- | --- |
| `<s:fielderror/>` | 输出一个或所有字段的错误信息（如姓名为空、年龄超出范围） |
| `<s:actionerror/>` | 输出与 Action 业务相关的全局错误信息 |


### 2. Struts.xml 核心配置
必须在 Struts 配置文件中添加 `input` 结果映射，确保验证失败时跳转到失败页面：

```xml
<result name="input">verifyView.jsp</result>
```

## 四、验证实现方式（两种核心方案）
### 方案一：使用 ActionSupport 实现验证（硬编码方式）
#### 1. 核心原理
通过继承 `ActionSupport` 类，重写 `validate()` 方法，在该方法中编写自定义验证逻辑，通过 `addFieldError()` 或 `addActionError()` 添加错误信息。

#### 2. 实现步骤
##### （1）创建 Action 类（EmployeeAction.java）
+ 路径：`./项目/src/main/java/action/EmployeeAction.java`
+ 核心代码：

```java
package action;
import com.opensymphony.xwork2.ActionSupport;

public class EmployeeAction extends ActionSupport {
    private String name;
    private int age;

    // getter 和 setter 方法
    public String getName() { return name; }
    public void setName(String name) { this.name = name; }
    public int getAge() { return age; }
    public void setAge(int age) { this.age = age; }

    // 业务方法（验证通过后执行）
    @Override
    public String execute() {
        return "employeeView"; // 跳转到成功页面
    }

    // 验证方法（自动触发，在 execute 前执行）
    @Override
    public void validate() {
        // 验证姓名：非空
        if (name == null || "".equals(name.trim())) {
            addFieldError("name", "名字是必须的");
        }
        // 验证年龄：28-65 岁之间
        if (age < 28 || age > 65) {
            addFieldError("age", "年龄必须在28到65岁之间");
        }
    }
}
```

##### （2）配置 Struts.xml
+ 路径：`./项目/src/main/resources/struts.xml`
+ 核心配置：

```xml
<struts>
  <package name="employeeDemo" namespace="/" extends="struts-default">
    <action name="employee" class="action.EmployeeAction">
      <result name="input">/verifyView.jsp</result> <!-- 验证失败跳转 -->
      <result name="employeeView">/employeeView.jsp</result> <!-- 验证成功跳转 -->
    </action>
  </package>
</struts>
```

##### （3）测试用例

| **访问地址** | **参数** | **验证结果** | **页面输出** |
| --- | --- | --- | --- |
| `http://127.0.0.1:8081/Struts2Test_war/employee.action` | 无 | 失败 | 名字是必须的年龄必须在 28 到 65 岁之间 |
| `http://127.0.0.1:8081/Struts2Test_war/employee.action?name=二狗` | name = 二狗 | 失败 | 年龄必须在 28 到 65 岁之间 |
| `http://127.0.0.1:8081/Struts2Test_war/employee.action?age=45` | age=45 | 失败 | 名字是必须的 |
| `http://127.0.0.1:8081/Struts2Test_war/employee.action?name=二狗&age=45` | name = 二狗 & age=45 | 成功 | name: 二狗age: 45 |


### 方案二：XML 实现验证（配置文件方式）
#### 1. 核心原理
无需在 Action 中编写硬编码验证逻辑，而是通过独立的 XML 配置文件定义验证规则，支持更多内置验证类型（如必填、邮箱、数值范围等），解耦验证逻辑与业务逻辑。

#### 2. 支持的内置验证类型
+ required：字段非空
+ requiredstring：字符串非空且去除空格后不为空
+ int：整数范围验证
+ stringlength：字符串长度验证
+ email：邮箱格式验证
+ regex：正则表达式验证
+ expression：OGNL 表达式验证

#### 3. 实现步骤
##### （1）创建 Action 类（Employee2Action.java）
+ 路径：`./项目/src/main/java/action/Employee2Action.java`
+ 核心代码（仅保留业务逻辑和属性封装）：

```java
package action;
import com.opensymphony.xwork2.ActionSupport;

public class Employee2Action extends ActionSupport {
    private String name;
    private int age;

    // getter 和 setter 方法
    public String getName() { return name; }
    public void setName(String name) { this.name = name; }
    public int getAge() { return age; }
    public void setAge(int age) { this.age = age; }

    // 业务方法
    @Override
    public String execute() {
        return "employeeView";
    }
}
```

##### （2）创建 XML 验证文件（Employee2Action-validation.xml）
+ 命名规则：`[Action类名]-validation.xml`（必须与 Action 类同名且在同一包下）
+ 路径：`./项目/src/main/java/action/Employee2Action-validation.xml`
+ 核心配置（验证姓名必填、年龄 29-64 岁）：

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE validators PUBLIC 
"-//Apache Struts//XWork Validator 1.0.3//EN" 
"http://struts.apache.org/dtds/xwork-validator-1.0.3.dtd">

<validators>
  <!-- 验证姓名：必填 -->
  <field name="name">
    <field-validator type="requiredstring">
      <message>名字是必须的</message>
    </field-validator>
  </field>

  <!-- 验证年龄：整数范围 29-64 -->
  <field name="age">
    <field-validator type="int">
      <param name="min">29</param>
      <param name="max">64</param>
      <message>年龄必须在${min}到${max}岁之间</message>
    </field-validator>
  </field>
</validators>
```

##### （3）配置 Struts.xml
+ 路径：`./项目/src/main/resources/struts.xml`
+ 核心配置：

```xml
<struts>
  <package name="employeeDemo2" namespace="/" extends="struts-default">
    <action name="employee2" class="action.Employee2Action">
      <result name="input">/verifyView.jsp</result> <!-- 验证失败跳转 -->
      <result name="employeeView">/employeeView.jsp</result> <!-- 验证成功跳转 -->
    </action>
  </package>
</struts>
```

## 五、两种实现方式对比
| **特性** | **ActionSupport 硬编码方式** | **XML 配置文件方式** |
| --- | --- | --- |
| 灵活性 | 高，支持复杂自定义逻辑 | 中等，依赖内置验证类型 + 正则 |
| 耦合度 | 与 Action 业务逻辑耦合 | 解耦，验证规则独立维护 |
| 维护性 | 需修改 Java 代码，重启应用 | 仅修改 XML，无需重启（视容器而定） |
| 适用场景 | 复杂业务验证（如多字段关联验证） | 简单常规验证（必填、范围、格式等） |
