## 一、核心配置文件概述
Struts2 应用程序的核心配置文件主要有 2 个，分别是 `web.xml` 和 `struts.xml`，二者各司其职，共同保障 Struts2 框架正常运作。

## 二、web.xml 文件
### （一）文件定位与作用
1. 本质：J2EE 标准配置文件，并非 Struts2 专属，但 Struts2 运行必需。
2. 核心作用：为 Web 应用提供接入点，配置 Struts2 过滤器，指定所有 URL 都由该过滤器解析。
3. 目录路径：`./项目/src/main/webapp/WEB-INF/web.xml`

### （二）关键配置代码
```xml
<!DOCTYPE web-app PUBLIC "-//Sun Microsystems, Inc.//DTD Web Application 2.3//EN"
"http://java.sun.com/dtd/web-app_2_3.dtd">
<web-app xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
  xmlns="http://java.sun.com/xml/ns/javaee"
  xmlns:web="http://java.sun.com/xml/ns/javaee/web-app_2_5.xsd"
  xsi:schemaLocation="http://java.sun.com/xml/ns/javaee http://java.sun.com/xml/ns/javaee/web-app_3_0.xsd"
  id="WebApp_ID" version="3.0">
  <display-name>Struts2Test</display-name>
  <!-- Struts2 过滤器配置 -->
  <filter>
    <filter-name>struts2</filter-name>
    <filter-class>org.apache.struts2.dispatcher.filter.StrutsPrepareAndExecuteFilter</filter-class>
  </filter>
  <!-- 过滤器映射：所有 URL 都被 Struts2 过滤器解析 -->
  <filter-mapping>
    <filter-name>struts2</filter-name>
    <url-pattern>/*</url-pattern>
  </filter-mapping>
</web-app>
```

+ 注意：过滤器映射路径为 `/*`，而非 `/*.action`，意味着所有请求都会经过 Struts2 过滤器。

## 三、struts.xml 文件
### （一）文件定位与作用
1. 核心作用：存储 Struts2 核心配置，可覆盖框架默认设置（如 `struts.devMode=false`），定义 Action、结果页面等关键信息。
2. 目录路径：`./项目/src/main/webapp/WEB-INF/classes/struts.xml`（文件名固定）  

### （二）基础配置示例
```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE struts PUBLIC "-//Apache Software Foundation//DTD Struts Configuration 2.3//EN" 
"http://struts.apache.org/dtds/struts-2.3.dtd">
<struts>
  <!-- 包配置 -->
  <package name="helloDemo" namespace="/" extends="struts-default">
    <!-- Action 配置 -->
    <action name="hello" class="action.HelloAction" method="execute">
      <!-- 结果页面配置 -->
      <result name="test">/testView.jsp</result>
    </action>
  </package>
</struts>
```

### （三）核心标签说明
#### 1. DOCTYPE（文档类型）
+ 所有 Struts2 配置文件必须包含正确的 DOCTYPE 声明，用于指定配置文件的语法规则和约束。
+ 示例中使用 Struts 2.3 版本的 DTD 约束，需与框架版本对应。

#### 2. \<struts> 标签
+ 根标记元素，所有配置内容都需包裹在该标签内。

#### 3. \<package> 标签
+ 作用：实现配置的分离与模块化，可声明不同功能模块的配置。
+ 核心属性：
    - `name`（必须）：包的唯一标识，不可重复。
    - `extends`（常用）：指定继承的父包，通常继承 `struts-default`（Struts2 内置默认包，提供基础功能）。
    - `abstract`（可选）：定义包为抽象包，值为 `true` 时，该包不能直接被用户使用（仅用于被其他包继承）。
    - `namespace`（可选）：Action 的唯一命名空间，**与 Action 名称共同构成访问 URL**。

#### 4. \<action> 标签
+ 作用：映射 URL 请求与对应的 Java 类（Action 类），定义请求的处理逻辑。
+ 核心属性：
    - `name`（必须）：Action 的唯一标识，与命名空间组合形成访问路径（如 `namespace="/"` + `name="hello"`，访问 URL 为 `http://xxx/hello`）。
    - `class`（必须）：Action 类的完整路径（如 `action.HelloAction`）。
    - `method`（可选）：指定 Action 中要执行的方法，默认值为 `execute()`，可自定义（如 `executeTest11()`）。

#### 5. \<result> 标签
+ 作用：定义 Action 方法执行后的跳转目的地（结果页面）。
+ 核心属性：
    - `name`（可选）：结果名称，需与 Action 方法返回的字符串一致，默认值为 `success`。
    - `type`（可选）：跳转类型，默认值为 `dispatcher`（请求转发到 JSP 页面），支持 `redirect`（重定向）等其他类型。

### （四）模块化配置方法
当 `struts.xml` 内容过于庞大时，可拆分为多个独立的 XML 文件，通过 `<include>` 标签导入主配置文件，实现模块化管理。

#### 示例：主 struts.xml 导入子配置文件
```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE struts PUBLIC "-//Apache Software Foundation//DTD Struts Configuration 2.0//EN" 
"http://struts.apache.org/dtds/struts-2.0.dtd">
<struts>
  <!-- 导入子配置文件 -->
  <include file="my-struts1.xml"/>
  <include file="my-struts2.xml"/>
</struts>
```

+ 子配置文件（如 `my-struts1.xml`）需遵循 Struts2 配置语法，可包含独立的 `<package>`、`<action>` 等标签。
