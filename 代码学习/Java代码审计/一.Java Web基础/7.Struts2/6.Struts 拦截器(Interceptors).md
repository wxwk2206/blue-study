## 一、前言
### 1. 核心定义
拦截器（Interceptor）是 Struts2 核心特性之一，是在 Action 执行之前、Result 执行之后进行功能处理的机制。

### 2. 核心作用
+ 预处理：Action 调用前执行逻辑（如参数处理、权限校验）
+ 后处理：Action 调用后执行逻辑（如日志记录、资源释放）
+ 异常处理：捕获异常并执行备用处理流程

### 3. 框架应用
Struts2 诸多核心功能基于拦截器实现，包括异常处理、文件上传、生命周期回调、数据验证等。

## 二、预定义拦截器
Struts2 内置多个常用拦截器，无需自定义即可直接使用，核心功能如下：

| **拦截器名称** | **核心功能** | **补充说明** |
| --- | --- | --- |
| params | 将请求参数设置到 Action 属性，并自动完成类型转换 | 数据绑定核心拦截器 |
| timer | 记录 ActionInvocation 后续执行时间（含 Action 运行时间） | 用于性能瓶颈分析，日志输出执行耗时 |
| logger | 日志输出待执行的 Action 信息 | 调试时快速定位目标 Action |
| exception（异常处理） | 映射 Action 抛出的异常，支持通过重定向自动处理 | 简化异常统一管理 |
| aliasInterceptor（别名） | 为参数设置别名，实现不同 Action 间同名参数映射 | 类似 “胶水”，在 Action 链中作用显著 |
| servletConfig（配置） | 为实现特定接口的 Action 设置属性 | 支持获取 servlet 相关参数（如 servletContext、session），需 Action 实现 ParameterAware 等接口 |
| i18n（国际化） | 在用户会话期间跟踪选定的区域 | 支持多语言切换功能 |
| prepare（预处理） | Action 方法执行前调用 prepare 方法 | 仅对实现 Prepareable 接口的 Action 有效 |
| chain（Action 链） | 将所有实例属性拷贝到当前对象的 valueStack 中 | 需与 Action 返回类型 “chain” 配合使用；实现 Unchainable 接口的对象除外 |
| debugging（调试） | 为开发人员提供多种调试屏幕 | 仅用于开发环境，方便问题排查 |
| scopedModelDriven | 为 Action 提供模型驱动支持 | 仅对实现 ScopedModelDriven 接口的 Action 生效 |
| modelDriven | 为 Action 提供模型驱动支持 | 仅对实现 ModelDriven 接口的 Action 生效 |
| fileUpload（文件上传） | 简化文件上传功能实现 | 无需手动处理上传流，框架自动封装 |
| checkbox（复选框处理） | 专门处理 checkbox 表单控件 | 解决未选中时无默认值的问题 |
| multiselect（多选） | 处理多选表单控件 | 与 checkbox 逻辑类似，未选中时设置默认值 |
| staticParams（静态参数） | 将 Action 配置的静态参数添加到 Action 中 | 若 Action 实现 Parameterizable 接口，静态参数会以 Map 形式设置 |


## 三、自定义拦截器
### 1. 实现方式
自定义拦截器需实现 `com.opensymphony.xwork2.interceptor.Interceptor` 接口（或继承 AbstractInterceptor 抽象类，简化实现）。

### 2. 核心接口方法

| **方法名** | **作用** | **类比** |
| --- | --- | --- |
| init() | 初始化相关资源 | 类似构造方法，拦截器创建时执行 |
| destroy() | 释放资源 | 类似析构方法，拦截器销毁时执行 |
| intercept(ActionInvocation invocation) | 核心处理逻辑 | 自定义功能的主要实现位置，返回 Result 字符串 |


### 3. intercept 方法关键说明
+ `invocation.invoke()`：继续执行后续拦截器→Action→Result，若不调用则终止流程（后续拦截器、Action 不再执行）
+ 方法执行顺序：
    1. `invocation.invoke()` 之前：Action 执行前的预处理逻辑
    2. `invocation.invoke()` 之后：Result 执行后的后处理逻辑
+ 返回值：若未执行 `invocation.invoke()`，返回值直接作为 Result 标识，触发对应视图处理

### 4. 自定义示例（完整流程）


#### （2）核心代码实现
1. 自定义拦截器类（MyInterceptor.java）
```java
package interceptor;
import com.opensymphony.xwork2.interceptor.AbstractInterceptor;
import com.opensymphony.xwork2.ActionInvocation;

public class MyInterceptor extends AbstractInterceptor {
    @Override
    public void init() {
        System.out.println("MyInterceptor初始化"); // 资源初始化
    }

    @Override
    public String intercept(ActionInvocation invocation) throws Exception {
        System.out.println("在Action执行之前"); // 预处理逻辑
        String result = invocation.invoke(); // 继续执行后续流程
        System.out.println("在Result运行之后"); // 后处理逻辑
        System.out.println("result值:" + result);
        return result;
    }

    @Override
    public void destroy() {
        System.out.println("MyInterceptor销毁"); // 资源释放
    }
}
```

2. 测试 Action 类（Test5Action.java）
```JAVA
package action;
import com.opensymphony.xwork2.ActionSupport;

public class Test5Action extends ActionSupport {
    @Override
    public String execute() throws Exception {
        System.out.println("Test5Action运行了");
        return "test5View"; // 返回 Result 标识
    }
}
```

3. 测试视图（test5View.jsp）

4. Struts2 配置（struts.xml）

```XML
<package name="test5Demo" namespace="/" extends="struts-default">
    <!-- 注册自定义拦截器 -->
    <interceptors>
        <interceptor name="myInterceptor" class="interceptor.MyInterceptor"/>
    </interceptors>

    <!-- 配置 Action 并引用拦截器 -->
    <action name="test5" class="action.Test5Action">
        <interceptor-ref name="params"/> <!-- 引用内置参数拦截器 -->
        <interceptor-ref name="myInterceptor"/> <!-- 引用自定义拦截器 -->
        <result name="test5View">/test5View.jsp</result> <!-- 绑定视图 -->
    </action>
</package>
```
