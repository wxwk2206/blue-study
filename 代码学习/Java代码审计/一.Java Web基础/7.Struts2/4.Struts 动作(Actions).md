## 一、前言
### 1. 核心地位
Actions 是 Struts2 框架的核心组件，适配任何 MVC（Model View Controller）框架，每个 URL 都会映射到特定的 Action，Action 提供处理用户请求的核心逻辑。

### 2. 核心功能
+ 数据传递：负责将请求中的数据传递到视图（如 JSP 等结果页面）。
+ 结果跳转：协助框架确定响应请求时应呈现的视图结果。

## 二、创建 Action
### 1. 核心要求
+ 必须包含无参数方法，返回值为 String 类型或 Result 对象。
+ 必须是 POJO（简单 Java 对象）。
+ 若未指定无参数方法，默认使用 `execute()` 方法。

### 2. 实现方式
#### （1）实现 Action 接口
+ 接口全路径：`com.opensymphony.xwork2.Action`
+ 接口内置常量（对应视图跳转标识）：

| **常量名** | **含义** |
| --- | --- |
| SUCCESS | 数据处理成功（对应成功页面） |
| NONE | 页面不跳转（返回 null） |
| ERROR | 数据处理错误（对应错误页面） |
| INPUT | 用户输入数据有误（用于表单校验，对应输入页面） |
| LOGIN | 权限认证（对应登录页面） |


+ 默认的Action方法：`String execute() throws Exception;`

#### （2）扩展 ActionSupport 类（推荐）
+ ActionSupport 类实现了包括 Action 接口在内的六个接口，提供更多实用方法。
+ 无需手动实现 `execute()` 方法（可重写），简化开发。

#### （3）自定义无继承类
+ 无需继承 ActionSupport 或实现 Action 接口。
+ 只需满足：包含 `public String execute() throws Exception {}` 方法即可。
+ 示例：

```java
public class Test1Action {
    public String execute() throws Exception {
        return "success";
    }
}
```

## 三、实操案例
### 案例目标
通过浏览器传入不同参数，控制 Action 返回不同视图（成功页面 / 错误页面）。

### 步骤拆解
#### 1. 创建成功视图（success1View.jsp）
+ 路径：`./项目/src/main/webapp/success1View.jsp`
+ 代码：

```plain
<%@ page contentType="text/html;charset=UTF-8" language="java" %>
<%@ taglib prefix="s" uri="/struts-tags" %> <!-- 引入 Struts2 标签库 -->
<html>
<head>
    <title>successTest1View</title>
</head>
<body>
    successTest1View,<s:property value="name"/> <!-- 显示传入的 name 参数 -->
</body>
</html>
```

#### 2. 创建错误视图（error1View.jsp）
+ 路径：`./项目/src/main/webapp/error1View.jsp`
+ 代码：

```plain
<%@ page contentType="text/html;charset=UTF-8" language="java" %>
<%@ taglib prefix="s" uri="/struts-tags" %>
<html>
<head>
    <title>error1View</title>
</head>
<body>
    error1View,<s:property value="name"/>
</body>
</html>
```

#### 3. 创建 Action 类（Test1Action.java）
+ 路径：`./项目/src/main/webapp/action/Test1Action.java`
+ 代码：

```java
package action;
import com.opensymphony.xwork2.ActionSupport;

public class Test1Action extends ActionSupport {
    private String name; // 接收请求参数 name
    // 重写 SUCCESS 常量，自定义成功标识
    public String SUCCESS = "successTest1";

    // 重写 execute 方法，实现业务逻辑
    @Override
    public String execute() throws Exception {
        if ("successTest1Value".equals(name)) {
            return SUCCESS; // 匹配成功，返回自定义成功标识
        } else {
            return ERROR; // 匹配失败，返回内置错误标识
        }
    }

    // getter/setter 方法（必须，用于接收请求参数）
    public void setName(String name) {
        this.name = name;
    }
    public String getName() {
        return this.name;
    }
}
```

#### 4. 配置 Struts2 核心配置文件（struts.xml）
+ 路径：`./项目/src/main/webapp/WEB-INF/classes/struts.xml`
+ 代码（在 `<struts></struts>` 标签内添加）：

```xml
<package name="test1Demo" namespace="/" extends="struts-default">
  <!-- 配置 Action：name 为访问标识，class 为 Action 全路径 -->
  <action name="test1" class="action.Test1Action">
    <!-- 配置结果映射：name 对应 Action 返回值，指向视图路径 -->
    <result name="successTest1">/success1View.jsp</result>
    <result name="error">/error1View.jsp</result>
  </action>
</package>
```

## 四、关键注意点
1. Action 类的属性必须提供 getter/setter 方法，否则无法接收请求参数。
2. 配置文件中 `<result>` 标签的 `name` 属性必须与 Action 中返回的字符串完全一致。
3. 引入 Struts2 标签库时，`uri="/struts-tags"` 必须正确，否则无法使用 `<s:property>` 等标签。
4. 若自定义 Action 未继承 ActionSupport，需手动实现 `execute()` 方法，否则会报错。
