## 一、核心定位：替代原生 JDBC 的 DriverManager
+ 实际项目中**不推荐使用原生 JDBC 的 DriverManager** 直接连接数据库，而是采用 `javax.sql.DataSource`（数据源）作为替代方案。
+ 核心优势：Web 服务启动时会**预先初始化数据源**，后续开发无需重复编写数据库连接相关代码，直接引用 DataSource 对象即可获取连接，简化开发流程。

## 二、DataSource 的实现与常见连接池
### 1. 核心基础
所有常用数据源 / 连接池均**基于 **`**javax.sql.DataSource**`** 接口实现**，保证了使用规范的统一性。

### 2. 常见实现（数据源 / 连接池）
+ 主流基础实现：C3P0、DBCP、Druid
+ 其他可选连接池（可自行拓展学习）：Proxool、BoneCP、HikariCP 等
