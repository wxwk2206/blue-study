## 一、核心概念与优势
动态加载和静态加载的区别为是否需要重新编译

### 1. 定义
Java 动态加载类是指在程序运行时，根据需求动态加载指定类并创建实例，无需在编译阶段就将所有类固定引入的机制。

### 2. 核心优势（对比静态加载）

| **特性** | **静态加载** | **动态加载** |
| --- | --- | --- |
| 代码耦合度 | 高，需硬编码引入类 | 低，通过配置或参数灵活加载 |
| 功能更新 | 需修改主程序代码 + 重新编译 | 仅需添加类文件 + 修改配置 / 参数 |
| 功能删除 | 需修改主程序代码 + 重新编译 | 仅需删除类文件 + 修改配置 / 参数 |
| 资源占用 | 加载所有类，资源消耗大 | 按需加载，节省资源 |


### 3. 典型应用场景
+ 功能模块分步开发（如 Office 组件先上线 Excel、Word，后续补充 Ppt）
+ 插件化架构（支持第三方插件动态接入）
+ 配置化功能切换（通过配置文件指定需加载的实现类）

## 二、实验设计与实现
### 1. 实验目标
创建 Office 组件，支持动态加载 Excel、Word、Ppt 功能类并执行对应功能。

### 2. 目录结构
```plain
反射/
└── java动态加载类1/
    ├── OfficeAble.java （接口类）
    ├── OfficeBetter.java （组件主类）
    ├── Excel.java （Excel功能类）
    ├── Word.java （Word功能类）
    └── Ppt.java （Ppt功能类）
```

### 3. 核心代码实现
#### （1）接口类：OfficeAble（功能类父接口）
定义所有功能类的统一方法，保证程序扩展性，所有功能类需实现该接口。

```plain
package 反射.java动态加载类1;

public interface OfficeAble {
    public void start(); // 统一功能方法
}
```

#### （2）组件主类：OfficeBetter（动态加载核心）
通过 `Class.forName()` 动态加载类，利用接口多态性调用功能方法。

```java
package 反射.java动态加载类1;

public class OfficeBetter {
    public static void main(String[] args) {
        try {
            // 1. 动态加载类（运行时通过参数指定类名）
            Class c = Class.forName("反射.java动态加载类1." + args[0]);
            // 2. 创建类实例（需强转为接口类型）
            OfficeAble oa = (OfficeAble) c.newInstance();
            // 3. 调用统一功能方法
            oa.start();
        } catch (Exception e) {
            e.printStackTrace(); // 捕获类未找到、实例化失败等异常
        }
    }
}
```

#### （3）功能类：实现接口
+ Excel 功能类

```java
package 反射.java动态加载类1;

public class Excel implements OfficeAble {
    public void start() {
        System.out.println("Excel ... start");
    }
}
```

+ Word 功能类

```java
package 反射.java动态加载类1;

public class Word implements OfficeAble {
    public void start() {
        System.out.println("Word ... start");
    }
}
```

+ Ppt 功能类

```java

    public void start() {
        System.out.println("Ppt ... start");
    }

```

## 四、关键技术点总结
1. **接口作用**：定义统一规范，使动态加载的类具备一致性，降低耦合。
2. **动态加载核心**：`Class.forName(String className)` 方法，通过全类名（包名 + 类名）在运行时加载类。
3. **实例化方式**：`Class.newInstance()` 方法创建类实例，需强转为接口类型以调用统一方法。
4. **异常处理**：需捕获 `ClassNotFoundException`（类未找到）、`InstantiationException`（实例化失败）等异常。

## 五、扩展说明
1. 新增功能类时，只需：
    - 实现 `OfficeAble` 接口
    - 编写 `start()` 方法
    - 将编译后的 `.class` 文件放入对应目录
    - 运行时指定类名参数（无需修改 `OfficeBetter` 代码）
2. 删除功能类时，只需：
    - 删除对应 `.class` 文件
    - 不再通过参数指定该类（无需修改主程序）
