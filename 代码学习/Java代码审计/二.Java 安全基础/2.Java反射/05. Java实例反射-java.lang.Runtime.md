# 前言
`java.lang.Runtime` 因为有一个 `exec`方法可以执行本地命令，所以该类常常用来执行本地系统命令。

通过前面我们已经学习了通过反射的各种方法，本章节以该类作为一个例子，实现反射调用`exec`方法。

# 不使用反射执行本地命令的代码
```java
package 反射;

import org.apache.commons.io.IOUtils;

public class ExecCmdTest {
    public static void main(String[] args) {
        // 输出命令执行结果
        try {
            System.out.println(
                IOUtils.toString(
                	Runtime.getRuntime().exec("whoami").getInputStream(),"UTF-8")
            );
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}
```

# 通过反射调用Runtime执行exec
```java
package 反射;

import org.apache.commons.io.IOUtils;

import java.io.InputStream;
import java.lang.reflect.Constructor;
import java.lang.reflect.Method;

public class ExecCmdTest {
    public static void main(String[] args) {
        try {
            // 获取Runtime类对象
            Class runtimeClass = Class.forName("java.lang.Runtime");

            // 获取构造方法
            Constructor runtimeConstructor = runtimeClass.getDeclaredConstructor();
            runtimeConstructor.setAccessible(true);

            // 创建Runtime类实例 相当于 Runtime r = new Runtime();
            Object runtimeInstance = runtimeConstructor.newInstance();

            // 获取Runtime的exec(String cmd)方法
            Method runtimeMethod = runtimeClass.getMethod("exec", String.class);

            // 调用exec方法 等于 r.exec(cmd); cmd参数输入要执行的命令
            Process p = (Process)runtimeMethod.invoke(runtimeInstance, "whoami");

            // 获取命令执行结果
            InputStream results = p.getInputStream();

            // 输出命令执行结果
            System.out.println(IOUtils.toString(results, "UTF-8"));
        } catch (Exception e) {
            e.printStackTrace();
        } 
    }
}
```

#  利用 Java 反射破坏单例模式  
```java
import java.lang.reflect.Constructor;
import java.lang.reflect.InvocationTargetException;

/**
 * 反射破坏Runtime单例模式的完整示例
 * 注意：
 * 1. Runtime类本身是单例设计（私有构造器+静态方法获取实例）
 * 2. 反射可以绕过私有构造器限制，创建多个实例
 * 3. 不同系统的exec命令需适配：Mac用"open ."，Windows用"cmd /c start ."，Linux用"xdg-open ."
 */
public class ReflectRuntimeExample {
    public static void main(String[] args) {
        try {
            // 1. 获取Runtime类的Class对象（反射入口）
            Class<Runtime> runtimeClass = Runtime.class;

            // 2. 获取私有的无参构造方法（getDeclaredConstructor获取所有构造器，包括私有）
            Constructor<Runtime> declaredConstructor = runtimeClass.getDeclaredConstructor();

            // 3. 取消访问检查，允许调用私有构造器（突破封装性的关键）
            declaredConstructor.setAccessible(true);

            // 4. 强行创建新的Runtime实例（正常情况下只能通过Runtime.getRuntime()获取）
            Runtime runtime1 = declaredConstructor.newInstance();
            System.out.println("第一个Runtime实例：" + runtime1.hashCode());

            // 5. 执行系统命令（适配不同操作系统）
            // Mac系统：open .  打开当前目录
            // Windows系统：cmd /c start .  打开当前目录
            // Linux系统：xdg-open .  打开当前目录
            runtime1.exec("cmd /c start ."); 

            // 6. 重复创建第二个Runtime实例，验证破坏单例
            Runtime runtime2 = declaredConstructor.newInstance();
            System.out.println("第二个Runtime实例：" + runtime2.hashCode());

            // 对比：正常单例获取的实例（和反射创建的实例不同）
            Runtime normalRuntime = Runtime.getRuntime();
            System.out.println("正常单例的Runtime实例：" + normalRuntime.hashCode());

        } catch (NoSuchMethodException e) {
            // 找不到指定构造器时抛出（比如传错参数类型）
            e.printStackTrace();
        } catch (InstantiationException e) {
            // 类是抽象类/接口，或无构造器时抛出
            e.printStackTrace();
        } catch (IllegalAccessException e) {
            // 访问权限不足时抛出（比如没调用setAccessible(true)）
            e.printStackTrace();
        } catch (InvocationTargetException e) {
            // 构造器执行时抛出异常（比如构造器内部有逻辑报错）
            e.printStackTrace();
        } catch (Exception e) {
            // 捕获exec命令执行的异常（比如命令不存在）
            e.printStackTrace();
        }
    }
}
```
