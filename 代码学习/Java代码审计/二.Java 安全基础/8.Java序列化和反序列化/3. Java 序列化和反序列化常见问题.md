# 0x01 serialVersionUID
## 0x01.1 serialVersionUID 有什么用
```plain
serialVersionUID的作用是验证版本一致性
如果serialVersionUID一致,说明它们的版本是一样的
如果serialVersionUID不一致,就说明版本不同,就无法运行或使用相关功能
```

## 0x01.2 serialVersionUID 不一致解决方法
```plain
在进行反序列化漏洞攻击的时候可能经常会遇到serialVersionUID不一致的情况
针对这种情况,解决方案也比较简单
例如:
cc1攻击过去现在因为serialVersionUID导致反序列化失败
解决方案: 下载cc1的jar包,指定jar包的版本,重新打包,然后在进行反序列化攻击
```

# 0x02 不参与反序列化的属性
## 0x02.1 transient 修饰符
```plain
在序列化时,如果我们不想序列化特定变量以满足安全约束
那么我们应该将该变量声明为transient,执行序列化时
JVM会忽略transient变量的原始值并将默认值保存到文件中
因此transient意味着不要序列化的意思
```

```plain
// 小结
1. transient修饰符只能作用于继承并实现了Serializable接口的类
2. transient修饰符仅适用于变量,不适用于方法和类
3. 被transient修饰的变量不能被序列化
4. 不管有没有transient修饰,static静态变量都不能被序列化
```

## 0x02.2 static 修饰符
```plain
static静态变量不是对象状态的一部分,因此它不参与序列化,所以不需要将静态变量声明为transient
注意: 不参与反序列化,但是值可以被修改,因此加不加transient,static静态变量的值都是可修改的
```

## 0x02.3 final 修饰符
```plain
final修饰符将直接通过值参与序列化,所以将final变量声明为transient变量不会产生任何影响
```

```plain
final故名思意即最终、不可变
final作为Java中的⼀个关键字,可以⽤来修饰类、类的⽅法、类的变量(局部变量和成员变量)
```

### 0x02.3.1 final修饰类
```plain
final修饰类: 
final用来修饰一个类,表示此类不能被其它类所继承
例如: Java中的String类、System类、StringBuffer类及Integer等包装类
注意: 被final修饰的类所有的方法都会被隐式的定义为final类型
```

```java
// 测试例子
final class A {
}

class B extends A {
}

运行结果: 会提示错误,不能被继承
```

### 0x02.3.2 final修饰类的⽅法
```plain
final修饰类的⽅法:
表明此方法不能被子类重写
比如: Object类中getClass()
```

```java
// 测试例子
class A {
    public final void print() {
        System.out.println("A");
    }
}

class B extends A {
    public void print() {
    }
}

运行结果: 错误,不能被重写
```

### 0x02.3.3 final修饰类的变量
```java
// final修饰的变量可以分为基本数据类型和引⽤数据类型类型两种

// 修饰基本数据类型变量的话,变量值在被初始化之后不能再被更改
public class Test1 {
    public static void main(String[] args) {
        final int a = 10;
        a = 11; // 更改final修饰的变量,会报错
    }
}





// 修饰形参,形参也不能被更改
public class Test1 {
    public static void main(final int a) {
        a = 10; // 更改final修饰的形参,会报错
    }
}





// 修饰引⽤数据类型变量的话,引⽤不能再指向其他对象,不过对象的内容是可以更改的
public class Test1 {
    int a = 10;

    public static void main(String[] args) {
        final Test1 t = new Test1();
        Test1 t2 = new Test1();
        t = t2; // 更改final修饰的对象,会报错
        t.a = 20; // 可以修改对象的内容
    }
}
```

```java
// final修饰局部变量和成员变量时⼜有不同

// 修饰局部变量时必须在使⽤前就对这个变量进⾏⼀次赋值,可以直接在定义时赋值,也可以在后续代码中赋值
public class Test1 {
    void methond() {
        // 定义的同时赋值
        final int a = 10;

        // 没有在定义的同时赋值
        final int b;
        b = 20;
    }
}





// 修饰成员变量在定义的时候没有赋值的话这个变量被称为“空⽩final变量”
// 空⽩final变量必须在构造⽅法或者静态代码块中赋值
public class Test1 {
    // 定义时初始化
    final int a = 10;

    final int b;
    final static int c;

    // 构造方法中赋值
    public Test1() {
        this.b = 20;
    }

    // 静态代码块中赋值
    static {
        c = 30;
    }
}
```

# 0x03 小例子
```java
// 创建个用于测试的类
package Test3;

import java.io.IOException;
import java.io.ObjectInputStream;
import java.io.Serializable;

public class User implements Serializable {
    private String name;
    private final String name2 = "小名-二狗";
    private static String name3 = "小名-狗蛋";
    private transient String name4 = "小名-村霸";
    private transient static String name5 = "小名-历狗";
    private transient int age;
    private String gender;
    private static final long serialVersionUID = 1L;

    public void setName(String name) {
        this.name = name;
    }

    public String getName() {
        return name;
    }

    public void setAge(int age) {
        this.age = age;
    }

    public int getAge() {
        return this.age;
    }

    public void setGender(String gender) {
        this.gender = gender;
    }

    public String getGender() {
        return this.gender;
    }

    @Override
    public String toString() {
        return "User{" +
                "name='" + this.name + "'" +
                ", name2='" + this.name2 + "'" +
                ", name3='" + name3 + "'" +
                ", name4='" + this.name4 + "'" +
                ", name5='" + name5 + "'" +
                ", age=" + this.age +
                ", gender='" + this.gender + "'" +
                "}";
    }

    // 反序列化类对象时,对age进行额外处理
    private void readObject(ObjectInputStream ois) throws IOException, ClassNotFoundException {
        age = 25;
        ois.defaultReadObject();
    }
}
```

```java
package Test3;

import java.io.*;
import java.lang.reflect.Field;
import java.lang.reflect.InvocationTargetException;

public class Demo2 {
    public static void main(String[] args) throws IOException, ClassNotFoundException, IllegalAccessException, InstantiationException, NoSuchFieldException, NoSuchMethodException, InvocationTargetException {
        Class aClass = Class.forName("Test3.User");
        User user = (User) aClass.newInstance();

        user.setName("大名-陈一发");
        user.setGender("男");

        // 使用了final修饰符,不参与反序列化,同时也无法修改值
        Field name2 = aClass.getDeclaredField("name2");
        name2.setAccessible(true);
        name2.set(user, "小名-二狗改");

        // 使用了static静态变量不是对象状态的一部分,因此它不参与序列化,但是反序列化以后是可以看到修改的值
        Field name3 = aClass.getDeclaredField("name3");
        name3.setAccessible(true);
        name3.set(user, "小名-狗蛋改");

        // 使用了transient修饰符,不参与反序列化
        // 因此反序列化时,不获取name4导致反序列化值为空
        Field name4 = aClass.getDeclaredField("name4");
        name4.setAccessible(true);
        name4.set(user, "小名-村霸改");

        // 使用了static修饰符,还使用transient修饰符
        // 但因为使用了static修饰符,static静态变量不是对象状态的一部分,因此它不参与序列化
        // 因此transient修饰符在这里是无效的,但是反序列化以后是可以看到修改的值
        Field name5 = aClass.getDeclaredField("name5");
        name5.setAccessible(true);
        name5.set(user, "小名-历狗改");

        // 使用了transient修饰符,不参与反序列化,赋值对于反序列化无效
        Field age = aClass.getDeclaredField("age");
        age.setAccessible(true);
        age.set(user, 22);

        System.out.println("当前User类: " + user.toString());

        // 序列化
        ObjectOutputStream obj = new ObjectOutputStream(new FileOutputStream("tempFile2"));
        obj.writeObject(user);

        // 反序列化并输出
        ObjectInputStream objectInputStream = new ObjectInputStream(new FileInputStream("tempFile2"));
        User u = (User) objectInputStream.readObject();
        System.out.println("反序列化后的User类: " + u);
    }
}




// 运行结果
// 注意: age的值,被修改是因为User类的readObject()方法进行了额外处理
当前User类: User{name='大名-陈一发', name2='小名-二狗', name3='小名-狗蛋改', name4='小名-村霸改', name5='小名-历狗改', age=22, gender='男'}
反序列化后的User类: User{name='大名-陈一发', name2='小名-二狗', name3='小名-狗蛋改', name4='null', name5='小名-历狗改', age=25, gender='男'}

```

# 0x04 小结
```plain
transient和static与final不参与反序列化
使用了transient修饰符的变量,可以在readObject反序列化时进行额外处理,来达到重新赋值的功能
使用了static修饰符的变量,不参与反序列化,因此加不加transient,static静态变量的值都是可修改的
使用了final修饰符将直接通过值参与序列化,所以将final变量声明为transient变量不会产生任何影响
```
