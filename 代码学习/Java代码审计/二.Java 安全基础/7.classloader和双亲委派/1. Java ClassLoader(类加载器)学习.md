## <font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">一、ClassLoader 概念</font>
<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">类加载器是负责将 Java 类（class 文件）动态加载到 JVM 内存中的组件，Java 程序需先启动 JVM，再通过它加载类并运行。</font>

## <font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">二、Class 的生命周期</font>
1. <font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">完整流程：加载 → 连接（验证、准备、解析）→ 初始化 → 使用 → 卸载</font>
2. <font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">关键特点：加载、验证、准备、初始化顺序确定，在某种情况下，解析可在初始化后执行（支持运行时绑定）；阶段交叉执行，非严格顺序完成。</font>

## <font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">三、ClassLoader 的分类</font>
### Bootstrap ClassLoader(<font style="color:rgb(18, 18, 18);">引导类加载器</font>)
```plain
BootstrapClassLoader是最顶层的加载类
负责加载“\jre\lib”目录(JDK内部的路径)或者被“-Xbootclasspath参数”所指定的路径中的所有类库

Windows下安装的Java,它加载“%JAVA_HOME%\jre\lib”里面的jar包
Mac下安装的Java,它加载“$JAVA_HOME/jre/lib”里面的jar包

比如:rt.jar,charsets.jar,resources.jar等....

引导类加载器无法被Java程序直接引用,用户在编写自定义类加载器时
如果想设置Bootstrap ClassLoader为其parent,可直接设置null
```

```plain
BootstrapClassLoader还是其它ClassLoader实例的父类
BootstrapClassLoader是JVM的一部分,它是由本机代码(C/C++)编写,在不同的平台上会有不同的实现
```

### Extention ClassLoader(扩展类加载器)
```plain
Extention ClassLoader是扩展类加载器
负责加载“\jre\lib\ext”目录(JDK内部的路径)或者被“java.ext.dirs系统变量”所指定路径中的所有类库

Windows下安装的Java,它加载“%JAVA_HOME%\jre\lib\ext”的jar包和class文件
Mac下安装的Java,它加载“$JAVA_HOME/jre/lib/ext”的jar包和class文件

扩展类加载器由引导类加载器加载,其父类加载器为引导类加载器,即parent=null

开发者可直接通过java.lang.ClassLoader中的getSystemClassLoader().getParent()方法,获取扩展类加载器
```

### Application ClassLoader(应用程序类加载器)
```plain
Application ClassLoader是应用程序类加载器
负责加载用户类路径(ClassPath)路径下的所有类和jar包,如未设置则应为应用程序当前路径
应用程序类加载器也是引导类加载器加载的,但是它的父类加载器是扩展类加载器
在一个应用程序中,应用程序类加载器一般是默认类加载器

开发者可直接通过java.lang.ClassLoader中的getSystemClassLoader()方法,获取应用程序类加载器,也可称它为系统类加载器
```

## <font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">五、ClassLoader 常用代码</font>
1. <font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">获取当前类的类加载器：</font>

```java
// 方式1
ClassLoader classloader = this.getClass().getClassLoader();
// 方式2
ClassLoader classloader2 = 类名称.class.getClassLoader();
```

2. <font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">查看类的加载器：</font>

```java
ClassLoader classloader = Class.forName("test.class").getClassLoader();
```

3. <font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">获取系统 / 扩展类加载器：</font>

```java
// 应用程序类加载器
ClassLoader applicationClassLoader = ClassLoader.getSystemClassLoader();
// 扩展类加载器
ClassLoader extentionClassloader = ClassLoader.getSystemClassLoader().getParent();
```

4. <font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">线程上下文类加载器（Web 环境推荐）：</font>

```java
// 设置
Thread.currentThread().setContextClassLoader(ClassLoader cl);
// 获取
ClassLoader classloader = Thread.currentThread().getContextClassLoader();
```
