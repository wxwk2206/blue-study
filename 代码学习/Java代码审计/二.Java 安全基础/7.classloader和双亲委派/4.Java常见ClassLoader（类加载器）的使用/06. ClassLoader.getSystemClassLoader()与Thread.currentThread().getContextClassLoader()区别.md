打个简单的比方,我们一个WEB程序,发布到Tomcat里面运行
Tomcat项目里的类是通过WebAppClassLoader类来加载的
所以它类的ClassLoader就会返回WebAppClassLoader

ClassLoader.getSystemClassLoader():
无论何时均会返回ApplicationClassLoader

Thread.currentThread().getContextClassLoader():
假如运行在非WEB的环境下,那么它返回ApplicationClassLoader
假如运行在Tomcat下的话,那么项目里的类是通过WebAppClassLoader类来加载的
所以它的返回会变成WebAppClassLoader
因此如果在WEB项目中,非特殊情况下一律使用该方法来获取ClassLoader,这样可以减少奇怪的错误
