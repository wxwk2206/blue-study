## 二、核心读取方式详解
### （一）基础 IO 流读取
 面向流：数据以 “流” 的形式连续传输，只能单向读取 / 写入（输入流读、输出流写）  

#### 1. FileInputStream（字节流）
+ **适用场景**：读取任意二进制文件（如图片、视频）或文本文件
+ **核心特点**：以字节为单位读取，需手动处理缓冲区
+ **关键代码**：

```java
package FileReadTest;
import java.io.ByteArrayOutputStream;
import java.io.File;
import java.io.FileInputStream;
import java.io.IOException;

public class FileInputStreamTestReadDemo
    public static void main(String[] args) {  
        File f = new File("C:\Users\22063\Desktop\bluek.txt");
    
        FileInputStream fis = null;
        //声明FileInputStream类型变量fis，用于后续操作文件输入流。
        try {   //try块用于包裹可能抛出异常的代码
            fis = new FileInputStream(f);
            //创建FileInputStream实例，打开目标文件并建立与文件的 “字节读取通道”
            int l;//声明int类型变量l，用于存储每次调用fis.read(bytes)方法时实际读取到的字节数。
            byte[] bytes = new byte[1024]; 
            //创建长度为 1024 的字节数组bytes，作为读取文件的 “缓冲区”
            ByteArrayOutputStream out = new ByteArrayOutputStream();
            // 循环读取，直到末尾（返回-1）
            while ((l = fis.read(bytes)) != -1) {
                out.write(bytes, 0, l); // 写入有效字节（0到l-1）
            }
            System.out.println(out.toString()); // 字节转字符串输出
        } catch (Exception e) {//捕获try块中所有可能抛出的异常
            e.printStackTrace();//打印异常详细信息
        } finally {//无论try块是否正常执行、是否抛出异常，finally块都会执行
            try {
                if (fis != null) fis.close(); // 强制关闭流
            } catch (IOException e) {
                e.printStackTrace();
            }
        }
    }
}
```

#### 2. FileReader（字符流）
+ **适用场景**：读取文本文件（避免字节流中文乱码问题）
+ **核心特点**：以字符为单位读取，直接处理文本内容
+ **关键代码**：

```java
package FileReadTest;
import java.io.FileReader;
import java.io.IOException;

public class FileReaderTestReadDemo {
    public static void main(String[] args) {
        FileReader fr = null;
        try {
            fr = new FileReader("/etc/hosts");
            int l;
            char[] buf = new char[6]; // 字符缓冲区
            StringBuilder out = new StringBuilder();
            while ((l = fr.read(buf)) != -1) {
                String str = new String(buf, 0, l); // 字符数组转字符串
                out.append(str);
            }
            System.out.print(out);
        } catch (Exception e) {
            e.printStackTrace();
        } finally {
            try {
                if (fr != null) fr.close();
            } catch (IOException e) {
                e.printStackTrace();
            }
        }
    }
}
```

### （二）NIO 工具类读取（Files）
+ **适用场景**：简化文本 / 二进制文件读取，JDK 7+ 推荐
+ **核心特点**：静态方法直接读取，无需手动处理流关闭

                       面向缓冲区，数据先写入缓冲区，再从缓冲区读取，支持双向操作

+ **关键代码**：

```java
package FileReadTest;
import java.nio.file.Files;
import java.nio.file.Path;
import java.nio.file.Paths;

public class FilesTestReadDemo {
    public static void main(String[] args) {
        Path path = Paths.get("/etc/hosts"); // 文件路径封装
        try {
            byte[] bytes = Files.readAllBytes(path); // 一次性读为字节数组
            String out = new String(bytes);
            System.out.println(out);
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}
```

### （三）随机访问文件读取（RandomAccessFile）
+ **适用场景**：需要随机访问文件任意位置的场景（如大文件分段读取）
+ **核心特点**：支持读写模式切换，可通过`seek()`定位读取位置
+ **关键代码**：

```java
package FileReadTest;
import java.io.ByteArrayOutputStream;
import java.io.File;
import java.io.IOException;
import java.io.RandomAccessFile;

public class RandomAccessFileTestReadDemo {
    public static void main(String[] args) {
        File f = new File("/etc/hosts");
        RandomAccessFile raf = null;
        try {
            // 模式：r(只读)、rw(读写)、rws(读写+内容同步)、rwd(读写+元数据同步)
            raf = new RandomAccessFile(f, "r");

            // 1. 关键：定位到第 100 字节（跳过前 100 字节）
            long skipPos = 100; // 要跳过的字节数（即起始读取位置）
            raf.seek(skipPos); // 指针移动到第 100 字节处
            // 2. 后续读取操作从第 100 字节开始
            
            int l;
            byte[] bytes = new byte[1024];
            ByteArrayOutputStream out = new ByteArrayOutputStream();
            while ((l = raf.read(bytes)) != -1) {
                out.write(bytes, 0, l);
            }
            System.out.println(out.toString());
        } catch (Exception e) {
            e.printStackTrace();
        } finally {
            try {
                if (raf != null) raf.close();
            } catch (IOException e) {
                e.printStackTrace();
            }
        }
    }
}
```

#### 2. FileUtils 工具类
+ **核心优势**：封装常用操作，一行代码完成读取，自动处理流关闭
+ **常用方法**：
    - `readFileToString(File file, String charset)`：读为字符串

**readFileToString**：适合小文本文件，直接获取完整字符串，需指定编码。

    readFileToByteArray(File file)：读为字节数组

**readFileToByteArray**：适合二进制文件或需要处理原始字节的场景,无默认编码,转字符串需手动指定。

   `readLines(File file, String charset)：读为字符串列表`

**readLines**：适合逐行处理文本文件（尤其是大文件），节省内存，需指定编码。

+ **示例代码 1：读为字符串**

```java
package FileReadTest;
import org.apache.commons.io.FileUtils;
import java.io.File;
import java.io.IOException;

public class FileUtilsTestReadDemo1 {
    public static void main(String[] args) throws IOException {
        File f = new File("/etc/hosts");
        String data = FileUtils.readFileToString(f, "UTF-8"); // 指定编码
        System.out.println(data);
    }
}
```

+ **示例代码 2：读为字节数组**

#### 3. IOUtils 工具类
+ **核心优势**：支持多种输入流（文件流、URL 流等）的读取转换
+ **常用方法**：
    - `toString(InputStream/URL, String charset)`：转字符串
    - `toByteArray(InputStream)`：转字节数组
    - `readLines(InputStream, String charset)`：转字符串列表
    - `read(InputStream, byte[])` ：返回实际读到的字节数** **
    - `readFully(InputStream, byte[])`：强制读满指定长度，读不满报错
+ **示例代码 1：读取 URL 响应**

```java
package FileReadTest;
import org.apache.commons.io.IOUtils;
import java.io.IOException;
import java.net.URL;

public class IOUtilsTestReadDemo1 {
    public static void main(String[] args) throws IOException {
        URL url = new URL("http://127.0.0.1:8088/test.txt");
        String data = IOUtils.toString(url, "UTF-8");
        System.out.println(data);
    }
}
```

+ **示例代码 2：读文件为字符串**

### （五） InputStream 基础字节输入流  
+  **适用场景**：所有字节类型数据的读取（文本 / 二进制文件、网络数据流等），是字节输入流的顶层抽象类，为所有字节输入流提供统一基础接口
+  **核心特点**：面向字节的阻塞式读取，无内置缓冲区（需手动创建字节数组作为缓冲区），所有具体字节输入流（如 FileInputStream、BufferedInputStream）均继承自它
+  **关键代码**：  

```java
package FileReadTest;
import java.io.BufferedInputStream;
import java.io.ByteArrayOutputStream;
import java.io.FileInputStream;
import java.io.InputStream;
import java.io.IOException;
public class InputStreamReadDemo {
    public static void main(String[] args) throws IOException {
        InputStream inputStream = null;
        try {
            inputStream = new BufferedInputStream(new FileInputStream("/etc/passwd"));
            byte[] buf = new byte[1024];
            int len;
            ByteArrayOutputStream out = new ByteArrayOutputStream();
            while ((len = inputStream.read(buf)) > 0) {
                out.write(buf, 0, len);
            }
            System.out.println(out.toString());
        } catch (IOException e) {
            e.printStackTrace();
        } finally {
            inputStream.close();
        }
    }
}
```



### （六）缓冲流优化读取（BufferedInputStream）
+ **适用场景**：大文件读取（减少磁盘 IO 次数，提升效率）
+ **核心特点**：内置缓冲区（默认 8KB），包装基础字节流使用
+ **关键代码**：

```java
package FileReadTest;
import java.io.BufferedInputStream;
import java.io.FileInputStream;
import java.io.IOException;

public class BufferedInputStreamReadDemo {
    public static void main(String[] args) {
        try {
            FileInputStream fis = new FileInputStream("/etc/passwd");
            // 包装基础流，启用缓冲
            BufferedInputStream bufferedInputStream = new BufferedInputStream(fis);
            StringBuilder dataBuilder = new StringBuilder();
            byte[] buffer = new byte[1024];
            int bytesRead;
            while ((bytesRead = bufferedInputStream.read(buffer)) != -1) {
                dataBuilder.append(new String(buffer, 0, bytesRead));
            }
            String data = dataBuilder.toString();
            System.out.println(data);
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
```

## 三、关键注意事项
1. **资源关闭**：所有流对象必须在 finally 块中关闭，或使用 try-with-resources 语法（JDK 7+）
2. **编码问题**：字符流 / 字符串转换时需指定编码（如 UTF-8），避免乱码
3. **缓冲区大小**：根据文件大小调整缓冲区（一般 1KB~8KB），过大可能占用过多内存
4. **异常处理**：文件读取可能抛出 IO 异常，必须捕获或声明抛出
5. **工具类选择**：
    - 简单读取推荐使用`Files`类（JDK 内置）
    - 复杂场景推荐`commons-io`（减少重复代码）
    - 大文件推荐`BufferedInputStream`（缓冲优化）
    - 随机访问推荐`RandomAccessFile`
