## 一、核心区别（JDK 动态代理 vs Cglib 代理）
| **特性** | **JDK 动态代理**                                         | **Cglib 代理**                                               |
| ------ | ---------------------------------------------------- | ---------------------------------------------------------- |
| 实现原理   | 基于接口，通过反射生成实现代理接口的匿名类，依赖 `InvocationHandler`<br/> 接口 | 基于 ASM 框架，修改目标类字节码，生成目标类的子类，依赖 `MethodInterceptor`<br/> 接口 |
| 代理目标   | 仅能代理实现了接口的类                                          | 可直接代理类（无需接口）                                               |
| 限制条件   | 必须实现接口                                               | 目标类 / 方法不能是 `final`<br/>（因采用继承方式，`final`<br/> 无法被重写）       |
| 核心机制   | 接口回调 + 反射                                            | 字节码增强 + 子类继承                                               |


## 二、Cglib 代理核心语法
### 1. 核心依赖
需引入 Cglib 相关依赖（如 Spring 内置 Cglib，可直接使用 `org.springframework.cglib` 包下的类）：

+ `Enhancer`：Cglib 核心工具类，用于生成代理子类
+ `MethodInterceptor`：回调接口，定义方法增强逻辑（类似 JDK 代理的 `InvocationHandler`）

### 2. 关键步骤
1. 定义目标类（无需实现接口）；
2. 实现 `MethodInterceptor` 接口，重写 `intercept` 方法，编写增强逻辑（如前置 / 后置处理）；
3. 通过 `Enhancer` 配置：设置父类（目标类）、绑定回调对象；
4. 生成代理对象，调用目标方法时自动触发增强逻辑。

## 三、实战案例
### 场景说明
目标：对用户 “看电影”“上洗手间” 的行为，在前后添加影院广告（增强逻辑）。

### 1. 目标类（被代理类）
```java
package 动态代理2;
public class People {
    // 看电影方法
    public void movie() {
        System.out.println("您正在观看电影《花园宝宝历险记》");
    }
    // 上洗手间方法
    public void toilet() {
        System.out.println("您冲进了洗手间");
    }
}
```

### 2. 代理类（增强逻辑实现）
```java
package 动态代理2;
import org.springframework.cglib.proxy.Enhancer;
import org.springframework.cglib.proxy.MethodInterceptor;
import org.springframework.cglib.proxy.MethodProxy;
import java.lang.reflect.Method;

// 实现 MethodInterceptor 接口，定义增强逻辑
public class Cinema implements MethodInterceptor {
    // 目标对象（被代理的对象）
    private Object target;

    // 绑定目标对象，生成代理对象
    public Object bind(Object target) {
        this.target = target;
        Enhancer enhancer = new Enhancer(); // Cglib 核心工具类
        enhancer.setSuperclass(target.getClass()); // 设置父类为目标类（生成子类代理）
        enhancer.setCallback(this); // 绑定回调对象（当前 Cinema 实例）
        return enhancer.create(); // 生成并返回代理对象
    }

    // 重写 intercept 方法，实现方法增强
    @Override
    public Object intercept(Object proxy, Method method, Object[] args, MethodProxy methodProxy) throws Throwable {
        // 前置增强：调用目标方法前执行（播放开始广告）
        System.out.println("------------------");
        System.out.println("该影院叫: " + this.getClass().getSimpleName());
        advertising1(); // 开始广告

        // 调用目标对象的原始方法
        Object result = method.invoke(target, args);

        // 后置增强：调用目标方法后执行（播放结束广告）
        advertising2(); // 结束广告
        return result;
    }

    // 自定义增强逻辑：开始广告
    public void advertising1() {
        System.out.println("开始广告: 电影准备开始了，只需要39元，快来买啊! 肯德基十翅一桶");
    }

    // 自定义增强逻辑：结束广告
    public void advertising2() {
        System.out.println("结束广告: 电影结束了，只需要39元，快买回家吃吧!");
    }
}
```

### 3. 测试类
```java
package 动态代理2;
public class Test {
    public static void main(String[] args) {
        // 1. 创建目标对象
        People people = new People();
        // 2. 创建代理对象（绑定目标对象）
        Cinema cinema = new Cinema();
        People peopleProxy = (People) cinema.bind(people);
        // 3. 调用代理对象的方法（自动触发增强逻辑）
        peopleProxy.movie(); // 代理看电影方法
        peopleProxy.toilet(); // 代理上洗手间方法
    }
}
```

## 四、核心总结
1. Cglib 代理的核心是**字节码增强**，通过生成目标类的子类实现代理，无需目标类实现接口；
2. 关键类 / 接口：
    - `Enhancer`：负责生成代理子类，需指定父类（目标类）和回调；
    - `MethodInterceptor`：回调接口，`intercept` 方法中编写增强逻辑（前置 / 后置 / 环绕增强）；
3. 注意事项：
    - 目标类 / 方法不能是 `final`，否则无法继承重写；
    - 增强逻辑写在 `intercept` 方法中，通过 `method.invoke(target, args)` 调用原始方法；
4. 适用场景：需代理无接口的类，或追求更高性能（字节码增强比反射效率高）的场景。
