## 一、前言
### （一）核心定位
静态代理是一种设计模式，无需额外依赖包，核心在于遵循特定实现思路即可。

### （二）关键注意点
1. 需先定义**接口或父类**，作为代理对象与目标对象的统一规范。
2. 代理对象与目标对象必须**实现相同接口或继承相同父类**。
3. 代理对象通过调用与目标对象相同的方法，间接触发目标对象的对应方法执行。

## 二、实现环境
无特殊依赖，基于 Java 基础环境即可实现。

## 三、实例演示（影院播放电影场景）
### （一）场景逻辑
+ 目标对象：电影公司（仅负责影片授权，与影院分票房，不处理额外事务）。
+ 代理对象：影院（代理电影播放，同时在影片前后播放广告获取额外收益）。
+ 核心需求：不修改电影公司（目标对象）逻辑，扩展 “播放广告” 功能。

### （二）代码实现步骤
1. **创建统一接口**（代理模式基础）：定义`Movie`接口，包含电影播放的核心方法`play()`。

```java
public interface Movie {
    void play();
}
```

2. **实现目标对象**：创建`RealMovie`类（电影公司），实现`Movie`接口，重写`play()`方法（核心功能：播放电影）。

```java
public class RealMovie implements Movie {
    @Override
    public void play() {
        System.out.println("播放授权影片");
    }
}
```

3. **实现代理对象**：创建`Cinema`类（影院），实现`Movie`接口，内部持有目标对象引用，重写`play()`方法（扩展功能 + 调用目标对象方法）。

```java
public class Cinema implements Movie {
    // 持有目标对象（电影公司）
    private Movie realMovie;

    // 构造方法注入目标对象
    public Cinema(Movie realMovie) {
        this.realMovie = realMovie;
    }

    @Override
    public void play() {
        // 扩展功能：播放前置广告
        System.out.println("播放电影前广告");
        // 调用目标对象核心方法
        realMovie.play();
        // 扩展功能：播放后置广告
        System.out.println("播放电影后广告");
    }
}
```

4. **运行测试**：创建测试类，通过代理对象`Cinema`调用`play()`方法，验证功能扩展效果。

```java
public class Test {
    public static void main(String[] args) {
        // 目标对象
        Movie realMovie = new RealMovie();
        // 代理对象（传入目标对象）
        Movie cinema = new Cinema(realMovie);
        // 通过代理对象执行方法
        cinema.play();
    }
}
```

## 四、优缺点总结
### （一）优点
在**不修改目标对象代码**的前提下，实现对目标对象功能的灵活扩展（符合 “开闭原则”）。
### （二）缺点
1. **扩展性差**：若目标接口（如`Movie`）新增方法，代理对象（`Cinema`）和所有目标对象（如`RealMovie`）都需同步修改，维护成本高。
2. **代码臃肿**：每个目标对象都需对应一个代理对象，若目标对象数量多（如 100 部影片对应 100 个`RealMovie`子类），需创建大量代理类，代码复用率低。
