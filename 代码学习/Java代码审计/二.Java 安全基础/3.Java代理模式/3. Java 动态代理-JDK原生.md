## 一、前言
### 1. 核心定位
动态代理是一种设计模式，与静态代理的**功能和目的一致**，核心差异在于 “动态” 特性：

+ 静态代理：代理对象在代码编写阶段直接定义，编译后确定
+ 动态代理：代理对象在程序**运行时动态创建**，无需硬编码

### 2. 核心优势
可统一处理代理对象的所有方法，无需逐个修改，大幅提升代码复用性和维护性。

### 3. 技术依赖
基于 Java 反射机制，核心依赖两个组件：

+ `java.lang.reflect.Proxy` 类（生成代理对象）
+ `InvocationHandler` 接口（处理代理逻辑）

## 三、核心语法
### 1. InvocationHandler 接口
#### （1）作用
每个代理实例都关联一个 `InvocationHandler` 实现类，代理对象的方法被调用时，会转发给该接口的 `invoke` 方法处理。

#### （2）核心方法
```java
public interface InvocationHandler {
    /**
     * @param proxy  被代理的目标对象
     * @param method 要调用的目标方法
     * @param args   方法调用时的参数数组
     * @return 方法执行后的返回值
     * @throws Throwable 方法执行过程中可能抛出的异常
     */
    Object invoke(Object proxy, Method method, Object[] args) throws Throwable;
}
```

+ 所有代理逻辑（如增强功能、方法转发）都在 `invoke` 方法中实现。

### 2. Proxy 类（java.lang.reflect.Proxy）
#### （1）作用
动态生成代理类实例，是创建代理对象的核心工具类，实现 `java.io.Serializable` 接口（支持序列化）。

#### （2）java.lang.reflect.Proxy包
```java
// Proxy动态产生的代理会调用InvocationHandler实现类,因此InvocationHandler是实际执行者
public class Proxy implements java.io.Serializable {
	...
    /**
     * @param loader		代理对象的类加载器
     * @param interfaces	代理类全部的接口
     * @param h				实现InvocationHandler接口的对象 
     * @return				返回代理对象生成的类实例
     * @throws IllegalArgumentException
     */
    @CallerSensitive
    public static Object newProxyInstance(ClassLoader loader,
                                          Class<?>[] interfaces,
                                          InvocationHandler h)
        throws IllegalArgumentException
    {
        if (h == null) {
            throw new NullPointerException();
        }

        final Class<?>[] intfs = interfaces.clone();
        final SecurityManager sm = System.getSecurityManager();
        if (sm != null) {
            checkProxyAccess(Reflection.getCallerClass(), loader, intfs);
        }

        /*
         * Look up or generate the designated proxy class.
         */
        Class<?> cl = getProxyClass0(loader, intfs);

        /*
         * Invoke its constructor with the designated invocation handler.
         */
        try {
            final Constructor<?> cons = cl.getConstructor(constructorParams);
            final InvocationHandler ih = h;
            if (sm != null && ProxyAccessHelper.needsNewInstanceCheck(cl)) {
                // create proxy instance with doPrivilege as the proxy class may
                // implement non-public interfaces that requires a special permission
                return AccessController.doPrivileged(new PrivilegedAction<Object>() {
                    public Object run() {
                        return newInstance(cons, ih);
                    }
                });
            } else {
                return newInstance(cons, ih);
            }
        } catch (NoSuchMethodException e) {
            throw new InternalError(e.toString());
        }
    }
    ...
}
```

#### （3）底层逻辑
1. 克隆传入的接口数组，校验安全权限
2. 查找或生成代理类（`getProxyClass0` 方法）
3. 获取代理类的构造器（参数为 `InvocationHandler`）
4. 通过构造器创建并返回代理实例

#### （3）实现代理对象（InvocationHandler 实现类）
```java
public class Cinema implements InvocationHandler {
    // 被代理的目标对象（可复用，支持不同接口）
    private Object target;

    // 绑定目标对象，生成代理实例
    public Object bind(Object target) {
        this.target = target;
        return Proxy.newProxyInstance(
            target.getClass().getClassLoader(),
            target.getClass().getInterfaces(),
            this  // 当前 Cinema 实例作为 InvocationHandler
        );
    }

    @Override
    public Object invoke(Object proxy, Method method, Object[] args) throws Throwable {
        System.out.println("------------------");
        System.out.println("该影院叫: " + this.getClass().getSimpleName());
        advertising1();  // 核心功能前增强（播放广告）
        Object result = method.invoke(target, args);  // 调用目标对象的核心方法
        advertising2();  // 核心功能后增强（播放广告）
        System.out.println("------------------");
        return result;
    }

    // 广告1（功能前）
    private void advertising1() {
        System.out.println("开始广告: 电影准备开始了,肯德基十翅一桶,只需要39元,快来买啊!");
    }

    // 广告2（功能后）
    private void advertising2() {
        System.out.println("结束广告: 电影结束了,肯德基十翅一桶,只需要39元,快买回家吃吧!");
    }
}
```

### 3. 关键结论
+ 代理类名格式：`com.sun.proxy.$ProxyN`（N 从 0 自增，不同接口生成不同代理类）
+ 统一增强：所有代理对象的方法调用，都会触发 `Cinema.invoke` 方法，实现统一增强

## 五、invoke 方法自动调用原理
### 1. 核心疑问
通过 `Proxy.newProxyInstance` 生成的代理对象（如 `movieProxy1`），是否是 `Cinema` 类的实例？

+ 答案：否！代理对象是 `$Proxy0`（动态生成的代理类）的实例

### 2. 代理类（$Proxy0）底层逻辑
反编译生成的 `$Proxy0.class`，核心结构如下：
```java
public final class $Proxy0 extends Proxy implements Movie {
    private static Method m3;  // 对应 Movie.play() 方法

    // 构造器：接收 InvocationHandler 实例（即 Cinema 对象）
    public $Proxy0(InvocationHandler var1) { super(var1); }

    // 实现 Movie.play() 方法
    public final void play() {
        try {
            // 调用父类（Proxy）的 h.invoke()，h 即绑定的 Cinema 实例
            super.h.invoke(this, m3, (Object[])null);
        } catch (RuntimeException | Error var2) {
            throw var2;
        } catch (Throwable var3) {
            throw new UndeclaredThrowableException(var3);
        }
    }

    // 静态代码块：初始化方法对象（equals、hashCode、toString、play）
    static {
        try {
            m3 = Class.forName("动态代理.Movie").getMethod("play");
            // 初始化 Object 类的 equals、hashCode、toString 方法
        } catch (NoSuchMethodException | ClassNotFoundException e) {
            throw new NoClassDefFoundError(e.getMessage());
        }
    }
}
```

### 3. 执行流程
1. `Proxy.newProxyInstance` 生成 `$Proxy0` 类并实例化，返回代理对象 `movieProxy1`
2. 调用 `movieProxy1.play()` → 触发 `$Proxy0.play()` 方法
3. `$Proxy0.play()` 内部调用 `super.h.invoke()`（`h` 是绑定的 `Cinema` 实例）
4. `Cinema.invoke()` 执行增强逻辑 + 调用目标对象方法（`realMovie1.play()`）
5. 流程结束，返回结果

## 六、核心总结
### 1. 动态代理优势
+ 代码复用：代理类（如 `Cinema`）逻辑固定，不会因业务扩展而膨胀
+ 支持 AOP 编程：分离核心业务与增强逻辑（如广告、日志、权限校验）
+ 解耦：例如 Web 开发中实现数据层与业务层分离

### 2. 关键特性（黑客必备，反序列化漏洞相关）
1. 代理类支持序列化（`Proxy` 实现 `Serializable` 接口）
2. 目标对象通过 `Proxy` 代理后，会变成代理对象（`cinema.bind()` 返回值）
3. 代理对象的**所有方法调用**，都会转发到关联的 `InvocationHandler.invoke` 方法
4. 代理对象会重写 `Object` 类的 3 个方法：`equals()`、`toString()`、`hashCode()`
5. 多代理类命名规则：`$Proxy0`、`$Proxy1`、`$Proxy2`...（按生成顺序自增）
