# <font style="color:rgb(0, 0, 0);">1 漏洞成因</font>
文件上传时,如果**文件校验不当**,攻击者可以非法上传一个可执行的脚本文件(如:jsp,jspx),获取服务器权限

# 2 审计策略
## 2.1 常规情况下审计策略
```plain
审计策略一: 
关注文件是如何进行文件后缀的验证的
如果是白名单,判断JDK版本,如果低于JDK1.7.0_40,可以尝试%00截断绕过(比如:1.jsp%00.txt)
如果是黑名单,尝试后缀大小写绕过黑名单(比如:1.JsP,Windows可以正常执行)
如果是黑名单,尝试Windows文件命名的机制(比如:“1.jsp.”,Windows会写入时自动改成1.jsp)
如果是黑名单,查看是黑名单后缀是否遗漏了(比如:禁止上传jsp,class,jar,没有禁止上传jspx)
```

```plain
审计策略二:
关注文件路径与文件名后缀是否用户可控制
如果文件路径用户可控制,那么可以使用“../”让文件保存在任意位置中
如果文件后缀用户可控制,并且使用黑名单,那么用户可以上传“../js/index.js”覆盖js文件进行钓鱼攻击
如果文件路径与文件后缀都可控制并且可上传jar,而且权限也够,那么可以尝试上传个恶意jar覆盖系统的jar
```

```plain
审计策略三:
查看安全校验的顺序,有一些上传的安全校验顺序有问题
它会先将文件保存在服务器,在进行安全检测,如果不通过检查在把文件删除
此时可以尝试在文件保存后触发报错终止流程,让本该被删除的文件保存下来
```

## 2.2 方便快速审计的污点跟踪类
```plain
方便快速审计的污点跟踪类
查找调用了这些类实现上传的业务代码,判断在写入时,后缀是否控制就可以快速审计出是否有漏洞了
例如:IDEA搜索“ServletFileUpload(”然后查看业务代码在最终写入时后缀是否可控
```

# 3 修复方法
```plain
1. 白名单检查文件扩展名(例如只允许:JPG,PNG,GIF后缀的文件进行上传)
2. JDK版本不要低于JDK1.7.0_40防止%00截断
3. 确保文件名不包含任何可能被解释为目录的子字符串(例如文件名:../../1.txt)
4. 重命名上传的文件以避免可能导致现有文件被覆盖的冲突
5. 隐藏上传文件的绝对路径
6. 将上传文件保存到图片服务器保存(比如:七牛云存储)
```

```plain
通用白名单文件类型:
图片类型: jpg、jpeg、bmp、png、gif
文件类型: pdf、doc、docx、xls、xlsx、xlsm，xlt、xltx、ppt、pptx
文本类型: txt(特例情况使用,尽量不要设置该白名单)
```

## 4.4 MultipartFile上传示例
```java
// 路径: ./SpringMVCTest2/src/main/com/test/controller/upload/MultipartFileUploadTest1.java
package test.controller.upload;

import org.springframework.stereotype.Controller;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.ResponseBody;
import org.springframework.web.multipart.MultipartFile;

import javax.servlet.http.HttpServletRequest;
import java.io.File;
import java.io.IOException;
import java.text.SimpleDateFormat;
import java.util.Date;
import java.util.UUID;

@Controller
@RequestMapping(value = "/MultipartFileUploadTest1")
public class MultipartFileUploadTest1 {
    @RequestMapping(value = "/upload", produces = "text/html;charset=UTF-8")
    @ResponseBody
    public String upload(MultipartFile uploadFile, HttpServletRequest request) throws IOException {
        // 上传的⽂件原名,如xxx.txt
        String originalFilename = uploadFile.getOriginalFilename();

        // 获取⽂件的扩展名,如txt
        String extendName = originalFilename.substring(originalFilename.lastIndexOf(".") + 1).trim().toLowerCase();

        // 设置不允许上传文件类型
        String suffixList = "jsp,java,class";
        for (String suffix : suffixList.split(",")) {
            if (suffix.equals(extendName)) {
                return "非法后缀上传失败:" + extendName;
            }
        }

        // 新的⽂件名字
        String newName = UUID.randomUUID().toString() + "." + extendName;

        // 文件保存地址
        String realPath = request.getSession().getServletContext().getRealPath("/uploads");

        // 解决⽂件夹存放⽂件数量限制,按⽇期存放
        String datePath = new SimpleDateFormat("yyyy-MM-dd").format(new Date());
        String uploadPath = realPath + "/" + datePath;
        File floder = new File(uploadPath);
        if (!floder.exists()) {
            floder.mkdirs();
        }

        // 把MultipartFile转换换成File
        // 保存文件在本地
        uploadFile.transferTo(new File(floder, newName));

        // 输出上传路径
        return "上传路径: " + uploadPath + "/" + newName;
    }
}
```
