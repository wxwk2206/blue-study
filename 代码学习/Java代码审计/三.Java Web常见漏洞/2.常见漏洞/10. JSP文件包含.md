# <font style="color:rgb(0, 0, 0);">0x01 漏洞成因</font>
```plain
程序开发人员通常会把可重复使用的代码写到单个文件中,在使用到这些代码时,可以通过文件包含函数进行调用
因此如果文件包含函数的参数外部可控,并且未过滤,这样就攻击者就可能可以非预期的文件

攻击者可能可以通过此功能造成如下安全问题:
1. 任意文件读取
2. 绕过基于Filter过滤器的权限验证
```

# 0x02 注意项
```plain
注意一: 
JSP文件包含分两种方式: 静态文件包含和动态文件包含
静态文件包含,不能使用变量来控制包含某个文件,所以不存在文件包含漏洞
动态文件包含,可以使用变量来控制包含某个文件,所以可能存在文件包含漏洞

注意二:
<jsp:include page="xxx" ></jsp:include>
可以解析JSP文件/JSPX文件/servlet请求

注意三:
<c:import url="xxx"></c:import>
可以解析JSP文件/JSPX文件/servlet请求/SpringMVC请求等...
可以通过file://进行文件读取

注意四:
<jsp:forward page="xxx"></jsp:forward>
可以解析JSP文件/JSPX文件/servlet请求/SpringMVC请求等...
对于非JSP文件扩展名时,会读取其源码并显示出来

注意五:
<% request.getRequestDispatcher("xxx").include(request,response); %>
可以解析JSP文件/JSPX文件/servlet请求/SpringMVC请求等...

注意六:
<% request.getRequestDispatcher("xxx").forward(request,response); %>
可以解析JSP文件/JSPX文件/servlet请求/SpringMVC请求等...
对于非JSP文件扩展名时,会读取其源码并显示出来
```

# 0x03 审计策略
```plain
想要审计该漏洞还是比较简单的,只需要按照特征寻找即可
查找到这些标签以后,查看外部是否可控,如果可控,并且未过滤,那就存在安全问题

静态文件包含(无法利用直接放弃):
<jsp:directive.include file="./111.txt"/>
<%@ include file="./111.txt" %>

动态文件包含:
page参数可控有安全问题
<jsp:include page="222.jsp" ></jsp:include>

JSTL库-动态文件包含:
url参数可控有安全问题
<%@ taglib uri="http://java.sun.com/jsp/jstl/core" prefix="c" %>
<c:import url="222.jsp"></c:import>

请求转发-1:
page参数可控有安全问题
<jsp:forward page="222.jsp"></jsp:forward>

请求包含:
getRequestDispatcher()可控有安全问题
<% request.getRequestDispatcher("222.jsp").include(request,response); %>

请求转发-2:
getRequestDispatcher()可控有安全问题
<% request.getRequestDispatcher("222.jsp").forward(request,response); %>
```

```java
package filter;

import javax.servlet.*;
import java.io.IOException;

public class TestFilter implements Filter {
    @Override
    public void doFilter(ServletRequest request, ServletResponse response, FilterChain chain) throws IOException, ServletException {
        response.getWriter().println("gggggggggggggg");
    }
}
```

```plain

<%-- 第四步 -->

    jspx-test-1-1-1-1-1
```

```java
// 第五步
// 添加用于测试访问的JSP HttpServlet类
// 路径: ./SpringMVCTest2/src/main/webapp/servlet/Test2.java
package servlet;

import javax.servlet.annotation.WebServlet;
import javax.servlet.http.HttpServlet;
import javax.servlet.http.HttpServletRequest;
import javax.servlet.http.HttpServletResponse;
import java.io.IOException;

@WebServlet("/test2")
public class Test2 extends HttpServlet {
    @Override
    protected void doGet(HttpServletRequest request, HttpServletResponse response) throws IOException {
        response.getWriter().write("test-2-2-2-2-2-2-2");
    }
}
```

```java
// 第六步
// 添加用于测试访问的Spring MVC的Controller类
// 路径: ./SpringMVCTest2/src/main/com/test/controller/Test3.java
package test.controller;

import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.RestController;

@RestController
public class Test3 {
    @RequestMapping("/test3")
    public String test3() {
        return "test-3-3-3-3-3-3-3";
    }
}
```

```xml
<!-- 第七步 -->
<!-- 使用这个测试拦截器 -->
```
