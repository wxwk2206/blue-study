# <font style="color:rgb(0, 0, 0);">1.漏洞成因</font>
Web站点常常需要根据不同的逻辑将用户引向到不同的页面，比较常见的就是登录认证成功之后将用户引导到登录之前的页面，这个过程中如果后台服务器在告知浏览器跳转时,**未对客户端传入的重定向地址进行合法性校验**，那么攻击者就可以构造恶意跳转的链接,向用户发起钓鱼攻击

# 2.漏洞作用
不能直接造成拿数据，删库等影响，是跳板漏洞，主要用于辅助攻击，

配合其他漏洞，如 XSS，CSRF，权限校验，

也可以用来钓鱼，窃取 token

# 3.审计策略
常见的容易出现URL重定向漏洞的功能有:

1. 用户登录/统一身份认证,通过后,会通过url=xxx的跳转到页面
2. 统一身份认证,退出时,会通过url=xxx的跳转到登录页面,未做规范时,攻击者可控
3. 用户分享功能,其它用户点击该功能跳转到别的页面中

```java
// JSP-重定向相关函数

// 搜索: response.sendRedirect/response.setHeader
<% response.sendRedirect("http://baidu.com"); %>
<% response.setHeader("refresh","0;URL=https://baidu.com"); %>
```

```java
// JSTL-重定向相关函数

// 搜索: <c:redirect
<c:redirect url="https://baidu.com"/>
```


# 4.修复方法
对传入的URL做校验,保证该URL来自于信任域

例如:对跳转的URL进行白名单检查,假设域名为baidu.com,那么除了baidu.com的域名,其它的域名都不进行跳转
