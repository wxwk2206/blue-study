# 1.漏洞成因
对于**用户的传递参数,没有进行过滤**,导致攻击者可以插入一些**恶意的JS语句**,在用户的浏览器中执行恶意JS脚本

攻击类型分为: 反射型XSS、DOM型XSS、存储型XSS

危害: 盗取用户信息、钓鱼、制造蠕虫等

# 2.漏洞类型
## 2.1反射型XSS
反射型：**一次性**，payload在URL中，需要诱导用户点击链接

<font style="color:#DF2A3F;">攻击代码在URL里,并且由服务器解析,输出在http响应中</font>

黑客需要诱惑用户点击一个恶意链接,触发XSS攻击
![[Pasted image 20260616115512.png]]

## 2.2DOM型XSS
DOM型：**纯前端**，payload不经过后端，只在浏览器DOM中执行

DOM型XSS与反射XSS很像

<font style="color:#DF2A3F;">区别就是DOM的攻击代码是在URL中,并且使用客户端接收,输出在DOM节点中</font>

![[Pasted image 20260616115525.png]]

## 2.3 存储型XSS
存储型：**持久化**，payload存在数据库中，所有访问该页面的用户都会触发

<font style="color:#DF2A3F;">存储型XSS的攻击代码是保存在服务端(数据库),随着用户访问-数据库获取数据加载,输出在http响应中</font>

例如说一个场景:

黑客在其他人的网站上写了一篇带有恶意JavaScript代码的文章

文章发表以后,所有访问该文章的用户,都会在他们的浏览器上执行黑客写入的恶意JavaScript代码

## 2.4不同类别之间的区别
| XSS类型 | 存储型XSS | 反射型XSS | DOM型XSS |
| --- | --- | --- | --- |
| 触发过程 | 1. 黑客构造XSS脚本保存到数据库<br/>2. 正常用户访问携带XSS脚本的页面 | 正常用户访问携带XSS脚本的URL | 正常用户访问携带XSS脚本的URL |
| 触发点 | 数据库 | URL | URL |
| 解析处 | 后端web应用 | 后端web应用 | 前端JavaScript |
| 输出位置 | HTTP响应中 | HTTP响应中 | 动态构造的DOM节点<br/>例如:js函数-innerHTML  |


# 3.审计策略
```plain
审计的要点就是定位用户的输入与输出,梳理数据交互以及前端展示的过程
找到对应输出与输出的地方以后就可以结合该代码现有的安全措施(输出编码、过滤器等)进行判断
例如是否存在绕过的可能,或者是没有任何安全防护可直接造成攻击
```

# 4.修复方法
```plain
自定义全局过滤器,对用户提交输入进行检查和过滤
对用户输入并用来展示的数据进行HTML转义,可用的工具类包括
org.springframework.web.util.HtmlUtils 
org.apache.commons.lang3.StringEscapeUtils
ESAPI.encoder().encodeForHTML 
等...
```

## 5.4存储型XSS
```plain
// 漏洞环境搭建-第一步
// 保存用户输入内容
// 目录: ./SpringMVCTest2/src/main/webapp/xss/xssTest3-insert.jsp
<%@ page import="java.nio.file.Files" %>
<%@ page import="java.nio.file.Paths" %>
<%@ page import="java.nio.charset.StandardCharsets" %>
<%@ page contentType="text/html;charset=UTF-8" language="java" %>
<%
    String input = request.getParameter("input");
    if (input != null && !input.equals("")) {
        String fileName = request.getServletContext().getRealPath("/xss/") + "/xssTest3.txt";
        Files.write(
                Paths.get(fileName),
                input.getBytes(StandardCharsets.UTF_8));
        out.println("输入成功");
    } else {
        out.println("请输入内容");
    }
%>
```

```plain
// 漏洞环境搭建-第二步
// 后端读取文件输出到HTML中
// 目录: ./SpringMVCTest2/src/main/webapp/xss/xssTest3-output.jsp
<%@ page import="java.nio.file.Files" %>
<%@ page import="java.nio.file.Paths" %>
<%@ page contentType="text/html;charset=UTF-8" language="java" %>
<%
    String fileName = request.getServletContext().getRealPath("/xss/") + "/xssTest3.txt";
    byte[] bytes = Files.readAllBytes(Paths.get(fileName));
    out.println(new String(bytes));
%>
```
