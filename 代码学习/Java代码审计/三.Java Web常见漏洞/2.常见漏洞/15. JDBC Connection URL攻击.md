# 0x01 JDBC简介
```plain
JDBC(Java Database Connectivity)是Java提供的标准API
用于连接和操作关系型数据库与其他编程语言相似
Java本身不实现对数据库的连接、查询、更新等操作,而是通过抽象出数据库操作的API接口(JDBC)来实现
不同的数据库提供商必须实现JDBC定义的接口,从而可以使用Java应用程序来连接和操作数据库
这种标准化的方法使得开发人员可以使用相同的API和代码来连接和操作不同的数据库
而不需要了解特定的数据库实现细节
```

# 0x02 JDBC Connection
```plain
JDBC定义了一个名为java.sql.Driver的接口类
用于实现对数据库的连接,为了完成数据库的连接操作,所有的数据库驱动包都必须实现这个接口
在Java中,可以使用java.sql.DriverManager.getConnection(xxx)方法来建立与数据库的连接
实际上,该方法间接调用了java.sql.Driver类的connect方法来实现数据库连接
连接成功后,会返回一个名为java.sql.Connection的数据库连接对象
所有的数据库查询操作都将依赖于这个Connection对象
通过这种方式,开发人员可以使用JDBC API提供的标准化方法
来连接和操作不同的数据库,从而简化数据库交互的过程
```

```plain
JDBC的连接格式:
jdbc://driver://host:port/database?配置name1=配置Value1&配置name2=配置Value2

JDBC的连接示例:
jdbc:mysql://localhost:3306/ekp?useUnicode=true&characterEncoding=UTF-8
```

# 0x03 漏洞成因
```plain
在一些业务后台中,会存在添加/修改/连接测试,数据源的功能,通过这个功能可以实时修改数据库配置
常见的功能就是: 输入新的数据库配置,点击测试,测试数据库连接是否正常的功能
在这种情况下,如果JDBC连接的URL可控时,配合一些低版本的数据库驱动包(比如:h2)就可以造成安全问题
```

# 0x04 审计策略
## 0x04.1 概述
```plain
想要挖掘这种漏洞
重点是关注数据库连接时的URL是否外部可控
并且源码中的数据库驱动包是否版本过低
如果URL外部可控,并且数据库驱动包比较旧没有更新,就表示可能有漏洞
```

## 0x04.2 方便快速审计的污点跟踪类
```plain
在审计时,关注这些这些类实现了数据库链接的业务代码
判断一下JDBC的URL是否外部可控,接着查看一下源码的数据库驱动包版本
就可以快速审计出是否有漏洞了

比如常见数据库驱动包:
mysql的mysql-connector-java
com.h2database的h2
org.postgresql的postgresql
om.ibm.db2的jcc
```

```plain
流行的JDBC连接池:
```

### 0x06.3.2 生成测试用的Controller
```java
// 路径: ./SpringMVCTest2/src/main/com/test/controller/jdbcConnectionURL/DatabaseTest.java
package test.controller.jdbcConnectionURL;

import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.RestController;

import java.sql.Connection;
import java.sql.DriverManager;
import java.sql.SQLException;
import java.sql.Statement;
import java.sql.ResultSet;

@RestController
@RequestMapping("/DatabaseTest")
public class DatabaseTest {
    @RequestMapping("/testDbConn")
    public String testDbConn(String driver, String connUrl, String username, String password) {
        Connection con = null;
        try {
            Class.forName(driver);
            con = DriverManager.getConnection(connUrl, username, password);

            // 用于mysql-connector-java低版本的反序列化测试使用
            // 在“5.x <= 5.1.10”这些版本下需要执行一次SQL语句查询才能进行反序列化
            Statement stmt = con.createStatement();
            ResultSet rs = stmt.executeQuery("SELECT * FROM users");

            if (con != null) {
                return "数据库连接测试成功!";
            } else {
                return "数据库连接测试失败!";
            }
        } catch (Exception e) {
            return "数据库连接异常!" + e;
        } finally {
            if (con != null) {
                try {
                    con.close();
                } catch (SQLException e) {
                    e.printStackTrace();
                }
            }
        }
    }
}
```
