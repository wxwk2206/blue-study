# 1.定义
XML用于标记电子文件使其具有结构性的标记语言，可以用来标记数据、定义数据类型，是一种允许用户对自己的标记语言进行**自定义**的源语言。

# 2.文档结构说明
XML文档结构包括XML声明、DTD文档类型定义（可选）、文档元素。

## 2.1 基本文档结构例子
```xml
<!--XML声明-->
<?xml version="1.0"?> 
<!--文档类型定义-->
<!DOCTYPE note [  <!--定义此文档是 note 类型的文档-->
<!ELEMENT note (to,from,heading,body)>  <!--定义note元素有四个元素-->
<!ELEMENT to (#PCDATA)>     <!--定义to元素为”#PCDATA”类型-->
<!ELEMENT from (#PCDATA)>   <!--定义from元素为”#PCDATA”类型-->
<!ELEMENT head (#PCDATA)>   <!--定义head元素为”#PCDATA”类型-->
<!ELEMENT body (#PCDATA)>   <!--定义body元素为”#PCDATA”类型-->
]]]>
<!--文档元素-->
<note>
<to>Dave</to>
<from>Tom</from>
<head>Reminder</head>
<body>You are a good man</body>
</note>
```

```plain
DTD(文档类型定义)的作用是定义XML文档的合法构建模块。
DTD可以在XML文档内声明,也可以外部引用。
```

## 2.2 DTD(文档类型定义)
### 2.2.1 内部声明DTD
```xml
<!DOCTYPE 根元素 [元素声明]>
```

### 2.2.2 引用外部DTD
```xml
<!DOCTYPE 根元素 SYSTEM "文件名">
<!DOCTYPE 根元素 PUBLIC "DTD名称" “DTD文件的URL”>
```

### 2.2.3 DTD中的一些重要的关键字
+ DOCTYPE(DTD的声明)
+ ENTITY(实体的声明)
+ SYSTEM、PUBLIC(外部资源申请)

# 4.DTD的实体类别介绍 
+ 内部实体
+ 外部实体

## 4.1 举例定义
### 4.1.1 内部实体定义例子
```xml
<!ENTITY 实体名称 "实体的值">
```

**内部实体**：直接在 XML 内部定义，比如：  

```plain
<?xml version="1.0"?>
<!DOCTYPE user [
  <!ENTITY name "张三">  <!-- 内部实体 -->
]>
<user>
  <name>&name;</name>  <!-- 引用实体，最终解析为“张三” -->
</user>
```

### 4.1.2 外部实体定义例子
```xml
<!ENTITY 实体名称 SYSTEM "URI">
```

**外部实体**：从 XML 文件**外部加载内容**，这就是 XXE 漏洞的核心来源，语法：  

```plain
<?xml version="1.0"?>
<!DOCTYPE user [
  <!ENTITY extEntity SYSTEM "file:///etc/passwd">  <!-- 引用本地文件 -->
]>
<user>
  <info>&extEntity;</info>  <!-- 解析外部实体内容 -->
</user>
```
