## 二、核心实现方式
### （一）使用 java.io.File 类
#### 1. 核心逻辑
通过 `File` 类的 `renameTo()` 方法实现文件移动，需先完成前置校验（文件存在性、目标是否为目录、父目录是否存在）。

#### 2. 完整代码
```java
package MoveDirectoryTest;

import java.io.File;

public class FileMoveDirectoryDemo1 {
    public static void main(String[] args) {
        // 源文件：当前目录下的 2.txt
        File f1 = new File("./2.txt");

        // 校验源文件是否存在
        if (!f1.exists()) {
            System.out.println("要移动的文件不存在");
            return;
        }

        // 目标文件：./test 目录下的 2.2.txt
        File f2 = new File("./test/2.2.txt");

        // 校验目标是否为目录（避免覆盖目录）
        if (f2.isDirectory()) {
            System.out.println("移动到目标位置的文件是目录,不能移动");
            return;
        }

        // 获取目标文件的父目录
        String parentPath = f2.getParent();
        File parentDir = new File(parentPath);

        // 父目录不存在则创建（包括多级目录）
        if (!parentDir.exists()) {
            parentDir.mkdirs();
            System.out.println("要移动到目标位置文件的父目录不存在,创建:" + parentPath);
        }

        // 执行文件移动（renameTo() 方法）
        f1.renameTo(f2);
    }
}
```

#### 3. 执行结果
根目录下的 `./2.txt` 被移动到 `./test/` 目录下，并改名为 `2.2.txt`。

### （二）使用 Apache Commons IO 的 FileUtils 类
需导入依赖：`org.apache.commons.io.FileUtils`，提供更简洁的文件 / 目录移动 API，支持目录创建、同名文件处理等场景。

#### 1. 场景 1：移动文件夹并重新命名
+ 功能：直接将源文件夹移动到目标路径，并修改文件夹名称
+ 代码：

```java
package MoveDirectoryTest;

import org.apache.commons.io.FileUtils;
import java.io.File;
import java.io.IOException;

public class FileUtilsMoveDirectoryDemo1 {
    public static void main(String[] args) throws IOException {
        // 源文件夹：./t
        File sourceFile = new File("./t");
        // 目标文件夹：./t2（新名称）
        File targetFile = new File("./t2");

        // 移动文件夹（若目标存在会覆盖，需注意数据安全）
        FileUtils.moveDirectory(sourceFile, targetFile);
    }
}
```

+ 执行结果：`./t` 文件夹被移动并改名为 `./t2/` 文件夹。

#### 2. 场景 2：将目录移动到指定目录下
+ 功能：将源目录作为子目录，移动到目标目录中（目标目录不存在可自动创建）
+ 代码：

```java
package MoveDirectoryTest;

import org.apache.commons.io.FileUtils;
import java.io.File;
import java.io.IOException;

public class FileUtilsMoveDirectoryDemo2 {
    public static void main(String[] args) throws IOException {
        // 源文件夹：./t
        File sourceFile = new File("./t");
        // 目标父目录：./t2
        File targetFile = new File("./t2");

        // 第3个参数 true 表示：目标父目录不存在时自动创建
        FileUtils.moveDirectoryToDirectory(sourceFile, targetFile, true);
    }
}
```

+ 执行结果：`./t` 文件夹被移动到 `./t2/` 目录下，最终路径为 `./t2/t/`。

#### 3. 场景 3：移动文件并重新命名
+ 功能：移动文件到指定路径，同时修改文件名；若目标路径存在同名文件会报错
+ 代码：

```java
package MoveDirectoryTest;

import org.apache.commons.io.FileUtils;
import java.io.File;
import java.io.IOException;

public class FileUtilsMoveDirectoryDemo3 {
    public static void main(String[] args) throws IOException {
        // 源文件：./1.txt
        File sourceFile = new File("./1.txt");
        // 目标文件：./2.txt（新名称，若已存在则抛异常）
        File targetFile = new File("./2.txt");

        // 移动文件并改名
        FileUtils.moveFile(sourceFile, targetFile);
    }
}
```

+ 执行结果：`./1.txt` 被移动并改名为 `./2.txt`（目标文件存在时抛出 IO 异常）。

#### 4. 场景 4：将文件移动到指定目录下
+ 功能：将源文件移动到目标目录中，保持原文件名；目标目录不存在可自动创建
+ 代码：

```java
package MoveDirectoryTest;

import org.apache.commons.io.FileUtils;
import java.io.File;
import java.io.IOException;

public class FileUtilsMoveDirectoryDemo4 {
    public static void main(String[] args) throws IOException {
        // 源文件：./1.txt
        File sourceFile = new File("./1.txt");
        // 目标目录：./t4
        File targetFile = new File("./t4");

        // 第3个参数 true 表示：目标目录不存在时自动创建
        FileUtils.moveFileToDirectory(sourceFile, targetFile, true);
    }
}
```

+ 执行结果：`./1.txt` 被移动到 `./t4/` 目录下，最终路径为 `./t4/1.txt`。

## 三、关键注意事项
1. **File 类的局限**：`renameTo()` 方法依赖操作系统，跨平台兼容性较差；移动大文件或跨磁盘时效率较低。
2. **FileUtils 优势**：API 简洁，无需手动处理父目录创建、跨平台兼容更好，支持更多场景（文件夹移动、文件到目录移动等）。
3. **同名文件处理**：`FileUtils.moveFile()` 若目标文件已存在会抛出 `IOException`；`FileUtils.moveDirectory()` 会覆盖目标目录（需谨慎使用）。
4. **异常处理**：`FileUtils` 的方法均声明抛出 `IOException`，需手动捕获或向上抛出。
5. **依赖导入**：使用 `FileUtils` 需引入 Apache Commons IO 依赖（Maven/Gradle），否则会报类找不到异常。
