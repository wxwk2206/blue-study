## 一、目录结构
```plain
FileDeleteTest/
├── FileDeleteDemo.java       // 基础File类删除文件
├── FileUtilsDeleteDemo1.java // FileUtils递归删除目录（抛异常）
├── FileUtilsDeleteDemo2.java // FileUtils静默删除（不抛异常）
├── FileUtilsDeleteDemo3.java // FileUtils清空目录内容（保留目录）
└── FileUtilsDeleteDemo4.java // FileUtils删除文件/目录（抛异常）
```

## 二、核心实现方式
### （一）使用 Java 原生 File 类
#### 1. 适用场景
删除单个文件（不支持递归删除目录）

#### 2. 代码实现
```java
package FileDeleteTest;
import java.io.File;

public class FileDeleteDemo {
    public static void main(String[] args) {
        // 指定要删除的文件路径
        File f = new File("C:\Users\22063\Desktop\bluek.txt");
        if (f.delete()) {
            System.out.println("删除:" + f.getName());
        } else {
            System.out.println("删除文件失败");
        }
    }
}
```

### （二）使用 Apache Commons-IO 的 FileUtils 工具类
#### 1. 前置依赖（Maven）
需在 `pom.xml` 中添加依赖：

```xml
<!-- https://mvnrepository.com/artifact/commons-io/commons-io -->
<dependency>
  <groupId>commons-io</groupId>
  <artifactId>commons-io</artifactId>
  <version>2.11.0</version>
</dependency>
```

#### 2. 四种常用删除场景

| **实现类** | **核心方法** | **功能描述** | **是否抛异常** | **执行结果示例** |
| --- | --- | --- | --- | --- |
| FileUtilsDeleteDemo1 | `FileUtils.deleteDirectory(f)` | 递归删除目录及所有子内容（文件 / 子目录） | 是 | 删除 `./t`<br/> 目录及所有内容 |
| FileUtilsDeleteDemo2 | `FileUtils.deleteQuietly(f)` | **静默**删除文件 / 目录（目录会递归删除），删除失败不报错 | 否 | 删除 `./t`<br/> 目录及所有内容 |
| FileUtilsDeleteDemo3 | `FileUtils.cleanDirectory(f)` | 仅**清空**目录内容（文件递归删除、子目录递归删除），保留原目录本身 | 是 | 保留 `./t4`<br/> 目录，删除其下所有内容 |
| FileUtilsDeleteDemo4 | `FileUtils.delete(f)` | 通用删除：文件直接删除，目录则递归删除所有内容，支持文件 / 目录两种类型 | 是 | 删除 `./t4`<br/> 目录及所有内容 |


#### 3. 各场景代码实现
##### （1）递归删除目录（抛异常）        FileUtils.deleteDirectory(f)
```java
package FileDeleteTest;
import org.apache.commons.io.FileUtils;
import java.io.File;
import java.io.IOException;

/**
 * 递归删除目录或文件，删除失败会抛出异常
 */
public class FileUtilsDeleteDemo1 {
    public static void main(String[] args) throws IOException {
        File f = new File("./t");
        FileUtils.deleteDirectory(f);
    }
}
```

##### （2）静默删除（不抛异常）        FileUtils.deleteQuietly(f)
```java
package FileDeleteTest;
import org.apache.commons.io.FileUtils;
import java.io.File;

/**
 * 若为目录，删除它和所有子目录；若为文件，直接删除；删除失败不抛异常
 */
public class FileUtilsDeleteDemo2 {
    public static void main(String[] args) {
        File f = new File("./t");
        FileUtils.deleteQuietly(f);
    }
}
```

##### （3）清空目录内容（保留目录）        FileUtils.cleanDirectory(new File("./t4"))
```java
package FileDeleteTest;
import org.apache.commons.io.FileUtils;
import java.io.File;
import java.io.IOException;

/**
 * 清除目录中的内容，不删除原目录；删除失败（文件不存在/无法删除）会抛异常
 */
public class FileUtilsDeleteDemo3 {
    public static void main(String[] args) throws IOException {
        FileUtils.cleanDirectory(new File("./t4"));
    }
}
```

##### （4）通用删除文件 / 目录（抛异常）        FileUtils.delete(f);
```java
package FileDeleteTest;
import org.apache.commons.io.FileUtils;
import java.io.File;
import java.io.IOException;

/**
 * 支持删除文件或目录：文件直接删除，目录递归删除所有内容；删除失败会抛异常
 */
public class FileUtilsDeleteDemo4 {
    public static void main(String[] args) throws IOException {
        File f = new File("./t4");
        FileUtils.delete(f);
    }
}
```

## 三、关键注意事项
1. **异常处理**：
    - 带异常抛出的方法（`deleteDirectory`、`cleanDirectory`、`delete`）需捕获 `IOException` 或声明抛出
    - `deleteQuietly` 无需处理异常，但需自行判断删除结果（返回布尔值）
2. **路径规范**：
    - 相对路径：`./` 表示项目根目录，需确保路径与程序运行目录一致
    - 绝对路径：推荐使用 `new File("D:/test/t")` 避免路径歧义
3. **权限问题**：
    - 若文件 / 目录被其他进程占用，或当前用户无删除权限，会导致删除失败
    - 抛异常的方法会明确提示失败原因，`deleteQuietly` 会静默失败（返回 `false`）
4. **目录删除区别**：
    - 原生 `File.delete()` 只能删除空目录
    - FileUtils 工具类支持非空目录的递归删除，无需手动遍历子文件
