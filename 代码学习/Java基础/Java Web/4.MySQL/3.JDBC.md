# 1.简述
JDBC：Java DataBase Connectivity ，就是使用Java操作关系型数据库的一套API

**本质：**

sun公司官方定义的一套操作所有关系型数据库的规范，即接口

各个数据库厂商去实现这套接口，提供数据库**驱动jar包**

我们使用这套接口（JDBC）编程，真正执行的代码是驱动jar包中的实现类

## 预编译失效场景
预编译不能100%防止SQL注入

**场景一：动态拼接表名，列名**

```java
// 危险：tableName是外部传入的，拼接进SQL结构
String tableName = request.getParameter("table");
String sql = "SELECT * FROM " + tableName + " WHERE id = ?"; // 表名拼接
PreparedStatement pstmt = conn.prepareStatement(sql);
pstmt.setInt(1, 123); // 仅参数值参数化
```

 在其中传入`<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">tableName = "users; DROP TABLE orders; --"</font>`

** 场景 2：参数化后又拼接参数值  **

<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">比如想实现 IN 查询，错误地把多个值拼到一个参数里：</font>

```java
// 危险：先拼接参数值，再参数化
String ids = request.getParameter("ids"); // 攻击者传入：1' OR '1'='1
String sql = "SELECT * FROM users WHERE id IN (?)";
PreparedStatement pstmt = conn.prepareStatement(sql);
pstmt.setString(1, ids); // 传入 "1' OR '1'='1"
```

<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">虽然用了预编译，但如果开发者后续自己拆分这个参数并拼接，依然会有注入风险；更糟的是，如果直接放弃参数化，用</font>`<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">String.format</font>`<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">拼接参数值，等于完全没用预编译。</font>

**<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">场景 3：业务逻辑漏洞导致的间接注入</font>**

<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">预编译只能防护 SQL 语法层面的注入，但如果业务逻辑有漏洞（比如权限校验绕过），即使 SQL 本身安全，攻击者也能通过其他方式达到注入的效果（比如越权查询数据）。</font>

## <font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);"> 正确的防护方式（让预编译发挥最大效果）</font>
<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">要接近 “100% 防护”，需要结合以下几点：</font>

1. **<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">所有动态值都参数化</font>**<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">：仅把 “数据值”（比如 id、用户名、条件值）作为参数传入，绝不拼接；</font>
2. **<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">白名单校验 SQL 结构</font>**<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">：如果必须动态指定表名 / 列名（比如多表查询），先做白名单校验（只允许指定的表名 / 列名）：</font>

```java
// 安全：先校验表名是否在白名单内
String[] allowedTables = {"users", "orders", "products"};
if (!Arrays.asList(allowedTables).contains(tableName)) {
    throw new IllegalArgumentException("非法表名");
}
String sql = "SELECT * FROM " + tableName + " WHERE id = ?";
```

3. **<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">数据库最小权限原则</font>**<font style="color:rgb(0, 0, 0);background-color:rgba(0, 0, 0, 0);">：给应用的数据库账号只分配必要权限（比如只允许 SELECT/INSERT，禁止 DROP/ALTER），即使出现注入，也能降低危害。</font>
