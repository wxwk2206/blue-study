## 1. 抽象类（Abstract Class）

### 1.1 抽象方法

当父类中的一个方法，**每个子类的实现都不同**，父类中无法写出有意义的方法体时，就应该定义为抽象方法。

```java
// 语法：没有方法体，直接用分号结束
public abstract 返回值类型 方法名(参数列表);
```

> [!note] 现实类比
> `Animal` 类有个 `eat()` 方法，但每个动物吃的都不一样。写 `{ System.out.println("吃"); }` 太敷衍 —— 干脆不写方法体，让子类各自实现。

```java
public abstract class Animal {
    public abstract void eat();  // 抽象方法：没有 {}
}
```

### 1.2 抽象类定义

一个类中**有抽象方法**，那这个类**必须是抽象类**。

```java
public abstract class 类名 {
    // 可以有普通成员变量
    // 可以有普通方法
    // 可以有抽象方法
    // 可以有构造方法（供子类 super() 调用）
}
```

### 1.3 抽象类的注意事项

| 规则 | 说明 |
|------|------|
| **不能实例化** | `new Animal()` ❌ —— 抽象类不能被直接创建对象 |
| **抽象方法必须在子类重写** | 子类必须重写所有抽象方法，除非子类也是抽象类 |
| **抽象类中不一定有抽象方法** | 可以有，也可以全是普通方法 |
| **有抽象方法的一定是抽象类** | 反过来说不成立 |
| **可以有构造方法** | 但不能 `new`，只能通过 `super()` 被子类调用 |
| **抽象方法不能用 `private`/`static`/`final`** | 因为它们都会阻止方法被重写 |

```java
public abstract class Animal {
    private String name;

    // 抽象类可以有构造方法
    public Animal(String name) {
        this.name = name;
    }

    // 抽象方法 —— 子类必须实现
    public abstract void eat();

    // 普通方法 —— 子类可直接继承使用
    public void sleep() {
        System.out.println(name + "在睡觉");
    }
}

public class Cat extends Animal {
    public Cat(String name) {
        super(name);  // 调用抽象类的构造方法
    }

    @Override
    public void eat() {  // 必须重写抽象方法
        System.out.println("猫吃鱼");
    }
}
```

## 2. 接口（Interface）

### 2.1 接口的概念

接口是一种**独立于继承体系的规则/规范**。如果说父类是"亲爹"（is-a 关系），那接口就是"干爹"（can-do 关系）——你可以认多个。

```mermaid
classDiagram
    class "《interface》Swim" {
        <<interface>>
        +swim()
    }
    class "《interface》Fly" {
        <<interface>>
        +fly()
    }
    class Animal {
        +eat()
    }
    class Dog {
        +swim()
    }
    class Bird {
        +fly()
        +swim()
    }
    class Duck {
        +fly()
        +swim()
    }
    Animal <|-- Dog
    Animal <|-- Bird
    Animal <|-- Duck
    Swim <|.. Dog : implements
    Fly <|.. Bird : implements
    Fly <|.. Duck : implements
    Swim <|.. Duck : implements
```

### 2.2 接口的定义与实现

```java
// 定义接口
public interface 接口名 {
    // JDK 7 及以前：只能有常量和抽象方法
}

// 实现接口
public class 类名 implements 接口名1, 接口名2, ... {
    // 必须重写所有接口的抽象方法（除非本类是抽象类）
}
```

```java
// 完整示例
public interface Swim {
    void swim();  // 默认是 public abstract
}

public class Dog extends Animal implements Swim {
    @Override
    public void swim() {
        System.out.println("狗刨式游泳");
    }
}

// 接口多实现
public class Frog extends Animal implements Swim, Jump {
    // 必须同时实现两个接口的所有抽象方法
}
```

### 2.3 接口成员的特点（JDK 7）

| 成员 | 默认修饰符 | 说明 |
|------|-----------|------|
| **成员变量** | `public static final` | 只能是常量，必须赋初始值 |
| **构造方法** | 无 | 接口**没有**构造方法 |
| **成员方法** | `public abstract` | 只能是抽象方法（JDK 7） |

```java
public interface MyInterface {
    // 成员变量：默认 public static final，必须赋值
    int MAX_COUNT = 100;  // 等价于：public static final int MAX_COUNT = 100;

    // 抽象方法：默认 public abstract
    void doSomething();   // 等价于：public abstract void doSomething();
}
```

### 2.4 接口与类的关系总结

| 关系 | 关键字 | 数量限制 |
|------|--------|:---:|
| 类与类 | `extends` | 单继承 |
| 类与接口 | `implements` | 多实现 |
| 接口与接口 | `extends` | 多继承 |

```java
interface A { void a(); }
interface B { void b(); }
interface C extends A, B { void c(); }  // 接口可以多继承接口

class MyClass implements C {
    // 必须实现 a(), b(), c() 三个方法
}
```

### 2.5 JDK 8+ 接口新特性

JDK 8 开始，接口可以包含带方法体的方法了——因为如果接口升级加新方法，所有实现类都得跟着改，这不现实。

#### 默认方法（default method）

```java
public interface MyInterface {
    // 默认方法：有方法体，实现类可以直接用，也可以重写
    public default void show() {
        System.out.println("接口中的默认方法");
    }
}

// 实现类可以选择重写，也可以不重写直接用
public class MyClass implements MyInterface {
    // show() 可以不重写，直接使用接口中的默认实现
}
```

> [!warning] 默认方法冲突
> 如果一个类实现了两个接口，而两个接口有**同名默认方法**，类必须自己重写解决冲突：
> ```java
> interface A { default void hello() { ... } }
> interface B { default void hello() { ... } }
>
> class C implements A, B {
>     @Override
>     public void hello() {
>         A.super.hello();  // 指定调用哪个接口的默认方法
>     }
> }
> ```

#### 静态方法（static method）

```java
public interface MyInterface {
    public static void utilMethod() {
        System.out.println("接口中的静态方法");
    }
}

// 调用：只能用接口名调用，不能用实现类对象调用
MyInterface.utilMethod();  // ✅
```

### 2.6 JDK 9+ 接口新特性：私有方法

```java
public interface MyInterface {
    // 供默认方法调用的私有方法
    private void commonLogic() {
        System.out.println("共用的内部逻辑");
    }

    // 供静态方法调用的私有静态方法
    private static void staticCommonLogic() {
        System.out.println("共用的静态内部逻辑");
    }

    default void methodA() {
        commonLogic();  // 调用私有方法
        System.out.println("方法A");
    }

    default void methodB() {
        commonLogic();  // 复用代码
        System.out.println("方法B");
    }
}
```

> [!tip] 为什么需要私有方法？
> 多个默认方法有重复代码时，抽成 `private` 方法——跟普通类的封装一个道理，只是以前接口不支持而已。

### 2.7 接口版本特性速查表

| JDK 版本 | 新增特性 |
|----------|----------|
| **JDK 7 及以前** | 常量 + 抽象方法 |
| **JDK 8** | `default` 方法（默认方法）、`static` 方法（静态方法） |
| **JDK 9** | `private` 方法、`private static` 方法 |

### 2.8 抽象类 vs 接口

| 对比维度 | 抽象类 (abstract class) | 接口 (interface) |
|----------|------------------------|-------------------|
| 关键字 | `abstract class` | `interface` |
| 成员变量 | 无限制 | 只能是 `public static final` 常量 |
| 构造方法 | 有 | 无 |
| 方法 | 可以有抽象方法，也可以有普通方法 | JDK 7：只有抽象方法<br/>JDK 8+：可加 default/static<br/>JDK 9+：可加 private |
| 继承/实现 | 单继承 | 多实现 |
| 设计理念 | "是什么" (is-a) — 共性抽取 | "能做什么" (can-do) — 行为规范 |
| 应用场景 | 模板方法模式、有状态的共性 | 解耦、定义 API 规范、多态 |

## 3. 内部类（Inner Class）

### 3.1 概述

内部类：在一个类的里面再定义一个类。

```java
public class Outer {
    // 外部类成员

    class Inner {
        // 内部类成员
    }
}
```

四种内部类：

| 类型 | 定义位置 | 重要程度 |
|------|----------|:---:|
| 成员内部类 | 类的成员位置 | 了解 |
| 静态内部类 | 类的成员位置 + `static` | 了解 |
| 局部内部类 | 方法内部 | 了解 |
| 匿名内部类 | 特殊的局部内部类 | ==重点== |

### 3.2 成员内部类

**特点：** 写在成员位置，属于外部类的成员，可以用 `private`、`default`、`protected`、`public`、`static` 修饰。

> [!note] JDK 版本注意
> JDK 16 以前，成员内部类**不能定义静态变量**；JDK 16 开始可以了。

#### 创建成员内部类对象

**方式一：** 外部类提供方法，对外返回内部类对象（适用于内部类被 `private` 修饰）

```java
public class Outer {
    private class Inner {
        public void show() {
            System.out.println("内部类方法");
        }
    }

    // 对外暴露的方法
    public Inner getInner() {
        return new Inner();
    }
}

// 使用
Outer o = new Outer();
Outer.Inner i = o.getInner();  // 通过方法获取
i.show();
```

**方式二：** 直接创建（内部类非 private 时）

```java
Outer.Inner oi = new Outer().new Inner();
```

#### 成员内部类访问外部类成员

存在同名变量时的区分：

```java
public class Outer {
    int a = 10;  // 外部类的成员变量

    class Inner {
        int a = 20;  // 内部类的成员变量

        public void show() {
            int a = 30;  // 局部变量

            System.out.println(a);            // 30：局部变量（最近）
            System.out.println(this.a);       // 20：内部类成员变量
            System.out.println(Outer.this.a); // 10：外部类成员变量
        }
    }
}
```

> [!tip] 访问层级
> ```
> a               → 局部变量
> this.a          → 内部类的成员变量
> Outer.this.a    → 外部类的成员变量
> ```

### 3.3 静态内部类

成员内部类前面加 `static`：

```java
public class Outer {
    static class Inner {
        public void show() {
            System.out.println("静态内部类");
        }
    }
}

// 创建对象（不需要外部类对象）
Outer.Inner oi = new Outer.Inner();
oi.show();
```

> [!important] 静态内部类的访问限制
> 静态内部类只能直接访问外部类的**静态成员**。要访问非静态成员，需要先创建外部类对象。

```java
public class Outer {
    int a = 10;              // 实例变量
    static int b = 20;       // 静态变量

    static class Inner {
        public void show() {
            // System.out.println(a);  // ❌ 不能直接访问实例变量
            System.out.println(b);     // ✅ 可以访问静态变量

            // 访问实例变量的方式：先创建外部类对象
            Outer o = new Outer();
            System.out.println(o.a);   // ✅
        }
    }
}
```

### 3.4 局部内部类

定义在方法里面的类，类似于方法的局部变量。

```java
public class Outer {
    public void method() {
        // 局部内部类
        class LocalInner {
            public void show() {
                System.out.println("局部内部类");
            }
        }

        // 只能在方法内部使用
        LocalInner li = new LocalInner();
        li.show();
    }
}
```

> [!note] 局部内部类特点
> - 外界无法直接使用，只能在方法内部创建对象
> - 可以直接访问外部类的成员
> - 可以访问方法内的局部变量（JDK 8+ 局部变量自动加 `final`）

### 3.5 匿名内部类（重点）

匿名内部类：**没有名字的内部类**，本质是"继承一个类或实现一个接口 + 重写方法 + 创建对象"——一步到位。

```java
new 类名或接口名() {
    // 重写方法
};
```

**使用场景：** 某个类/接口只需要用一次，不想单独写一个文件。

```java
// 场景：有一个 Swim 接口
public interface Swim {
    void swim();
}

// 传统做法：写一个实现类 → 创建对象 → 调用
// class Dog implements Swim { ... }  ← 需要单独一个文件，麻烦

// 匿名内部类做法：一步到位
public class Test {
    public static void main(String[] args) {
        // 创建匿名内部类对象，同时重写方法
        Swim s = new Swim() {
            @Override
            public void swim() {
                System.out.println("匿名内部类：正在游泳");
            }
        };

        s.swim();  // 输出：匿名内部类：正在游泳
    }
}
```

**完整剖析匿名内部类的本质：**

```java
// 这行代码等价于同时做了四件事：
new Swim() {                    // 1. new → 创建对象
    @Override                  // 2. 继承/实现 Swim 接口
    public void swim() {       // 3. 重写方法
        ...
    }
};                             // 4. 返回该对象

// ↑ 就是一个 "没有名字的 Java类 + 继承/实现 + 重写方法 + 创建对象" 的合体
```

> [!tip] 匿名内部类是 Lambda 的前置知识
> 后面学 Lambda 表达式时，你会发现 `() -> { ... }` 其实就是匿名内部类的简化版。
> ```java
> // 匿名内部类
> Swim s1 = new Swim() {
>     @Override
>     public void swim() { System.out.println("游泳"); }
> };
>
> // Lambda 化简（前提：接口只有一个抽象方法，即函数式接口）
> Swim s2 = () -> System.out.println("游泳");
> ```
