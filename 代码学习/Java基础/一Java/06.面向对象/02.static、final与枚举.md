## 1. static 关键字

`static` 表示**静态**，是 Java 的修饰符，用来修饰成员变量和成员方法。

### 1.1 static 修饰成员变量（静态变量 / 类变量）

**特点：**

- 被该类**所有对象共享**，不属于某个单独对象，而是属于类本身
- 随着类的加载而加载，优先于对象存在
- 只初始化一次，存储在方法区的静态区

**调用方式：**

| 方式 | 语法 | 推荐度 |
|------|------|--------|
| 类名调用 | `类名.静态变量名` | ==推荐== |
| 对象名调用 | `对象名.静态变量名` | 不推荐（容易误导） |

**代码示例：**

```java
public class Student {
    private String name;        // 实例变量：每个对象各有一份
    public static String teacher;  // 静态变量：所有对象共享
}

// 使用
Student.teacher = "王老师";  // 推荐：类名直接调用

Student s1 = new Student();
Student s2 = new Student();
s1.teacher = "李老师";
System.out.println(s2.teacher);  // 输出：李老师（共享同一份数据）
```

> [!note] 内存结构（静态变量 vs 实例变量）
> ```
> ┌──────── 堆内存 ────────┐     ┌────────── 方法区（静态区）──────────┐
> │                        │     │                                     │
> │ s1 → Student           │     │  Student.teacher = "李老师"          │
> │      name = "张三"      │     │                                     │
> │                        │     │  ← s1 和 s2 都指向同一个 teacher 值   │
> │ s2 → Student           │     │                                     │
> │      name = "李四"      │     │                                     │
> └────────────────────────┘     └─────────────────────────────────────┘
> ```

### 1.2 static 修饰成员方法（静态方法）

**特点：**

- 多用在**测试类**和**工具类**中（如 `Arrays`、`Collections`、`Math`）
- JavaBean 类中很少用静态方法（描述对象行为的方法不该是静态的）

**调用方式：**

| 方式 | 语法 | 推荐度 |
|------|------|--------|
| 类名调用 | `类名.静态方法名()` | ==推荐== |
| 对象名调用 | `对象名.静态方法名()` | 不推荐 |

### 1.3 静态方法访问规则

> [!important] 核心规则
> **静态方法中只能直接访问静态成员，不能直接访问非静态成员。**

| 场景 | 能否直接访问 | 解决方案 |
|------|:---:|----------|
| 静态方法 → 静态变量 | ✅ 可以 | 直接访问 |
| 静态方法 → 静态方法 | ✅ 可以 | 直接调用 |
| 静态方法 → 实例变量 | ❌ 不行 | 创建对象后通过对象访问 |
| 静态方法 → 实例方法 | ❌ 不行 | 创建对象后通过对象调用 |
| 实例方法 → 静态变量 | ✅ 可以 | 直接访问 |
| 实例方法 → 实例变量 | ✅ 可以 | 直接访问 / `this` |
| 静态方法中使用 `this` | ❌ 不行 | `this` 代表当前对象，静态方法没有对象上下文 |

```java
public class Demo {
    // 实例变量
    private String name = "张三";

    // 静态变量
    public static int count = 0;

    // 实例方法
    public void show() {
        System.out.println(name);   // ✅ 实例方法访问实例变量
        System.out.println(count);  // ✅ 实例方法访问静态变量
    }

    // 静态方法
    public static void method() {
        // System.out.println(name);  // ❌ 编译报错！静态方法不能直接访问非静态变量

        // ✅ 正确做法：创建对象后再访问
        Demo d = new Demo();
        System.out.println(d.name);

        System.out.println(count);  // ✅ 直接访问静态变量
    }
}
```

> [!tip] 为什么静态不能访问非静态？
> 静态成员随类加载就已存在，而实例成员要等到 `new` 对象才出现。先来的访问还没出生的——这不是为难人家嘛。

## 2. final 关键字

`final` 表示**最终、不可变**，可以修饰==类==、==方法==、==变量==。

### 2.1 final 修饰变量

| 变量类型 | 含义 | 示例 |
|----------|------|------|
| **局部变量** | 基本类型：值不可变；引用类型：地址不可变（对象内容可变） | `final int a = 10;` |
| **成员变量** | 必须手动赋值，且不能二次赋值 | `final String name = "固定";` |

```java
// 基本类型 final 变量
final int num = 10;
// num = 20;  // ❌ 编译错误！final 变量不可修改

// 引用类型 final 变量
final Student s = new Student("张三");
// s = new Student("李四");  // ❌ 编译错误！地址不可变
s.setName("李四");           // ✅ 地址不变，对象内容可以改
```

> [!warning] final 成员变量必须赋值
> ```java
> public class Demo {
>     final int a;      // ❌ 编译错误！成员变量没有默认值，必须手动赋值
>     final int b = 10; // ✅ 直接赋值
>
>     final int c;
>     public Demo() {
>         c = 20;       // ✅ 也可以在构造方法中赋值
>     }
> }
> ```

### 2.2 final 修饰方法

- final 方法**不能被重写（Override）**
- 适用于：方法逻辑已固定，不允许子类修改的场景

```java
public class Parent {
    public final void connect() {
        System.out.println("固定的连接逻辑");
    }
}

public class Child extends Parent {
    // @Override
    // public void connect() { ... }  // ❌ 编译错误！final方法不可重写
}
```

### 2.3 final 修饰类

- final 类**不能被继承**
- 典型的 final 类：`String`、`Integer`、`Math`

```java
public final class FinalClass { ... }

// public class Sub extends FinalClass { ... }  // ❌ 编译错误！
```

### 2.4 final 修饰符速查表

| 修饰对象 | 效果 | 典型场景 |
|----------|------|----------|
| 基本类型变量 | 值不可改（常量） | `final double PI = 3.14;` |
| 引用类型变量 | 引用地址不可改 | 固定绑定的配置对象 |
| 方法 | 不可被子类重写 | 模板方法模式的骨架方法 |
| 类 | 不可被继承 | `String`、工具类 |

## 3. 枚举（Enum）

> 当某个类的对象数量是**有限且固定**的（如星期、季节、订单状态），就不适合用普通 JavaBean —— 这时候用枚举。

### 3.1 为什么需要枚举

以订单状态为例，只有固定的六种状态：

| 状态常量 | 含义 |
|----------|------|
| `PENDING_PAYMENT` | 待付款 |
| `PENDING_DELIVERY` | 待发货 |
| `PENDING_RECEIPT` | 待收货 |
| `PENDING_EVALUATION` | 待评价 |
| `COMPLETED` | 已完成 |
| `CANCELLED` | 已取消 |

如果没有枚举，传统做法是用一堆常量：

```java
// ❌ 不好：常量可被随意赋值，类型不安全
public static final int PENDING_PAYMENT = 1;
public static final int PENDING_DELIVERY = 2;
// ...
int status = 999;  // 不会报错！但 999 不是有效状态
```

### 3.2 枚举定义格式

```java
public enum 枚举类名 {
    枚举项1(参数), 枚举项2(参数), ..., 枚举项N(参数);

    // 属性
    private 类型 变量名;

    // 构造方法（必须 private）
    private 枚举类名(参数) { ... }

    // 行为
    public 返回值 方法名() { ... }
}
```

### 3.3 完整示例

```java
public enum OrderStatus {
    PENDING_PAYMENT("待付款"),
    PENDING_DELIVERY("待发货"),
    PENDING_RECEIPT("待收货"),
    PENDING_EVALUATION("待评价"),
    COMPLETED("已完成"),
    CANCELLED("已取消");
    //                     ↑ 枚举项之间用逗号，最后一个用分号结束

    private String description;

    // 构造方法必须用 private（可省略，默认就是 private）
    private OrderStatus(String description) {
        this.description = description;
    }

    public String getDescription() {
        return description;
    }
}

// 使用示例
public class Test {
    public static void main(String[] args) {
        OrderStatus status = OrderStatus.PENDING_PAYMENT;
        System.out.println(status.getDescription());  // 输出：待付款

        // 遍历所有枚举值
        for (OrderStatus s : OrderStatus.values()) {
            System.out.println(s + " - " + s.getDescription());
        }

        // 根据名称获取枚举
        OrderStatus completed = OrderStatus.valueOf("COMPLETED");
    }
}
```

### 3.4 枚举的本质与注意事项

> [!important] 枚举的本质
> 枚举是一个**特殊的 Java 类**，编译器会把它编译成 `final class Xxx extends Enum`。

| 规则 | 说明 |
|------|------|
| 枚举项默认修饰符 | `public static final` — 所以枚举项本身就是该类的**常量对象** |
| 第一行必须是枚举项 | 枚举项之间用逗号分隔，最后以分号结尾 |
| 构造方法 | 必须是 `private`，防止外界创建新对象 |
| 编译器自动生成的方法 | `values()` — 获取所有枚举项的数组；`valueOf(String)` — 根据名称获取指定枚举项 |
| 不能继承其他类 | 枚举已经隐式继承了 `java.lang.Enum`（Java 单继承） |
| 可以实现接口 | 枚举可以实现多个接口 |

> [!tip] 枚举 vs 常量类
> 枚举是类型安全的——你传 `OrderStatus` 类型参数，别人就没法传个 `999` 糊弄你。用 `int` 常量？那就像在高速公路上骑自行车——不出事是运气好。
