# 1.函数定义
函数是一段可重复使用的代码块，接收输入（参数），执行操作，返回输出（返回值）。
```js
// ⭐ 一个函数的完整结构
function add(a, b) {     // function 关键字 + 名字 + (参数列表)
  return a + b;          // 函数体 + return 返回值
}

// 调用：名字 + (实际参数)
const result = add(3, 5);
console.log(result);     // → 8
```
# 2.三种声明函数的方式
## 2.1 函数声明（Function Declaration）
```js
// 最传统的方式，用 function 关键字
function greet(name) {
  return 'Hello, ' + name + '!';
}

console.log(greet('bluek'));  // → Hello, bluek!
```

## 2.2 函数表达式（Function Expression）
```js
// 把一个匿名函数赋值给变量
const greet = function(name) {
  return 'Hello, ' + name + '!';
};

console.log(greet('bluek'));  // → Hello, bluek!

// ⚠️ 注意末尾有分号！因为是赋值语句
```

## 2.3 箭头函数（Arrow Function）— ES6 推荐
```js
// 箭头函数是函数表达式的简写
const greet = (name) => {
  return 'Hello, ' + name + '!';
};

// ⭐ 简写：只有一个参数时可省略括号；函数体只有 return 时可省略 {} 和 return
const greet = name => 'Hello, ' + name + '!';

// 用模板字符串更优雅
const greet = name => `Hello, ${name}!`;

console.log(greet('bluek'));  // → Hello, bluek!
```

| 方式    | 语法                       | 特点            |
| ----- | ------------------------ | ------------- |
| 函数声明  | function fn() {}         | 有名字，可被**提升**  |
| 函数表达式 | const fn = function() {} | 匿名，不会提升       |
| 箭头函数  | const fn = () => {}      | 最简洁，this 行为不同 |
# 3.作用域（Scope）— 重点
作用域是变量可见的范围。一个变量在哪些地方能被访问，由作用域决定。
## 3.1 三种作用域

| 类型    | 谁创建的                 | 变量能被谁看见    |
| ----- | -------------------- | ---------- |
| 全局作用域 | 在最外层写的代码             | 所有人都能访问    |
| 函数作用域 | function 内部          | 只有函数内部能访问  |
| 块级作用域 | {} 花括号内（用 let/const） | 	只有花括号内能访问 |
## 3.2 代码对比
```js
// ① 全局变量
const globalVar = '我是全局的';

function test() {
  // ② 函数作用域变量
  const funcVar = '我是函数内的';

  console.log(globalVar);  // ✅ 能访问全局
  console.log(funcVar);    // ✅ 能访问自己

  if (true) {
    // ③ 块级作用域变量
    const blockVar = '我是块内的';
    console.log(blockVar);  // ✅ 块内能访问
  }

  console.log(blockVar);    // ❌ ReferenceError! 块外访问不到
}

test();
console.log(funcVar);        // ❌ ReferenceError! 函数外访问不到
```

## 3.3 作用域链（Scope Chain）
函数内部访问变量时，会按照从内到外的顺序查找
```js
const outer = '外层';

function middle() {
  const mid = '中层';

  function inner() {
    const inr = '内层';
    // 查找顺序：inr → mid → outer → 全局
    console.log(inr, mid, outer);  // ✅ 全都能访问
  }

  inner();
  console.log(inr);  // ❌ 内层的访问不到
}

middle();
```

# 4.var 的坑：变量提升
在 let 和 const 出现之前（ES6 之前），JS 只能用 var 声明变量。var 有两个臭名昭著的特性，至今仍在坑人：
## 4.1 变量提升（Hoisting）
```js
// ⚠️ var 声明的变量会被"提升"到作用域顶部
console.log(myVar);   // → undefined（不报错！）
var myVar = 10;
console.log(myVar);   // → 10

// 等价于 JS 引擎实际执行的：
var myVar;             // 提升到顶部，值为 undefined
console.log(myVar);    // → undefined
myVar = 10;
console.log(myVar);    // → 10
```
对比 let 的行为：
```js
console.log(myLet);   // ❌ ReferenceError! 暂时性死区
let myLet = 10;
```
## 4.2 var 没有块级作用域
```js
// ⚠️ var 声明的变量会"泄漏"到外层
if (true) {
  var leaked = '我跑出去了';
  let safe = '我只在块内';
}

console.log(leaked);  // → '我跑出去了'  ⚠️ var 变量泄漏！
console.log(safe);    // ❌ ReferenceError，let 有块级作用域
```

## 4.3安全方向⭐
老代码里大量用 var，变量提升和泄漏会导致意外的变量覆盖。比如：
```js
// 假设这是某个权限检查代码
function checkAdmin(user) {
  if (user.role === 'admin') {
    var isAllowed = true;
  }
  // ⚠️ 即使 user.role 不是 admin，
  // isAllowed 依然存在（值为 undefined）
  // 某些 if (isAllowed) 的判断可能被绕过
  return isAllowed;
}
```
如果用 let，isAllowed 只在 if 块内存在，更安全。

> [!important]  现代规则：
> 永远用 let 和 const，不用 var
> 读老代码遇到 var 时，脑内把它替换成"可能泄漏的变量"警惕一下
> const 优先，需要改的才用 let

# 5.案例：函数作用域的妙用
```js
// ⭐ IIFE：立即调用函数表达式（Immediately Invoked Function Expression）
// 老代码里用来创建私有作用域，避免污染全局
(function() {
  const privateVar = '别人访问不到我';
  console.log(privateVar);
})();

console.log(privateVar);  // ❌ ReferenceError

// 现代 JS 用块级作用域替代
{
  const privateVar = '别人也访问不到我';
  console.log(privateVar);
}
console.log(privateVar);  // ❌ ReferenceError
```
**逆向视角**：混淆代码中大量使用 IIFE 来隐藏逻辑、避免变量冲突。看到 (function(){...})() 这种结构，要知道它是立即执行的匿名函数。
