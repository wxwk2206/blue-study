##  抽象类
### 抽象方法
将共性的行为（方法）抽取到父类之后，因为每一个子类的方法体是不一样的，所有，在父类中**不能明确具体的方法体，**该方法就可以定义为抽象方法

```java
public abstract 返回值类型 方法名（参数列表）;
```

有abstract（抽象）关键字，子类方法必须重写（强制让子类按照某种格式重写）

### 抽象类
如果一个**类中存在抽象方法**，那么该类就**必须声明为抽象类**

```java
public abstract class 类名{...}
```

## 接口
<!-- 这是一张图片，ocr 内容为： -->
![](https://cdn.nlark.com/yuque/0/2026/png/62702379/1771942980057-bf7c7e80-a244-456b-b9d6-8ecb36b84e97.png)

接口：接口就是一个规则，而且是独立于继承体系以外的规则。（可以理解为干爹）

```java
public interface 接口名{...}
```

使用：接口和类之间的是实现关系，通过implements关键字表示

```java
public class 类名 implements 接口{...}
```

注意点：

+ 接口不能实例化
+ 接口的子类（实现类），要么重写接口中所有的抽象方法，要么实现类是一个抽象类
+ 一个类可以实现多个接口，也可以在实现一个继承的同时，实现多个接口

```java
public class 类名 implements 接口1，接口2{...}
public class 类名 extends 父类 implements 接口1，接口2{...}
```

### 接口中成员的特点
成员变量：只能是常量，默认修饰符：**public static final**

构造方法：没有

成员方法：只能是抽象方法。默认修饰符：**public abstract**

## 内部类
内部类：在一个类的里面，再定义一个类

<!-- 这是一张图片，ocr 内容为： -->
![](https://cdn.nlark.com/yuque/0/2026/png/62702379/1771949576828-b6258148-838e-4f12-888b-f5274b51876f.png)

内部类分为四种：成员内部类，静态内部类，局部内部类，匿名内部类

### 成员内部类（了解）
写在成员位置的，属于外部类的成员

成员内部类可以被一些修饰符所修饰，比如：private，默认，protected，public，static等

在成员内部类里面，jdk16以前不能定义静态变量，jdk16开始才可以定义静态变量

#### 获取成员内部类对象
1. 外部类编写方法，对外提供内部类对象（内部类被private修饰时，不能直接创建对象）

2. 直接创建

外部类名.内部类名 对象名 = 外部类对象.内部类对象；

Outer.Inner oi = new Outer（）.new Inner（）；

### 局部内部类（了解）
将内部类定义在方法里面就叫做局部内部类，类似于方法里面的局部变量

外界无法直接使用，需要在方法内部创建对象并使用

该类可以直接访问外部类的成员，也可以访问方法内的局部变量

### 匿名内部类（重点）
lambda表达式   方法引用

匿名内部类：就是一个没有名字的内部类

作用：在继承一个类或实现一个接口的时候**少写一个文件**，也是lambda的前置知识

```java
new 类名或接口名（）{
    重写方法;
};
```

 匿名内部类定义格式 = 没有名字的Java类+继承/实现 +重写方法+创建对象，是一个没有名字的Java类的对象
