## 一、结构体基础（day8-1）

### 1.1 概念

**结构体（Structure / struct）**：把若干个**不同数据类型**的变量打包在一起，形成一个复合类型。

> [!important] 细节
> 与数组对比：数组要求**同种类型**的连续存储；结构体允许**不同类型**字段共存，是 C 语言实现"面向对象思想"的底层基础。

| 特性     | 数组             | 结构体                |
| -------- | ---------------- | --------------------- |
| 字段类型 | 必须相同          | 可不同                |
| 访问方式 | `arr[i]` 下标    | `stu.name` 点运算符   |
| 内存布局 | 连续等长          | 按字段对齐填充         |
| 典型用途 | 同类数据集合      | 实体属性描述           |

### 1.2 定义语法

```c
#include <stdio.h>
#include <string.h>

// ⭐ 定义结构体类型 struct stu
struct stu{
    char sex;        // 性别（0男 / 1女）
    char name[16];   // 姓名（字符串数组）
    int age;         // 年龄
    float score;     // 成绩
};
```

> [!warning] 注意
> `struct stu` 是**类型名**，不是变量名。声明变量时必须带 `struct` 关键字：`struct stu arr[4];`，不能直接写 `stu arr[4];`（C++ 才允许）。

### 1.3 结构体数组与初始化

```c
int main()
{
    struct stu arr[4];   // ⭐ 结构体数组：一次装 4 个学生

    // ⭐ 字段赋值：char 数组用 strcpy，标量直接赋值
    arr[0].sex = 0;
    strcpy(arr[0].name, "青龙");   // 字符串赋值必须用 strcpy
    arr[0].age = 21;
    arr[0].score = 77.5f;

    arr[1].sex = 0;
    strcpy(arr[1].name, "白虎");
    arr[1].age = 21;
    arr[1].score = 90.5f;

    // ... 玄武、朱雀同理

    // ⭐ for 循环遍历结构体数组
    int i;
    for(i = 0; i < 4; i++)
    {
        printf("sex: %d\n", arr[i].sex);
        printf("name: %s\n", arr[i].name);
        printf("age: %d\n", arr[i].age);
        printf("score: %.1f\n", arr[i].score);
        printf("----------------\n");
    }
    return 0;
}
```

> [!bug] 常见错误
> `arr[0].name = "青龙";` ❌ 编译报错——数组名是地址常量，不能赋值。必须用 `strcpy(arr[0].name, "青龙");`。

## 二、typedef 与内存对齐（day8-2）

### 2.1 typedef 给结构体起别名

```c
#include <stdio.h>

// ⭐ typedef 起别名：之后可直接用 __stu 声明，省去 struct 关键字
typedef struct stu{
    char name[16];
    char a;
    int age;
    float score;
}__stu;

int main()
{
    struct stu li;   // 原始写法
    __stu hua;       // ⭐ 别名写法，等价于 struct stu hua;

    printf("sizeof(li)： %d \n", sizeof(li));
    printf("sizeof(hua)： %d \n", sizeof(hua));
    return 0;
}
```

> [!tip] typedef 的本质
> `typedef` 不是定义新类型，而是给**已有类型**起一个新名字（别名）。
> `struct stu` 和 `__stu` 在编译器眼里**完全等价**，sizeof 结果相同。

### 2.2 内存对齐原理

> [!abstract] 原理
> 结构体大小**不是字段简单相加**，而是按**对齐粒度**（默认 4 字节）进行填充。

| 字段     | 类型   | 占用字节 | 偏移量 |
| -------- | ------ | -------- | ------ |
| name[16] | char[] | 16       | 0–15   |
| a        | char   | 1        | 16     |
| (填充)   | —      | 3        | 17–19  |
| age      | int    | 4        | 20–23  |
| score    | float  | 4        | 24–27  |
| **合计** | —      | **28**   | —      |

> [!important] 关键规则
> - 每个字段的起始偏移量必须是其自身大小与对齐粒度中**较小者**的整数倍
> - `int` 占 4 字节，跟在 `char a`（1 字节）后面，需要补 3 字节填充
> - 结构体总大小必须是对齐粒度的整数倍

### 2.3 字段顺序的影响

```c
// 写法 A：char 紧跟 char[16]，浪费 3 字节填充
typedef struct stu {
    char name[16];
    char a;        // ← 这后面要补 3 字节
    int age;
    float score;
} __stu;           // sizeof = 28

// 写法 B：调整顺序，可能更省空间
typedef struct stu2 {
    char name[16];
    int age;       // 16 是 4 的倍数，无需填充
    char a;        // 末尾，按 4 字节对齐补 3 字节
    float score;
} __stu2;          // sizeof = 28（这个例子凑巧相同）
```

> [!tip] 工程经验
> 实际开发中，**按字段大小从大到小排列**可减少填充浪费。在嵌入式/网络协议场景下尤其重要。

## 三、结构体指针与函数封装（day8-3）

### 3.1 `.` 与 `->` 的区别

| 操作符 | 使用对象       | 示例              |
| ------ | -------------- | ----------------- |
| `.`    | 结构体**变量** | `li.name`         |
| `->`   | 结构体**指针** | `pstu->name`      |

```c
struct stu{
    char name[16];
    int age;
    float score;
};

// ⭐ 形参是结构体指针 → 内部用 -> 访问
void search(struct stu* pstu)
{
    if((pstu->name != NULL) && (pstu->score != 0))
    {
        printf("name: %s \n", pstu->name);   // 等价于 (*pstu).name
        printf("age: %d \n", pstu->age);
        printf("score: %.1f \n", pstu->score);
    }
    else
    {
        printf("this stu is not exist \n");
    }
}
```

> [!note] 说明
> `pstu->name` 是 `(*pstu).name` 的语法糖，写起来更简洁。两者**完全等价**。

### 3.2 增删改查 函数封装

将"增查改删"封装成函数，主调函数传结构体地址即可操作：

```c
// ⭐ Create：创建（写入数据）
void create(struct stu* pli, char* cname, int cage, float cscore)
{
    strcpy(pli->name, cname);   // 字符串用 strcpy
    pli->age = cage;
    pli->score = cscore;
}

// ⭐ Read：查询（search 函数见 3.1）

// ⭐ Update：修改
void modify(struct stu* pli, float cscore)
{
    pli->score = cscore;
}

// ⭐ Delete：逻辑删除（清零字段）
void del(struct stu* pli)
{
    memset(pli->name, 0, sizeof(pli->name));   // ⭐ memset 清零字符串
    pli->age = -1;                              // 用 -1 标记"已删除"
    pli->score = 0;
}
```

### 3.3 值传递 vs 地址传递

```c
int main()
{
    struct stu li, hua, han;

    // ⭐ 传地址：函数内部修改直接影响实参
    create(&li, "lilei", 20, 90.5);
    create(&hua, "lihua", 21, 87.5);
    create(&han, "hanmm", 22, 76.5);

    search(&li);          // 查询 li
    search(&han);         // 查询 han

    modify(&han, 92.5);   // 修改 han 的成绩
    search(&han);         // 再次查询，可见修改生效

    del(&hua);            // 删除 hua
    search(&hua);         // 查询失败：this stu is not exist

    return 0;
}
```

> [!important] 关键结论
> - 传**结构体变量本身** → 函数内拿到的是副本，修改不影响原数据，且大结构体拷贝开销大
> - 传**结构体指针**（地址） → 函数内直接操作原数据，效率高，是 C 语言的标准做法
>
> 这与 [[03. 指针]] 中"函数参数值传递"的本质一致——指针本身也是值传递，但传的是地址，所以能改到原数据。

> [!bug] `memset` 误用警告
> `memset(pli, 0, sizeof(pli))` ❌ 这里 `pli` 是指针，`sizeof` 得到的是指针大小（4 或 8 字节），不是结构体大小。应写 `sizeof(*pli)` 或对每个字段单独清零。

## 四、文件操作与数据持久化（day8-4）

**概括**：这段代码用**结构体 + 文件 I/O** 实现了一个极简版"学生成绩管理"：往 `stu.txt` 里写 6 条记录，再按姓名查回其中一条。核心是 `fopen / fwrite / fgets / fclose` 四个标准库函数的组合应用。

比喻：文件就像一个"硬盘上的笔记本"：`fopen` 是翻开本子（决定是写新内容还是读旧内容），`fwrite` 是动笔写，`fgets` 是逐行读，`fclose` 是合上本子。不合上就走的，数据可能丢页。

### 4.1 整体代码

> [!important] 细节
> 代码分为 5 个模块：结构体定义、整数转字符串 `myitoa`、浮点转字符串 `ftoa`、写一条记录 `record2file`、按名查找 `search_stu`，外加一个被 `#if 0` 屏蔽的基础 demo。

```c
#include <stdio.h>      // ⭐ 文件操作函数 fopen/fwrite/fclose/fgets
#include <string.h>     // ⭐ strlen/strstr 字符串处理
#include <stdlib.h>     // 标准库（本例未直接用，预留）

struct stu{
    char name[16];      // 姓名（最多 15 字符 + '\0'）
    int age;            // 年龄
    float score;        // 成绩
};

// ⭐ 模块 1：手动实现 itoa —— 两位整数转字符串
void myitoa(int i, char *a)
{
    *a = i / 10 + 48;        // 十位：除以 10，+48 转 ASCII 字符
    *(a+1) = i % 10 + 48;    // 个位：取余 10，+48 转 ASCII 字符
}

// ⭐ 模块 2：手动实现 ftoa —— 形如 90.5 的浮点数转字符串
void ftoa(float f, char *a)
{
    int tmp = f * 10;            // 放大 10 倍去掉小数点（90.5 → 905）

    *a = tmp / 100 + 48;         // 十位（905/100=9 → '9'）
    *(a+1) = (tmp / 10) % 10 + 48;  // 个位（90%10=0 → '0'）
    *(a+2) = '.';                // 小数点
    *(a+3) = tmp % 10 + 48;      // 小数位（905%10=5 → '5'）
    *(a+4) = '\0';               // ⭐ 字符串结束符，漏了会越界读
}

// ⭐ 模块 3：把一条学生记录写入文件（字段间用 4 空格分隔，行末换行）
void record2file(FILE* cfp, char* cname , int cage, float cscore)
{
    char* pspace = "    ";       // 4 空格作为字段分隔符
    char* pnewline = "\n";       // 换行符

    char sage[3] = {0};          // 两位整数 + '\0'
    char sscore[5] = {0};        // "90.5" + '\0'
    myitoa(cage, sage);          // age → 字符串
    ftoa(cscore, sscore);        // score → 字符串

    fwrite(cname,   1, strlen(cname),   cfp);  // 写姓名
    fwrite(pspace,  1, strlen(pspace),  cfp);  // 写分隔符
    fwrite(sage,    1, strlen(sage),    cfp);  // 写年龄
    fwrite(pspace,  1, strlen(pspace),  cfp);  // 写分隔符
    fwrite(sscore,  1, strlen(sscore),  cfp);  // 写成绩
    fwrite(pnewline,1, strlen(pnewline),cfp);  // 写换行
}

// ⭐ 模块 4：按姓名在文件中查找学生
void search_stu(FILE* cfp, char* name)
{
    char buf[24];                // 单行缓冲区

    while(cfp != (FILE*)EOF)     // ⚠️ 此判断有 bug，见 4.3 说明
    {
        fgets(buf, sizeof(buf)-1, cfp);    // ⭐ 逐行读取
        if(strstr(buf, name) != NULL)      // ⭐ 子串匹配
        {
            printf("%s \n", buf);
            break;
        }
    }
}

int main()
{
    FILE* fp = NULL;

    // ⭐ 阶段一：w+ 打开（读写 + 清空原内容），写入 6 条记录
    fp = fopen("./stu.txt", "w+");
    if(fp == NULL)               // ⭐ 必须检查打开失败
    {
        printf("fopen error \n");
        return 1;
    }

    record2file(fp, "lilei",    20, 90.5);
    record2file(fp, "lihua",    21, 87.5);
    record2file(fp, "jim",      20, 88.5);
    record2file(fp, "lucy",     19, 45.5);
    record2file(fp, "lily",     20, 89.5);
    record2file(fp, "xiaoming", 21, 55.5);
    fclose(fp);                 // ⭐ 写完必须关闭，刷新缓冲区

    // ⭐ 阶段二：r+ 打开（读写 + 保留原内容），按名查找
    fp = fopen("./stu.txt", "r+");
    if(fp == NULL)
    {
        printf("fopen error \n");
        return 1;
    }

    search_stu(fp, "xiaoming");
    fclose(fp);

    return 0;
}


#if 0
/*
    文件操作三步走（基础 demo，被 #if 0 屏蔽，需启用改成 #if 1）：
    1、创建文件
    2、往文件中写入数据。
    3、保存退出。
*/
int main()
{
    FILE* fp = NULL;
    char* ps = "hello world";

    fp = fopen("./a.txt", "w+");
    if(fp == NULL)
    {
        printf("fopen error \n");
        return 1;
    }

    fwrite(ps, 1, strlen(ps), fp);

    fclose(fp);

    return 0;
}
#endif
```

### 4.2 逐模块解释

#### 4.2.1 结构体定义

```c
struct stu{
    char name[16];
    int age;
    float score;
};
```

> [!note] 说明
> 这是后续写入文件的"记录格式"。注意 `name` 是 `char[16]` 数组，所以姓名不能超过 15 个字符（留 1 个给 `'\0'`）。本例的 `"xiaoming"`（8 字符）安全。

#### 4.2.2 整数转字符串 `myitoa`

> [!important] ASCII 转换核心公式
> **数字 `n` → 字符 `'n'`**：`char c = n + 48;`
>
> 因为 `'0'` 的 ASCII 码是 48、`'1'` 是 49、`'2'` 是 50……依次递增。
> 反向（字符 → 数字）：`int n = c - 48;`

| 输入 `i` | `i/10` | `i%10` | 输出字符串 |
| -------- | ------ | ------ | ---------- |
| 20       | 2      | 0      | `"20"`     |
| 21       | 2      | 1      | `"21"`     |
| 9        | 0      | 9      | `"09"` ⚠️  |

> [!bug] 局限性
> 这个函数**只支持两位数**。年龄 `9` 会被转成 `"09"`，单数字场景需要单独处理。更通用的做法是用标准库 `sprintf(sage, "%d", cage);`。

#### 4.2.3 浮点转字符串 `ftoa`

> [!tip] 转换思路
> 把 `90.5` 放大 10 倍成整数 `905`，然后用整数除法分别取出百/十/个位：
> - `905 / 100 = 9` → 十位 `'9'`
> - `905 / 10 % 10 = 0` → 个位 `'0'`
> - `905 % 10 = 5` → 小数位 `'5'`
> - 中间手动插一个 `'.'`

> [!bug] 同样的局限
> 只支持**两位整数 + 一位小数**的格式（如 90.5、87.5）。`100.0` 或 `9.99` 都会出错。生产环境请用 `sprintf(sscore, "%.1f", cscore);`。

#### 4.2.4 写入一条记录 `record2file`

> [!important] fwrite 四参数含义
> `fwrite(数据指针, 单元素字节数, 元素个数, 文件指针)`
>
> 本例固定写 `1, strlen(...)`：每个元素 1 字节，共 `strlen` 个元素 —— 等价于"按字节写入字符串"。

> [!note] 为什么不直接一次写完整行？
> 因为 `name/age/score` 三个字段类型不同（`char[]` / `int` / `float`），不能拼成一个字符串字面量。所以采用"分段写 + 分隔符"的策略，最终在文件里形成一行纯文本。

#### 4.2.5 按名查找 `search_stu`（⚠️ 含 bug）

| 函数        | 作用                           | 返回值                                |
| ----------- | ------------------------------ | ------------------------------------- |
| `fgets`     | 从文件读一行（保留 `\n`）      | 成功返回 `buf`，失败/EOF 返回 `NULL` |
| `strstr`    | 在 `buf` 中找子串 `name`        | 找到返回子串指针，未找到返回 `NULL`  |

> [!danger] ⚠️ 原代码循环条件错误
> `while(cfp != (FILE*)EOF)` 这个循环条件是**错的**，会导致两种问题：
>
> 1. **循环永不退出**：`fp` 是一个文件指针，它**永远不会变成 `EOF`**（EOF 是 `int -1`）。这个条件永远为真。
> 2. **读到文件尾后段错误**：`fgets` 到文件末尾返回 `NULL`，但代码没检查返回值就继续调用 `strstr(buf, name)`。`buf` 里残留上一行数据，找不到匹配时循环继续，再次 `fgets` 又返回 `NULL`……最终 `strstr` 可能访问非法内存崩溃。
>
> **正确写法**（用 `fgets` 返回值作为循环条件）：
> ```c
> while(fgets(buf, sizeof(buf), cfp) != NULL)
> {
>     if(strstr(buf, name) != NULL)
>     {
>         printf("%s \n", buf);
>         break;
>     }
> }
> ```
>
> 这样读完最后一行 `fgets` 返回 `NULL`，循环自动结束，无论是否找到都不会出问题。

#### 4.2.6 主流程：先写后读

> [!important] 关键：为什么要"先关闭再重新打开"？
> `w+` 模式打开后，文件位置指针在**文件开头**。连续写 6 条记录后，指针在文件末尾。如果直接调用 `search_stu`，会从末尾开始读 —— 什么都读不到。
>
> 两个方案：
> 1. **本例做法**：`fclose` 后再 `fopen("r+")`，指针重新回到开头
> 2. **更省事**：`rewind(fp);` 或 `fseek(fp, 0, SEEK_SET);` 把指针拨回头部，无需关文件

#### 4.2.7 `#if 0` 基础 demo

> [!tip] `#if 0` vs `/* */`
> - `/* */` **不能嵌套**：注释里再有 `/* */` 会提前结束
> - `#if 0` 是**预处理指令**，支持嵌套，且能用 `#if 1` 一键恢复整段代码
> - 调试时频繁切换的代码块，用 `#if 0 / #endif` 包起来更顺手

### 4.3 生成的数据文件 `stu.txt`

程序运行后，`stu.txt` 内容如下（字段间 4 空格，每行一条记录）：

```text
lilei    20    90.5
lihua    21    87.5
jim    20    88.5
lucy    19    45.5
lily    20    89.5
xiaoming    21    55.5
```

> [!note] 分隔符 vs 对齐
> 注意每行字段间是**固定 4 空格分隔**，不是对齐的——`jim`（3 字符）和 `xiaoming`（8 字符）后面的空格数不一样，看起来参差不齐。这是"分隔符"和"对齐"的区别：
> - **分隔符**（本例）：字段间固定字符，便于 `strstr` 等字符串函数处理
> - **对齐**（如 `fprintf("%-8s %2d %5.1f\n", ...)`）：视觉整齐，但代码更复杂

### 4.4 fopen 模式速查表

| 模式  | 读 | 写 | 文件存在时       | 文件不存在时 |
| ----- | -- | -- | ---------------- | ------------ |
| `"r"` | ✅ | ❌ | 从头读           | 失败返回 NULL |
| `"w"` | ❌ | ✅ | **清空原内容**   | 创建新文件    |
| `"a"` | ❌ | ✅ | **追加到末尾**   | 创建新文件    |
| `"r+"`| ✅ | ✅ | 从头读写（保留） | 失败返回 NULL |
| `"w+"`| ✅ | ✅ | **清空原内容**   | 创建新文件    |
| `"a+"`| ✅ | ✅ | 读从开头，写追加 | 创建新文件    |

> [!warning] 本例的模式选择
> - 第一次 `"w+"`：要写入 6 条新记录，且 `stu.txt` 可能已有旧数据，所以用 `w+` 清空重写
> - 第二次 `"r+"`：只读查询，不能改动数据，所以用 `r+`（保留内容）。其实纯查询用 `"r"` 就够了

### 4.5 文件操作三步走总结

> [!important] 文件 I/O 必须遵循的三步
> 1. **打开**：`fp = fopen(path, mode);` —— 必须检查 `fp == NULL`
> 2. **读写**：`fwrite` / `fread` / `fgets` / `fputs` / `fprintf` / `fscanf`
> 3. **关闭**：`fclose(fp);` —— 每条 `fopen` 路径都必须配对一个 `fclose`

> [!bug] 忘记 `fclose` 的后果
> - 缓冲区数据没刷到磁盘 → **文件内容丢失或不完整**
> - 文件句柄泄露 → 长期运行的程序可能耗尽系统文件描述符
> - Windows 上文件被占用 → 其他程序无法删除/修改该文件

### 4.6 潜在改进点

| 问题                              | 改进方案                                              |
| --------------------------------- | ----------------------------------------------------- |
| `myitoa` 只支持两位数             | 改用 `sprintf(sage, "%d", cage);`                     |
| `ftoa` 只支持 2 位整数+1 位小数   | 改用 `sprintf(sscore, "%.1f", cscore);`               |
| `search_stu` 循环条件错误         | 改为 `while(fgets(buf, sizeof(buf), cfp) != NULL)`    |
| 找不到学生时无提示                | 循环外加 `int found = 0;`，找到置 1，循环后判断        |
| 写完读需要关文件重开              | 用 `rewind(fp);` 一行搞定                             |
| 整条记录写多次 `fwrite`           | 用 `fprintf(fp, "%s    %s    %s\n", name, age, score);` |

> [!tip] 工程经验
> 实际开发几乎不会手写 `myitoa`/`ftoa`，标准库 `sprintf` / `snprintf` 已足够安全且功能完整。本例手写是为了**练手理解 ASCII 与数字的关系**，这是 C 语言基础练习的常见目的。

## 五、易错点总结

| 错误写法                                  | 正确写法                                  | 原因                          |
| ----------------------------------------- | ----------------------------------------- | ----------------------------- |
| `arr[0].name = "青龙"`                    | `strcpy(arr[0].name, "青龙")`             | 数组名是常量，不能赋值        |
| `struct stu li; ... stu li2;`             | `struct stu li2;` 或 `typedef ... __stu`  | C 中必须带 `struct`           |
| `memset(pli, 0, sizeof(pli))`             | `memset(pli, 0, sizeof(*pli))`            | `pli` 是指针，sizeof 得到 4/8 |
| `fopen` 后直接 `fwrite`                   | 先检查 `if(fp == NULL) return 1;`         | 打开可能失败，必须容错        |
| 忘记 `fclose(fp)`                         | 每条 fopen 路径都配对 fclose              | 文件可能丢失数据或句柄泄露    |
| `pstu.name`（指针用点）                   | `pstu->name` 或 `(*pstu).name`            | 指针必须用 `->`               |

> [!important] 今日核心
> 1. **结构体** = 不同类型字段的复合类型，是 C 语言组织数据的核心工具
> 2. **typedef** 简化类型名，**内存对齐** 决定 sizeof 结果
> 3. **结构体指针 + 函数封装** = C 语言版的"面向对象 CRUD"
> 4. **文件操作三步走** + 整数/浮点转字符串 = 数据持久化基础
> 5. **search_stu 循环条件 bug**：`fp != (FILE*)EOF` 永远为真，应改用 `fgets(...) != NULL`
