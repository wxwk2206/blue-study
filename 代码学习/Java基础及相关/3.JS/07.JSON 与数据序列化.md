# 1.JSON定义
JSON（JavaScript Object Notation，JavaScript 对象表示法）是一种纯文本数据格式，用键值对存储数据。它是前后端通信的通用语言——前端发 JSON，后端回 JSON。
# 2.双向转换：JS ↔ JSON
## 2.1 JSON.stringify() — JS 对象 → JSON 字符串
```js
const user = {
  name: 'bluek',
  age: 22,
  isAdmin: false
};

const json = JSON.stringify(user);
console.log(json);         // → '{"name":"bluek","age":22,"isAdmin":false}'
console.log(typeof json);  // → 'string'  ← JSON 本质是字符串！
```

## 2.2 JSON.parse() — JSON 字符串 → JS 对象
```js
const jsonText = '{"name":"bluek","age":22}';
const obj = JSON.parse(jsonText);

console.log(obj.name);     // → 'bluek'
console.log(typeof obj);   // → 'object'  ← 转换回了 JS 对象
```

一句话：stringify = 对象打包成字符串发送；parse = 收到的字符串解包成对象使用。两个操作互为逆运算。
# 3.stringify 的三个陷阱
## 3.1 哪些值会消失？
JSON.stringify 不是照相机——有些东西拍不出来：
```js
const data = {
  name: 'bluek',
  age: undefined,         // ⚠️ 会被跳过
  secret: function() {},  // ⚠️ 会被跳过
  now: new Date(),        // ⚠️ 转成 ISO 字符串
  amount: NaN,            // ⚠️ 转成 null
  nothing: null           // ✅ 保留
};

const result = JSON.stringify(data);
console.log(result);
// → '{"name":"bluek","now":"2026-06-19T10:00:00.000Z","amount":null,"nothing":null}'
//   undefined 和函数直接消失，NaN 变 null，Date 变字符串
```
规则：
- 直接删除：undefined、函数、Symbol（对象属性场景）
- 转为 null：NaN、±Infinity
- 转为 ISO 字符串：Date 对象
- 原样保留：字符串、数字、布尔、null、普通对象 / 数组
- **补充**：数组里：undefined/函数 都会变成 null，而非删除

> [!important] 重点
>如果你在对象里存了 token 或 password 但设为了 undefined 想隐藏它——stringify 确实会跳过，但 parse 回来时这个属性**直接没了**，不是变成 null。这可能导致后续代码判断 obj.token !== null 时出现意外行为。

## 3.2 循环引用 — 直接爆炸
```js
const a = {};
const b = { a: a };
a.b = b;           // a.b.a.b.a... 无限循环引用

JSON.stringify(a); // ❌ TypeError: Converting circular structure to JSON
```

## 3.3 数组里的 undefined，function(){} 变 null
```js
const arr = [1, undefined,function(){}, 3, null];
JSON.stringify(arr);  // → '[1,null,null,3,null]'  ← 变成 null（不是被跳过）
```

# 4.JSON.parse 的致命陷阱：原型链污染⭐
这是本课最重要的部分——**安全方向的直接应用**

## 4.1 原理
**Object.assign 只会污染目标对象本身，不会影响原型链**。真正危险的是**递归 merge**：
```
递归merge
Git 默认的合并算法，全称 recursive
原理：递归处理分支分叉历史，遇到多个共同祖先时，先递归合并中间节点，再做最终合并
```

```js
// ⭐ 首先定义危险的递归 merge 函数
// ⚠️ 注意：这里用 for...in 而非 Object.keys()，会遍历继承属性
//    这本身就是"危险"的一部分——正确实现应该加 hasOwnProperty 过滤
function dangerousMerge(target, source) {
//dangerousMerge 复用了递归 merge 的归并算法，是递归 merge 的不安全增强版；
  for (const key in source) {
    if (typeof source[key] === 'object' && source[key] !== null) {
      if (!target[key]) target[key] = {};
      dangerousMerge(target[key], source[key]);   // ← 递归！这是关键
    } else {
      target[key] = source[key];
    }
  }
}

// 攻击者构造的 payload
const payload = '{"__proto__": {"isAdmin": true}}';
//__proto__ 是 JavaScript 内置的原型属性,意图给所有对象的原型挂载 isAdmin: true
const userData = JSON.parse(payload);
const config = {};

// 💀 递归 merge 的执行轨迹
dangerousMerge(config, userData);

// 第1步：遍历 userData → key = "__proto__", value = {isAdmin: true}
// 第2步：value 是对象 → 递归进入
// 第3步：target["__proto__"] → 这是 config 的原型 → Object.prototype！
// 第4步：dangerousMerge(Object.prototype, {isAdmin: true})
// 第5步：Object.prototype.isAdmin = true ← 全局污染完成

console.log(config.isAdmin);  // → true
console.log({}.isAdmin);      // → true  ← 所有空对象都中招
console.log([].isAdmin);      // → true  ← 数组也中毒
```

| 方式            | 影响范围  | 原理                                                          |
| ------------- | ----- | ----------------------------------------------------------- |
| Object.assign | 仅目标对象 | config.\__proto\__ = {...} 只改了 config 一个对象的原型               |
| 递归 merge      | 全局    | 递归进入 config.\__proto\__（即 Object.prototype），直接在所有人共享的原型上写属性 |
****

> [!important] 为什么 for...in 本身也是问题的一部分？
> for...in 会遍历**继承的**属性（包括原型链上的）。如果 Object.prototype 已被污染，for...in 会拾取污染属性并继续传播——形成"污染扩散"。正确的 merge 应该用 Object.keys() 或加 hasOwnProperty 过滤。

> [!danger] 这就是原型链污染（Prototype Pollution）的经典入口！
> 攻击链路：用户输入 → JSON.parse → 递归 merge → \__proto\__ 被展开 → Object.prototype 被写入 → 全局污染
> **原理**：
> JSON.parse('{"\__proto__": {...}}') 解析出的对象带有一个名为 \__proto\__ 的自身属性。当递归 merge 遍历到这个键时，如果它递归进入嵌套对象，target["\__proto\__"] 会被解析为 Object.prototype（因为 target 是一般对象，它的 \__proto\__ 指向 Object.prototype），然后递归在其中写入恶意属性，达到全局污染。
> 
> **条件**：
> 1. 用户输入能被 JSON.parse 解析
> 2. 解析结果通过不安全的方式合并到关键对象上
> 3. 被污染的对象通过原型链影响业务逻辑
>
>  **修复**：
> 1. **使用 Object.create(null) 创建无原型对象存储数据**
> 2. merge 时过滤 \__proto\__ 和 constructor.prototype 键名
> 3. 合并后用 Object.freeze(Object.prototype) 冻结原型
> 4. 使用 Map 代替普通对象存储外部数据

# 5.实用技巧
## 5.1 深拷贝（Deep Clone）
```js
// ⭐ 一行代码实现深拷贝（有局限性但简单）
const original = { name: 'bluek', tags: ['js', 'sec'] };
const clone = JSON.parse(JSON.stringify(original));

clone.tags.push('xss');
console.log(original.tags);  // → ['js', 'sec']  原数组没变！✅
console.log(clone.tags);     // → ['js', 'sec', 'xss']

// ⚠️ 局限性：会丢失函数、undefined、Date、循环引用
```

## 5.2 格式化输出
```js
const data = { name: 'bluek', skills: ['JS', 'Python', 'Java'] };

// 第三个参数是缩进空格数
console.log(JSON.stringify(data, null, 2));
// {
//   "name": "bluek",
//   "skills": [
//     "JS",
//     "Python",
//     "Java"
//   ]
// }
```

## 5.3 replacer 参数 — 安全侧的「过滤器」
```js
// ⭐ stringify 的第二参数：选择要包含的属性（安全场景：防止泄露敏感字段）
const user = { name: 'bluek', password: 'secret123', role: 'admin' };

// 只序列化 name 和 role
const safe = JSON.stringify(user, ['name', 'role']);
console.log(safe);  // → '{"name":"bluek","role":"admin"}'  password 被过滤！
```

# 6.学习中的思考
JSON 只支持六种类型（string/number/boolean/null/array/object），函数、undefined、Symbol 都不支持。

思考：我有一个问题，json不支持这些类型，那找漏洞的时候，数据包类型从json改成其他的，会不会触发什么漏洞呢

ai回复：**是的，Content-Type 欺骗（Content-Type Confusion）是一大类漏洞。** 后端根据请求头的 Content-Type 决定用什么解析器解析 body。你告诉它「这是 JSON」，它就用 JSON 解析器；你骗它「这是 XML」，它可能换一个完全不同的解析器——而那个解析器的行为可能远比 JSON 危险。

# 7.核心要点

| 概念         | 一句话                                          |
| ---------- | -------------------------------------------- |
| JSON 本质    | 纯文本字符串，不是 JS 对象                              |
| stringify  | JS 对象 → JSON 字符串，会丢失 undefined/函数/Symbol<br> |
| parse      | JSON 字符串 → JS 对象                             |
| JSON 格式规则  | 键名双引号、无注释、无尾部逗号、仅六种类型                        |
| 深拷贝        | JSON.parse(JSON.stringify(obj))，有局限性         |
| 🔒 原型链污染入口 | JSON.parse + \__proto\__ + 不安全的 merge        |
| 🔒 防御      | Object.create(null)、过滤 \__proto\__ 键名        |
