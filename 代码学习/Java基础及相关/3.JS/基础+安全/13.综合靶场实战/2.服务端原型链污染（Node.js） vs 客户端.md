# 1.先回顾：客户端原型链污染
```js
// ⭐ 客户端 merge 污染（浏览器里）
function merge(target, source) {
  for (var key in source) {
    target[key] = source[key];  // ⚠️ 没过滤 __proto__
  }
}

// 攻击者传 JSON：
merge({}, JSON.parse('{"__proto__":{"isAdmin":true}}'));

// 结果：所有 {} 都有 isAdmin: true
var obj = {};
console.log(obj.isAdmin); // true ← 从原型链读到的
```
危害范围：**当前标签页**。刷新页面就没了。

# 2.服务端污染：完全不同的游戏
服务端原型链污染发生在 Node.js 进程里。一旦 Object.prototype 被污染，**整个进程的所有请求都受影响**，而且进程不重启就不会消失。
`比喻：客户端污染像你在自己家里墙上涂鸦——关上门别人看不到，刷新一下墙就恢复原样。服务端污染像你在整栋公寓的总水管里投毒——整栋楼的人都喝到毒水，你关自家水龙头没用，必须等物业来修水管。`

| 维度    | 客户端（浏览器）                    | 服务端（Node.js）                                 |
| ----- | --------------------------- | -------------------------------------------- |
| 污染范围  | 当前标签页                       | **整个进程**（所有并发请求共享）                           |
| 持续时间  | 刷新即消失                       | **进程不重启就一直存在**                               |
| 典型入口  | URL 参数 → JSON.parse → merge | HTTP body（JSON/表单）→ lodash/nconf 等合并 → 全局生效  |
| 最高危害  | DOM XSS / 权限绕过              | **RCE（远程代码执行）、任意文件读写、NoSQL 注入**              |
| 常见漏洞库 | 手工写的 merge 函数               | lodash < 4.17.11、hoek、nconf、dot-prop 等 npm 包 |
| 检测方式  | Console 查看 Object.prototype | HTTP 响应异常、错误信息泄露、定时黑盒探测                      |

**核心差异**：客户端污染你能刷新页面清理；服务端污染**你的攻击载荷留在了全局对象上**，后来访问的人都在"被污染的进程"里运行。一个请求造成的污染，可能影响成百上千个后续用户。
