# 1.原型链定义
	原型链是 JS 对象属性查找的「父对象链」。当你访问 obj.x，JS 引擎先在 obj 自身找；找不到就去它的原型找；还找不到就去原型的原型找……直到 null 为止。这条查找路径就叫原型链。
## 1.1 原型链长什么样
```
普通对象 obj 的原型链：

  obj  ──[[Prototype]]──→  Object.prototype  ──[[Prototype]]──→  null
  {}                          { toString, hasOwnProperty, ... }

  ↑ 自己创建的             ↑ 所有对象的共同祖先          ↑ 链的终点


数组 arr 的原型链（多一层）：

  arr  ──→  Array.prototype  ──→  Object.prototype  ──→  null
  []         { push, pop, map, ... }   { toString, ... }


函数 fn 的原型链：

  fn  ──→  Function.prototype  ──→  Object.prototype  ──→  null
  ƒ()       { call, apply, bind, ... }  { toString, ... }
```
## 1.2验证
```js
const obj = {};
const arr = [];
const fn = function() {};

// ⭐ 每个对象的原型
Object.getPrototypeOf(obj) === Object.prototype;     // → true
Object.getPrototypeOf(arr) === Array.prototype;      // → true
Object.getPrototypeOf(fn)  === Function.prototype;   // → true

// ⭐ 原型的原型
Object.getPrototypeOf(Array.prototype)    === Object.prototype;  // → true
Object.getPrototypeOf(Function.prototype) === Object.prototype;  // → true

// ⭐ 终点
Object.getPrototypeOf(Object.prototype);  // → null  ← 链到底了
```

# 2.\__proto\__ vs prototype — 最易搞混
混淆点:每个对象有 \__proto\__，每个函数有 prototype。
**Object.prototype — 原型对象,\__proto\__ — 原型链指针 —>指向Object.prototype**

| 属性          | 谁有   | 指向       | 作用              |
| ----------- | ---- | -------- | --------------- |
| \__proto\__ | 所有对象 | 自己的原型（爹） | 构成原型链           |
| prototype   | 只有函数 | 一个对象     | 作为 new 出来的实例的原型 |
## 2.1两者的关系
```js
function Person(name) {
  this.name = name;
}
// ⭐ Person 是函数，所以有 prototype
Person.prototype.sayHi = function() {
  return 'Hi, ' + this.name;
};

// ⭐ new Person() 创建实例
const bluek = new Person('bluek');

// 原型链是这样的：
// bluek.__proto__        === Person.prototype        ← true
// Person.prototype.__proto__ === Object.prototype    ← true
// Object.prototype.__proto__    === null             ← true

// ⭐ 关键：实例的 __proto__ 指向构造函数的 prototype
bluek.__proto__ === Person.prototype;  // → true ✅

// 所以 bluek 能调用 sayHi（在原型链上找到）
bluek.sayHi();  // → 'Hi, bluek'
```
⚠️ 记忆口诀：
- \__proto\__ 是「我爹是谁」——每个对象都有
- prototype 是「我儿子们的爹是谁」——只有函数有，用来给 new 出的实例当原型
## 2.2 为什么 __proto__ 是访问器属性
	__proto__ 不是对象自己的真属性，是个"假"属性。
```js
const obj = {};

// ⭐ obj.__proto__ 实际上是个 getter/setter
//    它定义在 Object.prototype 上
Object.getOwnPropertyDescriptor(Object.prototype, '__proto__');
// → { get: ƒ, set: ƒ, enumerable: false, configurable: true }
//get: ƒ -> 有 getter 函数 → 读取时触发;
//set: ƒ -> 有 setter 函数 → 写入时触发;
//enumerable: false -> 不可遍历（for...in 看不到它）;
//configurable: true -> 可以被删除/重新配置

// 推荐的替代写法（更安全、更明确）：
Object.getPrototypeOf(obj);       // 读取原型
Object.setPrototypeOf(obj, x);    // 设置原型
```
# 3.属性查找机制 — 原型链的核心
当你写 obj.x，JS 引擎的查找过程：

1. 检查 obj 自身有没有 x（own property）
2. 没有 → 查 obj.\__proto\__（原型）
3. 没有 → 查原型的 \__proto\__（原型的原型）
4. 一直查到 null（链终点）
5. 整个链都没有 → 返回 undefined
```js
// ⭐ 模拟 JS 引擎的属性查找
function getProperty(obj, key) {
  let current = obj;
  while (current !== null) {
    // 检查自身属性（不含继承的）
    if (Object.prototype.hasOwnProperty.call(current, key)) {
      return current[key];  // 找到了
    }
    current = Object.getPrototypeOf(current);  // ⭐ 往上爬一层
  }
  return undefined;  // 链到底都没有
}

const bluek = new Person('bluek');
getProperty(bluek, 'name');    // → 'bluek'（自身属性）
getProperty(bluek, 'sayHi');   // → ƒ（Person.prototype 上）
getProperty(bluek, 'toString');// → ƒ（Object.prototype 上）
getProperty(bluek, 'xyz');     // → undefined（整个链都没有）
```

> [!important]  关键认知
> 属性查找是**动态**的。你此刻**访问** obj.toString，JS 引擎此刻去爬原型链。如果原型链上的属性被改了，下次访问的结果就变了——这就是原型链污染能生效的根本原因。

# 4.原型链污染的完整原理
## 4.1 污染Object.prototype会影响所有对象的原因
```js
// ⭐ 所有的普通对象，原型链终点都是 Object.prototype
const a = {};
const b = { name: 'bluek' };
const c = new Person('test');

// 它们的原型链：
// a → Object.prototype → null
// b → Object.prototype → null
// c → Person.prototype → Object.prototype → null
//                  ↑
//           所有人都要经过这里！

// ⭐ 在 Object.prototype 上加属性
Object.prototype.hacked = true;

// ⭐ 现在所有对象都能访问到 hacked（通过原型链查找）
console.log(a.hacked);  // → true 💀
console.log(b.hacked);  // → true 💀
console.log(c.hacked);  // → true 💀
console.log([].hacked); // → true 💀 连数组也中招
console.log((function(){}).hacked);  // → true 💀 连函数也中招
```
## 4.2 完整攻击链路

| 步骤                    | 动作                                               | 涉及知识点<br>         |
| --------------------- | ------------------------------------------------ | ----------------- |
| ① 攻击者构造 payload       | '{"\__proto\__": {"isAdmin": true}}'             | 07.JSON<br>       |
| ② 后端 JSON.parse       | 解析出带 \__proto\__ 自身属性的对象                         | 07.JSON.parse<br> |
| ③ 递归 merge            | deepMerge(config, payload)，遇到 \__proto\__ 递归进入   | 07.递归 merge<br>   |
| ④ 钻进 Object.prototype | config.\__proto\__ 就是 Object.prototype，递归在它上面写属性 | ⭐ 本节原型链<br>       |
| ⑤ 全局污染                | Object.prototype.isAdmin = true                  | ⭐ 本节原型链<br>       |
| ⑥ 权限检查失效              | if (user.isAdmin) 通过原型链查到 true                   | 08.this + 本节<br>  |
| ⑦ for...in 扩散         | 遍历对象时拾取污染属性，继续传播                                 | 06.for...in 坑<br> |

> [!important]  原型链污染是 JS 特有的高危漏洞的原因！
> 其他语言（Java/C++）的继承是**编译期**决定的，运行时改不了父类。JS 的原型链是**运行时**的——你可以在**程序运行中途**往 Object.prototype 写属性，整个程序的所有对象**立刻**受影响。这种动态性是 JS 的强大之处，也是它最危险的地方。

# 5.constructor — 反向引用
原型对象上有一个 constructor 属性，指回构造函数：
```js
function Person(name) { this.name = name; }

// ⭐ Person.prototype.constructor 指向 Person 自己
Person.prototype.constructor === Person;  // → true

const bluek = new Person('bluek');
// ⭐ 通过实例能反查构造函数
bluek.constructor === Person;  // → true（通过原型链查找）
bluek.__proto__.constructor === Person;  // → true（直接在原型上）
```
**安全注意**：
constructor 也是可以被污染的入口！
```js
// ⭐ 攻击者除了 __proto__，还可以用 constructor.prototype
const payload = '{"constructor": {"prototype": {"isAdmin": true}}}';
// 如果 merge 函数递归进入 constructor → constructor.prototype
// 同样能写到 Object.prototype 上
```
所以防御时同时过滤 \__proto\__ 和 constructor！
# 6.Function 和 Object 的鸡生蛋问题
这是 JS 最烧脑的设计，看一眼就好，不用死记：
```js
// ⭐ Function 是个函数，所以它的 __proto__ 指向 Function.prototype
Function.__proto__ === Function.prototype;  // → true（自己指向自己！）

// ⭐ Object 也是个函数
Object.__proto__ === Function.prototype;    // → true

// ⭐ Function.prototype 的原型是 Object.prototype
Function.prototype.__proto__ === Object.prototype;  // → true

// ⭐ 所以最终的祖宗还是 Object.prototype
//    Object.prototype.__proto__ === null（终点）
```

```
函数和对象的「族谱」：

  Function  ──→  Function.prototype  ──→  Object.prototype  ──→  null
  Object    ──↗      ↑                      ↑
  Person    ──→  Person.prototype  ─────────┘
                     ↑
   new Person()  ──→  实例（__proto__ 指向 Person.prototype）
```
	一句话总结：Object.prototype 是万物之祖——所有对象的原型链最终都会指向它（除了 Object.create(null) 创建的）。这就是为什么污染它等于污染一切。

# 7.核心要点

| 概念               | 一句话                                                                                                            |
| ---------------- | -------------------------------------------------------------------------------------------------------------- |
| 原型链              | 对象属性查找的「父对象链」，自身 → 原型 → 原型的原型 → null<br>                                                                       |
| \__proto\__      | 每个对象都有，指向自己的原型（爹）<br>                                                                                          |
| prototype        | 只有函数有，作为 new 出的实例的原型<br>补充：prototype 就是函数对象上的一个普通属性名，值是个对象。它和 name、length 这些属性没本质区别，唯一特殊的是 new 操作会读取它来设置实例的原型。 |
| Object.prototype | 万物之祖，所有普通**对象**原型链的终点                                                                                          |
| constructor      | 原型对象上反向指回构造函数，也是污染入口<br>                                                                                       |
| 属性查找             | 动态的、运行时的，改原型链立刻影响所有对象<br>                                                                                      |
| 🔒 污染原理          | 往 Object.prototype 写属性 → 所有对象通过原型链查到<br>                                                                       |
| 🔒 防御            | 过滤 \__proto\__ 和 constructor；用 Object.create(null)<br>                                                         |
