# 1.this定义
概括：this 是函数运行时的一个隐藏参数，指向「调用这个函数的对象」。但具体指向谁，**取决于函数怎么被调用，不是函数怎么被定义**——这是它最反直觉的地方。
```js
function showThis() {
  console.log(this);
}

const obj = { name: 'bluek', show: showThis };

showThis();      // → Window/global  ⚠️ 直接调用
obj.show();      // → { name: 'bluek', show: ... }  ✅ 通过对象调用
```

> [!important]  JS 最反直觉的设计
> 同一个函数，this 居然能变。Java/C++ 里 this 永远指向当前实例，JS 不一样——它跟着**调用方式**走。

# 2.四种绑定规则
JS 有四条规则决定 this 指向谁，按优先级从低到高排：

| 规则       | 触发条件                                        | this 指向                        |
| -------- | ------------------------------------------- | ------------------------------ |
| ① 默认绑定   | 直接调用：fn()                                   | 全局对象（浏览器 window / Node global） |
| ② 隐式绑定   | 对象方法：obj.fn()                               | 调用的那个对象（obj）                   |
| ③ 显式绑定   | fn.call(obj) / fn.apply(obj) / fn.bind(obj) | 指定的那个对象                        |
| ④ new 绑定 | new fn()                                    | 新创建的空对象                        |

## 2.1 默认绑定（最低优先级）
```js
function foo() {
  console.log(this);  // ⚠️ 非严格模式：window / 严格模式：undefined
}

foo();  // 直接调用，没有任何对象 → 默认绑定

// 严格模式下
'use strict';
foo();  // this 是 undefined（更安全）
```

## 2.2 隐式绑定
```js
const user = {
  name: 'bluek',
  greet() {
    console.log(this.name);  // ⭐ this 指向调用者 user
  }
};

user.greet();  // → 'bluek'  ✅ user 调用了 greet

// ⚠️ 但如果把方法单独拿出来调用
const fn = user.greet;
fn();  // → undefined 或报错！this 不再指向 user
```

> [!important]   隐式绑定丢失 — 安全审计重点！
> 上面的 fn() 就是经典的「this 丢失」场景。代码审计时看到对象方法被赋值给变量再调用，要立刻警惕：**方法内依赖 this 的逻辑可能完全失效。**
```js
// 真实漏洞场景：权限检查方法被赋值给回调
const auth = {
  isAdmin: false,
  checkPermission() {
    return this.isAdmin;  // 依赖 this
  }
};

// ⚠️ 把 checkPermission 单独传给路由处理函数
router.get('/admin', auth.checkPermission);
// 等价于 fn = auth.checkPermission; fn();
// this 不再指向 auth，isAdmin 是 undefined → 权限检查失效
```

## 2.3 显式绑定 — call / apply / bind
三个函数可以**强制指定** this：

| 方法    | 语法                          | 立即调用？    |
| ----- | --------------------------- | -------- |
| call  | fn.call(obj, arg1, arg2)    | ✅ 立即     |
| apply | fn.apply(obj, [arg1, arg2]) | ✅ 立即     |
| bind  | const newFn = fn.bind(obj)  | 	❌ 返回新函数 |

```js
function greet(greeting, punctuation) {
  console.log(`${greeting}, ${this.name}${punctuation}`);
}

const user = { name: 'bluek' };

// ⭐ call：参数一个个传
greet.call(user, 'Hello', '!');   // → 'Hello, bluek!'

// ⭐ apply：参数用数组传
greet.apply(user, ['Hi', '.']);   // → 'Hi, bluek.'

// ⭐ bind：返回新函数，永久绑定 this
const boundGreet = greet.bind(user, 'Hey');
boundGreet('?');                   // → 'Hey, bluek?'
boundGreet('!!!');                 // → 'Hey, bluek!!!'（bind 的参数保留）
```

```js
//⭐ 修复 this 丢失：用 bind 把 this 锁死。
// 修复前面的权限检查漏洞
router.get('/admin', auth.checkPermission.bind(auth));
// 现在 this 永远是 auth，权限检查正常
```

## 2.4 new 绑定（最高优先级）
```js
function Person(name) {
  // ⭐ new 干了四件事：
  // 1. 创建一个新空对象 {}
//let instance = {};
  // 2. 把新对象的 __proto__ 指向 Person.prototype
//instance.__proto__ = Person.prototype;
  // 3. 把 this 指向新对象，执行函数体
//Person.call(instance, 'bluek');
  // 4. 如果函数没 return 对象，就返回这个新对象
//return instance;
  this.name = name;
  console.log(this);  // → 新对象 { name: 'bluek' }
}

const bluek = new Person('bluek');
```

# 3.箭头函数：this 的「叛逆者」
箭头函数（=>）**不遵守**上面四条规则。它的 **this** 是**词法**决定的——继承自定义时所在作用域的 this。
```js
// ⭐ 普通函数：this 跟着调用方式走
const obj = {
  name: 'bluek',
  regularFn() {
    console.log(this.name);
  },
  arrowFn: () => {
    console.log(this.name);   // ⚠️ 继承外层（全局）的 this
  }
};

obj.regularFn();  // → 'bluek'   ✅
obj.arrowFn();    // → undefined ⚠️ this 是 window，没有 name 属性
```

## 3.1 箭头函数的真实用途
```js
// ⭐ 箭头函数解决了「回调里 this 丢失」的问题
const timer = {
  count: 0,
  start() {
    // 普通函数：setTimeout 的回调 this 变成 window
    setInterval(function() {
      this.count++;        // ⚠️ this 是 window，count 没加到 timer 上
      console.log(this.count);  // → NaN
    }, 1000);
  }
};

const timer2 = {
  count: 0,
  start() {
    // 箭头函数：继承 start 方法的 this（即 timer2）
    setInterval(() => {
      this.count++;        // ✅ this 是 timer2
      console.log(this.count);  // → 1, 2, 3...
    }, 1000);
  }
};
```
 **选择标准**：
- 需要动态 this（对象方法、构造函数）→ 用普通函数
- 需要固定 this（回调、闭包内）→ 用箭头函数
# 4.优先级实例
四条规则同时存在时，按优先级生效：
```js
function show(label) {
  console.log(label, this.name);
}

const obj1 = { name: 'obj1' };
const obj2 = { name: 'obj2' };

// ① 默认 < 隐式
show('默认');          // → 默认 undefined/window
obj1.show = show;
obj1.show('隐式');     // → 隐式 obj1  ✅ 隐式覆盖默认

// ② 隐式 < 显式
obj1.show.call(obj2, '显式');  // → 显式 obj2  ✅ call 强行指定

// ③ 显式 < new
const bound = show.bind(obj1, 'bind');
new bound();          // → bind undefined  ⚠️ new 创建了新对象，覆盖 bind
// （新对象没有 name 属性，所以是 undefined）
```

# 5.安全方向：this 相关漏洞
## 5.1 权限检查失效（隐式绑定丢失）
```js
// ⭐ 真实漏洞模式
class AuthManager {
  constructor() {
    this.currentUser = null;
  }
  isAdmin() {
    return this.currentUser?.role === 'admin';
  }
}

const auth = new AuthManager();
auth.currentUser = { role: 'guest' };

// ⚠️ 危险：把方法当回调传
document.getElementById('btn').addEventListener('click', auth.isAdmin);
// 点击时 this 是 button 元素，不是 auth
// this.currentUser 是 undefined → isAdmin 返回 false（看似安全？）

// 更可怕的场景：
const check = auth.isAdmin;
if (check()) { /* 进入管理面板 */ }
// this.currentUser → undefined → undefined?.role === 'admin' → false
// 看起来返回 false 是安全的……但如果攻击者能影响 this 呢？
```
## 5.2 通过 call/apply 劫持 this
```js
// ⭐ 攻击者通过 call 控制 this，绕过检查
function checkPassword() {
  // 假设 this.password 是后端预设的密码
  return this.inputPassword === this.password;
}

const legitObj = { password: 'real_secret', inputPassword: 'wrong' };
checkPassword.call(legitObj);  // → false（正常）

// ⚠️ 攻击者构造自己的对象
const fakeObj = { password: 'attacker_knows', inputPassword: 'attacker_knows' };
checkPassword.call(fakeObj);  // → true！密码检查被绕过
```

> [!important] 审计要点
>看到函数内依赖 this.xxx 做权限判断，且该函数会被 call/apply/bind 调用或赋值给变量 → **立刻警惕 this 劫持**。攻击者可以构造对象 + .call() 让 this 指向自己的对象。

## 5.3 原型链 + this 组合攻击
```js
// ⭐ 原型链污染配合 this 劫持
function checkAdmin() {
  return this.isAdmin === true;
}

// 攻击者先污染原型链（07.JSON 与数据序列化中有提到）
Object.prototype.isAdmin = true;

// 现在任何对象调用 checkAdmin 都返回 true
const anyUser = {};
checkAdmin.call(anyUser);  // → true 💀
// 因为 anyUser.isAdmin 继承自 Object.prototype
```

# 6.核心要点

| 概念      | 一句话                                          |
| ------- | -------------------------------------------- |
| this 本质 | 函数运行时的隐藏参数，由**调用方式**决定                       |
| 四种规则    | 默认 < 隐式 < 显式 < new（优先级从低到高）                  |
| 隐式丢失    | obj.fn 赋值给变量再调用，this 变成 undefined/window<br> |
| 显式绑定    | call/apply 立即调用，bind 返回新函数<br>               |
| 箭头函数    | 不绑定 this，继承外层作用域的 this（词法）<br>               |
| 🔒 审计重点 | 方法当回调传、依赖 this 的权限检查、原型链+this 组合<br>         |
