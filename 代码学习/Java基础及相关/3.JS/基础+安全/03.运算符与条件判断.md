# 1.算数运算符
```
// 基本算术
10 + 3;    // → 13   加法
10 - 3;    // → 7    减法
10 * 3;    // → 30   乘法
10 / 3;    // → 3.333...  除法（永远返回浮点数，不像 Java 有整除）
10 % 3;    // → 1    取余（mod）—— 安全方向常用，比如判断奇偶
```
⚠️ 注意：**JS 的 / 没有整除**。10 / 3 在 Java 里是 3，在 JS 里是 3.333...。如果想整除，得用 Math.floor(10 / 3)。
## 1.1 + 号的双重身份
在 JS 里，+ 既是数字加法，又是字符串拼接（concatenation）
```
// 数字 + 数字 → 数学加法
5 + 3;            // → 8

// 字符串 + 任何 → 字符串拼接
'Hello' + ' ' + 'World';   // → 'Hello World'
'得分: ' + 100;            // → '得分: 100'（数字 100 被转成字符串）
'3' + 2;                   // → '32'  ⚠️ 不是 5！
```

> [!important] 安全方向重点
> '3' + 2 得到 '32' 而不是 5——这就是**隐式类型转换**（implicit type coercion）。当用户输入的数据（始终是字符串）参与运算时，这种转换可能导致意料之外的行为，是很多逻辑漏洞的根源。

### 隐式类型转换

三种目标类型：布尔值 (Boolean)、数字 (Number)、字符串 (String)
#### 一、转换为 布尔值（ToBoolean）

##### 1. 触发场景
逻辑判断类语法/运算符，会自动把表达式转为布尔：
- 分支：`if / else / while / do-while`
- 逻辑运算符：`! 、!! 、&& 、||`
- 三元运算符 `条件 ? a : b` 的条件部分

##### 2. 转换规则（核心：**只有 7 个假值，其余全为 true**）
| 原始值         | 转换结果    | 说明         |
| ----------- | ------- | ---------- |
| `false`     | `false` | 本身就是假      |
| `0` / `-0`  | `false` | 正负0都是假     |
| `""`（空字符串）  | `false` | 仅空串，非空串都是真 |
| `null`      | `false` | 空对象指针      |
| `undefined` | `false` | 未定义        |
| `NaN`       | `false` | 非有效数字      |

 **所有不在上表中的值，转布尔一律为 `true`**
重点易错示例：
```js
Boolean("0")      // true  非空字符串
Boolean(" ")      // true  空格也是非空
Boolean([])       // true  空数组（对象）
Boolean({})       // true  空对象
Boolean(Infinity) // true  无穷大
Boolean(Symbol()) // true  所有 Symbol
```

##### 3. 典型坑点
业务中想判断「空数据」，直接用隐式布尔判断会出错：
```js
let num = 0;
if(num) { /* 不会执行 */ } // 0 是假值，符合预期

let arr = [];
if(arr) { /* 一定会执行 */ } 
// 误区：以为空数组是空，实际对象转布尔永远为 true
```

#### 二、转换为 数字（ToNumber）
##### 1. 触发场景
算术运算、大小比较、宽松相等 `==`，绝大多数运算都会优先转数字：
- 算术符：`+ - * / % ++ --`（**二元+除外，二元+优先拼接字符串**）
- 比较符：`> < >= <=`
- 宽松相等：`== / !=`
- 一元正负号：`+val`、`-val`（一元+强制转数字）
	**⭐特例**：null，undefined在宽松相等 == 比较时，不会执行ToNumber数字转换
##### 2. 分类型详细转换规则
###### 1）原始类型转数字
| 原值 | 转换结果 | 示例 |
|------|----------|------|
| `undefined` | `NaN` | `Number(undefined) → NaN` |
| `null` | `0` | `Number(null) → 0` |
| `true` | `1` | `Number(true) → 1` |
| `false` | `0` | `Number(false) → 0` |
| 纯数字字符串 | 对应数字 | `Number("123") → 123` |
| 空串/全空格串 | `0` | `Number("") → 0`、`Number("   ") → 0` |
| 含非数字字符的字符串 | `NaN` | `Number("12a") → NaN`、`Number("abc") → NaN` |
| `NaN` | `NaN` | 自身不变 |
| `Infinity` | `Infinity` | 自身不变 |
补充：NaN不等于NaN

###### 2）引用类型（对象/数组）转数字
遵循**先转原始值，再转数字**的两步流程：
1. 执行 `valueOf()` → 不是原始值就执行 `toString()`
2. 把得到的原始值，再套用上面「原始类型转数字」规则

```js
// 1. 空数组
[] → valueOf()=[] → 调用toString() → "" → 转数字 → 0
Number([]) // 0

// 2. 单元素数字数组
[123] → toString() → "123" → 转数字 → 123
Number([123]) // 123

// 3. 多元素数组
[1,2] → toString() → "1,2" → 含非数字 → NaN
Number([1,2]) // NaN

// 4. 普通对象
{} → toString() → "[object Object]" → 非数字串 → NaN
Number({}) // NaN
```

##### 3. 高频易错案例
```js
// 1. null / undefined 差异
null > -1    // true  null→0
undefined > 0// false undefined→NaN，NaN和任何数比较都是false

// 2. 数组比较
[] == 0      // true  []→""→0
[123] == 123 // true
[1,2] == 1   // false [1,2]→"1,2"→NaN

// 3. 一元+强制转数字
+"123"  // 123
+""     // 0
+[]     // 0
+null   // 0
+undefined // NaN
```

#### 三、转换为 字符串（ToString）
##### 1. 触发场景
只有两类场景会优先转为字符串：
1. **二元加号 `+`**：只要**任意一侧是字符串**，整个表达式转为字符串做拼接
2. 隐式/显式字符串拼接、`alert()`、`document.write()` 等输出方法

##### 2. 分类型详细转换规则
###### 1）原始类型转字符串
| 原值 | 转换结果 | 示例 |
|------|----------|------|
| `null` | `"null"` | `String(null) → "null"` |
| `undefined` | `"undefined"` | `String(undefined) → "undefined"` |
| `true` | `"true"` | - |
| `false` | `"false"` | - |
| 数字 | 对应数字字符串 | `String(123) → "123"`、`String(NaN) → "NaN"` |

###### 2）引用类型（对象/数组）转字符串
同样**先转原始值**：`valueOf()` → `toString()`，最终得到字符串：
1. **数组**：元素用逗号拼接，空数组转为空串
   ```js
   String([])    // ""
   String([1])   // "1"
   String([1,2,3]) // "1,2,3"
   ```
2. **普通对象**：固定返回 `"[object Object]"`
   ```js
   String({}) // "[object Object]"
   ```

#### 四、三大转换 综合优先级总结
1. **布尔转换**：仅出现在逻辑判断、`!&&||if`，规则最简单（只记7个假值）
2. **字符串转换**：仅被 **二元 `+`（含字符串）** 触发，优先级高于数字转换
3. **数字转换**：除以上两种场景外，**所有运算默认转数字**（`==`、`><`、算术运算等）
#### 五、快速记忆口诀
1. 判真假：**七假其余全为真**（0、-0、""、null、undefined、NaN、false）
2. 做运算/比较：**统统转数字**，null变0、undefined变NaN
3. 加号拼接：**一见字符串，全转字符串**
4. 对象转换：**先 valueOf，后 toString**

## 1.2 模板字符串
用反引号 \` 包裹，用 ${} 嵌入变量，比 + 拼接清晰得多：
```
const name = 'bluek';
const score = 95;

// ❌ 老办法：用 + 拼接，眼花
const msg1 = '用户 ' + name + ' 的得分是 ' + score + ' 分';

// ✅ 新办法：模板字符串
const msg2 = `用户 ${name} 的得分是 ${score} 分`;

console.log(msg1 === msg2);  // → true，结果一样，但后者可读性碾压
```
# 2.比较运算符（核心！）

| 运算符 | 名称           | 行为                 |
| --- | ------------ | ------------------ |
| ==  | 松散比较（loose）  | 比较前会自动转换类型         |
| === | 严格比较（strict） | 类型不同直接返回 false，不转换 |
| !=  | 松散不等         | == 的反面             |
| !== | 严格不等         | === 的反面            |

## 2.1 == 的疯狂行为
```js
// ⚠️ 以下全部返回 true，因为 == 会自动转换类型
1 == '1';        // true   （字符串 '1' 转成数字 1）
0 == false;      // true   （false 转成数字 0）
0 == '';         // true   （空字符串转成数字 0）
0 == '0';        // true
null == undefined;  // true （这俩被认为"差不多"）
'' == false;     // true

// 更离谱的
[] == false;     // true   （空数组 → 空字符串 → 0）
[] == ![];       // true   （这条能让你怀疑人生）
```

## 2.2 === — 永远用它
```js
// ✅ 严格比较，类型不同直接 false，没有惊喜
1 === '1';          // false
0 === false;         // false
0 === '';            // false
null === undefined;  // false
```

> 	[!important] 黄金法则⭐
> 在 JS 中，永远用 \=== 和 !\==，除非你有明确的理由需要类型转换。做代码审计时，看到 == 就要警惕——它几乎总是一个潜在的漏洞点。
## 2.3 大小比较
```js
5 > 3;           // → true
5 >= 5;          // → true
5 < 3;           // → false
5 <= 5;          // → true

// ⚠️ 字符串比较按字典序（Unicode 码点）
'apple' < 'banana';   // → true
'10' < '9';           // → true！因为 '1' 的码点 < '9'（逐字符比较）
```

# 3.逻辑运算符

| 运算符  | 含义     | 示例              |
| ---- | ------ | --------------- |
| &&   | 且（AND） | 两边都 true 才 true |
| \|\| | 或（OR）  | 任一边 true 就 true |
| !    | 非（NOT） | 取反，true 变 false |
```js
const isLoggedIn = true;
const hasPermission = false;

isLoggedIn && hasPermission;  // → false （一个 true 一个 false）
isLoggedIn || hasPermission;  // → true  （至少一个 true）
!isLoggedIn;                  // → false （取反）
!hasPermission;               // → true
```
## 3.1 短路求值（Short-circuit）
&& 和 || 有一个关键特性：短路。
```js
// && : 左边为 false 时，右边根本不执行
true && console.log('执行了');   // → 打印 '执行了'
false && console.log('执行了');  // → 不打印，被短路

// || : 左边为 true 时，右边不执行
false || '默认值';    // → '默认值'（左边 false，返回右边）
true  || '默认值';    // → true（左边 true，短路，不看右边）
```
💡 实用技巧：|| 常用来设默认值：const name = userInput || '匿名用户'。如果 userInput 为空字符串、null、undefined 等 falsy 值，就用 '匿名用户'。
