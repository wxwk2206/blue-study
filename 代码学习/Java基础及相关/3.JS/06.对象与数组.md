# 1.对象（Object）— 键值对的集合
对象是一组「键（key）→ 值（value）」的映射，用花括号 {} 包裹。每个键是一个属性名，每个值可以是任意类型——数字、字符串、函数、甚至另一个对象。
## 1.1 创建和访问
```js
// ⭐ 对象字面量创建
const user = {
  name: 'bluek',        // 键: 值
  age: 22,
  role: 'security researcher',
  isActive: true
};

// ⭐ 两种访问方式
console.log(user.name);    // → 'bluek'     点运算符（最常用）
console.log(user['role']); // → 'security researcher'  方括号（键有空格或动态时用）
```
## 1.2 点运算符 vs 方括号

| 方式   | 语法         | 使用场景                 |
| ---- | ---------- | -------------------- |
| 点运算符 | obj.key    | 键名是合法标识符（无空格、无特殊字符）  |
| 方括号  | obj['key'] | 	键名有空格/特殊字符，或键名存在变量里 |
```js
// 为什么需要方括号？看这
const obj = {
  'my-key': 1,           // 键名有短横线
  'full name': 'bluek'   // 键名有空格
};
obj['my-key'];     // ✅ 能用方括号
obj['full name'];  // ✅ 能用方括号
obj.my-key;        // ❌ 语法错误！
```
## 1.3 增删改属性
```js
const user = { name: 'bluek' };

// 增：直接赋值
user.age = 22;

// 改：同样直接赋值
user.name = 'admin';

// ⭐ 删：delete 关键字
delete user.age;

console.log(user);  // → { name: 'admin' }
```
⚠️ 注意：const 声明的对象属性可以修改，只是不能把变量指向另一个对象。const user = {...}; user = {}; 才会报错。这是面试经典坑。

## 1.4 对象里放函数 → 方法
```js
const calculator = {
  value: 0,
  // ⭐ 对象里的函数叫「方法」（method）
  add(n) {
    this.value += n;       // ⭐ this 指向 calculator 自己
    return this;
  },
  getValue() {
    return this.value;
  }
};

calculator.add(5).add(3);  // 链式调用
console.log(calculator.getValue());  // → 8
```
this 是什么？在对象方法里，this 指向调用该方法的那个对象。这里是 calculator。

# 2.数组（Array）— 有序列表
数组是用 [] 包裹的有序数据集合，每个元素通过数字索引（从 0 开始）访问。
## 2.1 基础操作
```js
// ⭐ 创建数组
const fruits = ['apple', 'banana', 'orange'];

// 访问（索引从 0 开始）
console.log(fruits[0]);  // → 'apple'
console.log(fruits[2]);  // → 'orange'
console.log(fruits[99]); // → undefined ⚠️ JS 不抛异常！

// 修改
fruits[1] = 'grape';

// 长度
console.log(fruits.length);  // → 3
```
> [!important]  安全方向
> 数组越界不抛异常，返回 undefined。这是 JS 和 Java/C 的显著区别。如果代码没有检查 undefined 就继续操作，可能导致逻辑漏洞——比如 fruits[99].toUpperCase() 会因为 undefined 没有 .toUpperCase() 而抛 TypeError，但 if (fruits[99] === 'admin') 只是静默返回 false。

## 2.2 增删改查 — 核心方法

| 方法         | 作用   | 示例（arr = [1, 2, 3]）            |
| ---------- | ---- | ------------------------------ |
| push(v)    | 尾部添加 | arr.push(4) → [1,2,3,4]        |
| pop()      | 尾部删除 | arr.pop() → 返回 3，arr 变 [1,2]   |
| unshift(v) | 头部添加 | arr.unshift(0) → [0,1,2,3]     |
| shift()    | 头部删除 | arr.shift() → 返回 1，arr 变 [2,3] |
```js
const arr = [1, 2, 3];

arr.push(4);    // → [1, 2, 3, 4]  尾部加
arr.pop();      // → 4             尾部删，返回被删的
arr.unshift(0); // → [0, 1, 2, 3]  头部加
arr.shift();    // → 0             头部删，返回被删的

console.log(arr);  // → [1, 2, 3]  回到原样
```

## 2.3 splice — 万能手术刀
splice(起始索引, 删除个数, 添加项1, 添加项2...) — 在任意位置增删：
```js
const items = ['a', 'b', 'c', 'd', 'e'];

// ⭐ 删除：从索引 1 开始删 2 个
items.splice(1, 2);         // → ['a', 'd', 'e']

// ⭐ 插入：从索引 2 开始删 0 个，插入 'x' 和 'y'
items.splice(2, 0, 'x', 'y');  // → ['a', 'd', 'x', 'y', 'e']

// ⭐ 替换：从索引 3 开始删 1 个，插入 'z'
items.splice(3, 1, 'z');       // → ['a', 'd', 'x', 'z', 'e']
```
## 2.4 slice — 不改变原数组的切片
```js
const arr = [1, 2, 3, 4, 5];
arr.slice(1, 3);   // → [2, 3]   从索引 1 到 3（不含 3）
arr.slice(2);      // → [3, 4, 5] 从索引 2 到末尾
arr.slice();       // → [1, 2, 3, 4, 5] 浅拷贝整个数组
console.log(arr);  // → [1, 2, 3, 4, 5] 原数组不变
```

# 3.遍历数组：四种方式

| 方式       | 语法                    | 返回什么      | 能否中断    |
| -------- | --------------------- | --------- | ------- |
| for...of | for (const v of arr)  | —         | ✅ break |
| forEach  | arr.forEach(v => ...) | undefined | ❌ 不能    |
| map      | arr.map(v => ...)     | 新数组       | ❌ 不能    |
| filter   | arr.filter(v => ...)  | 新数组（筛选后的） | 	❌ 不能   |
```js
const nums = [1, 2, 3, 4, 5];

// ⭐ for...of：遍历值，可 break
for (const n of nums) {
  if (n > 3) break;
  console.log(n);              // → 1, 2, 3
}

// ⭐ forEach：对每个元素执行操作，无返回值
nums.forEach(n => console.log(n * 2));  // → 2, 4, 6, 8, 10

// ⭐ map：转换每个元素，返回新数组
const doubled = nums.map(n => n * 2);   // → [2, 4, 6, 8, 10]

// ⭐ filter：筛选符合条件的元素，返回新数组
const evens = nums.filter(n => n % 2 === 0);  // → [2, 4]
```
- 需要中断循环 → for...of
- 需要产生新数组 → map（转换）或 filter（筛选）
- 只是执行副作用（打印/发请求）→ forEach
# 4.结合实战：对象数组
现实中对象和数组通常一起出现——一个数组里存一堆对象：
```js
// ⭐ 典型的 API 返回数据结构
const users = [
  { id: 1, name: 'bluek', role: 'admin' },
  { id: 2, name: 'jack',  role: 'user' },
  { id: 3, name: 'tom',   role: 'user' }
];

// ⭐ 获取所有用户名（map）
const names = users.map(u => u.name);     // → ['bluek', 'jack', 'tom']

// ⭐ 筛选管理员（filter）
const admins = users.filter(u => u.role === 'admin');  // → [{ id:1, ... }]

// ⭐ 查找某个用户（find）
const jack = users.find(u => u.name === 'jack');       // → { id:2, name:'jack', ... }
```

> [!important]  代码审计视角
> 上面的 u.role \=== 'admin' ,这是之前学的严格比较。如果这里用了 \==（松散比较），攻击者可能通过类型混淆让普通用户也被识别为管理员。数据结构操作 + 松散比较 = 安全漏洞。

# 5.for...in 的坑（安全方向重要）

for...in 遍历对象的**所有可枚举属性**——包括**原型链上的**：
```js
const obj = { a: 1, b: 2 };

// ⭐ 给所有对象的原型加个属性
Object.prototype.hidden = '污染';

// for...in 会遍历到原型链
for (const key in obj) {
  console.log(key);  // → 'a', 'b', 'hidden'  ⚠️ 原型上的也遍历出来了！
}
```
⭐**这就是原型链污染的危害之一**！攻击者往 Object.prototype 注入属性后，所有对象的 for...in 都会意外遍历到它。如果业务代码用 for...in 做关键逻辑，行为会完全错乱。
> [!important]  Object.prototype
> - Object.prototype 是所有普通对象的根原型，原型链顶端，Object.prototype.\__proto\__ === null (代表原型链走到头了，没有上一级原型)。
> - 提供 toString、hasOwnProperty 等基础方法，所有对象默认继承。
> - 禁止直接修改，会影响全局所有对象。

正确的遍历方式：
```js
// ✅ 方法1：Object.keys()——只返回自身属性
const keys = Object.keys(obj);        // → ['a', 'b']  不含原型

// ✅ 方法2：for...of + Object.keys()
for (const key of Object.keys(obj)) {
  console.log(key, obj[key]);         // → a 1, b 2
}

// ✅ 方法3：hasOwnProperty 过滤
for (const key in obj) {
  if (Object.prototype.hasOwnProperty.call(obj, key)) {
    console.log(key);                 // → 'a', 'b'  只自身属性
  }
}
```
 **总结**：遍历数组用 for...of；遍历对象自身属性用 Object.keys()。不要把 for...in 当默认选择——它的原型链行为在安全审计时是减分项。

# 6.核心要点

| 概念      | 一句话                                                                 |
| ------- | ------------------------------------------------------------------- |
| 对象      | { key: value } 键值对集合，键是字符串                                          |
| 访问属性    | obj.key（常用）或 obj['key']（动态/特殊键名）                                    |
| 数组      | [item1, item2] 有序列表，索引从 0 开始<br>                                    |
| 越界行为    | undefined，不抛异常（和 Java 不同）                                           |
| 核心方法    | 增删：push/pop/unshift/shift/splice；遍历：map/filter/forEach/for...of<br> |
| 遍历对象    | 用 Object.keys()，别用 for...in（会遍历原型链）<br>                             |
| 🔒 审计重点 | 数组越界 → undefined；for...in → 原型链污染入口<br>                             |
