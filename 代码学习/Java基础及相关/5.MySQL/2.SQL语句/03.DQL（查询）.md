DQL英文全称是Data Query Language（数据查询语言），用来查询数据库表中的记录。
关键字：SELECT
```MySQL
select
    字段列表
from
    表名列表
where
    条件列表
group by
    分组字段列表
having  #是分组之后的过滤条件，专门用来对 GROUP BY 分组聚合后的结果做筛选
    分组后条件列表
order by
    排序字段列表
limit
    分页参数
```
- 基本查询（select...from...）
- 条件查询（where）
- 分组查询（group by）
- 排序查询（order by）
- 分页查询（limit）
## 1.基本查询select...from
```sql
-- 查询多个字段
select 字段1,字段2,字段3 from 表名;

-- 查询所有字段(通配符)
select * from 表名;

-- 为查询字段设置别名, as关键字可以省略
select 字段1 [as 别名1], 字段2 [as 别名2] from 表名;

-- 去除重复记录
select distinct 字段列表 from 表名;
```
SELECT  后面直接跟的是「要查的内容」可以是：1. 列名，2. 常量/数字，3. 函数，4. 表达式

## 2.条件查询where
```sql
-- 条件查询
select 字段列表 from 表名 where 条件列表 ;
```

### 条件
| 比较运算符 | 功能 |
| ---- | ---- |
| `>` | 大于 |
| `>=` | 大于等于 |
| `<` | 小于 |
| `<=` | 小于等于 |
| `=` | 等于 |
| `<>` 或 `!=` | 不等于 |
| `between ... and ...` | 在某个范围之内(含最小、最大值) |
| `in(...)` | 在in之后的列表中的值，多选一 |
| `like` 占位符 | 模糊匹配(`_`匹配单个字符，`%`匹配任意个字符) |
| `is null` | 是null |

| 逻辑运算符 | 功能 |
| ---- | ---- |
| `and` 或 `&&` | 并且（多个条件同时成立） |
| `or` 或 `||` | 或者（多个条件任意一个成立） |
| `not` 或 `!` | 非，不是 |

## 3.分组查询group by
### 聚合函数
聚合函数：将一列数据作为一个整体，进行纵向计算

| 函数    | 功能   |
| ----- | ---- |
| count | 统计数量 |
| max   | 最大值  |
| min   | 最小值  |
| avg   | 平均值  |
| sum   | 求和   |

**注意**：所有的聚合函数不参与null值的统计
count使用方式：count（字段）  count（\*） count（常量）
```sql
--  统计该企业员工的平均薪资 - avg
select avg(salary) from emp;

--  统计该企业员工的最低薪资 - min
select min(salary) from emp;

--  统计该企业员工的最高薪资 - max
select max(salary) from emp;
```

### 分组查询
```sql
-- 分组查询
select 字段列表 from 表名 [where 条件列表] group by 分组字段名 [having 分组后过滤条件];
```
**where与having的区别：**
1.执行时机不同：where是分组之前进行过滤，不满足where条件，不参与分组；而having是分组之后对结果进行过滤。
2． 判断条件不同：where不能对聚合函数进行判断，而having可以。
```sql
--  先查询入职时间在 '2015-01-01'（包含）以前的员工，并对结果根据职位分组，获取员工数量大于等于2的职位
select job, count(*) from emp where entry_date <= '2015-01-01' group by job having count(*) >=2;
```
执行顺序：where>聚合函数>having

DQL语句中where与having的区别？
- 执行时机不同（where -> group by -> having）
- 判断条件不同（having后可以用聚合函数，where不可以）

## 4.排序查询order by

```sql
-- 排序查询
select 字段列表 from 表名 [where 条件列表] [group by 分组字段名 having 分组后过滤条件] order by 排序字段 排序方式;
```
**排序方式**：升序（asc），降序（desc）；默认为升序asc，可以不写

注意：如果是多字段排序，当第一个字段值相同时，才会根据第二个字段进行排序。

### SQL注入相关：
**order by 常数** 是按第常数列排序，如 order by 3 就是按第三列排序

以此方法可以知道数据表中有多少列

在新版本中 **order by 常数** ，当常数超过列数时不会报错，而是按常数排序，即不排序

**新方法：用union select null**，...试字段数
