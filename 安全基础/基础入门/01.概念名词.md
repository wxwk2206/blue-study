## 域名
### 1.什么是域名：域名就是地址名称，
如：www.baidu.com

### 2.在哪注册域名：第三方平台，域名注册商，
如：万网

### 3.什么是二级域名，多级域名
以 `a.b.c.com` 为例：

+ `.com` → 顶级域名
+ `c.com` → 二级域名
+ `b.c.com` → 三级域名
+ `a.b.c.com` → 四级域名

前面加得越多，级别越多，统称多级域名。

### 4.URL和域名的关系
URL是完整地址，域名是地址中的关键部分

| **维度** | **域名** | **URL** |
| --- | --- | --- |
| 范围 | 仅指向 “服务器” | 指向 “服务器上的具体资源” |
| 组成 | 单一字符串（如 [baidu.com](https://baidu.com)） | 多部分组合（协议 + 域名 + 路径 + 参数等） |
| 作用 | 定位服务器 | 定位服务器上的具体资源 |
| 例子 | `jd.com` | `https://jd.com/phone/iphone.html` |


### 5.域名发现对安全测试的意义
子站与主站解析的IP服务器地址一样时，以子站作为突破口进行渗透测试，突破到一定权限后就可以获取到主站权限

#### 主站
主站是什么，以 `baidu.com` 举例：

+ 主站：`baidu.com`（不带任何前缀，或带 www）
+ 其他都是子站 / 分站：
    - `map.baidu.com` 地图站
    - `tieba.baidu.com` 贴吧站
    - `pan.baidu.com` 网盘站

它们都属于百度，但只有 [baidu.com](https://baidu.com) 是主站。

#### 旁站入侵（Side-site Attack）
**一句话：同一台服务器上，还跑着别的网站，先攻破那个弱的，再拿到整台服务器权限。**

一台服务器（IP）上，可能放了 N 个网站：

+ [www.A.com](https://www.A.com)（你想打的目标，很安全）
+ [www.B.com](https://www.B.com)（别人的垃圾站，很弱）
+ [www.C.com](https://www.C.com)（小站点，漏洞多）

它们**共享同一台服务器**。

旁站思路：

1. 扫目标网站的 **同 IP 服务器** 有哪些站
2. 找这些站里**最薄弱的一个**
3. 攻破这个弱站
4. 因为**同服务器**，拿到服务器权限
5. 再横向移动，拿到目标站数据

## DNS
### 1.什么是DNS
域名系统服务协议（DNS）是一种分布式网络目录服务，主要用于域名与IP地址的相互转换

### 2.本地HOSTS与DNS的关系
**本地HOSTS可以重定向解析IP地址，**

本地HOSTS位于"C:\Windows\System32\drivers\etc\hosts"

### 3.CDN是什么，与DNS的关系
CDN：内容分发网络，就是把视频、图片、网页**提前复制到全国各地的节点**，访问时，自动连**距离最近、最快的那台机器**。

作用：加速、不卡、减轻源站压力、**隐藏真实服务器 IP**

扫描到CDN节点的IP，而不是源站时，只能扫描到节点的缓存

### 4.DNS常见安全攻击手段
#### 1. DNS 劫持（最常见）
+ 把你要访问的域名，**解析成黑客指定的假 IP**
+ 你以为上官网，其实上钓鱼站
+ 例子：路由器被黑、DNS 被改、运营商劫持

#### 2. DNS 污染（DNS 投毒）
+ 黑客**污染 DNS 缓存**
+ 让整个地区、整个运营商都解析到错误 IP
+ 特点：**改本机 DNS 没用，依然错**

#### 3. DDoS 攻击 DNS
+ 把 DNS 服务器打崩
+ 一崩，所有人都**无法解析域名 → 全网瘫痪**
+ 常见：UDP 泛洪、放大攻击

#### 4. DNS 域传送漏洞（渗透必考）
+ 管理员配置错误
+ 黑客直接拿到**整个域名的所有解析记录**
+ 子域名、邮件服务器、内网 IP 全泄露
+ 命令：`nslookup > server x.x.x.x > ls -d domain.com`

#### 5. DNS 隧道（DNS Tunnel）
+ 黑客**把数据藏在 DNS 请求里**传出去
+ 绕过防火墙、不出网机器也能外带数据
+ 内网渗透、远控常用

#### 6. DNS 重绑定（DNS Rebinding）
+ 先解析公网 IP，过一会偷偷换成**内网 IP**
+ 绕过浏览器同源策略
+ 用来**攻击内网设备**

#### 7. 缓存投毒
+ 让本地 DNS 缓存**假记录**
+ 下次所有人再查，都自动跳到假网站

## 脚本语言
### 1.常见脚本语言类型
#### 1. 通用型脚本语言（全能型，覆盖多场景）
+ **Python**
    - 核心特点：语法简洁、跨平台、库生态极丰富（网络、爬虫、数据分析、自动化、渗透测试全覆盖）。
    - 典型场景：渗透测试（写 POC/EXP、批量扫描）、运维脚本、网络爬虫、AI 开发、自动化测试。
    - 安全领域应用：你之前关注的 DNS 检测、CDN 绕过、子域名爆破等，几乎都能用 Python 快速实现（比如用`requests`库做 HTTP 请求，`dnspython`库解析 DNS）。
+ **JavaScript (JS)**
    - 核心特点：前端浏览器标配，也可通过 Node.js 运行在后端（全栈脚本），异步编程能力强。
    - 典型场景：前端交互、后端 API 开发（Node.js）、前端渗透（XSS 漏洞利用、DOM 劫持）、自动化脚本。
    - 安全领域应用：前端漏洞测试（XSS 脚本编写）、Node.js 做后端渗透测试脚本、DNS 隧道客户端开发等。
+ **Perl**
    - 核心特点：文本处理能力极强，正则表达式原生支持，早期渗透测试 / 运维的 “神器”。
    - 典型场景：日志分析、文本处理、早期渗透脚本（比如 Metasploit 里很多模块基于 Perl）。
    - 现状：现在逐渐被 Python 替代，但老的安全工具 / 脚本仍大量使用。

#### 2. 运维 / 系统管理脚本语言（针对系统操作）
+ **Shell 脚本（Bash/Zsh 等）**
    - 核心特点：直接调用操作系统命令，专为 Linux/Unix 系统管理设计，轻量、贴合系统底层。
    - 典型场景：Linux 运维自动化（批量启停服务、备份文件、修改配置）、服务器初始化、简单的渗透测试前置脚本（比如批量执行系统命令）。
    - 实用举例：你之前改 DNS 的`ipconfig /flushdns`（Windows 批处理）、`netsh`命令，本质就是 Shell / 批处理脚本的范畴。
+ **PowerShell**
    - 核心特点：Windows 系统的 “高级 Shell”，兼容.NET 框架，可调用 Windows API，功能远超传统批处理（.bat）。
    - 典型场景：Windows 运维自动化、Windows 渗透测试（比如权限提升、内网横向移动脚本）。
    - 安全领域：黑客常用来写无文件攻击脚本，绕过杀毒软件，也是你在 Windows 上改 DNS、查网络配置的核心脚本工具。

#### 3. 网页 / 服务器端脚本语言（专用于 Web 开发）
+ **PHP**
    - 核心特点：专为 Web 开发设计，轻量、与 MySQL 数据库适配极佳，部署简单。
    - 典型场景：中小型网站后端（比如论坛、博客、电商网站）、Web 漏洞（SQL 注入、文件包含）的主要载体。
    - 安全关联：你之前问的 “旁站入侵” 中，很多漏洞（比如 PHP 文件上传漏洞）都是针对 PHP 脚本的。
+ **Ruby**
    - 核心特点：语法优雅、面向对象，Ruby on Rails 框架让 Web 开发极快。
    - 典型场景：Web 开发（Rails）、渗透测试工具（Metasploit 框架基于 Ruby 开发）。
+ **ASP/**[**ASP.NET**](https://ASP.NET)
    - 核心特点：微软推出的 Web 脚本，运行在 IIS 服务器上，适配 Windows 服务器。
    - 典型场景：Windows 服务器的 Web 开发，也是 Web 渗透中常见的攻击目标（比如 ASP 木马）。

#### 4. 数据库脚本语言（专用于数据库操作）
+ **SQL**（结构化查询语言，虽不算传统脚本，但常被归为 “数据库脚本”）
    - 核心特点：操作数据库的标准语言，可写存储过程、触发器等脚本。
    - 典型场景：数据查询 / 修改、数据库运维，也是 SQL 注入攻击的核心（黑客构造恶意 SQL 脚本注入 Web 应用）。

## 后门
 绕过所有验证，**偷偷进入服务器 / 网站**的通道  

1. **Web 后门（最常见**）  
上传一个脚本文件到网站目录，访问就能控制服务器。  
你前面学的都能对应上：  
PHP 后门：一句话木马、phpspy  
ASP/ASPX 后门：asp 木马  
JSP 后门：jsp 大马、jsp 一句话（JavaWeb 专用）  
Python 后门：py 脚本  
特点：  只要能访问网页，就能控制服务器，旁站入侵、文件上传漏洞后必用
2. **系统后门（拿到服务器权限后用）**  
在操作系统里留入口：  
开隐藏账号  
开 3389 远程桌面  
写 计划任务  
开 SSH 公钥登录（Linux）  
远控木马：向日葵、TeamViewer、远控程序
3. **端口后门**  
偷偷开一个端口，只有黑客知道：  开 4444、8888、9999 等端口  
防火墙放行 ，直接连接控制
4. **启动项后门**  
开机自动运行：  
Windows：注册表、启动文件夹  
Linux：/etc/init.d、systemd、crontab
5. **第三方服务后门**  
数据库后门（SQL Server 开启 xp_cmdshell）  
Tomcat 后台弱口令  
Redis 无密码 → 直接写 SSH 公钥
6. **逻辑后门（代码里藏的）**

```java
if(username == "admin" && password == "123456"){
  // 正常登录
}
if(username == "test" && password == "abc@123"){
  // 后门，直接登录
}
```

## WEB
### 1.web的组成框架模型
网站源码：分脚本类型，应用方向

操作系统：Windows，Linux

中间件（搭建平台）：apache，iis，tomcat，nginx等

数据库：access，MySQL，mssql，Oracle，Sybase，db2，postsql等

### 2.web相关安全漏洞
**WEB源码类对应漏洞**：SQL注入，上传，xss，代码执行，变量覆盖，逻辑漏洞，反序列化等

**WEB中间件对应漏洞**：Tomcat 弱口令、Tomcat 任意文件上传 / 包含、Nginx 解析漏洞等

**WEB数据库对应漏洞**： 弱口令、默认账号  ， 数据库未授权访问、外网暴露端口等  

**WEB系统层对应漏洞**：Windows/Linux 系统弱口令、远程桌面 RDP/SSH 弱口令等

**其他第三方对应漏洞**： 框架漏洞 ， 插件漏洞  ， 依赖组件   第三方接口、支付接口、第三方登录接口安全缺陷等

**APP或PC应用结合类**： 接口越权、接口参数篡改、APP 抓包重放、签名绕过， CSO 客户端漏洞、本地权限提升、配置文件泄露、客户端协议漏洞、内网穿透漏洞等
